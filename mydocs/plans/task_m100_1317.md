# Task M100 #1317 수행계획서 — SVG 적분기호 path 렌더로 폰트 독립 bbox 확보

> PR #1314 (Task #1313) 후속. 메인테이너 시각 판정 보류 해소.

## 1. 배경

PR #1314 (적분기호 글리프 확대 및 상·하한 밀착)이 메인테이너(edwardkim) 시각 판정에서 보류됨.

- ✅ Canvas/WASM: 적분기호 크기·상하한 배치 개선
- ❌ **SVG export: 적분 상한/하한 위치가 어긋나 보임**
- 재현: `samples/3-10월_교육_통합_2022.hwp` 9페이지 (`rhwp export-svg ... -p 8`)

## 2. 근본 원인 (코드 검증 완료)

| 위치 | 내용 |
|------|------|
| `layout.rs:550-553` | 적분 상·하한을 **고정 offset**(`sup fs*0.21`, `sub fs*0.55`, `top_pad fs*0.13`)으로 배치. 적분 글리프 박스를 `height = fs × INTEGRAL_SCALE(2.5)` **명목 크기**로 가정 |
| `svg_render.rs:97-111` | `MathSymbol(∫)`을 `<text font-size=lb.height font-family="'Latin Modern Math',...">`로 출력. **폰트 임베딩 없음** |
| `ttfs/` | 수식 폰트(Latin Modern Math/STIX) **부재** (NotoSansKR만 존재) → `--embed-fonts`로도 임베딩 불가 |

→ SVG는 뷰어 대체 폰트(보통 Times)의 ∫를 그리는데, 그 실제 visual 높이가 명목 박스(`fs*2.5`)와 달라 layout 고정 offset으로 배치된 상·하한이 글리프 끝에서 떠 보인다.

Canvas/WASM은 실제 웹폰트 메트릭이 명목 박스에 근사하게 맞아 정상으로 보였던 것.

## 3. 해결 방향 (채택: 2안)

SVG 경로에서 적분기호 ∫를 폰트 `<text>`가 아닌 **path/shape로 렌더링**하여 글리프 visual bbox를 **폰트 독립·결정적**으로 만든다. 상·하한 attach point를 그 결정적 bbox에 맞춰 재정렬한다.

### 핵심 원칙

- **정합 범위**: Canvas만이 아니라 SVG export + native PNG/Skia 까지 같은 기준으로 정합 (메인테이너 요구).
- **HWP3 무관**: 본 변경은 공통 렌더러(`src/renderer/equation/`) 범위. 포맷별 분기 없음.
- **정답지**: `pdf/` (한글 2022) 9페이지·11페이지 적분. (Linux 환경 → pdf/ 1차 정답지)

## 4. 영향 범위 (예상)

| 파일 | 변경 성격 |
|------|----------|
| `src/renderer/equation/svg_render.rs` | ∫ `<text>` → path 렌더 (MathSymbol 분기 + BigOp 적분 분기) |
| `src/renderer/equation/layout.rs` | (필요 시) 적분 글리프 visual bbox·attach point를 명시 상수화하여 path와 offset이 동일 기준 공유 |
| `src/renderer/equation/canvas_render.rs`, `skia/equation_conv.rs` | path 기준 정합 검증 후 필요 시 동기화 |

> 실제 단계 분할·세부 변경은 **구현 계획서**(`task_m100_1317_impl.md`)에서 확정.
