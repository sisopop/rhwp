## 1. 미해결 문제 (v1 진단 확정)
PR #1368(A/C)로 p22 그림 overflow 는 해소했으나 **콘텐츠 단/페이지 배치**는 미해결:

| 케이스 | 현상 | 누적 정확도 | 필요 |
|--------|------|------------|------|
| p17 pi=894 "C×C" | 좌단 하단 (한컴=우단 상단) | 단0 **과소**(렌더 990.8 ≈ 누적 999.6, 한 줄 더 fit) | 더 |
| p21 pi=1127 | p22 단0 상단 (한컴=p21 우단 하단) | 단1 **+64px 과대**(수식 saved-delta 가 vpos 압축 간격 과다 계상) | 덜 |

**두 케이스 반대 방향.** 단일 규칙 acc 보정 3종 모두 실증 회귀:
①`line_adv_sum` 전면→2022 +166px ②uncapped→질문흐름 10건 ③vpos-span 전면→전 문서 과충전.

## 2. 근본 원인
각 미주 para 의 실제 렌더 높이가 제각각(saved-delta / line_adv_sum / vpos-span). 렌더러는
para 별 `HeightCursor::vpos_adjust`(height_cursor.rs:146)로 배치하나, typeset 누적
`compute_en_metrics`(typeset.rs)는 이를 saved-delta + caps 로 **근사**할 뿐 → divergence.
caps(`stale_forward_vpos`/`capped_new_endnote_advance`/`compact_local_rewind`)는 그 근사의
패치 누적이며, exam 별 하드튜닝되어 어느 방향으로도 일괄 조정 불가.

## 3. SSOT 원칙 (후보 A)
**typeset 누적 per-para = 렌더러 vpos_adjust 가 산출할 실제 렌더 높이.** 두 경로가 **동일
높이 함수**를 공유(근사 아님). caps 는 SSOT 정확화로 불필요해지면 단계적 제거.

### 핵심 난점 — 시점 차이
- 렌더러 `vpos_adjust` 는 **렌더 시점**(페이지 확정 후) per-para y 보정.
- typeset 누적은 **페이지네이션 시점**(배치 결정용) 사전 높이 필요.
→ vpos_adjust 의 입력(prev para seg, vpos 연속성, trailing-ls bridge 게이트)을 **페이지네이션
시점에 동일 계산**하는 예측 함수 `endnote_para_vpos_advance(prev, this, ctx)` 추출 필요.
v1 의 `EnAdvance`/`RHWP_EN_SSOT` 플래그·`EN_SSOT`/`EN_ACC` 계측·`task1363_ssot_diff.py`
하니스 재사용.

## 4. 단계 (잠정, 다회차)
1. **Stage 1 — vpos_adjust 입력 분해**: `vpos_adjust` 가 per-para 높이를 정하는 모든 입력

   backward clamp)을 페이지네이션 시점 가용성 기준으로 분류. 예측 가능/불가 항목 식별.
2. **Stage 2 — 공유 예측 함수 추출**: `endnote_para_vpos_advance` 를 height_cursor 와 typeset
   양쪽이 호출하도록 추출(렌더는 검증, typeset 은 예측). `RHWP_EN_SSOT=A2` 플래그 게이트.
3. **Stage 3 — caps 제거 + 재튜닝**: SSOT 정확화로 무력화되는 cap 분기부터 1개씩 제거,
