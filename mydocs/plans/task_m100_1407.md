## 1. 배경

#1382 4단계 전수 검증에서 분리. 143E 문단 0.14(머리말 + 하이퍼링크 필드 + 새번호
복합 문단)에서 newNum 슬롯이 원본 위치(텍스트 끝)에서 fieldEnd 뒤(텍스트 앞)로 이동
— ir-diff char_offsets[3] 27→35 (8유닛). #1382 이전부터 존재하는 기존 결함.

## 2. 근본 원인 (1단계 조사 완료 — spot 확정)

원본 IR 문단 0.14:
- text="김영훈 기자(…)" (30 chars), char_offsets=`[16,17,18, 27..53]`
- controls=`[Header, Field/Hyperlink, NewNumber]`, field_ranges=`[0..3 ctrl_idx=1]`
- newNum 은 원본 XML 상 **텍스트 끝**(pos 54)

`render_runs` 메인 루프(`section.rs:431~`)는 `char_pos >= expected+8` 갭이 보이면
controls 순서대로 다음 슬롯을 방출한다. idx=3 에서 char_pos=27 갭(실제론 fieldEnd
자리)을 만나 **slots[2]=NewNumber 를 가로채 방출** → 이후 텍스트 +8 밀림.

- newNum 은 placeholder 가 없어(파서 `section.rs:3891`) char_offsets 에 위치 정보가
  안 남는다 → controls 순서 + 8유닛 갭 추론만으로 배치 → fieldEnd 갭과 혼동.
- autoNum(#1382)은 placeholder 공백으로 위치 명시 → 정확. newNum 은 별개 계열.

상세: `mydocs/troubleshootings/hwpx_newnum_slot_after_text.md`

## 3. 수정 방향 (구현계획서에서 확정)

핵심: 메인 루프에서 **fieldEnd 갭이 슬롯 방출보다 우선**해야 하는 위치를 판별.
fieldEnd(field_ranges 유래 8유닛 슬롯)는 controls 에 대응 컨트롤이 없으므로, 현재
char_pos 갭이 fieldEnd 의 것인지(아직 안 닫힌 field_range 의 end 위치인지) 먼저 확인하고
그 경우 슬롯 방출을 양보한다. 텍스트 끝 슬롯(newNum)은 남은 슬롯 일괄 방출(line 559)
경로로 자연 배치.

대안·세부는 구현계획서에서 슬롯/필드 우선순위 규칙으로 정밀화.
