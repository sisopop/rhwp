## 2. 현재 재검증 결과

이슈 본문은 `samples/hwpx/tb-tac-img-01.hwpx`를 재현 파일로 적고 있으나, 최신 `upstream/devel`에는 해당 파일이 없다. 대신 저장소에 이미 있는 `samples/hwpx/hy-001.hwpx`가 본문 조건과 같은 구조를 가진다.

확인한 구조:

- 첫 표 / 첫 행 / 첫 셀 `name="기관명"`
- 셀 `vertAlign="CENTER"`
- 같은 문단에 TAC picture와 텍스트 `광부`가 함께 존재
- picture `treatAsChar="1"`
- line segment `vertsize="2834"`, `textheight="2834"`, `baseline="2409"`

현재 rhwp render tree에서 첫 셀의 image는 cell clip 아래쪽으로 밀린다.

- cell clip: 대략 `y=79.35`, `h=41.55`
- text line: 대략 `y=81.2`, `h=37.8`
- image: 대략 `y=102.6`, `h=37.8`
- 결과: image 하단이 cell clip 밖으로 잘려 한컴 PDF 기준과 다르다.

한컴 PDF 기준은 Git LFS pointer 대신 GitHub media URL에서 실제 PDF를 받아 확인했다.

## 3. 관련 기존 작업

- #974: `hy-001` 글상자 내부 TAC 그림/공백 렌더링 guard

- #1285: 셀 안 TAC 표/텍스트 inline sequence 정렬 guard

이번 결함은 글상자 내부가 아니라 표 셀 내부의 `paragraph_layout` inline picture y 계산과 관련이 높다. 특히 `tac_picture_label_extra_px`가 TAC picture 높이와 line height가 거의 같은 경우 image y를 아래로 더 미는 경로가 `광부`처럼 실제 텍스트가 있는 셀 문단에도 적용되는지 확인해야 한다.
