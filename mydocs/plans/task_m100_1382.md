## 2. 현황 분석 (2026-06-12 — 코드 확인 완료)

### 결함 본체: 같은 파서 안의 두 폭 집계 축

| 축 | 위치 | `\u{0012}`(AUTO_NUMBER) 폭 | 사용처 |
|----|------|---------------------------|--------|
| offsets 조립 루프 | `parser/hwpx/section.rs:642~648` | **8 유닛** jump (+placeholder 공백, #1050 HWP PARA_TEXT 정합) | char_offsets, serializer 슬롯 추론 |
| `calc_utf16_len_from_parts` | `parser/hwpx/section.rs:4918` | **1 유닛** (`\u{0002}~\u{0004}`만 8, `\u{0012}`는 `_` 분기) | char_shapes 경계 위치 |

→ 같은 문단의 char_shapes 경계와 char_offsets가 서로 다른 좌표계에 놓인다.
serializer(RunSplitter + 슬롯 시스템)는 offsets 축 위에서 동작하므로 1유닛 축의
char_shapes 경계가 어긋난 위치에 cut을 만들고, autoNum 슬롯이 변위된다.

### 발현 2패턴 (실증 완료)

1. **143E 각주**: 원본 char_shapes `[(0,10),(2,11)]`(1유닛 축) → 재파싱 `[(0,10),(1,11)]`
   — 경계 1유닛 시프트, baseline IR_DIFF xfail
2. **ta-pic 캡션** (#1387 판정): autoNum 슬롯 문단 끝 방출 + 재파싱 끝 placeholder
   공백 1자 — 한컴은 ctrl XML 위치(끝)에 번호 렌더, rhwp-studio는 placeholder 문자
   위치(mid-text)에 렌더 → 두 렌더러 표시 불일치

### 트러블슈팅 사전 검색 결과

  선례: "자기 정합 ≠ 한컴 호환" 입증 기록. 본 타스크 4단계 한컴 판정 게이트의 근거.
- autoNum 폭 자체의 선행 실패 기록 없음 (#1050이 offsets 축 8유닛 정합의 권위).

### 열린 PR 충돌 확인

## 3. 정정 방향 (스펙 근거)

HWP 5.0 PARA_TEXT에서 AUTO_NUMBER(0x12)는 확장 컨트롤 — 8 WCHAR 점유. offsets 축의
8유닛(#1050에서 한컴 정합 확정)이 정본이고, `calc_utf16_len_from_parts`의 1유닛이
결함이다. **`\u{0012}`를 8유닛 분기에 추가**하여 char_shapes 경계를 offsets 축에
일관화한다. 1단계에서 스펙 표 + 143E 원본 char_shapes 실측으로 교차 확정 후 진행
(추정 금지 룰).

## 4. 목표와 완료 조건

1. char_shapes 경계와 char_offsets가 같은 축(8유닛)에 놓인다.
2. 143E 각주 roundtrip char_shapes 대칭 — `XFAIL_1378_RECURSIVE`에서 제거, baseline 승격.
3. ta-pic 캡션 autoNum 슬롯이 mid-text 원위치에 방출 — 재파싱 텍스트 완전 일치
   (#1387 테스트의 trim_end 완화 제거·승격).
4. 전수 배치 신규 실패 0, 렌더링 회귀 0 (char_shapes 위치 소비처 영향 검증 포함).
5. 한컴 판정: ta-pic 캡션 번호가 "&lt;그림 1&gt;" 위치에 표시.

## 5. 단계 개요 (구현계획서에서 상세화)

1. **측정/추적**: 두 발현 패턴의 메커니즘 정밀 추적 (RunSplitter cut 경로), 전수
   `\u{0012}`·`\u{0002}~\u{0004}` 분포, 스펙 교차 확정, char_shapes 소비처(렌더러/
   레이아웃) 축 의존성 조사
2. **파서 정정**: `calc_utf16_len_from_parts` `\u{0012}` 8유닛 + 동류 비일관 토큰
   일괄 점검 + 단위 테스트
3. **xfail 승격 + 테스트 승격**: 143E `XFAIL_1378_RECURSIVE` 제거(가드가 강제),

## 6. 위험과 대응

| 위험 | 대응 |
|------|------|
| char_shapes 축 변경이 렌더러/레이아웃의 위치 소비처에 회귀 유발 | 1단계에서 소비처 전수 조사 (HWP5 파서도 8유닛 축이므로 HWPX만 비일관이었을 가능성이 높음 — 일관화가 오히려 정합 방향). 전수 시각 회귀는 4단계 SVG + 한컴 판정 |
