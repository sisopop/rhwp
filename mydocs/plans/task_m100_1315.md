### 사전 점검 결과

- **열린 PR 충돌**: PR #1366 (idaeho, hwp2hwpx CLI)이 `src/serializer/hwpx/` 일부를 수정하나, 현재 수정 요청(최신 devel 기준 재작성) 상태. 본 타스크는 serializer 본체 수정을 최소화하고 검증 하네스·문서가 중심이므로 충돌 위험 낮음. serializer 수정이 필요해지면 충돌 파일(`section.rs`, `context.rs`, `table.rs`)을 피해 별도 보고한다.
- **트러블슈팅 사전 검색**: `hwpx_lineseg_reflow_trap.md`(LINE_SEG 재계산 함정), `hwpx_footnote_save_5round_oracle_method.md`(5-round oracle 기법), `hwpx_equation_save_eqedit_spec_errata.md` 등 직렬화 관련 기록 확인. roundtrip 다회 반복(2-round 이상) 안정성 검사를 하네스에 반영한다.
- **이슈 주의사항 반영**: `tac-img-02.hwpx` 류 복합 샘플은 full-document roundtrip oracle로 쓰지 않고 등급표에 명시한다.

## 수행 범위

이슈의 작업 범위 5개 항목을 그대로 따른다.

1. **roundtrip 검증 하네스 정리** — 전수 샘플 대상, 패키지 구조 + 재파싱 가능 여부 검사
2. **샘플 등급화** — baseline / xfail / oracle 부적합 분류
3. **최소 호환성 기준 정의** — rhwp-studio 열림, 한컴에디터 열기 오류 없음(작업지시자 시각 판정), 핵심 구조 보존
4. **회귀 검증 도구화** — `output/poc/` 산출 명령 + header/section 주요 속성 비교
5. **문서화** — baseline 범위·제한 + roundtrip 검증 절차 기록

## 수행 방법 (단계 개요 — 상세는 구현 계획서에서)

- **1단계: 전수 인벤토리 + 현황 측정**
  `samples/hwpx/` 전체 `.hwpx`에 대해 roundtrip 실행 → 파싱 실패 / 직렬화 실패 / 재파싱 실패 / IrDiff 건수를 표로 수집. 코드 수정 없이 측정 우선.
- **2단계: 검증 하네스 구축**
  전수 샘플을 도는 통합 테스트(또는 xtask/CLI 서브커맨드) + 패키지 구조 검사(필수 엔트리, BinData 카운트) 추가. 기존 `roundtrip_ir_diff` 재사용, 누적-만-가능 원칙 유지.
- **3단계: 등급화 + xfail 처리**
  1단계 측정 결과 기반으로 baseline pass 목록 / xfail 목록 확정. xfail은 사유(미지원 기능)와 함께 테스트에 명시.
- **4단계: 회귀 도구화 + 산출물 생성**
  roundtrip HWPX를 `output/poc/task1315/`에 생성하는 명령 정리. 대표 샘플 산출물을 작업지시자 시각 판정(rhwp-studio + 한컴에디터)용으로 전달.
- **5단계: 문서화 + 최종 보고**
  `mydocs/manual/hwpx_roundtrip_baseline.md`(가칭)에 검증 절차·등급표·제한 사항 기록. full fidelity oracle 오해 방지 가이드 포함.

> 한컴에디터 열기 검증은 작업지시자 직접 수행 항목이므로, 4단계에서 산출물 전달 후 판정을 요청한다.
