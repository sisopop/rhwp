## 2. 현황 분석 (2026-06-12 — #1379 머지 후 재실측)

이슈 등록 시점과 현재 코드 사이에 #1378/#1379가 머지되어 **이슈 기재 범위 일부가
이미 해소**되었을 가능성이 높다. 착수 시점 spot 실측 (ta-pic-001-r, #1379 4단계 rt):

| 축 | 이슈 기재 | 현재 실측 | 판단 |
|----|----------|----------|------|
| 셀·글상자 합성 lineseg | vertsize=1000 고정 | 셀 문단 lineseg(1200/1020/360) **원본 일치** | #1379 공유 경로 전환으로 해소 추정 — 1단계 전수 확정 |
| 본문 값 변형 | lh=21974→19924 등 | 미확인 (당시 실측치는 #1315 기준) | 1단계 재측정 |
| 빈 문단 fallback | — | RT에 `vertsize=1000 … horzsize=0` 합성 잔존 | 잔존 결함 — 출처(파서 미적재 vs serializer fallback) 분류 필요 |
| 페이지 수 변화 4샘플 | form-002 10→17 등 | 미확인 | 1단계 재측정 |

참고: ta-pic-001-r의 lineseg 1개 소실은 표 캡션 문단 소실(#1387) 기인으로 본 타스크
범위 밖. 페이지 여백 기인 시프트는 #1388 범위.

### serializer 경로 현황

#1379 이후 본문·셀·글상자 세 경로 모두 `render_paragraph_parts()`(section.rs) 공유:
IR `line_segs` 비어있지 않으면 `render_lineseg_array_from_ir`, 비어있으면
`render_lineseg_array_fallback` 합성. 따라서 잔존 결함 후보는:

1. **IR이 비는 경우** — 파서가 lineseg를 적재하지 않는 문단 유형 (빈 문단 등)
2. **IR 값 자체가 변형되는 경우** — 파서 적재 시점 변형
3. **방출 시 값 가공** — serializer 필드 매핑 오류

1단계에서 전수 측정으로 후보를 분류·확정한다.

### 게이트 공백

baseline 게이트(`diff_documents`)는 lineseg 값을 비교하지 않는다 (매뉴얼 1절 명시).
lineseg 비교를 게이트에 동승할지는 1단계 측정 결과(불일치 규모)를 보고 결정한다 —
합성 fallback 문서·2-round 안정성과의 상호작용 주의.

### 트러블슈팅 사전 검색 결과

- `hwpx_lineseg_reflow_trap.md` — HWPX lineseg는 최소 정보만 가질 수 있고, reflow
  재계산은 신뢰 불가 (TAC 표 높이 혼입 전례). **원본 값 보존이 원칙**, 재계산 금지
- `task178_second_attempt_hancom_rejection.md` — 자기 roundtrip 통과 ≠ 한컴 호환,
  최종 판정은 한컴에디터(작업지시자)
- 2-round 게이트의 존재 이유가 lineseg 다회 저장 드리프트 검출 (매뉴얼 7절)

### 열린 PR 충돌 확인

PR #1366(idaeho)이 `section.rs`의 lineseg 인접 영역(`replace_first_linesegs` 호출
순서, `table.rs` lineseg IR 방출)을 수정하나, **수정 요청 + 범위 축소 안내 후 응답
대기 상태**이며 해당 hunk는 드랍 권장 대상이다 (2026-06-12 코멘트). 본 타스크가
선행하고, 충돌 발생 시 컨트리뷰터에게 재안내한다.

## 3. 목표와 완료 조건

1. 셀·글상자 합성 lineseg 제거 확인 — #1379 해소분 전수 확정 (잔존 시 본 타스크 정정)

   렌더 페이지 수 일치
5. 대표 샘플 SVG 비교에서 lineseg 기인 시프트 해소 (#1387/#1388 기지 차이 제외)
6. baseline 전수(54건)·기존 테스트 무회귀 + 2-round 안정성 유지
