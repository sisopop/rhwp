## 1. 배경

#1379 4단계 SVG 비교(ta-pic-001-r)에서 검출. 원본 표의 캡션 subList(문단
"&lt;그림 1&gt; 의정활동 모니터링 시스템 예시", autoNum 컨트롤 포함)가 roundtrip 후
소실된다. #1388 해소 후 ta-pic-001-r의 원본 vs RT 잔존 차이는 캡션 18자(단일 행)
**하나뿐**이므로, 본 타스크 해소 시 좌표 완전 일치가 기대치다.

## 2. 현황 분석 (2026-06-12 — 코드 확인 완료)

  영역 무관 — 충돌 없음.

### 인접 항목 (범위 검토)

- `ShapeComponent.caption`(그림/도형 캡션, shape.rs:634)은 표와 별개 경로 —
  1단계 측정에서 전수 분포만 확인하고, 표 외 캡션은 범위 밖이면 이슈 분리 제안.
- `Caption.vert_align` 은 파서가 읽지 않는다 — OWPML 속성 존재 여부를 1단계에서
  확인하고, 파서 보강 필요 시 범위 포함 여부를 승인받는다.

## 3. 목표와 완료 조건

1. HWPX 저장 시 표 캡션(`hp:caption` 속성 + subList 문단)이 원본대로 방출된다.
2. `diff_documents`의 Table 재귀가 caption(존재 여부 + 속성 + 문단 재귀: char_shapes/
   controls/linesegs 3종 동승)을 비교하여 게이트 사각이 해소된다.
3. `samples/hwpx` 전수 roundtrip에서 caption diff 0, 기존 xfail 외 신규 실패 0.
4. ta-pic-001-r 원본 vs RT SVG 좌표 완전 일치 (잔존 18좌표 해소).

## 4. 단계 개요 (구현계획서에서 상세화)

3. **게이트 동승**: Table 재귀에 caption 비교 추가 (#1378/#1379/#1380 3종 비교 동승)
