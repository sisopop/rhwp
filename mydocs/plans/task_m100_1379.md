## 2. 현황 분석

### serializer 세 경로의 비대칭

| 경로 | 위치 | 현재 동작 |
|------|------|----------|
| 본문 | `section.rs render_runs()` (359) | **완비** — UTF-16 슬롯 추론 + `render_control_slot()` 디스패치 (Table/Picture/Shape/Equation/Footnote/Field/Form 등 15종) + #1378 run 분할 |
| 표 셀 | `table.rs write_sub_list()` (221) | `render_text_runs()` — 텍스트만 run 분할, **컨트롤 미방출** |
| 글상자 | `shape.rs write_draw_text_paragraph()` (223) | 동일 — 텍스트만, **컨트롤 미방출** |

핵심 방향: 셀·글상자 문단 직렬화를 본문 `render_runs()` 경로로 통합한다.
`render_runs`는 이미 `&mut SerializeContext`를 받고 표/그림 재귀 호출 구조(셀 안 표)도
`render_control_slot → write_table → write_sub_list` 상호 재귀로 자연스럽게 성립한다.

### 확장 범위 — rect/drawText 직렬화 미비 (한컴 열기 거부 원인)

tbox-v-flow-01 원본 vs rt diff (이슈 #1379 코멘트의 표):

| 항목 | IR 보존 여부 | serializer 현황 |
|------|-------------|----------------|
| rect 꼭짓점 `pt0`~`pt3` | **보존** (`RectangleShape.x_coords/y_coords [i32;4]`) | `write_rect`(shape.rs:36) 미방출 — **한컴 거부 유력 1순위** |
| `lineShape` | 보존 (`DrawingObjAttr.border_line`) | 미방출 |
| `fillBrush` | 보존 (`DrawingObjAttr.fill`) | 미방출 |
| `shadow` | 보존 (`DrawingObjAttr.shadow_*`) | 미방출 |
| `instid` | 보존 (`DrawingObjAttr.inst_id`) | `common.instance_id`로 대체 출력 (원본값 무시) |
| offset/orgSz/curSz/flip/rotationInfo/renderingInfo | **조사 필요** (HWPX 파서의 IR 적재 여부) | 미방출 |
| 요소 순서 (원본: lineShape…drawText→pt0-3→sz/pos/outMargin) | — | rt는 sz/pos/outMargin→drawText 순 — 한컴 기대 순서로 정렬 필요 |
| drawText subList `textDirection` | **미보존** (`TextBox`에 필드 없음 — `list_attr` raw u32만) | "HORIZONTAL" 하드코딩 (shape.rs:202) — VERTICALALL 세로쓰기 소실 |
| drawText `name`/`editable`, rect `numberingType` | 조사 필요 | 미방출/하드코딩 |

IR이 보존하지 않는 항목(textDirection 등)은 모델 필드 추가 + HWPX 파서 적재가 선행되어야
한다 — 1단계 조사에서 항목별 보존 여부를 확정하고 범위를 분류한다.

### 게이트 공백

baseline 게이트(`diff_documents`)는 컨트롤 수/내용을 비교하지 않아 subList 컨트롤
소실을 통과시킨다 (의도된 한계, 매뉴얼 1절). 본 타스크에서 **문단별 컨트롤 수·타입
시퀀스 비교**(본문+재귀)로 게이트를 강화하여 소실을 회귀 게이트에 고정하는 것을 검토한다.

### 트러블슈팅 사전 검색 결과

- `task178_second_attempt_hancom_rejection.md` — **rhwp 자기 roundtrip 통과 ≠ 한컴 호환**.
  한컴은 결손 필드에 엄격. 최종 판정은 한컴에디터(작업지시자)
- `picture_save_hancom_compatibility.md` — (HWP5 사례) 한컴 정상 파일과의 요소 단위
