## 1. 배경

### 1.1 정책 결정 (작업지시자)

> "그냥 hwpx 의 경우 저장을 disable 시키는 건 어떤가요? 그리고 사용자에게 모던한 방법으로 알려주면 되요"

이전 검토 (모달 → 차단 모달) 거쳐 **UI 차원 비활성화 + 모던 안내** 로 정책 확정.

### 1.2 위험 (재확인)

#178 두 번째 시도 결과:
- HWPX 출처를 HWPX 로 저장 시 표/이미지/스타일 손실 가능 (#182 직렬화기 미완성)
- HWPX 출처를 HWP 로 저장 시 한컴 거부 (#197 완전 변환기 필요)
- PR #189 머지 후 Ctrl+S 가 같은 파일 직접 덮어쓰기 → **원본 손상 위험 증가**

→ 본 배포 (v0.2.1) 에서는 HWPX 저장 자체를 비활성화하여 사용자 데이터 보호.

## 2. 목표

### 2.1 1차 목표

HWPX 출처 문서의 저장 진입점 (`file:save`, `hwpctl/SaveAs`) 을 비활성화하고, 사용자에게 정중한 안내를 다층적으로 제공.

### 2.2 성공 기준

- HWPX 출처 → 저장 메뉴 / 도구 상자 저장 버튼 / Ctrl+S 모두 비활성 (시각적 + 동작)
- HWPX 출처 → hover 툴팁으로 안내
- HWPX 출처 → 상태 표시줄에 베타 모드 메시지
- HWPX 출처 → 로드 직후 1회 토스트
- HWP 출처 → 모달/툴팁/토스트 없음, 기존 동작 보존 (회귀 0)
- #197 완료 시 본 차단 제거 가능하도록 명확한 위치/이름

## 3. 통합 전략 원칙

1. **UI 차원 차단** — `canExecute` 가드로 진입점 자체 비활성. 모달 X.
2. **다층 안내** — 툴팁 (hover 시) + 상태바 (상시) + 토스트 (로드 직후 1회). 정중한 톤.
3. **기존 메커니즘 재사용** — `canExecute(ctx)`, `#status-bar`, 신규 toast 컴포넌트 (또는 기존 활용)
4. **회귀 우선** — HWP 출처 무영향
5. **트러블슈팅 사전 검색** — #178 정독, 추가 0건
6. **chrome-ext-update-01 (What's New) 분리** — 신규 이슈 #199 로 등록 (v0.2.1 포함)

## 5. 구현 단계 (3단계)

### Stage 1 — `canExecute` 가드 + 다층 안내 UI

- `file:save.canExecute` 갱신 (sourceFormat 직접 호출):
  ```typescript
  canExecute: (ctx) => ctx.hasDocument && services.wasm.getSourceFormat() !== 'hwpx'
  ```
  (또는 services 접근이 canExecute 에 안 되면 ctx 유지 + 시점 결정)
- `hwpctl/SaveAs` 도 동일 가드 (다른 진입점)
- 도구 상자 / 메뉴의 저장 버튼 disabled 시 `title` 속성으로 hover 툴팁
- 상태 표시줄: HWPX 로드 시 `sb-message` 갱신 (기존 메커니즘 활용)
- 신규 토스트 컴포넌트 (`ui/toast.ts`):
  - 우상단 슬라이드 인
  - 자동 페이드 (예: 8초)
  - 사용자 닫기 액션 (× 버튼)
- HWPX 로드 직후 토스트 1회 호출
- 단위 테스트: `canExecute` 분기 + 토스트 컴포넌트 표시/닫기

완료: 단위 테스트 그린, TypeScript 컴파일 통과, vite dev 시각 확인.

### Stage 2 — 빌드 검증

- rhwp-studio 빌드 (`npm run build`)
- rhwp-chrome dist 빌드 (`npm run build`)
- TypeScript 컴파일 0 에러

완료: 빌드 산출물 정상.

### Stage 3 — 작업지시자 수동 검증 + 보고서

검증 케이스:
- HWPX 파일 열기 → 저장 메뉴/버튼/Ctrl+S 비활성 확인
- HWPX 파일 열기 → hover 툴팁 표시
- HWPX 파일 열기 → 상태바 베타 메시지
- HWPX 파일 열기 → 로드 직후 토스트 1회 (자동 페이드 + 닫기 버튼 동작)
- HWP 파일 열기 → 위 모든 안내 없음, 정상 저장

완료: 작업지시자 검증 통과 + 보고서 + merge.
