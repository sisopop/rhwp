## 배경

Task #331 의 시도는 typeset advance 를 `total_height` → `height_for_fit` 으로 바꾸고 layout 측 trail_ls 를 제외하는 단순 fix 였으나, layout 의 **clamp pile 버그**(`paragraph_layout.rs:807-816`)와 **header 영역 측정 다중성**이 latent 상태로 존재했기에 회귀(글자 겹침)를 일으켜 commit `078717f` 로 revert 되었다.

근본 원인은 typeset / layout / vpos correction / header(표·Shape) 측정기가 **서로 다른 advance 모델**로 누적해 ~60px drift 가 발생, 한 쪽만 손대면 반대편의 buffer 가 사라져 latent 버그가 노출된다는 점이다.

## 목표

typeset 과 layout 이 같은 데이터(=`height_for_fit` + LINE_SEG vpos)를 advance 의 단일 source of truth 로 사용하도록 통합. 마지막 단계에서 Task #331 의 원 의도(`pi=26 + 보기 ①②③` 가 page 1 col 1 에 fit, PDF 일치)가 자연 해결된다.

## 범위

### 수정 대상 (예상)

- `src/renderer/typeset.rs` — page advance 누적
- `src/renderer/paragraph_layout.rs` — per-paragraph advance, clamp pile 버그
- `src/renderer/layout.rs` — vpos correction (`vpos_end` trail_ls 처리, 양방향 보정)
- `src/renderer/header.rs` 또는 표 매니저 (`heightmeasurer.rs` 등) — header drift 검증
- 기존 lib 테스트 calibration, golden SVG baseline 일부 갱신

### 제외

- HWP 파서 자체의 LINE_SEG/ParaShape 변경
- TypesetEngine 의 페이지 분배 알고리즘 자체 재설계 (재 pagination 루프 → Plan B, 본 task 범위 아님)
- 폰트 메트릭 DB 변경 (drift 의 원인이지만 라이선스/가용성 문제로 별도 트랙)

## 접근 원칙

> typeset 과 layout 은 **같은 advance 모델 = `height_for_fit` (trail_ls 제외) + LINE_SEG vpos** 를 사용한다. vpos correction 은 양방향이지만 collapse 회귀를 막는 가드 조건을 명시한다. layout 이 overflow 하면 그리지 않고 typeset 에 신호를 돌려 보내거나(또는 단순 stop drawing) **clamp pile 은 절대 만들지 않는다**.

## 단계 (5단계)

이슈 본문의 sub-task 를 그대로 5 단계로 매핑한다. 각 단계는 회귀 테스트(샘플별 export-svg + golden + lib) 를 포함한다.

### Stage 1 — typeset advance 를 `height_for_fit` 기반으로 변경

- `typeset.rs` 의 `cur_h += fmt.total_height` → `+= fmt.height_for_fit`
- 영향: 페이지 분배 시 trailing_ls 만큼 더 이상 over-advance 하지 않음
- 회귀 검증:
  - 단독 적용 시 layout drift 가 더 커질 가능성 → Stage 1 만 적용한 상태에서 21_언어 page 1 col 1 의 pi=10/26 상태와 글자 겹침 여부 측정
  - golden 6 개 baseline 갱신 또는 유지 결정

### Stage 2 — layout per-paragraph advance 를 `height_for_fit` 와 정합

- `paragraph_layout.rs` 의 마지막 visible 줄 trail_ls 를 advance 누적에서 제외
- 영향: layout 측 누적이 typeset 과 동일 모델
- 회귀 검증: vpos correction 이 다시 ls 를 더하는 케이스(`layout.rs:1392`)와 충돌 여부 → Stage 3 와 묶어 한꺼번에 검증해야 함

### Stage 3 — vpos correction `vpos_end` trail_ls 제외 + 양방향 보정 + 가드

- `layout.rs:1392` 의 `end_y >= y_offset - 1.0` 조건 변경 시 발생한 collapse 회귀(`pi=10/pi=11 동일 y`)를 막을 가드 추가
- 후보: 인접 문단 간 최소 advance 보장(`y_offset_next > y_offset_prev + min_advance`), 또는 vpos 가 누락/0 인 라인은 보정 건너뛰기
- 회귀 검증:
  - col 1 collapse 케이스 직접 재현 (이전 시도에서 회귀했던 케이스)
  - HWP 호환 케이스 (form-002 등) 광범위 회귀

### Stage 4 — clamp pile 버그 수정

- `paragraph_layout.rs:807-816` 의 `text_y = col_bottom - lh` + clamp 가 piling 을 만드는 부분
- 정책 변경: overflow 라인은 **그리지 않음**(skip) + typeset 에 overflow signal (가능하면)
- 콘텐츠 손실 방지: typeset 이 overflow signal 받지 못하더라도 Stages 1~3 가 정합되면 발생 빈도 자체가 사라져야 함을 검증
- 회귀 검증:
  - Task #321 v5/v6 가 검증한 multi-table/tac-case 시나리오
  - 글자 겹침 0 (모든 샘플)

### Stage 5 — header (표/Shape) 영역 측정 통합

- 표 매니저(`heightmeasurer.rs` 또는 동등 모듈) 와 typeset 의 header 추정 동기화
- 영향: 시작점 ~17px drift 제거 → Stages 1~4 의 효과가 페이지 첫 문단부터 정확히 적용
- 회귀 검증:
  - 21_언어 page 1 col 1 의 pi=26 + 보기 ①②③ fit (PDF 일치) ✅ ← Task #331 원 의도
  - pi=10 partial 글자 겹침 없음 ✅
  - 표가 있는 모든 샘플 회귀
