# Task #354 수행계획서 — rhwp-firefox/build.mjs CodeQL 경고 해소

## 이슈

[#354](https://github.com/edwardkim/rhwp/issues/354) — Code Scanning Alert #17 (CodeQL `js/shell-command-injection-from-environment`) 해소.

## 배경

`rhwp-firefox/build.mjs:18` 에서 `execSync(cmd, ...)` 가 shell 문자열 보간으로 명령을 받음. `__dirname` 기반 절대 경로가 문자열에 끼어들어 CodeQL 이 경고 발생. 빌드 도구 한정이라 실제 보안 위험은 낮으나, **저장소를 공백 포함 경로에 클론한 경우 빌드 실패** 가능 → 실용적 버그.

## 목표

1. CodeQL Alert #17 해소
2. 공백 포함 경로에서도 빌드 정상 동작

## 범위

### 수정 대상

- `rhwp-firefox/build.mjs` (단일 파일, `run()` 함수 + 1 호출부)

### 제외

- `rhwp-chrome/build.mjs` 등 다른 확장 빌드 스크립트 — 본 task 범위 외 (별도 점검 후 후속 task 가능)
- 코드 동작 변경 — 인터페이스만 변경, 빌드 결과물 동일

## 접근 원칙

> `execSync` (shell 보간) → `execFileSync` (인자 배열, `shell: false`) 전환. 셸 인젝션 차단 + 공백 경로 안전 처리.

## 단계 (3단계)

### Stage 1 — 코드 수정

`rhwp-firefox/build.mjs`:

```js
// Before
import { execSync } from 'child_process';
function run(cmd, cwd = __dirname) {
  console.log(`> ${cmd}`);
  execSync(cmd, { stdio: 'inherit', cwd });
}
run(`npx vite build --config ${resolve(__dirname, 'vite.config.ts')}`, studioDir);

// After
import { execFileSync } from 'child_process';
function run(cmd, args, cwd = __dirname) {
  console.log(`> ${cmd} ${args.join(' ')}`);
  execFileSync(cmd, args, { stdio: 'inherit', cwd, shell: false });
}
run('npx', ['vite', 'build', '--config', resolve(__dirname, 'vite.config.ts')], studioDir);
```

### Stage 2 — 빌드 검증

- `cd rhwp-firefox && rm -rf dist && npm run build` 정상 통과
- `dist/manifest.json` version, structure 변경 전과 동일
- `npx web-ext lint --source-dir=rhwp-firefox/dist` errors 0
- 공백 경로 호환성 (선택, 필수 아님): `/tmp/path with space/` 에 임시 클론 후 빌드 시도
