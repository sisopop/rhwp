## 현상

1. **#111**: 체크박스가 있는 표 셀에 커서 진입이 되지 않는다.
2. **#112**: 체크박스를 클릭해도 체크 상태가 토글되지 않는다.

## 원인 분석

### #111 — 셀 커서 진입 불가

`src/document_core/queries/cursor_rect.rs`의 `collect_runs()`가 `RenderNodeType::TextRun` 노드만 수집한다.
양식 컨트롤은 `RenderNode::FormObject`로 렌더링되어 TextRun이 생성되지 않으므로,
해당 셀의 `cell_runs`가 비어 있어 hitTest가 올바른 커서 위치를 반환하지 못한다.

### #112 — 체크박스 클릭 토글 불가

체크박스 토글 로직은 `rhwp-studio/src/engine/input-handler.ts`의 `handleFormObjectClick()`에 이미 구현되어 있다.
그러나 `getFormObjectAt()`이 폼 컨트롤을 감지하려면 hitTest가 먼저 해당 셀에 커서를 위치시켜야 한다.
#111이 해결되지 않으면 클릭이 폼 컨트롤 핸들러에 도달하지 못한다.

즉 **#111이 #112의 선행 조건**이다.

## 수정 범위

- `src/document_core/queries/cursor_rect.rs` — 양식 컨트롤 셀 hitTest 처리
- `src/document_core/queries/form_query.rs` — `get_form_object_at_native()` 동작 확인
- `rhwp-studio/src/engine/input-handler-mouse.ts` — 클릭 흐름 확인

## 수정 방향

### 1단계: #111 — 셀 커서 진입

`cursor_rect.rs`의 셀 클릭 감지 로직에서 해당 셀에 TextRun이 없더라도,
셀의 bbox 내에 폼 컨트롤이 있으면 커서를 셀 내부로 진입시키도록 수정한다.

### 2단계: #112 — 체크박스 클릭 토글

`input-handler-mouse.ts`의 클릭 흐름에서 `getFormObjectAt()` 호출이
셀 커서 진입과 독립적으로 동작하는지 확인하고, 필요 시 클릭 흐름을 조정한다.
