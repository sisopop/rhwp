# Task M100-1108 수행 계획서

## 1. 작업 개요

GitHub Issue #1108:

```text
HWPX exam_social 문단 시작 문단번호 출력 누락
```

`samples/hwpx/exam_social.hwpx`를 rhwp-studio에서 렌더링하면 문단 시작의 문단번호가 출력되지
않는다. 같은 문서의 HWP 정답지인 `samples/exam_social.hwp`는 문단 시작에서 문단번호가 정상
출력된다.

작업지시자 추가 관찰:

```text
1. HWP 쪽은 rhwp-studio에서 문단 시작 문단번호가 출력된다.
2. 1페이지 문단 시작 지점도 HWP와 HWPX가 다르다.
3. 정답은 HWP이다.
4. HWPX에서 문단 정보 일부가 미처리되는 것으로 추정한다.
```

따라서 이번 이슈는 단순 glyph 출력 문제가 아니라, HWPX 문단 정보, 번호 정보, 들여쓰기/여백,
paragraph start position 관련 IR 매핑 누락을 함께 조사한다.

## 2. 대상 파일

```text
source HWPX:
  samples/hwpx/exam_social.hwpx

oracle HWP:
  samples/exam_social.hwp
```

## 3. 목표

```text
1. HWP와 HWPX의 문단번호 출력 차이를 페이지/문단 단위로 특정한다.
2. 같은 위치의 HWP/HWPX paragraph IR을 비교한다.
3. HWPX에서 누락되거나 잘못 매핑된 문단 속성을 찾는다.
   - numbering/bullet/auto number relation
   - para shape reference
   - left margin / indent / tab / line segment
   - 문단 시작 x/y 위치에 영향을 주는 필드
4. HWPX 렌더링이 HWP 정답지와 같은 문단번호 및 문단 시작 위치를 갖도록 수정한다.
5. 회귀 테스트와 시각 판정용 SVG/Web 산출물을 만든다.
```

## 4. 범위

이번 이슈의 범위:

```text
HWPX -> Document IR -> paragraph numbering/layout renderer
```

이번 이슈의 비범위:

```text
HWPX -> HWP 저장
HWP5 저장 record contract
문단번호 편집 UI
```

HWP 저장 경로의 문제가 발견되면 별도 이슈로 분리한다.

## 5. Stage 1 계획 — 문단번호/문단 시작 위치 차이 trace

출력 위치:

```text
output/poc/hwpx/task1108/stage1_para_number_trace/
```

작업:

```text
1. HWP 정답지와 HWPX의 1페이지 SVG를 같은 조건으로 export한다.
2. 문단번호가 출력되는 HWP 문단과 누락되는 HWPX 문단을 paragraph index 기준으로 찾는다.
3. 해당 문단들의 IR을 비교한다.
   - text
   - para_shape_id
   - char_shape_id
   - numbering/bullet 관련 필드
   - lineSeg/lineSegArray
   - margin/indent/tab
   - controls before paragraph text
4. HWPX XML 원문에서 해당 문단의 번호/문단모양 속성을 확인한다.
5. 원인 후보를 3개 이하로 압축한다.
```

산출물:

```text
hwp_page1.svg
hwpx_page1.svg
paragraph_number_diff.md
paragraph_ir_diff.md
hwpx_source_para_trace.md
```

## 6. Stage 2 계획 — 최소 수정 후보

Stage 1에서 확인한 누락 필드를 기준으로 HWPX parser/IR/renderer 중 가장 좁은 위치에 수정한다.

원칙:

```text
1. HWP 정답지를 기준으로 하되, HWPX XML 원문에 존재하는 의미 정보를 우선 사용한다.
2. HWP 렌더링 경로는 변경하지 않는다.
3. HWPX 문단번호 출력과 문단 시작 위치를 함께 검증한다.
4. #1100에서 해결한 바탕쪽/머리말/쪽번호 동작은 회귀시키지 않는다.
```
