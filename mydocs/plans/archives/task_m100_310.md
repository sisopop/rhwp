# Task #310 수행계획서: LINE_SEG vpos 노출 + 검증 도구 추가

상위 Epic: #309
브랜치: `task306` (사용자 지시로 유지)

## 목적

LINE_SEG `vpos` 데이터를 페이지네이션 검증 도구에 노출하여, 4개 샘플(21_언어/exam_math/exam_kor/exam_eng)의 HWP 의도 vs 현 엔진 결과 차이를 데이터로 확인한다. **동작 변경 없음** — 회귀 위험 zero. Epic #309 의 2단계 설계 근거 확보가 본 타스크의 유일한 목표.

## 배경

Task #306 분석 결과, HWP 파일은 LINE_SEG 마다 `vpos`(세로 위치)를 저장하며 `vpos=0` 리셋이 새 열/쪽 시작을 표시한다. 우리 엔진은 파라그래프 단위 자체 높이로 배치하여 vpos와 어긋남 누적. 21_언어 +4쪽 과잉 발생.

현황 코드:
- `src/document_core/queries/rendering.rs:1300` — `dump_page_items` (pi/높이/텍스트만 출력, vpos 미포함)
- `src/renderer/svg.rs:1666` — `render_debug_overlay` (문단/표 경계만 표시, vpos 정보 없음)
- `src/renderer/pagination/state.rs:32` — `page_vpos_base` 이미 존재, vpos 데이터 자체는 IR에 보존되어 있음

## 작업 범위

### 1. `dump-pages` 출력에 vpos 정보 추가

**대상**: `src/document_core/queries/rendering.rs::dump_page_items`

각 `FullParagraph`/`PartialParagraph` 항목에 첫 LINE_SEG의 `vpos`와 마지막 `vpos`를 함께 출력. `vpos=0` 리셋 줄이 포함된 경우 마커(`[vpos-reset]`) 표시.

출력 예 (현행):
```
FullParagraph  pi=44  h=37.3 (sb=16.0 lines=21.3 sa=0.0)  "..."
```
출력 예 (개선):
```
FullParagraph  pi=44  h=37.3 (sb=16.0 lines=21.3 sa=0.0)  vpos=75116..76932  "..."
FullParagraph  pi=50  h=21.3 (sb=0.0 lines=21.3 sa=0.0)  vpos=89644 [vpos-reset@line5]  "..."
```

`PartialParagraph`도 해당 줄 범위의 vpos 출력. `Table`/`Shape`는 컨테이너 문단의 vpos만.

### 2. `--debug-overlay`에 vpos=0 리셋 위치 표시

**대상**: `src/renderer/svg.rs::render_debug_overlay`

LINE_SEG에서 `vpos=0` 인 줄(첫 줄 제외)을 가진 문단을 찾아, 해당 줄 시작 y좌표에 노란 가로 점선 + `vpos-reset s{section}:pi={pi}:line={n}` 라벨을 그린다. 기존 문단/표 경계 표시는 유지.

### 3. 4개 샘플 vpos 패턴 분석 보고서 작성

**산출물**: `mydocs/tech/line_seg_vpos_analysis.md`

각 샘플에 대해:
- 총 vpos=0 리셋 개수
- HWP 의도 열/쪽 경계 위치 목록 (vpos=0 리셋 지점, 섹션:pi:line)
- 현 엔진이 끊은 페이지/열 경계 위치
- 일치/불일치 패턴 요약

특히:
- **21_언어**: 어긋남이 어디서 시작되어 어떻게 누적되는지
- **exam_math**: 일치 패턴 (왜 회귀 없이 잘 맞는지)
- **exam_kor / exam_eng**: 중간 케이스
