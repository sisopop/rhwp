## 1. 문제 / 근본 원인

페이지네이터(typeset)의 cut/추정 높이가 렌더러(layout) 실측 height 보다 작아, '여유 있음'
판정 후 렌더 시 본문 하단을 초과(LAYOUT_OVERFLOW). 현재 16건(섹션0 12, 섹션1 4).

### 1-1. 진단 (page 27 = SIR-002, used 875.7px vs 렌더러 960.4px, ~85px 과소)

| 항목 | 타입 | 비고 |
|------|------|------|
| pi=238 | PartialTable(cont) | 표 측정 |
| pi=239 | FullParagraph | `fmt_total=35.2` vs `vpos_h=17.3`, diff **+17.9**(sb 4.0 + trailing_ls 13.9) |
| pi=240 | Table(TAC) | 표 측정 |
| pi=241 | FullParagraph | diff +17.9 |
| pi=242 | PartialTable(행0) | 표 측정 — 본문 바닥 19.2px 초과 |

- 문단은 페이지네이터가 +17.9px **과다**계상(단단 trailing_ls 누적, #359 정책).
- 그럼에도 합계가 ~85px **과소** → **표(pi=238/240/242) 측정이 과소**가 주범.
- `RHWP_TYPESET_DRIFT`는 문단 전용 → **표 드리프트 계측 도구 부재**(Stage 1에서 추가).

## 2. 목표

페이지네이터 표/문단 측정 = 렌더러 height 가 되도록 단계적 정합 → 본문 하단
LAYOUT_OVERFLOW 최소화(page-larger 제외 이상적 0). 사용자 보고 28·35·49쪽(및 WASM 대응)
해소.

## 3. 범위

### 포함
- 표(PartialTable/Table) cut/advance 높이 ↔ 렌더러 `resolve_row_heights`/MeasuredTable 정합.
- 단단 문단 누적(trailing_ls/sb/vpos) 페이지네이터↔렌더러 정합.
- 정합 도구(표 드리프트 계측) 추가.

### 범위 외
- 본문보다 큰 단일 항목(page-larger: pi=323, pi=567) 내부 분할 — 별도 이슈.
- HWP3 전용 경로(CLAUDE.md 규칙).

## 4. 성공 기준

## 5. 단계 (4~6단계, 진단 우선)

1. **Stage 1 진단 도구 + 드리프트 분해**: 표 측정 계측(`RHWP_TABLE_DRIFT`: 페이지네이터
   행/표 advance 높이 vs 렌더러 row height per 표). page 27 의 ~85px 과소를 항목·행 단위로
   분해해 과소 발생 함수 특정. (코드: 진단 출력 추가만)

3. **Stage 3 표 잔여축 + 문단 누적 정합**: 남은 표 축 + 단단 trailing_ls/sb 누적 정합.
