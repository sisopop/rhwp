## 개정 배경

단계1 (`task_m100_146_stage1.md`) 에서 `mutool draw -F stext` 로 PDF 문자 좌표를 직접 추출해 SVG 좌표와 정밀 비교한 결과, 원본 수행계획서의 버그 후보 ②(Justify 미반영), ③(Hanging indent 어긋남)은 **실제 버그가 아님**이 확인되었다.

### 재확인된 좌표 일치 (144dpi → pt 환산, SVG 96dpi → pt 환산)

| 항목 | PDF 좌표(pt) | SVG 좌표(pt 환산) | 차이 | 판정 |
|------|-------------|------------------|------|------|
| 본문 line1 "범" 끝점 | 538.15 | 538.27 | 0.12 pt | **Justify 정상** |
| 본문 line2 "조" 시작점 | 85.99 | 86.03 | 0.04 pt | **Hanging indent 정상** |
| 제목 "국" 시작점 | 79.04 | 70.80 | **-8.24 pt** | **□ 글자 폭 측정 오류** |

### 실제 버그 진단

`src/renderer/layout/text_measurement.rs:845-858` `is_fullwidth_symbol()` 함수의 범위 리스트에서 **U+25A0-U+25FF (Geometric Shapes 블록, □/■/▲/◆ 등 포함) 누락**.

결과:
- `is_cjk_char('□')` = false
- `is_fullwidth_symbol('□')` = false
- 따라서 `base_w = font_size * 0.5` (반각 처리) → 21.33px 대신 10.67px
- PDF(한컴 HY헤드라인M 폰트 메트릭)는 15.95pt=21.27px 로 측정
- **제목 줄 전체가 좌측으로 ≈11px 밀려 보이고, □ 다음 공백이 사라진 것처럼 보이는 착시 현상 유발**

Geometric Shapes는 HWP 문서에서 **섹션 머리 기호**(□ 1. / □ 가. / ○ ㅇ 등)로 매우 자주 쓰이므로 범위 추가는 타당하다.

## 목표 (축소)

- **유일한 목표**: `is_fullwidth_symbol` 에 Geometric Shapes 블록(U+25A0-U+25FF) 범위 추가
- 인접 관련 블록(화살표 U+2190-U+21FF, 분수 U+2150-U+215F 등)은 **이번 범위에서 제외** (실사례 확인 없음 / 오남용 우려). 본 타스크에서는 Geometric Shapes 한정.

**범위 (IN)**
- `is_fullwidth_symbol` 범위 1줄 추가
- 단위 테스트: □(U+25A1), ■(U+25A0), ▲(U+25B2), ◆(U+25C6), ○(U+25CB) 폭이 font_size 로 측정됨을 확인
- 재현 검증: `samples/text-align.hwp` 제목의 "국" 시작 x 가 PDF 기준 ±1pt 이내

**범위 (OUT) — 이유 재정리**
- Justify 반영 (원본 버그 후보 ②): 이미 정상 동작 확인
- Hanging indent 보정 (원본 버그 후보 ③): 이미 정상 동작 확인
- 자간 × 공백 상호작용 (원본 버그 후보 ①): 실제 원인이 아님
- 한컴·한양 proprietary 폰트 번들/치환: 별도 전략 문서 범위

## 단계 축약 (2단계)

단계1 (샘플 편입·파이프라인 고정)은 완료됨.

- **단계2 (신규 핵심)**: `is_fullwidth_symbol` 범위 추가 + 단위 테스트 + SVG 재생성 대조

## 검증 방법

1. `cargo test --lib` 전체 통과
2. `is_fullwidth_symbol` 신규 유닛 테스트 통과
3. `text-align.hwp` 재출력 후 제목 □→국 간격이 PDF 대비 ±1px (반올림 포함) 로 수렴
4. `cargo test --test svg_snapshot` 실패 샘플 없음 또는 변경된 샘플 전부 "PDF 기준으로 가까워짐" 증빙
5. `cargo clippy -- -D warnings` 통과
