# Task #403 수행 계획서

**제목**: AI 워크로드를 위한 FFI 인터페이스 개발

**마일스톤**: M100 (v1.0.0)

**Issue**: https://github.com/edwardkim/rhwp/issues/403

**브랜치**: `local/ffi-csharp` (`local/devel`에서 분기)

---

## 1. 배경

`export-text`, `export-markdown` 개발 이후 console 앱으로 동작하는 방식으로 진행하였으나 프로세스를 통한 작업으로 인한 외부 실행 문제로 FFI를 통합적 개발의 필요성이 요구됨.

## 2. 구현 방안

`wasm`과 다른 워크로드이므로 기존 `lib.rs`는 유지하고 별도의 `bindings`를 구성하여 각 언어별 또는 사용자별 독립된 ffi 인터페이스를 유지하고 구현한다.
각 언어별 구현은 `bindings` 이하에 각 언어별 폴더에 생성한다.

## 3. 개발 사항

`bindings` 폴더에 `lib.rs`는 AI 워크로드 또는 서버측 워크로드에 적합한 export 목록을 포함하며 각 언어별 폴더에 Native Binding을 구현한다.
현재 Native Binding 구현은 csharp만 구현하며 기타 필요에 의해 추가될 수 있다.
binding 기능은 `export-text`, `export-markdown`에 한정한다.
추후 계속 추가될 수 있다.

## 4. 검증

ffi 연동 후 각 언어별 앱에서 `export-text`, `export-markdown`을 동작할 경우 파일 자체는 입력 받은 폴더 위치에 생성되며 추출한 파일 목록이 반환되어야 한다.


## 5. 위험도

- **중**: 네이티브 인터페이스이므로 메모리 할당과 해제에 신경써야하며 호출시 반드시 예외처리와 메모리 해제를 고려해야 한다.
