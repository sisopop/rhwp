## 현상

`form-002.hwpx`를 SVG로 내보내면 양식 컨트롤(체크박스 등)이 렌더링되지 않는다.

## 원인

HWPX 파서(`src/parser/hwpx/section.rs`)의 `<hp:run>` 파싱 분기에 양식 컨트롤 요소 처리가 없다.
`_ => {}` 분기로 무시된다.

HWP 바이너리 파서에는 `FormObject` / `FormType` 모델과 렌더러가 완비되어 있으므로,
HWPX → IR(`Control::Form`) 변환만 추가하면 기존 렌더러를 그대로 재사용할 수 있다.

## 수정 범위

`src/parser/hwpx/section.rs` 단일 파일

## HWPX XML 구조

```xml
<hp:run charPrIDRef="12">
  <hp:checkBtn caption="원천기술형" value="UNCHECKED"
               name="CheckBox1" foreColor="#000000" backColor="#FFFFFF"
               enabled="1" tabOrder="181" ...>
    <hp:sz width="2268" height="1134"/>
    <hp:formCharPr charPrIDRef="0" .../>
  </hp:checkBtn>
</hp:run>
```

## HWP ↔ HWPX 매핑

| FormType | HWP 코드 | HWPX 태그 | 속성 타입 |
|----------|----------|-----------|-----------|
| PushButton | `tbp+` | `<hp:btn>` | AbstractButtonObjectType |
| CheckBox | `tbc+` | `<hp:checkBtn>` | AbstractButtonObjectType |
| RadioButton | `tbr+` | `<hp:radioBtn>` | AbstractButtonObjectType |
| ComboBox | `boc+` | `<hp:comboBox>` | ComboBoxType + `<listItem>` 자식 |
| Edit | `tde+` | `<hp:edit>` | EditType + `<text>` 자식 |

## 수정 방향

`<hp:run>` Start 이벤트 분기에 5개 태그 핸들러 추가:

1. 태그명으로 `FormType` 결정
2. 속성에서 `name`, `caption`, `foreColor`, `backColor`, `enabled`, `value` 파싱
3. `<hp:sz>` 자식에서 `width`, `height` 읽기
4. ComboBox: `<listItem>` 자식에서 항목 텍스트 수집
5. Edit: `<text>` 자식 텍스트 읽기
6. `Control::Form(Box::new(form))`으로 변환하여 `para.controls`에 추가
7. `text_parts`에 `"\u{0002}"` 삽입 (컨트롤 위치 마커)
