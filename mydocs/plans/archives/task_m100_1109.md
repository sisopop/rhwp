## 1. 증상

GitHub Actions CI (`.github/workflows/ci.yml` 의 `build-and-test` job) 의 `Run tests` step:

```
##[error]No space left on device : '/home/runner/actions-runner/cached/2.334.0/_diag/pages/...log'
```

- 실패 run: `26359599960` (2026-05-24 11:13)
- 영향 PR: #1085 (Task #1042)

## 2. 본질 분석 (사전 조사 결과)

### 2.1 실패 영역

- `cargo test --verbose` step — 28 examples (`examples/*.rs`) 모두 컴파일 + 모든 integration test binary 생성
- `examples/` 누적 영역:
  - 본 프로젝트 누적 진단/inspection 도구 28 개 (build_issue_*/diag_*/dump_*/inspect_* + repro_*)
  - 본인 추가 9 개 (Task #1058 + #1061 + #1064 + #1067)

### 2.2 누적 영역

- `actions/cache@v4` 의 `target/` cache 점진 누적 (Cargo.lock 변경 시 부분 추가)
- ubuntu-latest runner 디스크 한도 ~14GB
- 누적 영역: cache + 빌드 산출물 + apt-get install (libfontconfig + libfreetype) + diag log

### 2.3 WASM 무관

- `wasm-build` job 은 `workflow_dispatch || tag` 만 실행
- 본 PR 빌드는 `build-and-test` job 만 동작
- 작업지시자 가설 ("WASM 이미지 누적") 은 본 실패 영역 아님

### 2.4 ubuntu-latest 의 사전 설치 toolchain (제거 가능 ~10GB)

GitHub-hosted runner `ubuntu-latest` 의 사전 설치 영역:
- `/usr/share/dotnet` — .NET SDK ~2GB
- `/usr/local/lib/android` — Android SDK ~14GB (큰 영역)
- `/opt/ghc` — Haskell GHC ~5GB
- `/opt/hostedtoolcache/CodeQL` — CodeQL ~3GB

본 프로젝트는 Rust 전용 — 위 모두 미사용.

## 3. 해결 방향 (작업지시자 결정 (c))

`build-and-test` job 의 checkout step 직후에 "Free disk space" step 추가:

```yaml
- name: Free disk space
  run: |
    sudo apt-get clean
    sudo rm -rf /usr/share/dotnet
    sudo rm -rf /usr/local/lib/android
    sudo rm -rf /opt/ghc
    sudo rm -rf /opt/hostedtoolcache/CodeQL
    sudo docker image prune --all --force || true
    df -h
```

기대 효과:
- 사전 설치 toolchain ~24GB 회수
- runner 가용 디스크 한도 대폭 확보
- 본 영역 외 build/test/clippy 단계는 변경 없음

## 4. 적용 범위

- `build-and-test` job — 필수 (본 실패 영역)
- `wasm-build` job — 동일 패턴 (선제적 적용, tag/manual 시 동일 영역)
