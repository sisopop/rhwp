# Task #1012 수행 계획서 — wrap 옵션 paragraph 의 z-order 정합

이슈: [#1012](https://github.com/edwardkim/rhwp/issues/1012)
배경: 작업지시자 [PR #1011](https://github.com/edwardkim/rhwp/pull/1011) 진행 중 발견

## 1. 문제 진단

`samples/test-image.hwp` page 1: 4 종 wrap 옵션 (자리차지/글앞으로/어울림/글뒤로) Picture 가 단일 paragraph 의 controls 로 포함됨. paragraph text = "자리차지   글앞으로   어울림   글뒤로". 한컴 viewer 는 각 그림 아래에 해당 라벨 텍스트를 표시.

### 1-1. 한컴 viewer (정답)

| 그림 | wrap 모드 | z-order 효과 | 라벨 텍스트 |
|------|----------|--------------|------------|
| Picture[2] | 자리차지 (위아래, TopAndBottom) | 텍스트 위/아래 — 겹침 없음 | 표시 ✓ |
| Picture[5] | 글앞으로 (InFrontOfText) | image 가 text 위 (overlay) | 텍스트는 image 뒤 (시각 일부 가림) |
| Picture[3] | 어울림 (Square wrap) | 텍스트가 image 주변 wrap — 겹침 없음 | 표시 ✓ |
| Picture[4] | 글뒤로 (BehindText) | image 가 text 뒤 | **텍스트 우선 표시 (image 위) ✓** |

### 1-2. rhwp 현재 (결함)

`./target/release/rhwp export-svg samples/test-image.hwp -p 0` 결과 SVG 구조:
- 7~26번 line: 20 개 `<text>` (라벨 글자 + tac marker)
- 27~30번 line: 4 개 `<image>` (그림 4종)

→ **모든 image 가 모든 text 뒤에 그려짐 (z-order 무시)**. SVG 는 후속 요소가 위에 그려지므로 모든 라벨이 image 에 의해 가려짐.

특히 **글뒤로 (Picture[4])** 는 text 가 image 위에 표시되어야 하는 wrap 모드인데, 현재 render 는 image 가 text 위에 (잘못된 방향).

## 2. Root cause 가설

`src/renderer/layout.rs` 의 paragraph 렌더링 pass 에서:
- text run nodes 가 먼저 tree 에 push
- image/shape control nodes 가 그 뒤에 push

paragraph 내 controls 의 wrap 모드별 z-order 분기가 누락. HWP spec 의 z-order:
- **BehindText**: text 가 image 위 (image 먼저 그림, 텍스트 나중)
- **InFrontOfText**: image 가 text 위 (image 나중 그림)
- **TopAndBottom / Square / Tight**: 본문 흐름 외부 — z-order 무관

## 3. 변경 범위 (예상)

| 영역 | 변경 |
|------|------|
| `src/renderer/layout.rs` 또는 `paragraph_layout.rs` | wrap 모드별 image/shape 의 render 순서 분기 |
| (선택) `PageRenderTree` 빌드 구조 | 같은 paragraph 의 BehindText image → text 앞쪽 insert / InFrontOfText image → text 뒤쪽 push |

목표: paragraph 내 controls 중 BehindText 인 Picture/Shape 만 텍스트 nodes 보다 먼저 tree 에 push (또는 별도 z-layer 그룹). 나머지는 기존 순서 유지.
