# Task M100-1090 수행 계획서

## 1. 작업 개요

GitHub Issue #1090:

```text
[hwpx2hwp] el-school-001.hwpx 표 셀 배경 이미지 HWP 저장 지원
```

이 이슈는 #1064의 후속/서브 이슈다. 범위는 HWPX 문서에서 표 > 셀에 지정된 배경
그림을 HWP로 저장할 때 HWP5 셀/BorderFill 계약에 맞게 보존하는 것이다.

## 2. 대상 샘플

```text
samples/hwpx/el-school-001.hwpx
samples/el-school-001.hwp
saved/111el-school-001.hwp
```

각 파일의 의미:

```text
samples/hwpx/el-school-001.hwpx
  - HWPX 입력 원본

samples/el-school-001.hwp
  - 한컴 변환/정답 HWP

saved/111el-school-001.hwp
  - 현재 rhwp-studio에서 저장한 재현 HWP
```

## 3. 현재 관찰

작업지시자 확인 결과:

```text
한컴 원본/정답:
  셀 테두리/배경 > 그림 > 채우기 유형 = 크기에 맞추어

현재 rhwp 저장 결과:
  셀 테두리/배경 > 그림 > 채우기 유형 = 바둑판식으로-세로/왼쪽
```

따라서 이번 이슈의 첫 번째 핵심은 단순히 배경 이미지 유무가 아니라, HWPX의 셀 배경
이미지를 HWP5로 저장할 때 그림 채우기 유형과 정렬/타일링 속성을 정확히 매핑하는 것이다.

## 4. 가설

현재 저장기는 표 셀 배경 이미지 자체는 일부 인식하지만, HWP5 BorderFill 또는 셀 배경
그림 payload 안의 다음 값을 한컴 정답지와 다르게 기록하고 있을 가능성이 높다.

```text
1. image fill mode
   - stretch / fit-to-size / tile 구분

2. tile direction
   - none / horizontal / vertical / both

3. alignment
   - left/top/center/right/bottom 등 기준점

4. HWPX XML의 cell background image 속성에서 HWP5 BorderFill picture fill 속성으로 가는 매핑
```

## 5. 진행 전략

이번 작업은 기존 hwpx2hwp 이슈와 동일하게 정답지 oracle 비교 방식으로 진행한다.

```text
1. HWPX 원본, 한컴 정답 HWP, rhwp 저장 HWP를 모두 비교한다.
2. 문제가 되는 셀과 해당 BorderFill/배경 그림 record를 특정한다.
3. 정답지 HWP와 rhwp 저장본의 payload 차이를 decode한다.
4. "크기에 맞추어"를 만드는 최소 HWP5 contract를 확정한다.
5. 구현 후 한컴 에디터와 rhwp-studio 양쪽에서 재검증한다.
```

```text
  - borderfill_picture_diff.md
  - hwpx_cell_background_source.md

```

확인 항목:

```text

2. samples/el-school-001.hwp 정답지에서 해당 셀이 참조하는 BorderFill 번호
3. saved/111el-school-001.hwp에서 같은 셀이 참조하는 BorderFill 번호
4. 정답지와 저장본의 BorderFill picture fill payload 차이
5. 한컴 UI의 "크기에 맞추어"와 payload bit/field의 대응 후보
```
