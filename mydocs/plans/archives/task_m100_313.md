# Task #313 수행계획서: TypesetEngine을 main pagination으로 전환

상위 Epic: #309
선행: #310 (도구), #311 (vpos-reset 부정), #312 (TypesetEngine 발견)
브랜치: `task313` (task312에서 분기)

## 목적

`DocumentCore::paginate`에서 `Paginator` → `TypesetEngine` 으로 진입점 전환하여 4샘플 모두 PDF에 가까운 페이지네이션 달성. **21_언어 19쪽 → 15쪽 (PDF 일치)** 가 핵심 목표.

## 배경 / 데이터 근거

#312 검증 결과:

| 샘플 | Paginator | TypesetEngine | PDF |
|------|-----------|---------------|-----|
| 21_언어 | 19 | **15** | 15 ✓ |
| exam_math | 20 | 20 | 20 ✓ |
| exam_kor | 25 | 24 | (미보유) |
| exam_eng | 11 | 9 | (미보유) |

TypesetEngine은 이미 코드베이스에 존재하며 (`src/renderer/typeset.rs`), debug 빌드에서 검증 모드로 동작 중. 결과 페이지 수가 더 정확.

## 작업 범위

### 1. TypesetEngine 출력 호환성 검토

`PaginationResult` 출력 필드 비교 (Paginator vs TypesetEngine):
- `pages: Vec<PageContent>` ✓ 양쪽 모두 채움
- `wrap_around_paras` — typeset.rs:418 에서 `Vec::new()`로 비움 (Paginator는 채움)
- `hidden_empty_paras` — typeset.rs:418 에서 `HashSet::new()`로 비움
- 페이지별 `active_header/footer`, `page_number_pos`, `page_hide`, `footnotes` — typeset이 채우는지 확인 필요

미채움 필드는 SVG 렌더링/페이지 표시에 문제 야기 가능 → 식별 + 보완 필요.

### 2. 단순 진입점 전환 + SVG 출력 검증

`src/document_core/queries/rendering.rs::paginate()` 에서:
```rust
// 기존
let result = paginator.paginate_with_measured_opts(...);

// 변경
let result = typesetter.typeset_section(...);
```

4샘플로 SVG 생성 + 시각 검증:
- 21_언어: 15쪽 + 본문/머리말/꼬리말/표 정상 표시
- exam_math: 20쪽 + 표 처리 정상
- exam_kor: 24쪽
- exam_eng: 9쪽

### 3. 누락 필드 보완

2단계에서 식별된 미채움 필드를 TypesetEngine 에 추가. 예:
- header_footer collection (이미 typeset.rs:334 에서 수집 중인 듯)
- wrap_around_paras 처리

### 4. 4샘플 + cargo test 회귀 검증

- `cargo test`: 992 passed 유지
- 4샘플 페이지 수 + 시각 회귀 0
- E2E 테스트 (rhwp-studio) 통과

### 5. 부속물 정리 결정

- `--respect-vpos-reset` 실험 플래그 (#311) — 보존 / 제거
- `PaginationOpts.respect_vpos_reset` 필드
- `Paginator` 코드 — fallback 보존 / 제거
- TYPESET_VERIFY 검증 코드 — 제거 (역할 종료)
