# Task M100 #955 수행계획서 — rustfmt 정규화 및 cargo fmt 적용 정책 수립

## 1. 작업 배경

`#949` 작업 중 `cargo fmt`가 기능 변경과 무관한 광범위 diff를 만드는 문제가 확인되었다.
현재 저장소에는 `rust-toolchain.toml`, `rustfmt.toml`, `.rustfmt.toml`이 없고, CI도
`dtolnay/rust-toolchain@stable`을 사용하므로 rustfmt 기준이 시간에 따라 달라질 수 있다.

이 상태에서 기능 브랜치가 `cargo fmt`를 실행하면 다음 문제가 발생한다.

```text
- 기능 변경 diff와 포맷 diff가 섞인다.
- 진단 출력 문자열 wrapping이 실제 로직 변경처럼 보인다.
- 컨트리뷰터별 rustfmt 버전 차이로 PR diff가 커진다.
- CI에 cargo fmt --check를 바로 넣으면 기존 코드가 대량 실패한다.
```

따라서 이번 타스크는 기능 구현이 아니라 저장소의 포맷 거버넌스를 확정하는 작업이다.

## 2. 목표

```text
1. rhwp의 rustfmt 기준을 명시한다.
2. 컨트리뷰터가 언제 cargo fmt를 실행해야 하는지 문서화한다.
3. 기능 변경과 포맷 변경을 같은 커밋에 섞지 않는 정책을 확정한다.
4. 저장소 전체 포맷 정규화를 단일 전용 커밋으로 수행한다.
5. 정규화 이후 CI에서 cargo fmt --check를 적용할 수 있게 한다.
```


## 4. 정책 방향

이번 타스크의 핵심 원칙은 다음과 같다.

```text
1. rustfmt 기준은 저장소가 소유한다.
2. 기능 변경 커밋과 포맷 정규화 커밋은 분리한다.
3. 전체 cargo fmt는 포맷 전용 브랜치/커밋에서만 수행한다.
4. 정규화 이후 신규 PR은 cargo fmt --all -- --check를 통과해야 한다.
5. rustfmt 버전은 CI와 로컬이 같은 기준을 쓰도록 고정 또는 명시한다.
```
