- **작성일**: 2026-04-17

- **긴급도**: 최우선

## 1. Context

현재 rhwp-studio에서 HWPX로 연 문서를 편집·저장하면 **표·이미지·스타일·글꼴이 모두 손실**된다. 원인은 `src/serializer/hwpx/`의 불균형:

- **section.rs**: 텍스트·탭·소프트브레이크는 IR 기반 동적 생성 ✅
- **header.rs**: 13줄짜리 `include_str!("templates/empty_header.xml")` 하드코딩 ❌
- **section.rs**: `secPr/pagePr/grid`는 `empty_section0.xml` 템플릿 치환 ❌
- **미구현**: `<hp:tbl>`, `<hp:pic>`, BinData ZIP 엔트리, charPrIDRef/paraPrIDRef 매핑

그리고 **구조적 함정**: `mod.rs:61-66`에 `sections.len() == 1 && bin_data_content.is_empty()` 분기가 있어 빈 문서 테스트는 템플릿 경로로 통과하지만 실문서는 동적 경로에서 실패한다 — "단편적 성공의 거짓-양성".

수식 개선·폰트 적용 등 배포 대기 기능이 누적됐으나 이 문제가 해결되지 않으면 사용자 파일 손실이 발생해 v1.0.0 배포 불가. 최우선 처리.

## 2. 통합 전략 원칙 (단편적 실패 방지)

1. **Two-pass + `SerializeContext`**: 1-pass 스캔으로 ID 풀(char_shapes, para_shapes, border_fills, numberings, fonts_per_lang, bin_data_map, style_ids) 구성. 2-pass에서 header/section/content/manifest가 동일 컨텍스트만 읽는다. `ctx.assert_all_refs_resolved()`로 section의 모든 `charPrIDRef` 등이 header에 등록됐는지 런타임 단언.
2. **빈 문서 특수 분기 제거**: `mod.rs:61-66` 삭제. 빈 Document도 동적 경로. `EMPTY_CONTENT_HPF`/`empty_header.xml`/`empty_section0.xml`는 테스트 fixture로만 보존.
3. **누적 라운드트립 하네스 우선**: Stage 0에서 `tests/hwpx_roundtrip_integration.rs` 먼저 구축. 각 Stage는 이전 샘플도 재통과해야 완료. 하네스 통과 없이 merge 금지.
4. **IR 의미 비교 기반 검증**: `parse→serialize→parse` IR을 원본 IR과 필드 단위 비교(`IrDiff`). 바이트 비교는 브리틀. `rhwp diff a.hwpx b.hwpx` CLI 확장 보조.
5. **3-way BinData 단언**: `<hp:pic>` binaryItemIDRef ↔ `content.hpf` opf:item id ↔ ZIP entry 경로 — 3개 집합의 합집합=교집합을 단언.
6. **#181(SVG 스냅샷)과 독립 진행**: #181은 렌더 결과 회귀, #182는 직렬화 정확도. 계층이 달라 독립 하네스 병존.
7. **한컴 OWPML 공식 오픈소스를 스펙 참조원으로 활용**: [hancom-io/hwpx-owpml-model](https://github.com/hancom-io/hwpx-owpml-model) (Apache 2.0, 292개 클래스)을 **뼈대 층 기본값·enum 값·태그 전체 집합의 권위 있는 출처**로 사용. rhwp와 비교하여 커버리지 실측·누락 점검. 상세: `mydocs/tech/hwpx_hancom_reference.md`.
8. **한컴 DVC 검증기를 보조 게이트로 활용**: [hancom-io/dvc](https://github.com/hancom-io/dvc) (Apache 2.0, HWPX 서식 검증 DLL)를 Stage 5 완료 시점에 Windows VM에서 수동 검증 보조 게이트로 사용. Rust 포팅은 본 타스크 범위 밖 — 별도 후속 이슈로 분리됨. 상세: `mydocs/tech/hwpx_dvc_reference.md`.

## 3. 목표

HWPX로 연 문서를 편집 후 HWPX로 저장했을 때 **표·이미지·스타일·글꼴이 보존**되어 한컴2020에서 정상 렌더링되도록 한다. 각 Stage는 자동화된 IR 라운드트립 하네스를 통과해야 완료.

### 성공 기준

1. 단위 테스트 전체 통과 (`cargo test`)
2. 통합 라운드트립 하네스 통과 (`cargo test --test hwpx_roundtrip_integration`) — 빈문서/텍스트/표/이미지 누적 샘플 IrDiff 0
3. `SerializeContext::assert_all_refs_resolved()` 전 경로 통과
4. 한컴2020 수동 오픈·렌더 (보조 게이트)

## 4. 단계 분할 (5 Stage)

### Stage 0 — 라운드트립 하네스 + 분기 제거 + 한컴 기본값 상수화 (기반 공사)

- `src/serializer/hwpx/mod.rs`: `sections.len() == 1 && bin_data_content.is_empty()` 분기 제거
- 추가: `src/serializer/hwpx/context.rs` (`SerializeContext` + 1-pass 스캐너)
- 추가: `src/serializer/hwpx/roundtrip.rs` (`roundtrip_ir_diff`)
- 추가: `src/serializer/hwpx/fixtures.rs` (테스트 대조용 상수)
- 추가: `tests/hwpx_roundtrip_integration.rs` (blank_hwpx.hwpx IrDiff 0)
- 추가: `src/serializer/hwpx/canonical_defaults.rs` — **한컴 OWPML 공식 기본값 상수 테이블** (뼈대 층 속성의 정본). Apache 2.0 출처 명시.

**완료 기준**: 기존 단위 테스트 10개 그린 + `blank_hwpx.hwpx` 라운드트립 IrDiff 0 + `assert_all_refs_resolved` 통과 + `canonical_defaults.rs` 주요 10+개 속성 기본값 등록

### Stage 1 — header.xml IR 기반 동적 생성

- `header.rs` 13줄 → ~400줄. `HeaderWriter<'a>` 구조체 + 9개 하위 writer 함수
- `fixtures.rs`에 `default_doc_info_resources()`

**완료 기준**: `blank_hwpx.hwpx` + `ref_empty.hwpx` IrDiff 0 + 전 참조 resolve

### Stage 2 — section.xml 동적화 + charPrIDRef 매핑

- `section.rs` 템플릿 치환 제거. `SectionWriter`가 루트부터 빌드
- 추가: `paragraph.rs` (`write_paragraph` + `ParagraphChild` enum)
- `Paragraph.char_shape_refs`로 `<hp:run charPrIDRef>` UTF-16 범위 분할

**완료 기준**: 기존 문단/탭/줄바꿈 테스트 유지 + `ref_text.hwpx` IrDiff 0 + 다중 run 일치

### Stage 3 — Table 직렬화 (`<hp:tbl>`)

- 추가: `table.rs` (`write_table`). `table.common` 기반 (attr 비트 연산 금지)
- `<hp:tr>`/`<hp:tc>` 내부 `<hp:subList>` → `write_paragraph` 재귀

**완료 기준**: `hwp_table_test.hwp` 라운드트립 IrDiff 0 + 중첩 표 inner paragraph 보존 + 미등록 borderFillIDRef는 단언으로 잡힘

### Stage 4 — Picture + BinData ZIP 엔트리

- 추가: `picture.rs` (`write_picture`)
- `ctx.bin_data_map` + `mod.rs`에서 `BinData/image{N}.*` ZIP 엔트리 write
- 추가: `manifest.rs` (`META-INF/manifest.xml` 동적화)
- `content.rs` 연결

**완료 기준**: `pic-in-head-01.hwp`/`pic-crop-01.hwp` 라운드트립 + PNG Blake3 일치 + 3-way 단언 통과

### Stage 5 — 도형·필드 + 대형 실문서 스모크

- 추가: `shape.rs` (`<hp:rect/line/container/textart>` 등)
- 추가: `field.rs` (`<hp:fieldBegin/End>`, 각주/미주 최소 세트)
- 대형 샘플 3건 하네스 추가

**완료 기준**: `2025년 1분기 해외직접투자 보도자료f.hwpx` 외 2건 `IrDiff::allowed(shape_raw=true)` 통과 + 한컴2020 수동 오픈 성공

## 7. 위험 요소 & 완화

| 위험 | 완화 |
|---|---|
| IR char_shape 인덱스 vs HWPX `id` 속성 충돌 | `IdPool<T>` 양방향 맵만 노출 |
| 템플릿↔동적 결과 불일치 (빈 문서) | Stage 0 `blank_hwpx.hwpx` IrDiff 0 게이트 + `default_doc_info_resources` 보강 |

| BinData 3-way 동기화 누락 | `assert_all_refs_resolved`에서 집합 동일성 단언 |
| 이전 단계 하네스 회귀 | 누적만, 삭제·완화 금지 |
| pyhwpx 없는 CI | Rust 통합 테스트가 1차 게이트, pyhwpx는 수동 보조 |
