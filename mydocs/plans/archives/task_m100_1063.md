## 2. 증상

용지 설정 대화창의 "용지 방향" 섹션에서 `세로`와 `가로` 라디오 카드의
가이드 아이콘이 시각적으로 구분되지 않는다. 기대 동작은 `세로` 아이콘은
세로 비율 문서 박스, `가로` 아이콘은 가로 비율 문서 박스로 명확히 보이는 것이다.

이슈 본문 기준 재현 경로:

```bash
cd rhwp-studio && npx vite --host 0.0.0.0 --port 7700
# 브라우저에서 rhwp-studio 접속 → 임의 HWP 파일 또는 새 문서 열기
# 메뉴 [쪽] → [용지 설정] 또는 F7
# "용지 방향" 섹션의 세로 / 가로 아이콘 비교
```

## 4. 원인 가설

1. SVG markup 자체의 `width`/`height`는 다르지만, `.icon-radio-icon` 또는 주변 flex/card 레이아웃에서 실제 렌더 크기 차이가 약해진다.
2. SVG root에 명시적 class/style이 없어 전역 또는 인접 CSS와 결합될 때 intrinsic size가 기대만큼 보존되지 않을 수 있다.
3. `landscape` SVG의 외곽 비율은 가로이나 카드 폭/높이와 내부 선 표현이 세로 아이콘과 비슷해, 사용자 관찰상 두 아이콘이 같은 방향으로 인식될 수 있다.

## 5. 목표

- `세로` / `가로` 아이콘의 실제 SVG bounding box 비율을 DOM 수준에서 구분 가능하게 만든다.
- `가로` 아이콘은 텍스트 라벨 없이도 가로 방향 박스로 식별 가능해야 한다.
- 기존 PageSetupDialog 동작, 용지 방향 변경 시 폭/길이 교환, 제본 아이콘 레이아웃은 유지한다.
- Rust/WASM, 문서 파서/렌더러, HWP 저장 경로는 수정하지 않는다.

## 6. 범위

### 포함

- `rhwp-studio/src/ui/page-setup-dialog.ts`
  - 용지 방향 SVG에 안정적인 class 또는 식별자를 부여
  - 필요 시 landscape 아이콘 내부 표현을 더 명확한 가로 문서 형태로 보정
- `rhwp-studio/src/styles/dialogs.css`
  - `.icon-radio-icon svg`의 display/size 정책 명시
  - 방향 아이콘의 width/height 보존 규칙 추가
- `rhwp-studio/e2e/` 또는 기존 테스트 체계
  - 용지 설정 대화창에서 portrait/landscape 아이콘의 실제 DOM bounding box 비율 검증

### 제외

- PageDef 모델, WASM API, 저장/불러오기 동작 변경
- HWP/HWPX/HWP3 포맷 파서 변경
- 한컴 호환 저장 contract 변경
- 용지 설정 대화창 전체 UX 재설계

## 7. 검증 기준

1. `rhwp-studio` TypeScript/Vite 빌드 통과.
2. 새 E2E 또는 동등한 자동 검증에서:
   - `portrait` 아이콘 bounding box는 `height > width`
   - `landscape` 아이콘 bounding box는 `width > height`
   - 두 아이콘의 비율 차이가 작은 오차가 아닌 명확한 차이로 확인
3. 브라우저 시각 확인:
   - 용지 설정 대화창에서 `세로`와 `가로` 아이콘이 즉시 구분됨
   - desktop/mobile viewport에서 텍스트/아이콘 겹침 없음
