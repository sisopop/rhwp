# Task #280 수행계획서 — 수식 SVG 렌더링 폰트 스택 재정렬

## 이슈

[#280](https://github.com/edwardkim/rhwp/issues/280) — 수식 SVG 렌더링: 폰트가 볼드처럼 보임 (Cambria Math 폴백) — 폰트 스택 재정렬

## 목표

`samples/equation-lim.hwp` 등 수식 포함 HWP 파일의 SVG 렌더링에서 글자가 "볼드처럼 두껍게" 보이는 문제를 해소한다. 한컴 PDF 출력(HyhwpEQ 독점 폰트) 에 **시각적으로 근접**한 가는 세리프 렌더링이 목표. 픽셀 단위 일치는 불가능(독점 폰트 접근 불가) 하므로 "충분히 근접한 모양" 선을 유지한다.

## 배경 (원인 분석 요약)

### 재현 사례

- 샘플: `samples/equation-lim.hwp` (`lim _{h→0} {f(2+h)-f(2)} over h`)
- 정답: `samples/equation-lim.pdf` (한컴 Office 로 생성, HyhwpEQ 폰트 서브셋 임베딩)
- 사용자 제보: "폰트가 볼드, 크기가 다름"

### 조사 결과

1. **폰트 두께 (볼드처럼 보이는 원인)** — `src/renderer/equation/svg_render.rs:11` 의 `EQ_FONT_FAMILY`:
   ```
   'Latin Modern Math', 'STIX Two Math', 'Cambria Math', 'Pretendard', serif
   ```
   Windows 기본 환경에서 앞 두 폰트는 미설치, **Cambria Math** 가 매칭. Cambria Math 는 수학 디스플레이용 무거운 획 폰트라 일반 세리프보다 확연히 굵어 보임.

2. **`lim` 크기 차이는 `lim` 특별 취급이 아님** — PDF 내부에서 `lim` 은 110pt, 본문은 92pt 로 렌더링되어 "함수명 1.2x 확대 규칙" 이 있는지 의심했으나, 다음 근거로 **폰트 설계 차이에 의한 보정**임을 확인:
   - `exam_math.hwp` 의 `"b= log 2"` 수식 bbox 높이 = 1125 HWPUNIT (font_size * 1.02)
   - 단순 `"1"`, `"f(1)"` 과 동일 → `log`, `sin`, `cos` 는 본문과 같은 크기
   - PDF 의 110/92 는 HyhwpEQ 의 ASCII 글리프 vs PUA 수식 글리프 em 박스 차이를 HWP 엔진이 보정한 것
   - **결론: `lim` 등에 1.2x 배율 적용은 하지 않음**

3. **부차적 차이 (이 이슈 범위 외)**:
   - 괄호 `(` `)` 를 SVG path 로 그리는 `draw_stretch_bracket` 의 `paren_w = fs * 0.3` 가 다소 큼
   - 마이너스 기호 위치가 살짝 어긋남
   - → 별도 이슈 후보

## 수정 방침

### Phase 1: 폰트 스택 재정렬 (이 이슈 범위)

`EQ_FONT_FAMILY` 를 가는 세리프 우선으로 재정렬.

```rust
// 변경 전
const EQ_FONT_FAMILY: &str = " font-family=\"'Latin Modern Math', 'STIX Two Math', 'Cambria Math', 'Pretendard', serif\"";

// 변경 후 (제안)
const EQ_FONT_FAMILY: &str = " font-family=\"'STIX Two Text', 'Latin Modern Roman', 'Times New Roman', 'Times', 'Cambria', serif\"";
```

**설계 근거**:
- `STIX Two Text` — STIX 프로젝트의 텍스트 버전 (Math 는 굵음, Text 는 세리프 본문용). 설치된 환경에서 이상적.
- `Latin Modern Roman` — TeX 계열 클래식 세리프. 설치된 환경(LaTeX 설치 머신) 에서 PDF 와 가장 유사.
- `Times New Roman` / `Times` — 거의 모든 시스템 기본 세리프. Windows/Mac/Linux 어디서나 매칭.
- `Cambria` — Office 설치 환경 예비. (Cambria Math 아님 — 일반 Cambria 는 세리프 본문 폰트)
- `serif` — 최종 폴백.
- **Cambria Math 제거** — 이게 현재 볼드 인상의 원인이므로 폴백 후보에서 제외. 수학 전용 기호(∫, ∑ 등) 가 필요하면 브라우저가 폰트 글리프 폴백으로 자동 처리.

### Phase 2 (이 이슈 범위 아님, 후속 이슈 후보)

- 괄호 path 의 폭 비율 조정 (`paren_w = fs * 0.3` → 실측 기반)
- 마이너스 기호 위치 미세 조정

## 리스크 및 영향 범위

- **영향 범위**: 수식 SVG 렌더링만. 다른 렌더러(text, table, shape 등) 영향 없음.
- **호환성**: 폰트 이름을 변경해도 SVG 자체 구조는 동일. 기존 스냅샷이 실제 text 의 `font-family` 속성 문자열을 포함하면 업데이트 필요.
- **주의**: `canvas_render.rs` 는 Phase 1 범위 밖 (브라우저 canvas API 의 `font` 프로퍼티는 문자열 그대로 전달되므로 재정렬 일관성 검토 필요). 별도 검토.
