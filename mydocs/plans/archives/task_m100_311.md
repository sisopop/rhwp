# Task #311 수행계획서: 페이지네이션에서 LINE_SEG vpos-reset을 단/페이지 경계로 강제

상위 Epic: #309
선행: Task #310 (분석 도구) 완료
브랜치: `task311` (devel에서 분기)

## 목적

LINE_SEG `vertical_pos == 0` 리셋(이하 vpos-reset)을 페이지네이션 엔진의 단/페이지 강제 경계로 처리하여, **21_언어 SVG 19쪽 → 15쪽 (PDF 일치)** 달성. 다른 3개 샘플은 회귀 0.

## 배경 / 데이터 근거

Task #310 분석 결과(`mydocs/tech/line_seg_vpos_analysis.md`):

| 샘플 | SVG 쪽 | vpos-reset | FullParagraph 내부 reset |
|------|--------|-----------|--------------------------|
| 21_언어 | 19 (+4) | 13 | **7** ← 수정 대상 |
| exam_math | 20 ✓ | 0 | 0 |
| exam_kor | 25 | 8 | 0 |
| exam_eng | 11 ✓ | 0 | 0 |

21_언어 +4쪽의 직접 원인은 7건의 "FullParagraph 내부 vpos-reset" — HWP는 문단 중간을 끊을 의도였으나 우리 엔진은 통째로 한 단/페이지에 배치. 다른 샘플은 이 패턴 0건 → 좁은 분기로 회귀 없이 해결 가능.

## 작업 범위

### 코드 변경 (`src/renderer/pagination/engine.rs`)

`paginate_text_lines` 함수의 적합성 검사 직전(라인 ~601)에 다음 로직 추가:

1. `para.line_segs` 에서 첫 vpos-reset 위치 검출 (`line_index > 0 && vertical_pos == 0`)
2. vpos-reset 발견 시:
   - reset 직전 줄까지를 PartialParagraph 로 현재 단/페이지에 배치 (가용 공간이면)
   - `advance_column_or_new_page()` 호출
   - reset 줄부터 나머지는 다음 단/페이지로 (재귀적으로 추가 vpos-reset 처리)
3. vpos-reset 미존재 시 기존 로직 그대로 (변화 없음)

### CLI 옵션 추가

`--respect-vpos-reset` 플래그 (기본 off, 검증 후 on 전환):

```bash
rhwp export-svg sample.hwp --respect-vpos-reset    # 21_언어 15쪽
rhwp export-svg sample.hwp                          # 기본: 19쪽 (기존 동작)
```

플래그 도입은 단계적 회귀 검증을 위한 안전장치. 4개 샘플 + 기존 회귀 테스트 통과 확인 후 별도 sub-issue 또는 본 타스크 마지막 단계에서 기본 on 전환.

### 옵션 plumbing

`Paginator` API 옵션 전달 경로:
- 기존: `paginate_with_measured_opts(..., hide_empty_line: bool)`
- 변경: `PaginationOpts` 구조체로 전환 (`hide_empty_line`, `respect_vpos_reset` 통합)

기존 호출자는 한 군데(rendering.rs)이며, 호환을 위해 `paginate_with_measured` (기본값) 유지.

3. 4개 샘플 검증 + 기본 on 전환 + 분석 보고서 갱신

## 위험 / 대응

| 위험 | 대응 |
|------|------|
| `paginate_text_lines` 줄 분할 루프가 vpos-reset 무관하게 자체 cursor 운영 | vpos-reset를 함수 진입 시점에 사전 검출하여 `forced_breaks: Vec<usize>` 로 변환, 루프 내부에서 break 지점으로 사용 |
| Multi-column 문서에서 기존 column-break 로직과 충돌 | vpos-reset 검출 시 multi-column path와 single-column path 모두 동일하게 적용. 우선순위는 vpos-reset > column flag |
| 마지막 줄(line_count-1)에 vpos-reset (이론상 가능) | 분리 후 마지막 줄만 다음 페이지에 — 기존 PartialParagraph 분리 로직 재사용 |
| 옵션 off 상태에서 코드 경로 변경으로 인한 미세 회귀 | 옵션 off 시 기존 함수 흐름과 100% 동일하도록 가드 (early-return) |
