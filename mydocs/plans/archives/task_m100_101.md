## 배경

hwpspec.hwp 19페이지(global_idx=18)에서 pi=78 표(26행×3열)가 분할될 때
PartialTable이 페이지 하단을 11.2px 초과하는 LAYOUT_OVERFLOW가 발생한다.

```
LAYOUT_OVERFLOW: page=6, col=0, para=78, type=PartialTable, y=1045.2, bottom=1034.1, overflow=11.2px
```

## 현상

- 19페이지: `PartialTable pi=78 ci=0 rows=0..20`  ← 11.2px 오버플로우
- 20페이지: `PartialTable pi=78 ci=0 rows=20..26 cont=true`

## 구조 데이터

```
표: 26행×3열, 쪽나눔=RowBreak
크기: 38253×9139 HU (134.9×32.2mm)
outer_margin: bottom=7.0mm (1984 HU)
행 높이(h≠0인 행):
  r=0: 1459 HU  (헤더)
  r=4: 2976 HU
  r=6: 1176 HU
  r=7: 1176 HU
  r=19: 1176 HU
  나머지: h=0 (최소 행 높이)
```

## 원인 가설

PartialTable rows=0..20의 렌더 높이 계산 시 px 변환 과정에서 누적 오차가 발생하여
실제 배치 높이(y=1045.2 + 렌더높이)가 페이지 하단(1034.1)을 11.2px 초과하는 것으로 추정.

가능한 원인:
1. 행 높이 HU→px 변환 시 반올림 누적 오차
2. 표 상단/하단 외부 여백(outer_margin) 포함 계산 오류
3. PartialTable 높이 계산과 피트 판단 높이 계산의 불일치

## 수행 목표

1. rows=0..20 PartialTable의 렌더 높이가 페이지 하단을 초과하지 않도록 수정
2. LAYOUT_OVERFLOW 제거
3. 회귀 0건 유지
