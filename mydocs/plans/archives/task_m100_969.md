# Task #969 수행 계획서 — HWPX preset lineSegArray HWP5 typeset 정합

- 이슈: [#969](https://github.com/edwardkim/rhwp/issues/969)
- 부모 이슈: [#942](https://github.com/edwardkim/rhwp/issues/942) (증상 A 분리)
- 선행: Task #936 v2 Stage 7.8 (root cause 진단)
- 브랜치: `local/task969`
- 마일스톤: v1.0.0

## 1. 배경

`samples/hwp3-sample16-hwp5.hwpx` 가 한컴 PDF (64 페이지) 대비 **72 페이지** (+8 inflate). 같은 원본의 HWP3/HWP5 변환본은 62 페이지로 정합.

Stage 7.8 에서 `pi=395 cur_h=985.5 > avail=967.3 (18.2 px overflow)` 식별. 후속 IR 비교 (2026-05-18) 로 **root cause 확정**.

## 2. Root cause (확정)

`rhwp ir-diff samples/hwp3-sample16-hwp5.hwpx samples/hwp3-sample16-hwp5.hwp --summary`:

```
59건  line_segs count       ← 가장 큰 차이
12건  cc
 7건  pos
...
```

`󰏅` (PUA 항목 부호) 로 시작하는 paragraph (376, 380, 383, 386, 395, 400, 403, 407, …) 에서:

| 항목 | HWPX | HWP5 |
|------|------|------|
| line_segs count | **1** (preset) | **0** (typeset 위임) |
| ls[0] lh | 1299 HU (= 17.3 px) | — |
| ls[0] ls | 779 HU (= 10.4 px) | — |
| paragraph height | preset 기반 | typeset 동적 계산 |

ParaShape 동일 (`ps_id=32 align=Justify spacing before=2264 line=160/Percent`). 차이는 **lineSegArray 의 IR 사전 emit 여부**.

paragraph 당 ~27.7 px 차이 × 8 개 누적 = **+8 페이지 inflate** (Stage 7.8 의 18.2 px overflow 와 정합).

## 3. 수정 방향 후보

| 후보 | 설명 | 위치 | Risk |
|------|------|------|------|
| **D1** | HWPX 파서가 lineSegArray emit 안 함 | `src/parser/hwpx/section.rs` 등 | 중 (정합 case 회귀) |
| **D2** | `format_paragraph` 가 preset LineSeg 무시 | `src/renderer/typeset.rs` | 중 |
| **D3** | HWPX 파서의 LineSeg 변환 식 보정 | `src/parser/hwpx/` | 낮 |
| **D4** | typeset 이 preset LineSeg height 만 무시 | `src/renderer/typeset.rs` | 낮-중 |

구현 계획서에서 후보 1 개 확정.
