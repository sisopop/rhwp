# 수행계획서 — Task #1217: Dependabot 결합 의존성 그룹화

- **이슈**: #1217 (M100 / v1.0.0)
- **브랜치**: `local/task1217`
- **성격**: 의존성 인프라 개선 (내부 타스크, 소스 코드 무변경 — 설정 1파일)
- **작성일**: 2026-06-01

## 1. 배경과 문제

PR #1215(pdf-writer 0.12→0.15)가 CI Build & Test / Analyze(rust) **FAILURE**.

- `src/renderer/pdf.rs` 는 **svg2pdf + pdf-writer 를 함께** 사용 — svg2pdf 가 생성한
  `pdf_writer::Chunk` 를 받아 `renumbered_chunks` 로 재배치(pdf.rs:79, 109).
- 두 크레이트는 **동일 pdf-writer 타입을 공유**해야 함.
- svg2pdf 0.13.0(최신)은 pdf-writer **0.12** 에 고정.
- pdf-writer 단독 0.15 bump 시 그래프에 0.12/0.15 공존 →
  `error[E0308]: mismatched types — multiple different versions of crate pdf_writer` (로컬 재현 확증).

Dependabot 기본 동작은 결합 크레이트를 **개별 PR** 로 올림 → 동일 함정 반복.
Cargo.toml 의 동일 결합 후보: `usvg 0.45`, `resvg 0.47`, `ttf-parser 0.25` (resvg 패밀리, 버전 동반 필요).

## 2. 목표

`.github/dependabot.yml` 에 **결합 의존성 그룹** 추가로 단독 bump 깨짐 PR 을 원천 차단.

- 결합 크레이트는 1 PR 로 동반 bump → svg2pdf 가 새 pdf-writer 지원 시에만 그룹 통과.
- 개별 무관 crate(serde 등)는 기존대로 개별 PR 유지 (작은 배치 회전 철학 정합,
  `feedback_small_batch_release_strategy`).
- CI Build & Test 가 최종 게이트(이미 작동).

## 3. 적용 범위

| 생태계 | 그룹명 | 패턴 | 근거 |
|--------|--------|------|------|
| cargo | `resvg-stack` | svg2pdf, pdf-writer, usvg, resvg, ttf-parser | pdf-writer/usvg 타입 공유, 버전 결합 |
| npm | `vite-stack` | vite, vite-plugin-pwa | vite-plugin-pwa 1.x ↔ vite 8 peer 결합 |

- 기존 `target-branch: devel`, `schedule: weekly`, `open-pull-requests-limit: 10` 유지.
- `groups` 는 같은 ecosystem 블록 안에 둠.

## 4. 검증 방법

설정 파일 변경이므로 빌드/테스트 대상 아님. 검증은 다음으로 한정:

1. **YAML 문법 검증**: `python -c "import yaml; yaml.safe_load(open('.github/dependabot.yml'))"` 통과.
2. **스키마 정합**: Dependabot `groups` 공식 스펙(patterns/exclude-patterns)과 1:1 대조.
3. **회귀 영향 없음 확인**: 소스/Cargo.toml/package.json 무변경 → 빌드·테스트 무영향(자명).
4. **사후 관찰(보고서에 명시)**: 다음 주간 Dependabot 실행 시 resvg-stack/vite-stack
   그룹 PR 로 묶이는지는 머지 후 GitHub 동작으로 확인(이번 타스크 검증 범위 밖, 후속 관찰 항목).
