## 1. 문제 정의

### 취약점 발견 경위

GitHub CodeQL 자동 보안 스캔(Alert #12)에서 `rhwp-chrome/content-script.js`의 XSS 취약점이 검출됨.

Safari 확장(`rhwp-safari/src/content-script.js`)은 Task #84에서 `createElement` + `textContent` 패턴으로 이미 수정 완료. Chrome 확장은 동일 수정 미적용 상태.

### 취약점 내용

**파일**: `rhwp-chrome/content-script.js`

**취약점 1 (주요)** — HTML 문자열 조합 후 innerHTML 삽입 (line 100~143):

```javascript
let html = '';
html += `<div class="rhwp-hover-title">${title}</div>`;      // XSS
html += `<div class="rhwp-hover-meta">${meta.join(' · ')}</div>`;  // XSS
html += `<div class="rhwp-hover-info">${info.join(' · ')}</div>`;  // XSS
html += `<div class="rhwp-hover-category">${category}</div>`;// XSS
html += `<div class="rhwp-hover-desc">${description}</div>`; // XSS (CodeQL #12)
html += `<div class="rhwp-hover-thumb"><img src="${thumbnail}">`;  // XSS
html += `<div class="rhwp-hover-title">${fileName}</div>`;   // XSS
card.innerHTML = html;  ← 위험 지점
```

외부 입력값(`data-hwp-*` 속성, anchor.href에서 추출한 파일명)이 이스케이프 없이 HTML 문자열에 삽입됨.

**취약점 2 (보조)** — 썸네일 dataUri를 innerHTML로 삽입 (line 192, 209):

```javascript
thumbDiv.innerHTML = `<img src="${cached.dataUri}" alt="미리보기">`;
thumbDiv.innerHTML = `<img src="${response.dataUri}" alt="미리보기">`;
```

---

## 2. 수정 방향

Safari content-script.js의 안전한 패턴(`createElement` + `textContent`) 동일 적용.

**핵심 원칙**:
- 텍스트 콘텐츠: `div.textContent = text` (HTML 삽입 없음)
- 이미지: `createElement('img')` → `img.src = url`
- HTML 문자열 조합(`innerHTML` 대입) 완전 제거

---

## 3. 수정 범위

| 항목 | 현재 | 수정 후 |
|------|------|---------|
| card 생성 (hover card 본문) | `innerHTML = html` | DOM API로 각 요소 생성 |
| 썸네일 img (사전 지정) | `img src` 포함 html 문자열 | `createElement('img')` |
| 썸네일 img (자동 추출 후 삽입) | `thumbDiv.innerHTML = ...` | `createElement('img')` |
| 썸네일 img (Service Worker 응답) | `thumbDiv.innerHTML = ...` | `createElement('img')` |
