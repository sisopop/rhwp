## 1. 배경

인라인 수식(treat_as_char)이 텍스트와 겹치는 문제가 발생한다. 수식 컨트롤이 문단에서 차지하는 조판 영역(너비)이 실제 렌더링 너비와 불일치하기 때문이다.

### 원인 분석

`composer.rs` 라인 130-132에서 수식의 `tac_controls` 너비를 `eq.common.width`(HWP 저장값, HWPUNIT)로 설정한다. 이 값은 한컴 편집기가 계산한 원래 수식 크기이다. 그러나 rhwp의 수식 레이아웃 엔진(`EqLayout`)이 계산하는 실제 렌더링 너비와 불일치하면, 문단 조판에서 수식에 할당된 영역이 실제보다 작거나 커서 주변 텍스트와 겹치거나 빈 공간이 생긴다.

**핵심**: 수식 조판 영역의 너비를 HWP 저장값이 아닌, rhwp 레이아웃 엔진이 산출한 실제 렌더링 너비로 대체해야 한다.

## 2. 목표

- 인라인 수식의 조판 영역(너비/높이)을 레이아웃 엔진 산출값과 일치시켜 텍스트 겹침 해결
- #139에서 식별한 레이아웃 상수(BIG_OP_SCALE, MATRIX 간격 등) 보정

## 3. 구현 단계 (3단계)

### 1단계: 인라인 수식 조판 영역 너비 보정

- `composer.rs`에서 수식의 `tac_controls` 너비를 레이아웃 엔진 산출값으로 대체
- 수식 스크립트 → AST → LayoutBox 계산 → 실제 너비 사용
- 높이도 동일하게 보정

### 2단계: 레이아웃 상수 보정

- `layout.rs` 상수 조정 (한컴 기준값 측정 결과 반영)
- BIG_OP_SCALE, MATRIX_COL_GAP, MATRIX_ROW_GAP 등

### 3단계: 테스트 및 검증

- `cargo test` 전체 통과
- exam_math.hwp SVG 출력으로 수식-텍스트 겹침 해소 확인
- output/svg/ 폴더에 검증용 SVG 생성

## 4. 검증 방법

- exam_math.hwp 1~2페이지 SVG 출력 → 수식과 텍스트 겹침 없음 확인
- `cargo test` 전체 통과
