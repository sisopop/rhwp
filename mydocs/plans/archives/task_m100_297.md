# Task #297: exam_math.hwp 12쪽 바탕쪽 "* 확인 사항" 박스 위치 오류 — 수행계획서

## 배경

GitHub Issue #297은 `samples/exam_math.hwp` 12쪽 우단(30번 문제) 콘텐츠가 PDF 대비 아래로 밀린다고 보고했다. 최초 증상 보고는 "동전 그림(pi=33) 위치 어긋남"이었으나, 조사 결과 **pi=33 동전 이미지 자체는 PDF와 y좌표가 일치(≈497px)**하고, 실제 드리프트는 **바탕쪽(master page)의 "* 확인 사항" 표**에서 발생한다.

따라서 본 타스크의 정정된 제목 및 범위는:
> **"exam_math.hwp 바탕쪽 '* 확인 사항' 표가 PDF 대비 약 89px 아래로 배치되는 현상 수정"**

## 증상 데이터

### 좌표 비교 (12쪽 우단, 96dpi PNG 기준)

| 요소 | PDF y | SVG y | 차이 |
|------|-------|-------|------|
| 동전 이미지 상단 (pi=33) | ~495 | 497.25 | ≈0 |
| 동전 이미지 하단 | ~645 | 640.95 | ≈0 |
| **"* 확인 사항" 박스 상단** | **~1270** | **1359.4** | **+89px (밀림)** |
| body_area 하단 (기준선) | 1360.7 | 1360.7 | — |

### 바탕쪽 표 원본 스펙 (모든 섹션/페이지 동일)

```
vert=Paper/101954 HU  vert_align=Top
horz=Paper/85 HU      horz_align=Center
size=58408x2196 HU    treat_as_char=false  wrap=TopAndBottom
rows=1 cols=3         attr=0x082a2400
```

- `101954 HU = 359.6mm = 1359.4px` (용지 상단 기준)
- `394mm - 359.6mm = 34.4mm` ≈ 꼬리말(13) + 하단 여백(21) = **본문 하단 경계선**
- PDF 렌더 박스 실측 높이: 1360.7 - 1270 ≈ **90px** (내용물 3~4행 포함)

## 원인 가설

`vertical_offset=101954 HU`의 해석이 rhwp와 한컴 뷰어에서 다름:

- **rhwp 현재**: 표의 **TOP**을 y=101954 HU에 배치 → SVG에서 표가 본문 하단 경계선에서 시작되어 페이지 바닥으로 밀림
- **한컴 뷰어(PDF)**: 표의 **BOTTOM**을 y=101954 HU에 맞춤 → 표가 본문 하단 경계선에 닿도록 위로 확장되며 "* 확인 사항" 헤더가 본문 하단보다 위에 위치

근거:
1. 101954 HU가 정확히 **본문 하단 경계선**과 일치하는 것은 우연이 아님 — HWP가 "이 좌표에 표의 바닥이 닿도록 하라"고 기록한 것
2. `wrap=TopAndBottom` + 바탕쪽 문맥에서 표는 일반적으로 페이지 하단(꼬리말 위)에 고정되는 배치 패턴

1. **HWP 스펙 재확인** — `vertical_offset`의 기준점이 실제로 TOP인지, 혹은 `vert_align` 해석이 다른지
2. **다른 샘플에서 동일 패턴 회귀 확인** — 바탕쪽 `wrap=TopAndBottom` 표가 있는 다른 문서에서도 PDF와 일치 여부
3. **`vert_align` 외 값(Center/Bottom) 처리** — 샘플이 없다면 보수적 수정으로 한정
