### 2.2 한컴 UX 목표 (작업지시자 명시 + 표 UX 동일 적용)

| 클릭 위치 | 한컴 동작 (목표) | 표 UX 참조 (이미 구현) |
|----------|------------------|-------------------------|
| 글상자 **외부 경계선** (tolerance ~5px) | 글상자 객체 선택 | `isTableBorderClick` (input-handler.ts:1105) |
| 글상자 내부 (텍스트 위) | 즉시 텍스트 편집 진입 | 셀 내부 cursor 진입 |
| 글상자 내부 (텍스트 없음/패딩) | 즉시 텍스트 편집 진입 (첫 paragraph 시작) | 빈 셀 클릭 (cell_bboxes 매칭) |
| 글상자 안 표/이미지/안 글상자 | 즉시 해당 객체 hit (글상자 가로채기 없음) | 셀 안 인라인 객체 hit |
| 글상자 외곽 근처 (밖에서) | 글상자 객체 선택 | `findTableByOuterClick` (input-handler.ts:1130) |
| 글상자 내부 편집 중 Esc | 글상자 객체 선택 (편집 모드 탈출) | ✅ **이미 구현** (`enterPictureObjectSelectionDirect`, input-handler-keyboard.ts:937) |
| 글상자 객체 선택 중 Enter/더블클릭 | 텍스트 편집 진입 | `enterTableObjectSelection` → `enterCell` 패턴 |

### 2.3 표 UX 재사용 — 정확한 매핑 분석

표 UX 의 구성 요소가 **이미 모두 일반화 가능** 한 패턴:

#### A. `isTableBorderClick` (`input-handler.ts:1105`)

```typescript
const bbox = this.wasm.getTableBBox(sec, ppi, ci);  // ← 글상자: getShapeBBox 신규
const tolerance = 5;
const nearLeft = Math.abs(pageX - bbox.x) <= tolerance;
// ... 4 경계 + 직교 범위 검사
```

**글상자 적용**: `getTableBBox` 대신 `getShapeBBox` 신규 wasm API 필요. 경계선 검사 로직 자체는 그대로 재사용 가능.

#### B. `findTableByOuterClick` (`input-handler.ts:1130`)

```typescript
// 클릭한 paragraph + 인접 paragraph(±2) 의 controls 중 표 검사
for (let offset = 0; offset <= 2; offset++) { ... }
```

**글상자 적용**: 동일 패턴 — 클릭 paragraph + 인접 paragraph 의 Shape 컨트롤 검사 + `getShapeBBox` 외곽 근처 판별.

#### C. 셀 내부 cursor 진입 (`input-handler-mouse.ts:617~660`)

```typescript
const hit = this.wasm.hitTest(pageIdx, pageX, pageY);
// 표 경계선 → enterTableObjectSelectionDirect
// 셀 내부 paragraph hit → cursor 그대로 이동
```

**글상자 적용**: 현재 hit_test_native 는 글상자 안 텍스트만 인식. **글상자 안 빈 영역** (패딩) 도 인식하도록 `collect_runs` 에 `RenderNodeType::TextBox` 노드 수집 추가 → cell_bboxes 와 동등 처리.

#### D. Esc 패턴 (`input-handler-keyboard.ts:933~963`)

```typescript
if (e.key === 'Escape') {
  if (inTextBox) {
    // 글상자 편집 → 객체 선택 (이미 구현)
    this.cursor.enterPictureObjectSelectionDirect(sec, ppi, ci, objType);
  } else if (inCell) {
    // 셀 편집 → 표 객체 선택 (이미 구현)
    this.cursor.enterTableObjectSelection();
  }
}
```

**이미 구현**. 본 task 영역 변경 없음.

#### E. 객체 선택 → 편집 진입 (`input-handler-mouse.ts:866~896`)

현재 글상자 객체 선택 시 **더블클릭** 으로만 텍스트 편집 진입. 한컴 UX 는 **객체 선택 상태에서 다시 클릭 (또는 Enter)** 으로 진입 — 표 UX 와 동일.

표의 `enterCell` 패턴 (line 627: 셀 외곽 클릭 → 객체 선택, 다시 셀 내부 클릭 → 셀 cursor) 과 정합.

4. **`rhwp-studio/src/engine/input-handler.ts:1105`** `isTableBorderClick` — 글상자 대응 `isShapeBorderClick` 헬퍼 필요
