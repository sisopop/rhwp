## 1. 배경

이슈 [#608](https://github.com/edwardkim/rhwp/issues/608) (@almet) 의 "Publish releases" 요청에 대해 v0.7.10 부터 CLI 바이너리 릴리즈 (`rhwp` 실행파일) 를 GitHub release assets 으로 첨부하기로 약속.

**v0.7.10 릴리즈 흐름의 본질 영역**:
- 본 사이클 (5/1 ~ 5/6) 외부 PR 22건 + 메인테이너 정정 누적의 **첫 patch 릴리즈**
- 본 task 가 v0.7.10 의 신규 기능 영역 (CLI binary release) 로 작업지시자 안내 정합

## 2. 목적

- HWP → SVG / PNG / PDF / TEXT / Markdown 변환을 결정적 자동화 파이프라인 / CI / 공공 자산 워크플로우에서 사용 가능하게 함
- 한컴 환경 없이 바이너리 한 개로 사용 가능 (source of truth)
- 본 프로젝트 정체성 ("All HWP, Open for Everyone" / "닫힌 포맷의 벽 깨기") 정합

## 3. 본 환경 인프라 점검

### 3.1 기존 GitHub Actions 워크플로우

`.github/workflows/`:
- `ci.yml` — 빌드/테스트 (PR + push)
- `codeql.yml` — 보안 분석
- `deploy-pages.yml` — GitHub Pages 배포
- `npm-publish.yml` — npm 패키지 배포
- `render-diff.yml` — Canvas visual diff (PR #498 도입)

→ 본 task 는 **신규 워크플로우 추가** (`release-binary.yml`).

### 3.2 Cargo.toml 의 binary 설정

```toml
[[bin]]
name = "rhwp"
path = "src/main.rs"

[[bin]]
name = "font-metric-gen"
path = "src/tools/font_metric_gen.rs"
```

→ `rhwp` binary 는 이미 정의됨, 코드 변경 0.

### 3.3 현재 빌드 산출물

- `target/release/rhwp` (13 MB, x86_64-unknown-linux-gnu)
- 본 환경: WSL2 Ubuntu, rustc 1.93.1

## 4. 기술 명세 (이슈 #612 본문 정합)

### 4.1 빌드 타깃 (4 플랫폼 matrix)

| 플랫폼 | 타깃 트리플 | runner | 산출물 |
|------|---|---|---|
| Linux | `x86_64-unknown-linux-gnu` | `ubuntu-latest` | `rhwp-vX.Y.Z-linux-x86_64.tar.gz` |
| macOS Intel | `x86_64-apple-darwin` | `macos-13` (Intel) | `rhwp-vX.Y.Z-macos-x86_64.tar.gz` |
| macOS Apple Silicon | `aarch64-apple-darwin` | `macos-14` (M1) | `rhwp-vX.Y.Z-macos-aarch64.tar.gz` |
| Windows | `x86_64-pc-windows-msvc` | `windows-latest` | `rhwp-vX.Y.Z-windows-x86_64.zip` |

### 4.2 아카이브 내용

- `rhwp` (Windows 는 `rhwp.exe`)
- `LICENSE`
- `README.md` (한글 본문)
- **`README_EN.md` (영문 본문)** — 작업지시자 안내 정합. 외부 영어권 사용자 (이슈 #608 @almet 등 Freedom of Press / Pelican / Mozilla 출신 영역) 정합

→ 본 환경 한국어 / 영어 dual readme 정합 (`mydocs/manual/` + `mydocs/eng/manual/` 패턴과 동일).

### 4.3 무결성 검증

- `SHA256SUMS.txt` — 모든 아카이브의 SHA-256 체크섬 (release page 동봉)

### 4.4 CI 트리거

- `release` 태그 (`v*`) 푸시 시 자동
- 매트릭스 빌드 → `softprops/action-gh-release` 로 자산 첨부

### 4.5 native-skia feature 영역

PR #599 의 `native-skia` feature 는 **본 task 의 기본 release binary 에는 미포함**:
- 이유: skia-safe 의존성이 release binary 크기 + 빌드 시간 증가 + 플랫폼별 라이브러리 영역
- v0.7.10 의 기본 binary 는 SVG / PDF / Text / Markdown 변환만 지원
- PNG (Skia) 는 별도 feature build 또는 후속 task (`v0.7.11` ?) 에서 검토

→ 작업지시자 결정 영역 — 작업지시자 결정 시 본 task 본문 갱신.

## 5. 구현 옵션 (이슈 #612 본문 정합)

### 옵션 A — `cargo dist` (권장 — 이슈 본문 정합)

장점:
- 표준화된 release 도구 (axodotdev/cargo-dist)
- 자동 generate `.github/workflows/release.yml`
- Linux / macOS / Windows 매트릭스 + checksums 자동
- Homebrew tap / Cargo install 도 추가 옵션 확장 가능
- Astral (uv / ruff) / pyo3 등 주요 Rust 프로젝트 채택

단점:
- 외부 도구 의존성
- 학습 곡선

### 옵션 B — 자체 GitHub Actions matrix

장점:
- 외부 도구 의존성 없음
- 더 세밀한 제어 (signing / SBOM / 추가 자산)
- 본 환경 ci.yml 패턴 정합

단점:
- workflow 작성 + 유지보수 부담
- Homebrew / Cargo install 등 후속 옵션 별도 작업

## 7. 잠재 우려 영역

### 7.1 macOS code signing

- macOS 는 unsigned binary 실행 시 Gatekeeper 경고
- 본 task 영역 비포함 (별도 Apple Developer ID 인증 영역)
- README 에 "first run: `xattr -d com.apple.quarantine rhwp`" 안내 권장

### 7.2 Windows 의 SmartScreen

- Windows Defender SmartScreen 이 unsigned 바이너리 차단 가능
- 본 task 영역 비포함 (코드 서명 인증서 별도)
- README 에 "first run: SmartScreen → 추가 정보 → 실행" 안내 권장

### 7.3 빌드 시간

- 4 플랫폼 매트릭스 빌드: 각 ~10 분 = 총 ~40 분 (parallel)
- Skia feature 포함 시 빌드 시간 대폭 증가 (skia-safe 컴파일)

### 7.4 종속성 — fontconfig / freetype 등

- Linux: 시스템 fontconfig 의존 (font-config-dev install 필요)
- macOS / Windows: 시스템 폰트 영역 (별도 의존성 없음)
- → workflow 의 platform 별 dependency install 단계 영역
