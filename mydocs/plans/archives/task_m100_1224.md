# 수행계획서 — Task #1224: 폰트 충실도 (한컴 돋움 폴백 글리프 과대·과굵)

- **이슈**: #1224 (M100 / v1.0.0)
- **브랜치**: `feature/issue-1224-font-fidelity` (base: `stream/devel` 4f5a8e22)
- **성격**: 렌더 시각 충실도 개선 — 폰트 폴백/임베딩 (레이아웃·메트릭 무변경)
- **작성일**: 2026-06-01

## 1. 배경과 문제

`samples/3-09월_교육_통합_2023.hwp` 6쪽 문26 등에서 레이아웃(글자 크기 9pt, advance,
줄나눔, 페이지 분할)은 PDF·스펙과 **정확히 일치**하나, **글리프가 PDF보다 크고 두껍게**
렌더된다.

| | 글리프 잉크 높이 (em=15px 기준) | em 대비 |
|--|--|--|
| rhwp (Noto Sans CJK KR 대체) | 13px | 0.87 |
| PDF (한컴 돋움) | 10px | 0.67 |

### 1.1 메커니즘 확증 (코드 레벨)

SVG 텍스트는 `svg.rs::draw_text` 에서 **클러스터 단위 개별 `<text>`** 로 방출되며:

- **위치**: `compute_char_positions`(메트릭 DB advance) 기반 `char_x` 로 절대 배치 → 자간 정확
- **가로폭**: `svg_text_length_attrs`(`textLength`/`lengthAdjust`) 로 글리프 폭을 advance 에 제약
- **세로 잉크/굵기(em-fill)**: **제약 수단 없음** — 폴백 폰트 디자인이 전적으로 결정

즉 위치·자간·가로폭은 모두 제어되고, **유일한 미제어 차원이 세로 em-fill** 이다. 따라서
폴백 폰트만 한컴 돋움에 가까운 em-fill 로 바꾸면 레이아웃·메트릭 변경 없이 시각이 교정된다.

### 1.2 폴백 실태

- `renderer/mod.rs::generic_fallback` sans 체인:
  `'Malgun Gothic','맑은 고딕','Apple SD Gothic Neo','Noto Sans KR','Pretendard',…,sans-serif`
- 이 리눅스/CI 환경엔 **Noto Sans CJK KR 만 설치** (Pretendard·나눔·Spoqa 전무) →
  rsvg/브라우저가 체인을 따라가다 **Noto 로 렌더** → em-fill 0.87.
- 한컴 돋움(Haansoft Dotum)·MS 맑은 고딕은 저작권 폰트라 번들·시스템 설치 불가.

### 1.3 핵심 시사점

체인 **우선순위만** 바꿔도, 그 폰트가 시스템에 없으면 rsvg 는 다시 Noto 로 폴백한다.
**결정적 충실도를 보장하려면 적합 폰트를 SVG 에 임베딩(서브셋)** 해야 한다. 따라서 본
타스크는 "후보 실측 → 폰트 선정 → 체인 우선순위 + 임베딩 기본셋" 을 모두 다룬다
(이슈 후보방향 1·2·3 전체, 작업지시자 확정 범위).

## 2. 목표

한컴 돋움/맑은 고딕 계열 본문이 PDF 정답지와 **시각적으로 정합**하도록, em-fill 이
한컴 돋움(~0.67)에 근접한 **오픈소스 대체폰트를 선정**하고 폴백 체인 우선순위 조정 +
임베딩 경로에 반영한다. **레이아웃·메트릭 DB·줄나눔·페이지수는 불변**을 보장한다.

## 3. 적용 범위

| 영역 | 변경 | 비고 |
|------|------|------|
| 후보 폰트 실측 | em-fill / 잉크 박스 측정 보고서 | PDF 정답지 대비 |
| `renderer/mod.rs::generic_fallback` | sans(돋움/고딕) 체인에 선정 폰트 우선 삽입 | serif·mono 체인은 범위 밖(별도 판단) |
| 임베딩 경로 (`svg.rs` `--font-style`/`--embed-fonts`) | 대체폰트 자동 참조/서브셋 기본 반영 | `--font-path` 탐색·서브셋 파이프라인 재사용 |
| `ttfs/FONTS.md` / `tech/font_fallback_strategy.md` | 메트릭(advance) ↔ 렌더 글리프(대체폰트) 시각 정합 가이드 | 문서화 |

### 범위 밖 (별개 처리)

- serif(바탕/명조) 계열 폴백 충실도 — 본 건은 돋움/고딕에 한정(이슈 증상 범위). 필요 시 후속 이슈.
- 메트릭 DB advance 수치 변경 — 레이아웃 정확하므로 손대지 않음.
- WASM 웹(`rhwp-studio`) `font-loader.ts` 체인 — SVG 네이티브 경로 우선. 웹 반영은 후속 검토.
- 저작권 폰트 번들 — 불가. 오픈소스(OFL/무료배포)만 사용.

## 4. 접근 (측정 주도 선정)

후보 오픈소스 sans(em-fill 작은 순 추정): **Spoqa Han Sans Neo**, **Gowun Dodum**,
**IBM Plex Sans KR**, **Pretendard**(이미 번들), Noto Sans KR(현 폴백, 비교 기준).

1. 각 후보의 한글 음절 대표 글리프 잉크 박스 높이 / em 비율을 실측 (`fc-list`+pyft/ftview
   또는 동일 em SVG 렌더 후 픽셀 측정) → 한컴 돋움(PDF) 0.67 과 편차 최소 폰트 선정.
2. 선정 폰트의 라이선스(OFL/무료 상업 허용)·한글 커버리지(11,172)·굵기 축 확인.
3. 폴백 체인에 Noto **앞**으로 삽입 + 임베딩 기본 참조 등록.

## 5. 검증 방법

1. **시각 회귀**: `export-svg samples/3-09월_교육_통합_2023.hwp -p 5` →
   `pdf/3-09월_교육_통합_2022.pdf` 6쪽과 픽셀/육안 비교. em-fill 0.67±0.05 목표.
2. **레이아웃 불변**: 변경 전후 `dump-pages` 문단 배치·높이·페이지수 **동일** 확인
   (글리프 폰트만 교체, 메트릭 무변경 → 자명하나 회귀 표로 증빙).
3. **임베딩 산출**: `--embed-fonts` 서브셋 크기·글자수 로그 + 오프라인 렌더 정상.
4. **회귀 스위트**: `cargo test` 전체 통과(특히 `test_generic_fallback`).
5. **다문서 표본**: 돋움/고딕 본문 포함 샘플 2~3종 추가 시각 확인.

## 7. 리스크

- **0.67 em-fill 오픈소스 부재 가능성**: 현대 한글 고딕 대부분 0.8+. 최근접 폰트로
  편차 최소화하되, 도달 불가 시 한계를 보고서에 명시(완전 일치 아닌 근접 목표).
- **임베딩 폰트 미보유 환경**: `--font-path` 로 후보 폰트 탐색 경로 확보 필요. 없으면
  서브셋 실패 → local() 폴백(현 동작 유지, 회귀 아님).
- **체인 변경의 타 폰트 영향**: 돋움/고딕 키워드 분기 내로 한정 → serif·PUA 매칭 무영향 확인.
