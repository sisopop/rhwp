```typescript
private scrollCaretIntoView(rect: CursorRect): void {
    const caretDocY = pageOffset + rect.y * zoom;
    const scrollTop = this.container.scrollTop;
    const viewHeight = this.container.clientHeight;
    if (caretDocY < scrollTop + margin) {
      this.container.scrollTop = Math.max(0, caretDocY - margin);
    } else if (caretDocY + caretHeight > scrollTop + viewHeight - margin) {
      this.container.scrollTop = caretDocY + caretHeight - viewHeight + margin;
    }
}
```

## 3. 정정 영역 후보

### 가설 A: `scrollCaretIntoView` 가드

caret 위치가 사용자 의도적 scroll 영역 (사용자가 scrollbar / wheel / page-up/down 등 으로 명시적 scroll 한 후) 시 `scrollCaretIntoView` skip.

- 영역: caret 변경 (cursor move) 가 동반 안 된 mouseup → `scrollCaretIntoView` 호출 자체 skip
- 정정 위치: `updateCaret` 안 의 가드 또는 `scrollCaretIntoView` 안 의 가드

### 가설 B: `onMouseUp` 의 `updateCaret` 호출 영역 좁힘

`onMouseUp` 끝 `updateCaret()` 호출 자체가 무차별. caret 변경 trigger 가 있는 경우 (selection drag 종료 등) 만 호출.

- 영역: `onMouseUp` 끝의 `this.updateCaret()` 를 조건 가드 (`if cursor moved during drag`) 로 좁힘
- 정정 위치: `onMouseUp` ([input-handler-mouse.ts:1385](rhwp-studio/src/engine/input-handler-mouse.ts#L1385))

### 가설 C: scrollbar mouseup 식별

mouseup event 의 target 이 scrollbar 영역 인지 식별 후 skip.

- 영역: 브라우저 scrollbar 의 정확한 식별 (`event.target` 이 scroll container 인 경우 + clientX 가 scrollbar 영역)
- 정정 위치: `onMouseUp` 시작 부분 가드
- 한계: scrollbar 영역 식별이 브라우저별 상이 (Chrome/Safari/Firefox)

### 권장 영역

가설 A (선호): caret 위치/cursor 가 변경되지 않았으면 `scrollCaretIntoView` 자체 skip. PR #718 의 패턴 (`updateCaretDuringDrag` → `updateCaretNoScroll` 분기) 와 정합. 영향 좁힘 (`feedback_hancom_compat_specific_over_general` 정합).
