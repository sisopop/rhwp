## 2. 결함 영역

### 결함 1 — HWP3 외부 file path 그림 Document IR 누락

- HWP3 spec offset 74 그림 종류 (0=외부 파일, 1=OLE, 2=Embedded, 3=Drawing) + offset 83~339 그림 파일 이름 (256 byte) 인식
- 본 환경 HWP3 파서 (`mod.rs:844~857`) 이미 인식하지만 `pic.image_attr.bin_data_id` 만 부여, 외부 file path 자체 IR 미보존
- 결과: 외부 file path 그림 본 환경 SVG 출력 시 image 표시 안됨
- 한컴 한글 2024 정합: placeholder (점선 사각형 + 깨진 이미지 아이콘) 표시

### 결함 2 — 폰트 매핑 차이

- `hwp3-sample10.hwp` paragraph 4 ("Technical Bulletins") CharShape font_face 본 환경 (신명 디나루) vs 한컴 정합 (돋움)
- 본 환경 HWP3 파서가 글꼴 이름 array (7개 언어별) 또는 CharShape font index 매핑 결함

## 3. 사전 진단 (Stage 1 검증 영역)

### 가설 A: Document IR 확장 (외부 file path 필드)

### 가설 B: Renderer placeholder 처리

외부 file path 그림 (binary 데이터 부재) 인 경우 SVG/Canvas 출력 시 placeholder (점선 사각형 + 아이콘) 표시.

### 가설 C: HWP3 파서 글꼴 이름 array 정합화

본 환경 HWP3 파서가 글꼴 이름 array (7개 언어별, kchar array[40] 반복) 정확 파싱하는지 확인. CharShape font_face index → 글꼴 이름 매핑이 한컴 정합인지.
