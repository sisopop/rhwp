## 1. 배경 / 증상

`samples/3-09월_교육_통합_2022.hwpx` 9~11쪽 미주 마커 **"문N)"** 가 **"N)"** 로 렌더링됨 (접두문자 '문' 탈락).
한글 2022 PDF(`pdf/3-09월_교육_통합_2022.pdf`)에는 "문4)", "문5)" 로 표시됨.

SVG 검증: 9쪽 미주 영역 마커가 `"4）"` (접두 없음, 접미 전각 '）'만)로 출력 확인.

## 2. 근본 원인 (확정)

HWPX 본문 미주 요소는 접두/접미문자를 코드포인트 숫자로 인코딩:

```xml
<hp:endNote number="1" prefixChar="47928" suffixChar="65289" instId="...">
```
- `prefixChar="47928"` = 0xBB38 = **'문'**
- `suffixChar="65289"` = 0xFF09 = **'）'**(전각)

미주 마커 문자열은 `src/renderer/typeset.rs:60` `format_endnote_marker_text()` 가
`Endnote.before_decoration_letter`(접두) + 번호 + `after_decoration_letter`(접미) 로 조립한다.
**렌더 경로는 이미 접두문자를 지원**한다.

그러나 HWPX 파서가 `suffixChar` 만 읽고 `prefixChar` 분기가 없다:

| 함수 | 파일:라인 | suffixChar | prefixChar |
|------|----------|:---------:|:----------:|
| `parse_ctrl_endnote` | `src/parser/hwpx/section.rs:3920` | O (3930→3935) | **X (누락)** |
| `parse_ctrl_footnote` | `src/parser/hwpx/section.rs:3884` | O (3896→3901) | **X (누락)** |

→ `before_decoration_letter` 가 0 으로 남고 → `note_decoration_char(0)` = `None` → 접두 빈 문자열 → '문' 탈락.

> 참고: `parse_note_pr_children`(endNotePr/footNotePr shape, 라인 777/784) 과
> `parse_ctrl_autonum`(라인 3974/3978) 은 prefixChar/suffixChar 양쪽 모두 파싱한다.
> **이번 누락은 `parse_ctrl_endnote`/`parse_ctrl_footnote` 두 함수에 국한**된다.

## 3. 수정 범위 (최소)

두 함수에 `suffixChar` 와 대칭으로 `prefixChar` 분기를 추가한다.
u16(코드포인트 숫자) 파싱 → `before_decoration_letter` 에 매핑.

- **모델 변경 불필요** (`Endnote.before_decoration_letter`, `Footnote.before_decoration_letter` 이미 존재)
- **렌더러 변경 불필요** (`format_endnote_marker_text` 이미 접두 사용)
- HWP3 전용 모듈 무관, 공통 모듈 분기 추가 없음
