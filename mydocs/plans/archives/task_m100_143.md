# Task #143: LaTeX 분수·제곱근 입력 최소 지원 — 수행계획서

## 목표

수식 편집기 미리보기에서 사용자가 LaTeX 형태의 기본 수식을 붙여넣었을 때, `\frac{1}{2}` 같은 핵심 구조가 문자 그대로 렌더링되는 문제를 우선 수정한다.

## 현재 증상

임시 프로브로 확인한 현재 동작:

```text
\frac{1}{2}
→ Row([Text("\\"), Text("frac"), Number("1"), Number("2")])
```

즉 `\frac` 가 명령어로 해석되지 않고 백슬래시와 텍스트로 분리되어, 화면에는 분수 대신 `\frac12` 형태의 잘못된 결과가 보인다.

## 범위

이번 PR은 Issue #143 전체 범위 중 가장 영향이 큰 최소 조각만 처리한다.

- `\frac{a}{b}` → `EqNode::Fraction`
- `\sqrt{x}` → `EqNode::Sqrt`
- `\sqrt[n]{x}` → `EqNode::Sqrt { index: Some(...) }`
- `\pm`, `\alpha` 처럼 기존 한컴 명령 매핑이 있는 백슬래시 명령은 기존 `lookup_symbol` 경로 재사용

## 비범위

- `\begin{matrix}` / `\end{matrix}` 환경 파서
- `\mathbb`, `\mathcal` 등 AST 확장이 필요한 LaTeX 전용 서체
- 수식 편집 UI의 명시적 한컴/LaTeX 모드 토글
- 전체 LaTeX 문법 호환

## 접근

1. 토크나이저에서 `\` 뒤 ASCII 알파벳 시퀀스를 `Command` 토큰으로 읽는다.
2. 파서에서 `frac` 계열을 두 그룹 인자로 읽어 기존 `EqNode::Fraction` 으로 변환한다.
3. 파서에서 `sqrt` 뒤 대괄호 인덱스를 선택적으로 읽는다.
4. 기존 한컴 스크립트 테스트를 함께 실행해 회귀를 확인한다.

## 검증 기준

1. 신규 테스트가 수정 전 실패하고 수정 후 통과한다.
2. `\frac{1}{2}` 가 `EqNode::Fraction` 으로 파싱된다.
3. `x=\frac{-b \pm \sqrt{b^2}}{2a}` 에 분수, ±, 제곱근이 모두 포함된다.
4. `cargo test --lib equation` 통과.
5. 실제 SVG 미리보기 결과를 PNG 스크린샷으로 남긴다.
