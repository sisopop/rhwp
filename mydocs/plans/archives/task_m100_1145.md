## 2. 문제 요약

`RowBreak` 표가 페이지 하단에 걸릴 때 rowspan 블록 중간에서 `PartialTable` 분할이 발생한다.
문제 샘플에서는 제목 셀이 `row_span=2`인데 분할 경계가 `end_row=1`이 되어,
1페이지 말미와 2페이지 시작에 같은 제목 셀이 중복 출력된다.

## 4. 원인 후보

공통 조판 경로의 `typeset_block_table()`에서 `RowBreak` 표는 rowspan 보호 블록 정책을 비활성화한다.

현재 조건:

```text
protected = !mt.allows_row_break_split()
  && block_size >= 2
  && block_size <= BLOCK_UNIT_MAX_ROWS
```

따라서 `page_break=RowBreak`이면 `protected=false`가 되고,
rowspan 블록 내부인 `end_row=1`도 합법적인 행 경계 분할 후보가 된다.

`layout_partial_table()`은 fragment 범위와 row-spanned 셀이 겹치면 해당 셀을 포함하므로,
제목 셀이 이전 fragment와 다음 fragment에 모두 렌더링된다.

## 5. 구현 방향

`RowBreak` 표라도 분할 후보가 rowspan 블록 내부이면 다음 정책을 적용한다.

1. 후보 `end_row`가 rowspan 블록 내부인지 판정한다.
2. 블록 전체가 현재 남은 페이지 영역에 들어가면 `end_row`를 블록 끝으로 확장한다.
3. 블록 전체가 들어가지 않으면 블록 시작 전으로 되돌려 다음 페이지로 넘긴다.
4. fresh page에도 들어가지 않는 oversized 블록은 기존 block cut 경로를 허용한다.

핵심은 `RowBreak`의 일반 행 경계 분할 정책은 유지하되,
rowspan 블록 내부 절단만 막는 핀셋 처리로 제한하는 것이다.
