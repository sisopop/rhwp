# Task 400: 텍스트 폭 측정 정합성

## 수행 목표

rhwp의 텍스트 폭 측정 결과가 한컴 원본 LINE_SEG의 text_start와 일치하도록 폰트 메트릭을 보정한다. Task 398 베이스라인에서 확인된 text_start 음수 오차(-2~-4 UTF-16)의 근본 원인을 규명하고 해결한다.

## 배경

Task 398 측정 결과:
- text_start 오차가 일관적으로 **음수** → rhwp가 한컴보다 문자 폭을 좁게 측정
- 이로 인해 한 줄에 더 많은 글자를 넣어 줄바꿈 위치가 달라짐
- lseg-03-spacing에서 4줄→5줄 불일치의 근본 원인

## 구현 계획

### 1단계: 문자별 폭 차이 분석 도구

- lseg-01-basic.hwp의 원본 text_start로부터 줄별 텍스트 폭을 역산
- rhwp `estimate_text_width()` 결과와 비교
- 한글/영문/공백/특수문자 각각의 폭 차이 패턴 식별

### 2단계: 내장 폰트 메트릭 보정

- 바탕체 메트릭에서 발견된 폭 차이를 보정
- 한컴 원본 text_start를 기준으로 역공학
- 보정 후 Task 398 테스트로 일치율 변화 측정

### 3단계: 일치율 개선 확인

- 통제 샘플(lseg-01~06) + 기존 샘플 재측정
- text_start 일치율, 줄 수 일치율 개선 확인
- 개선 리포트 작성

## 산출물

- 폭 차이 분석 결과
- 폰트 메트릭 보정 코드
- 일치율 개선 리포트 (`mydocs/working/task_400_result.md`)
