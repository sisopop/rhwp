# 수행계획서 — Task M100-1187: BookReview.hwp 글상자 내용 영역 밖 출력 회귀

## 1. 이슈

GitHub Issue #1187.

`samples/basic/BookReview.hwp` 1쪽에서 큰 점선 사각형 글상자 하단 밖으로 뒤쪽 목차 텍스트가 출력되고, 우측 하단 저자 정보 박스와 겹치는 회귀를 수정한다.

## 2. 재현 기준

- 기준: `upstream/devel`
- 확인 커밋: `1eb76529fce21d0a5330720f7b458831c5252fdf`
- 재현 명령:
  - `rhwp export-svg samples/basic/BookReview.hwp -p 0 --debug-overlay --show-control-codes --show-grid`
  - `rhwp dump samples/basic/BookReview.hwp -s 0 -p 12`

관찰 결과:

- 큰 글상자 rect 는 `y=509`, `height=503` 으로 하단이 약 `1012px`.
- 같은 글상자의 뒤쪽 문단은 `y=1057`, `y=1094` 까지 출력됨.
- 생성 SVG 에는 body clip 만 있고 글상자/도형 clipPath 가 없음.
- HWP dump 기준 첫 사각형 글상자는 15개 문단을 포함하며, 뒤쪽 문단의 `line_seg.vertical_pos` 가 글상자 내부 높이를 초과함.

## 3. 진단

현재 `layout_textbox_content` 는 글상자 내부 영역(`inner_area`)을 계산하지만, 글상자 콘텐츠를 별도 clip 그룹으로 감싸지 않고 shape node 자식으로 직접 배치한다.

출력 경로도 Body 와 TableCell clip 만 처리한다. `RenderNodeType::TextBox` 와 paint layer 의 `GroupKind::TextBox` 는 존재하지만, 글상자 clip 영역을 표현하거나 적용하는 경로가 없다.

이 때문에 원본 HWP 의 글상자 내부 배치값이 글상자 높이를 초과하는 경우, rhwp SVG/paint 출력에서는 텍스트가 도형 밖으로 그대로 보인다.

## 4. 목표

글상자 내부 콘텐츠가 글상자 내부 영역을 벗어나 보이지 않도록 렌더링 clip 을 적용한다.

핵심 원칙:

- overflow 문단을 삭제하거나 임의로 재배치하지 않는다.
- 원본 `line_seg.vertical_pos` 기반 배치 해석은 유지한다.
- 글상자 도형의 배경/테두리와 글상자 내부 텍스트 clip 을 분리한다.
- SVG exporter 와 paint layer 경로의 동작을 맞춘다.

## 5. 작업 범위

| 항목 | 범위 |
|------|------|
| 렌더 모델 | 글상자 콘텐츠 clip 영역을 표현할 수 있는 노드/메타데이터 정리 |
| 레이아웃 | `layout_textbox_content` 에서 글상자 텍스트 자식을 clip 가능한 컨테이너 아래에 배치 |
| SVG 출력 | 글상자 clipPath 를 생성하고 콘텐츠 그룹에 적용 |
| paint layer | 글상자 콘텐츠 clip 을 LayerNode 로 반영 |
| 회귀 테스트 | `BookReview.hwp` 기반으로 글상자 밖 텍스트 출력 재발 방지 |

## 6. 제외 범위

- HWP 파서의 TextBox/LINE_SEG 구조 변경
- 글상자 overflow 문단의 연결 글상자 재분배 로직 변경
- 세로쓰기/다단/표 레이아웃의 일반 재설계
- unrelated golden SVG 대량 갱신
- 한컴 원본과 다른 임의 줄바꿈/문단 높이 보정

## 7. 검증 게이트

- `cargo build --bin rhwp`
- 신규 또는 보강 회귀 테스트 통과
- `cargo test --tests` 중 관련 테스트 통과
- `cargo fmt --check`
- `samples/basic/BookReview.hwp` SVG 재생성 후 큰 글상자 하단 밖 목차 텍스트가 보이지 않는지 확인
- SVG 산출물에 글상자 clipPath 또는 동등한 clip 적용이 존재하는지 확인

## 8. 리스크

- 글상자 clip 을 도형 전체에 걸면 배경/테두리까지 잘릴 수 있으므로 콘텐츠에만 적용해야 한다.
- 기존 Body/TableCell clip 과 중첩될 수 있으므로 clip group 닫힘 순서와 레이어 트리 구조를 주의해야 한다.
- author box 는 z=4 로 별도 도형이다. 첫 글상자 overflow 만 가려져야 하며, author box 자체가 사라지면 안 된다.
- 연결 글상자나 세로쓰기 케이스는 단순 overflow 삭제로 처리하면 후속 회귀를 만들 수 있다.

## 9. 예상 단계

구현계획서에서 3~5단계로 확정한다.

1. RED 회귀 테스트와 현재 출력 구조 고정
2. 글상자 콘텐츠 clip 렌더 모델/레이아웃 적용
3. SVG 출력 clip 적용
4. paint layer clip 적용
5. BookReview 시각 검증과 최종 회귀 테스트 정리
