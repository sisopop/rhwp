# Task #617 수행 계획서 — 표 셀 padding 축소 휴리스틱 오작동

## 1. 배경

`exam_kor.hwp` 16/27/36번 `<보기>`·`<자료>` 글상자에서 본문이 셀 테두리에 거의 닿아 표시되는 시각적 오류.

근본 원인은 `src/renderer/layout/table_layout.rs:860 shrink_cell_padding_for_overflow` 의 자연 폭 추정 부정확성:

- `estimate_text_width` 가 양쪽 정렬 한국어 본문의 효과 폭을 과대 추정
- 음수 letter_spacing 만 clamp 하고 CharShape `ratio` (장평) / `spacing` (자간) 의 효과 폭은 부분 반영
- 1.15× 임계를 넘기면 padding(850 HU ≈ 3 mm)을 1 px 까지 축소

같은 표 구조 / 같은 padding / 같은 ParaShape 인 25번(정상) vs 27번(깨짐)의 갈림이 결정적 증거 (텍스트 분량 차이만 존재).

## 2. 목표

1. 16/27/36번 보기 박스 좌·우 padding 이 시각적으로 복원되어 본문과 테두리 사이 약 3 mm 간격이 보일 것.
2. shrink 휴리스틱이 본래 보호하려던 케이스(IR `line_segs` 미흡으로 글자가 셀 밖으로 흘러나오는 비정상 입력)는 그대로 보호할 것.
3. 회귀 방지: 기존 샘플(특히 KTX 목차 등 Task #279/#347 에서 거론된 비대칭 padding 케이스) 에서 변경 전과 동일하게 렌더되는지 확인.

비목표: 표 셀 전체 패딩 모델 재설계, table.padding/cell.padding 우선순위 정책 변경.

## 3. 접근 방식

핵심 관점: **HWP IR 의 `line_segs` 가 이미 셀 폭 안으로 자간을 분배해 줄바꿈을 확정한 상태**라면, 추가적인 자연 폭 비교는 부정확한 추정으로 오작동만 유발한다.

세 단계로 나누어 수정:

### A. shrink 비활성화 조건 추가 (주된 수정)

`shrink_cell_padding_for_overflow` 진입부에서, 모든 단락의 모든 `line_segs` 가 채워져 있고 (HWP 가 줄바꿈을 결정한 상태) 줄 수가 1 이상이면 padding 을 그대로 반환.

근거: HWP 가 양쪽 정렬·자간 분배로 line_segs 를 확정한 텍스트는 셀 가용 폭(= cell_w − 좌·우 padding)에 이미 fit 한 상태. 자연 폭 추정이 임계를 넘기더라도 실 렌더는 fit 하므로 padding 을 깎으면 오히려 오버 페인팅.

### B. 자연 폭 추정 정확도 보정 (잔여 케이스)

A 가 적용되지 않는 케이스(line_segs 가 비어 있는 비정상 입력) 에서도 추정이 한국어 본문에 대해 더 정확하도록:

- CharShape `ratio` (글자 너비 %) 및 `spacing` (자간 %) 을 effective_width 에 반영
- 임계를 1.15 → 1.30 으로 완화 (한국어 양쪽 정렬 분배 폭 안전 마진)

### C. 최소 padding 정책 강화

shrink 가 실제로 트리거 되는 비정상 케이스에서도 padding 이 1 px 까지 깎이지 않고 원래의 30% 이상 보장.

## 4. 단계 구분 (구현 계획서에서 상세화)

- Stage 1: 재현 픽스처 + 단위 테스트 추가 (실패 확인)
- Stage 2: A — line_segs 기반 shrink skip 구현
- Stage 3: B + C — 추정 정확도 + 최소 padding 보정
- Stage 4: 회귀 검증 (전체 샘플 SVG diff, 수동 시각 확인)

## 5. 검증 방법

1. 단위 테스트: `shrink_cell_padding_for_overflow` 입력 케이스
   - line_segs 채워진 입력 → padding 그대로
   - line_segs 비어 있고 텍스트 폭 초과 → 종전처럼 축소 (단 30% 하한)
   - 일반 정상 케이스 (Task #347 KTX 사례) → 변경 없음
2. 시각 검증:
   - `rhwp export-svg samples/exam_kor.hwp -p 5,8,16` 로 16/27/36번 박스 좌·우 여백 복원 확인
   - 25번(원래 정상)·기타 박스 회귀 없음 확인
3. 전체 샘플 SVG 비교: 변경 전/후 diff 가 의도한 박스에만 발생하는지 확인.
