# Task #656: typeset/layout height 측정 모델 통일 (Task #485 후속) — 수행 계획서

## 배경

Task #485 (synam-001.hwp 분할 표 셀 마지막 줄 클립) 정정에서 `compute_cell_line_ranges` break 조건에 `SPLIT_LIMIT_EPSILON = 2.0px` 휴리스틱 마진을 도입.

이 epsilon 은 **typeset 의 `split_end_limit = avail_content` 추정과 layout 의 line_h 누적 사이의 미세 어긋남** 을 흡수하는 임시방편. 본 타스크는 그 구조적 본질을 통일하는 후속 작업.

## 수행 목표

`mydocs/troubleshootings/typeset_layout_drift_analysis.md` §근본 해결안 비교 의 **A. 단일 모델 통합** 안을 단계적으로 적용:

1. typeset advance = `height_for_fit` (Task #331 의 1단계 — 단단/다단 통일)
2. layout per-paragraph advance = `height_for_fit` (Task #331 의 2단계)
3. vpos correction 의 `vpos_end` 에서 trail_ls 제외, 또는 양방향 보정
4. clamp pile 버그 → "stop drawing on overflow" 로 변경 + typeset 에 overflow signal 발행
5. 정합 입증 후 Task #485 의 `SPLIT_LIMIT_EPSILON` 제거

## 우선순위 / 본질

이슈 자체 우선도 **낮음** — 현재 epsilon 휴리스틱이 결함 해소 + 회귀 없음. 다른 분할 표 결함 누적 또는 폰트 변경으로 epsilon 부족 케이스 발생 시 우선도 상승.

**구조 부채 해소** 가 본 타스크의 본질이며, 시각 결함 해소가 아니다. 따라서 **회귀 0** 이 필수 조건이며, 회귀 발생 시 즉시 단계 후퇴 (revert) 한다.
