## 3. 진단 방향

### 3.1 머리말 영역 정밀 record 비교

- 정답지 vs 저장본의 머리말 영역 (record #138~182) record-by-record byte diff
- ODD (홀수쪽) vs EVEN (짝수쪽) 비교 — EVEN 정상이므로 contract 차이가 trigger 단서
- SHAPE_COMPONENT (239 bytes) 의 byte 단위 diff — 어느 field 가 한컴 줄나눔/높이 계산에 영향

### 3.2 저장 전용 계약 후보

- SHAPE_COMPONENT 의 도형 크기/렌더링 매트릭스 (current size 등)
- 글상자 LIST_HEADER 의 text margin / max_width
- 글상자 내부 paragraph 의 line_seg vertical size
- TABLE 의 cell margin / inner contract

### 3.3 #1110 누적 자료 활용

- `output/poc/hwpx2hwp/task1110/stage28_*` (정답지 vs stage28 저장본 = 현재 코드 동일)
- stage28 의 oracle inventory + needle32 + shape bundles

### 3.4 페이지번호(AutoNumber) 생성·배치 경로 조사 (작업지시자 지시 추가)

**HWPX 원본 구조** (`Contents/section0.xml`):
```
<hp:header id="1" applyPageType="ODD">         ← 홀수쪽 머리말 (문제)
  └ <hp:tbl> <hp:tc> <hp:rect> <hp:drawText>   ← 표/셀/사각형/글상자
      └ <hp:p paraPrIDRef="53" charPrIDRef="63">
          └ <hp:ctrl><hp:autoNum num="1" numType="PAGE"></hp:ctrl>
            <hp:t/>                            ← 빈 텍스트 (autoNum 단독)
```

**ODD vs EVEN 글상자/문단 차이** (trigger 핵심):

| 항목 | ODD (홀수, 문제) | EVEN (짝수, 정상) |
|------|-----------------|-------------------|
| rect width | **4252** (좁음) | 15941 (넓음) |
| rect height | 3685 | 3685 |
| horzAlign | RIGHT | LEFT |
| **paraPrIDRef** | **53** | 64 |
| paraPr align | RIGHT | JUSTIFY |
| **paraPr breakNonLatinWord** | **BREAK_WORD** | KEEP_WORD |
| lineSpacing | 165% | 165% |

→ 좁은 글상자(4252) + paraPr 53 (`BREAK_WORD`) 조합에서 한컴이 페이지번호 줄나눔.

**파싱·직렬화 경로**:
- 파싱: `src/parser/hwpx/section.rs:3569` `b"autoNum"` → `parse_autonum_attrs` → `Control::AutoNumber`
- 어댑터: `src/document_core/converters/hwpx_to_hwp.rs` — 이미 AutoNumber 관련 materialize 다수
  - `autonum_fwspace_range_tag_materialized` (AutoNumber 뒤 fixed-width space PARA_RANGE_TAG)
  - `autonum_fwspace_char_shape_offsets_materialized` (char shape start_pos 보정)
  - `header_footer_fwspace_control_materialized`
  - `master_page_autonum_placeholder_removed`

→ **이전 round1 의 fixed-width space 처리가 이 영역**. 한컴 줄나눔/높이 계산과 직결 가능성 우선 조사.
