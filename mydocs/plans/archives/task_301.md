# Task #301 수행계획서: exam_math.hwp 12쪽 정규분포 z-table 이중 렌더링 버그

- **마일스톤**: 미지정

## 1. 현상

`samples/exam_math.hwp` 페이지 12 좌측 컬럼 #29 문제 안에 배치된 정규분포표(z-table)의 셀 텍스트가 SVG 출력에서 **두 번 그려져 시각적으로 겹침**.

대상 표:
- 위치: 섹션 1, 문단 27, 컨트롤 0 (`pi=27 ci=0`)
- 형태: 5행 × 2열, 크기 146.9×114.4px
- 속성: `wrap=Square`, `treat_as_char=false`
- 셀 내용: `z` / `P(0≤Z≤z)` 헤더 + `(0.5, 0.1915)`, `(1.0, 0.3413)`, `(1.5, 0.4332)`, `(2.0, 0.4772)`

## 2. 재현 절차

```bash
cargo run --release --bin rhwp -- export-svg samples/exam_math.hwp -p 11 -o output/svg/exam_math_p12/
```

확인 방법:
```bash
grep -c '0\.1915\|0\.3413\|0\.4332\|0\.4772' output/svg/exam_math_p12/exam_math_012.svg
# 기대값: 4 (각 1회) / 실제값: 8 (각 2회)
```

또는 SVG 라인 297~352 직접 확인. 각 셀당 동일 텍스트의 `<g transform>` 블록이 2개씩 생성됨.

좌표 차이 패턴:
- y 오프셋: 일정하게 약 -1.31 (둘째 복사가 1.31 위로 이동)
- x 오프셋: 셀 정렬에 따라 다름 (`0.5`는 +2.51, `0.1915`는 +28.86)

## 3. 원인 가설

세 가지 후보:

### 가설 A: 다단(multi-column) 페이지에서 wrap=Square 표 중복 배치
- 페이지 12는 2단 레이아웃 (`단 0`/`단 1`)
- z-table은 `단 0`의 19개 항목 중 하나
- `pagination/engine.rs`의 `is_wrap_around_table` 처리에서 양쪽 단에 중복 배치되는 로직 존재 가능성
- 그러나 결과 SVG에서 표는 좌측 컬럼 영역(x≈343-490)에만 나타나므로, **표 자체가 두 번 배치됐다기보다는 셀 내부 컨텐츠만 두 번 렌더링**된 것으로 보임

### 가설 B: 표 셀 컨텐츠 렌더링 중복 호출
- `layout/table_layout.rs` 또는 `layout/table_cell_content.rs`에서 셀 내부 paragraph를 두 번 처리
- y 오프셋이 항상 -1.31로 고정 → 두 경로(예: 1차 측정 + 2차 실제 렌더)가 서로 다른 y 보정값을 사용

### 가설 C: SVG 렌더링 경로에서 cell-clip 그룹이 중복 생성
- SVG 출력(`renderer/svg.rs`) 단계에서 cell 단위로 그룹 emit이 두 번 호출

## 4. 영향 범위

- **재현 조건 추정**: `wrap=Square` + 다단 레이아웃 + 표 셀 내 텍스트
- **다른 파일 영향**: `samples/` 내 다단+wrap=Square 표가 있는 다른 문서에서도 동일 버그 가능
- **WASM 영향**: `web_canvas.rs`도 동일 IR을 사용하므로 영향 가능 (검증 필요)
- **PDF/native 출력**: 동일 IR 기반이면 동일 증상 가능

## 5. 검증 방법

### 5.1 회귀 방지 자동 테스트
- `output/re/` 재현검증 스위트 또는 새 통합 테스트로 z-table 셀 텍스트가 1회만 출력됨을 검증

### 5.2 수동 검증
1. 수정 후 위 재현 명령 재실행 → grep 결과 4가 되는지 확인
2. SVG를 PNG로 변환하여 PDF와 시각 비교
3. `samples/` 내 다른 다단+표 문서 회귀 확인 (최소 2~3개 샘플)

### 5.3 IR 비교
- 동일 문서의 HWPX가 있다면 `rhwp ir-diff`로 IR 단계에서는 차이 없음을 확인 (IR이 아닌 렌더 단계 버그임을 입증)
