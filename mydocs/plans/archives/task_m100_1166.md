# 수행계획서 — Task M100-1166: HWPX 가로 편집 용지 무시 결함

## 1. 이슈

#1166 (bug, 외부 제보). 가로 편집 용지 HWPX 가 rhwp 에서 세로로 렌더링됨.
`getPageDef()` 가 `landscape:false` + 세로 치수 반환.

## 2. 근본 원인 (진단 완료)

### 권위 매핑 (hwplib ForSecPr.java:151-158)
- `WIDELY` = 세로(Portrait) → `landscape=false`
- `NARROWLY` = 가로(Landscape) → `landscape=true`
- width/height 는 항상 짧은변/긴변 (HWP5 동일 규약). landscape=true 면 렌더가 swap.

샘플 검증: para-001(세로)=WIDELY, landscape-001(가로)=NARROWLY.

### 결함 위치 (HWPX 경로만 — HWP5 는 정상)
1. **파싱** `parser/hwpx/section.rs:234`: `b"landscape" => { /* 무시: landscape=false 유지 */ }` — 잘못된 가정. 가로 무시.
2. **직렬화** `serializer/hwpx/section.rs:72` + `templates/empty_section0.xml`: `pagePr landscape="WIDELY" width="59528" height="84186"` **하드코딩** 템플릿 사용. IR page_def 미반영 → HWPX 저장 시 가로/세로 + 용지 크기 손실.

> HWP5 파싱/직렬화(`control.rs serialize_page_def`)는 landscape flag 처리 정상. 본 작업 범위 제외.

## 4. 섹션별 처리 (작업지시자 요구)

rhwp 는 이미 `Document.sections[].section_def.page_def` 로 **섹션별 PageDef** 보유.
파싱은 각 secPr 의 pagePr 을 해당 섹션 page_def 에 반영(현 구조 유지),
직렬화도 섹션별 page_def 로 출력하면 섹션별 다른 방향 자연 충족.
