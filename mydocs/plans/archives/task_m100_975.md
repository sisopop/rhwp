# 수행 계획서 — Task #975: PageBackground 이미지 fill_mode 렌더링 보정

- 이슈: [#975](https://github.com/edwardkim/rhwp/issues/975)
- 마일스톤: M100 / v1.0.0
- 브랜치: `task-975-page-background-fill-mode` (base = upstream/devel `39d90d9d`)

## 1. 증상

Tistory HWP 2010 워터마크 예제 `143E433F503322BD33.hwp`를 열면 파란 배경 이미지가 페이지 전체 크기로 늘어나 렌더링된다.

덤프 기준 해당 이미지는 일반 그림 개체가 아니라 페이지 배경/BorderFill 이미지다.

```text
border_fill[2] fill_type=Image  image(bin_id=3, mode=Center)
```

## 2. 현재 코드 관찰

`src/renderer/layout.rs`의 `build_page_background()`는 `PageBackgroundImage.fill_mode`에 `img_fill.fill_mode`를 보존한다.

```text
PageBackgroundImage {
    data: c.data.clone(),
    fill_mode: img_fill.fill_mode,
}
```

하지만 렌더러의 PageBackground 경로는 이 값을 사용하지 않는다.

| 파일 | 현재 동작 | 문제 |
|------|----------|------|
| `src/renderer/svg.rs` | `preserveAspectRatio="none"` + page bbox 전체 `<image>` | `Center` 등 배치 모드 무시 |
| `src/renderer/web_canvas.rs` | `draw_image(..., bbox 전체)` | `Center` 등 배치 모드 무시 |

일반 `ImageNode` 경로에는 이미 `ImageFillMode` 처리 함수가 있다.

- SVG: `render_image_node()` → `render_positioned_image()` / `render_tiled_image()`
- Web Canvas: `draw_image_with_fill_mode()`

## 3. Root cause 가설

파서와 레이아웃은 `ImageFillMode::Center`를 보존하고 있으며, 최종 렌더링 단계에서 PageBackground 이미지가 무조건 `FitToSize`처럼 처리된다.

따라서 본 task의 1차 root cause는 렌더러의 PageBackground 전용 이미지 출력 경로가 일반 이미지 fill mode 처리 경로와 분리되어 있는 것이다.
