# 수행계획서 — HeightMeasurer ↔ cell_units 측정 정합

- 타스크: GitHub #1022 (M100, v1.0.0)
- 브랜치: `local/task1022` (`local/devel` 분기)
- 작성일: 2026-05-20
- 선행: task993 (분할 표 컷 모델, merge `45584a25`) — 회귀 진단 인계

## 1. 배경

task993 의 컷 walk(`advance_row_cut`)이 비분할 행에서까지 `cell_units` 측정을 누적하면서, `MeasuredTable.row_heights`(렌더러 `resolve_row_heights`)와 행마다 미세하게 어긋난다. 베이스라인(`8be5e0c2`) `LAYOUT_OVERFLOW` 0건이 task993 머지 후 **50건**으로 회귀.

상세: `mydocs/working/task_m100_993_findings_layout_overflow.md`.

## 2. 문제

`HeightMeasurer::measure_section`(`MeasuredCell.line_heights` / `total_content_height`)와 `table_layout::cell_units`(`compose_paragraph` + `corrected_line_height` + `cell.height` 필러)가 같은 셀 콘텐츠에 대해 다른 px 합을 산출한다. 두 측정은 베이스라인 px 모델에서는 paginator 가 mt 만 쓰고 renderer 도 mt 만 써서 일관성이 유지됐으나, task993 의 cut walk 가 cell_units 측정으로 paginator 를 옮긴 뒤 비분할 행에서까지 drift 누적이 LAYOUT_OVERFLOW 50건을 노출.

## 3. 작업 범위

- 포함:
  - `HeightMeasurer`(MeasuredCell)와 `cell_units` 가 같은 셀에 대해 동일 px 합을 산출하도록 측정 시스템 통일.
  - 페이지네이터·렌더러 양쪽이 단일 측정 함수를 호출하도록 정합 — task993 의 단일 권위 원칙 확장.
  - `LAYOUT_OVERFLOW` 회귀 0건 복원.
- 제외:
  - task993 컷 디스크립터 스키마(`RowCut` start_cut/end_cut) 유지.
  - `advance_row_cut` 의 split 결정 의미 보존(vpos 리셋 = hard break, intra-row 분할).
  - 비공개 샘플 커밋, HWP3, 파서.

1. 측정 시스템 조사 — `HeightMeasurer::measure_*` 와 `cell_units` 의 정확한 알고리즘 차이 추출, 항목별 측정 분기 매핑.
