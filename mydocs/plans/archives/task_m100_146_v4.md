# 수행계획서 v4: Heavy display 폰트 face → font-weight=bold 매핑

- **타스크**: [#146](https://github.com/edwardkim/rhwp/issues/146)
- **마일스톤**: M100
- **브랜치**: `local/task146`
- **작성일**: 2026-04-23
- **이전 버전**: v1/v2 (Geometric Shapes), v3 (TAC 표 선행 공백)

## 1. 배경

v3 완료 이후 작업지시자 추가 지적: **폰트 볼드 처리 안되는 버그**.

PDF(한컴) 제목 "□ 국어 변화…" 는 HY헤드라인M (Display Heavy) 으로 렌더되어 시각적으로 bold. rhwp SVG 는 HY헤드라인M 이 설치 안된 환경에서 `Malgun Gothic` regular 로 fallback → 시각적 bold 소실.

### 원인 분석 (완료)

| 경로 | 상태 |
|------|------|
| HWP CharShape.bold 파싱 (`doc_info.rs:525` `(attr & 0x02) != 0`) | **정상** |
| Resolver (`style_resolver.rs:342` `bold: cs.bold`) | **정상** |
| SVG 출력 (`svg.rs:1787` `if style.bold { font-weight="bold" }`) | **정상** |
| `biz_plan.hwp` bold 출력 개수 (17~642) | **정상** |

**→ 메커니즘은 정상**. 증상의 실체는 **heavy display face 가 폰트 치환 시 regular 로 떨어지는 시각적 괴리**.

### 유력 heavy face 목록 (한컴/한양 Display-Heavy 계열)

- HY헤드라인M, HYHeadLine M, HYHeadLine Medium
- HY견고딕, HY견명조, HY견명조B
- HY그래픽, HY그래픽M

이 faces 는 HWP CharShape.bold=false 이어도 본래 **글꼴 자체가 heavy**. browser 에서 fallback 될 때 font-weight=bold 를 강제하면 Malgun Gothic Bold 등으로 렌더되어 PDF 시각에 근사.

## 2. 목표

`is_heavy_display_face(&str) -> bool` 헬퍼를 추가하고, SVG 출력 시 `style.bold || is_heavy_display_face(&style.font_family)` 로 bold 적용.

**범위 (IN)**
- heavy face 판정 헬퍼 (8개 face 화이트리스트)
- SVG 출력 분기 (svg.rs:215, 1346, 1420, 1787 4곳) 에 반영
- 단위 테스트: heavy face 리스트별 bold 판정 검증
- 재현 검증: `samples/text-align.hwp` 제목이 PDF 와 bold 시각 근사

**범위 (OUT)**
- 실제 HY헤드라인M 폰트 번들/배포 (라이선스 이슈)
- WASM/Canvas 렌더 경로 동일 적용 (본 타스크 SVG 한정, 별도 조치 시 web_canvas.rs 추가 가능)
- 실제 글리프 메트릭 재측정 (이미 is_fullwidth_symbol 로 처리됨)
