# Task M100-1099 수행 계획서

## 1. 작업 개요

GitHub Issue #1099:

```text
[hwpx2hwp] exam_kor.hwpx HWP 저장 파일손상 contract 분리 처리
```

#1098에서 `exam_kor.hwpx` 계열의 HWPX 렌더링은 HWP 정답지와 유사한 수준까지 개선했다.
이번 이슈는 렌더링이 아니라 HWPX를 HWP로 저장할 때 한컴 에디터가 파일손상으로 판정하는
HWP5 저장 contract 문제를 분리해서 처리한다.

## 2. 대상 파일

```text
source HWPX:
  samples/hwpx/exam_kor.hwpx
  samples/hwpx/exam-kor-1p.hwpx
  samples/hwpx/exam-kor-2p.hwpx
  samples/hwpx/exam-kor-3p.hwpx
  samples/hwpx/exam-kor-4p.hwpx

oracle HWP:
  samples/exam_kor.hwp
  samples/exam-kor-1p.hwp
  samples/exam-kor-2p.hwp
  samples/exam-kor-3p.hwp
  samples/exam-kor-4p.hwp
```

## 3. 선행 결론

```text
1. HWPX 렌더링은 #1098에서 먼저 개선했다.
2. 1페이지 축소 샘플 저장 HWP에서도 한컴 에디터 파일손상 판정이 발생했다.
3. 따라서 저장 실패 원인은 전체 페이지 수가 아니라 1페이지에도 존재하는 HWP5 record contract
   누락 또는 잘못된 record 관계일 가능성이 높다.
4. rhwp-studio 재로드 성공은 충분 조건이 아니다. 판정 기준은 한컴 에디터 정상 로딩이다.
```

## 4. 목표

```text
1. 1페이지 축소 샘플에서 파일손상 원인을 먼저 좁힌다.
2. HWPX→HWP 저장 결과와 한컴 정답 HWP의 record/stream 차이를 비교한다.
3. 저장기에서 누락된 HWP5 control, DocInfo, SectionDef, ColumnDef, master page, tail/count/level
   contract를 특정한다.
4. 최소 수정 후보를 만들고 한컴 에디터 판정을 통해 검증한다.
5. 1페이지 성공 후 2/3/4페이지, 최종 `exam_kor.hwpx` 순서로 확장한다.
```

## 5. Stage 1 계획 — 1페이지 파일손상 contract trace

출력 위치:

```text
output/poc/hwpx2hwp/task1099/stage1_1p_contract_trace/
```

작업:

```text
1. `samples/hwpx/exam-kor-1p.hwpx`를 현재 저장기로 HWP 저장한다.
2. 생성 HWP가 rhwp에서 reload 가능한지와 한컴 에디터 파일손상 판정 결과를 기록한다.
3. 정답 HWP `samples/exam-kor-1p.hwp`와 생성 HWP의 다음 항목을 비교한다.
   - FileHeader / DocInfo stream inventory
   - BodyText section record tree
   - SectionDef / ColumnDef / master page 관련 record
   - LIST_HEADER / PARA_HEADER / CTRL_HEADER level, parent, size, tail
   - BinData, BorderFill, CharShape, ParaShape count/reference
4. diagnostics 도구가 제공하는 summary/diff 산출물을 함께 저장한다.
5. 파일손상 지점 후보를 record 단위로 3개 이하로 압축한다.
```

산출물:

```text
generated_reload.md
stream_inventory_diff.md
record_tree_diff.md
section_masterpage_contract_diff.md
contract_violation_candidates.md
```

## 6. Stage 2 계획 — 최소 후보 생성

Stage 1에서 특정된 contract 후보를 기준으로 판정용 HWP를 만든다.

원칙:

```text
1. 이미 성공한 HWPX 렌더링 수정은 재검증 대상으로 되돌리지 않는다.
2. 성공 후보는 반드시 `exam-kor-1p`에서 먼저 한컴 에디터 정상 로딩을 통과해야 한다.
3. 1페이지가 실패하면 2/3/4페이지 후보를 생성하지 않는다.
4. rhwp-studio 정상 렌더링만으로 성공 판정하지 않는다.
```

## 7. 검증 계획

필수 검증:

```text
cargo fmt --check
cargo check
cargo test 관련 회귀 테스트
cargo build --bin rhwp
```

한컴 판정:

```text
1. `exam-kor-1p` 저장 HWP가 파일손상 없이 열리는가
2. 페이지 바탕쪽, 지문 박스, 다단/본문 배치가 #1098 렌더링 기준을 깨지 않는가
3. 1페이지 성공 후 2/3/4페이지 축소 샘플이 순차적으로 열리는가
4. 최종 `exam_kor.hwpx` 저장 HWP가 열리는가
```

```text
3. 확장 샘플과 최종 샘플에 대한 다음 단계 전략이 명확해진다.
4. 작업지시자 판정을 통과한다.
```
