# Task #888 수행 계획서

## 1. 이슈 요약

**Issue #888**: Integrate #854 POC fixes for HWPX IR clone HWP save compatibility

**목표**: #854 POC 에서 한컴 에디터 정답 판정까지 확인한 HWPX -> IR -> HWP 저장 호환 정정을 `local/devel` 기반 통합 브랜치에 선별 반영한다.

## 2. 배경

#854 POC 는 `basic-table-01.hwpx`, `expense_report.hwpx` 를 중심으로 다음 결함을 단계적으로 분리했다.

- HWPX 출처 table raw payload 누락으로 한컴 에디터 파일 손상 판정 발생
- TAC table 배치가 종이 왼쪽으로 붙는 문제
- 문단/문자 BorderFill 의 투명 no-fill 배경 무늬가 한컴에서 잘못 해석되는 문제
- HWPX `secPr/pageBorderFill` 이 HWP `SectionDef` page border/fill 로 보존되지 않아 쪽 배경이 미출력되는 문제

Stage 14 에서 작업지시자 판정 기준으로 정답지와 맞는 결과를 확보했다.

## 3. 통합 원칙

- POC 산출물 전체를 머지하지 않고 검증된 최소 코드만 이식한다.
- raw BodyText/DocInfo graft 방식은 제외한다. Stage 13 에서 텍스트 박스화, 병합 셀 border 오염, 투명 border 같은 부작용이 확인되었다.
- table 호환성은 IR adapter 의 materialization 으로 해결한다.
- page background 호환성은 HWPX section parser 의 `pageBorderFill` 정합 매핑으로 해결한다.
- 기존 `local/devel` dirty 상태를 만들지 않기 위해 별도 worktree/중간 브랜치에서 진행한다.

## 4. 적용 범위

### 4.1 HWPX -> HWP adapter

- Hancom table: `row_sizes`, outer margin, page break, table record attr, ctrl header attr/height materialization
- TAC table: table ctrl attr + row cell-count 기반 row size materialization
- paragraph/char BorderFill no-fill 정규화
- adapter report 필드 확장

### 4.2 HWPX section parser

- `hp:pageBorderFill` 파싱
- page border/fill attr bit 매핑
- `SectionDef.page_border_fill`, `extra_page_border_fills` 보존

### 4.3 테스트/샘플

- `samples/hwpx/basic-table-01.hwpx`
- `samples/hwpx/expense_report.hwpx`
- `tests/hwpx_to_hwp_adapter.rs` 에 POC 확정 회귀 테스트 추가

## 5. 비범위
