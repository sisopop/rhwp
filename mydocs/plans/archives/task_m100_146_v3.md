## 1. 추가 배경

v2 수정(Geometric Shapes U+25A0-U+25FF 전각 처리)으로 제목 좌표가 PDF 와 일치하게 수정되었으나, 같은 샘플 `samples/text-align.hwp` 의 **표 위치가 body_left 에 밀착**하는 현상이 남아 있음이 작업지시자 추가 지적으로 확인되었다.

### 증상

| 항목 | PDF (pt→px 환산) | rhwp SVG | 차이 |
|------|----------------|----------|------|
| 표 외곽 좌측 | ≈ 112 px (body_left + 36 px) | 75.58 px (body_left) | **-36 px** |

HWP 원문 문단 0.2 는 `"    "` (선행 공백 4개) + 표(TAC=true, wrap=TopAndBottom) 로 구성. PDF 는 공백 4개를 advance 해 표를 오른쪽으로 밀어내지만, rhwp 는 공백을 무시하고 표를 body_left 에 배치.

### 원인 규명

1. `pagination` 엔진은 TAC 표 문단(para 2)에 `PageItem::Table` 만 발행하고 `PageItem::FullParagraph` 는 발행하지 않음.
2. 따라서 `paragraph_layout.rs:1249-1676` 의 TAC 분기(선행 공백 → 표 배치 로직)가 **호출되지 않음**.
3. 결과적으로 `tree.set_inline_shape_position` (line 1638) 이 호출되지 않아 `inline_shape_position` 가 비어있음.
4. `layout.rs:1948-1959` 에서 `is_tac && inline_pos.is_none()` 케이스의 `tbl_inline_x = None` 으로 fallback, 표는 `col_area.x = body_left` 에 배치됨.

### 관련 소스

- `src/renderer/pagination/engine.rs:833, 1139` — `PageItem::Table` 발행 지점
- `src/renderer/layout.rs:1886-1918` — 호스트 문단 텍스트 렌더링 (`!is_tac && !text_already_laid_out` 가드로 TAC 시 스킵)
- `src/renderer/layout.rs:1948-1959` — `tbl_inline_x` 계산
- `src/renderer/layout.rs:1993-2001` — `layout_table` 호출 지점

## 2. 목표

`samples/text-align.hwp` 의 표 외곽 좌측 x 가 한컴 PDF (±3 px) 이내로 수렴.

**범위 (IN)**
- TAC 표 문단에서 선행 텍스트(특히 공백) 폭을 계산해 표 x 좌표에 반영
- 단위 테스트: 선행 공백 N개 + TAC 표 조합의 inline x 값 검증
- 회귀 검증: `svg_snapshot` 영향 정량화 + golden 갱신(해당 시)

**범위 (OUT)**
- 비-TAC(anchored) 표 수직/수평 offset: 이미 정상 동작
- 표 내부 셀 텍스트 · 셀 병합 · 쪽 나눔: #249/#251 등 별건 기조치
- HY헤드라인M / 휴먼명조 등 폰트 실체 번들: 폰트 전략 별도 범위

## 3. 수정 방식 비교

| 방식 | 장점 | 단점 | 판정 |
|------|------|------|------|
| A. 페이지네이션이 TAC 문단에 `FullParagraph + Table` 쌍 발행 | 일관된 흐름 (비-TAC 와 동일 경로) | 기존 `!is_tac` 가드 재설계 필요, 회귀 위험 큼 | ✗ |
| **B. layout_table_item 에서 inline_pos 없을 때 선행 텍스트 폭을 직접 계산** | 변경 범위 최소, 영향 국소적 | 텍스트 측정 함수 재사용 필요 | **✓ 채택** |
| C. paragraph_layout 를 TAC 문단에도 호출 | 기존 TAC 분기 그대로 재사용 | 이중 표 렌더 위험, 가드 신설 필요 | ✗ |

**방식 B 선택**: `layout.rs:layout_table_item` 내부에서 `is_tac && inline_pos.is_none()` 분기를 추가하여 문단 `composed.lines` 의 첫 번째 줄에서 TAC 위치 이전의 텍스트/공백 폭을 합산, `tbl_inline_x = col_area.x + effective_margin + leading_w` 로 설정.
