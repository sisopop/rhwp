# 수행계획서 — Task #1236: 미주(해설) 영역 줄간격 압축

- **이슈**: #1236 (M100 / v1.0.0)
- **브랜치**: `feature/issue-1236-endnote-line-spacing` (base: `stream/devel` f83c43b5)
- **성격**: 레이아웃 버그 — 미주 영역 줄간격(line spacing) 계산
- **작성일**: 2026-06-02

## 1. 배경과 증상

`samples/3-11월_실전_통합_2022.hwpx` 의 **풀이(미주) 페이지(10~14쪽)** 에서 본문 줄간격이
한컴 PDF 정답지보다 **좁게** 렌더된다. 작업지시자가 문8·문11·문15·문19·문22·문24 등 다수
지점을 지적했으며, 모두 미주(해설) 영역이다.

## 2. 조사 결과 (확인 완료)

- **미주 영역 한정**: 문제 페이지(1~9쪽, 일반 문단)는 PDF 와 줄간격 정합(정상, 4쪽 시각 확인).
  미주 페이지(10쪽~)만 줄간격이 좁다(12쪽 문19 시각 확인 — rhwp 촘촘 vs PDF 여유).
- **별도 레이아웃 경로 정황** (`dump-pages`):
  - 일반 문단: `FullParagraph  h=20.0 (… lh=14.0 ls=6.0 …)` — 줄높이(lh)+줄간격(ls) 분해 표시.
  - 미주 문단: `FullParagraph[미주]  pi=…  vpos=…` — lh/ls 분해 **없이 vpos 만** 표시.
- 연속 단문단 vpos 델타 ≈ 1352 HWPUNIT 로 PDF 대비 작음.
- 표(TopAndBottom) 주변에서 특히 두드러짐(문15·문19 지적과 정합).
- 미주 레이아웃은 `src/renderer/layout/picture_footnote.rs` / `paragraph_layout.rs` 경로로 추정.

## 3. 목표

미주(해설) 영역의 줄간격을 일반 문단과 동일한 규칙(ParaShape line spacing)으로 계산해
한컴 PDF 정답지와 정합시킨다. **문제 페이지(정상) 회귀 없음**을 보장한다.

## 4. 적용 범위

| 영역 | 내용 |
|------|------|
| 조사 | 미주 라인 레이아웃의 줄간격(ls) 계산 위치 특정, 일반 문단과 차이 규명 |
| 회귀 여부 | bisect 로 최근 도입/장기 여부 확인 |
| 수정 | 미주 줄간격 계산을 ParaShape 기반으로 정합 (`layout/` 미주 경로) |
| 검증 | 미주 풀이 페이지 ↔ PDF 줄간격 정합 + 문제 페이지 회귀 없음 |

### 범위 밖

- 각주(footnote) 줄간격 — 본 건은 미주(endnote) 한정(필요 시 동일 점검은 별도 판단).
- 표 자체의 outer_margin(상하 여백) — 표-문단 간격이 별개 원인이면 분리 검토.
- 일반 문단 줄간격 — 정상이므로 무변경.

## 5. 검증 방법

1. **시각 정합**: `export-svg 3-11월_실전_통합_2022.hwpx -p {9,10,11,13}` ↔ PDF 동일 페이지
   줄간격 대조(문8·문11·문15·문19 지점).
2. **수치 정합**: `dump-pages` 로 미주 문단 줄간격(vpos 델타)이 일반 문단 규칙(lh+ls)과
   동일하게 계산되는지 확인.
3. **회귀**: 문제 페이지(1~9쪽) 줄간격 불변 + `cargo test --lib` 통과.
4. **다문서**: 다른 미주 포함 문서(있으면) 1~2종 추가 확인.
