# Task #69: 오픈소스 대체 폰트 메트릭 보정 — 수행계획서

## 목표

Task #67에서 도입한 오픈소스 대체 폰트(Pretendard, D2Coding, GowunBatang, GowunDodum)의 메트릭 데이터를 생성하여 텍스트 폭 측정의 정확도를 높인다.

| 폰트 | 메트릭 있음 | 비고 |

## 구현 단계

### 1단계: font-metric-gen으로 메트릭 생성

- `web/fonts/`의 woff2는 메트릭 추출 불가 (TTF 필요)
- Google Fonts에서 TTF 다운로드 또는 woff2→TTF 변환
- `cargo run --bin font-metric-gen -- --dir <경로> --output src/renderer/font_metrics_data.rs`

### 2단계: 메트릭 별칭(alias) 매핑 추가

- `font_metrics_data.rs`에서 Pretendard → 맑은 고딕 폴백 등 별칭 확인
- style_resolver의 폰트 치환 체인과 메트릭 조회가 일치하는지 검증

### 3단계: 렌더링 비교 검증

- 기존 샘플 문서로 줄바꿈 위치, 문자 간격 비교
- `cargo test` 전체 통과
