## 배경

#976은 `samples/복학원서.hwp`의 중앙 JPEG 워터마크를 한컴 PDF 참고 렌더에 가까운 opaque PNG로 선보정하고, SVG / WebCanvas / rhwp-studio overlay 경로에서 후단 filter, opacity, mix-blend-mode 중복 적용을 막았다.

하지만 현재 구현은 워터마크 JPEG 판정과 baked PNG 생성이 `src/renderer/svg.rs` 중심에 있고, legacy SVG / WebCanvas / Studio overlay JSON 경로가 각각 유사한 재판정과 후처리 생략 정책을 가진다. 반면 native Skia PNG export와 browser CanvasKit renderer는 `PageLayerTree` / `PaintOp::Image`를 직접 replay하는 방향으로 이동 중이다.

따라서 #992처럼 native Skia renderer 안에 #976 처리를 개별 추가하면 단기 증상은 해결할 수 있지만, renderer별 image payload 의미가 계속 갈라질 위험이 있다. 이번 작업은 baked watermark를 문서 IR 변경이 아니라 `PageLayerTree` 단계의 visual projection으로 일반화해 replay renderer들이 같은 payload를 쓰게 하는 것을 목표로 한다.

## 작업 목표

- 워터마크성 JPEG 판정과 baked PNG 변환을 renderer 공통 resolver로 이동한다.
- `PaintOp::Image` 생성 단계 또는 그 직전에서 이미지 visual payload를 resolve한다.
- resolved baked watermark에는 후단 `ImageEffect`, brightness/contrast, opacity, blend 정책이 중복 적용되지 않도록 의미를 명시한다.
- native Skia PNG export와 browser CanvasKit direct renderer가 같은 resolved image payload를 사용한다.
- Studio overlay JSON이 renderer별 별도 bake 판정 없이 같은 resolved 결과를 사용하게 한다.
- legacy SVG는 기존 #976 동작을 유지하되, 위험이 낮으면 같은 resolver 호출로 연결한다.

## 포함 범위

### Stage 1 — 현재 경로와 계약 진단

- #976 이후 최신 `devel` 기준으로 워터마크 bake 관련 코드 위치를 정리한다.
- `RenderNodeType::Image` → `PaintOp::Image` → PageLayerTree JSON → native Skia / browser CanvasKit / Studio overlay 경로를 추적한다.
- 현재 `PaintOp::Image`와 `ImageNode`가 가진 필드로 resolved bytes / MIME / baked 여부를 표현할 수 있는지 확인한다.
- `samples/복학원서.hwp` 1페이지에서 중앙 워터마크가 PageLayerTree JSON과 native Skia 경로에서 어떻게 노출되는지 확인한다.
- 일반 JPEG/PNG/PCX 이미지에 오탐 적용될 수 있는 지점을 분리한다.

**산출물**: `mydocs/working/task_m050_1016_stage1.md`

## 제외 범위

- `Document` IR 또는 parser 단계에서 원본 JPEG를 baked PNG로 변경하지 않는다.
- 일반 JPEG/PNG/PCX 이미지의 렌더링 정책을 바꾸지 않는다.
- legacy SVG 전체를 PageLayerTree renderer로 전환하지 않는다.
- Task #975의 PageBackground fill mode 보정과 섞지 않는다.
- HWP3 전용 분기를 추가하지 않는다.
- 이슈 close는 작업지시자 승인 전에는 수행하지 않는다.

## 위험 및 대응

| 위험 | 가능성 | 대응 |
|------|--------|------|
| resolved payload를 문서 IR처럼 취급해 원본 의미가 손상됨 | 중간 | `Document`/parser 변경 금지, PageLayerTree visual projection으로 제한 |
| `PaintOp::Image` 확장으로 JSON 소비자 호환성 깨짐 | 중간 | additive 필드와 schema minor bump 검토, 기존 필드 유지 |
| legacy SVG와 PageLayerTree 경로가 일시적으로 다른 resolver를 사용 | 중간 | 기존 #976 테스트 유지, 연결 위험이 크면 후속 이슈로 분리 |
| Skia/CanvasKit에서 effect 생략 정책이 누락되어 이중 보정 발생 | 중간 | baked watermark flag를 테스트로 검증 |
