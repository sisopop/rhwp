### 1.2 estimate_text_width 호출처 (PUA 영향 영역)

| 파일:라인 | 호출 컨텍스트 | 영향 |
|----------|-------------|------|
| `composer.rs:920` | LineSeg cs/lh 검증 (`estimate_line_seg_width`) | 줄간격 계산 |
| `layout.rs:3444` | wrap=Square 호스트 텍스트 폭 추정 | Square wrap 영역 |
| `layout.rs:3510/3516/3522` | line 시작 estimation | layout pass |
| `table_layout.rs:860` | 셀 컨텐츠 폭 측정 | 셀 정렬 |
| `table_layout.rs:1657/1814/1840/1922` | inline shape 의 text_before 폭 | inline shape 좌표 |

→ PUA 영역 텍스트가 들어간 **모든 layout 측정 경로** 가 영향. 자모 시퀀스 (3-4 char) 가 PUA 1-char 의 폭으로 측정 → 실제 표시 폭과 다름.

### 1.3 가능한 가시 영향

1. **Square wrap / 줄바꿈 위치** — 자모 시퀀스가 PUA 1-char 보다 넓으면 (대부분의 폰트 메트릭에서 가능) 줄에 더 많은 글자 배치된 것으로 인식, 실제 출력은 우측 overflow
2. **줄간격** — 자모 합자 height vs PUA char height 차이 (lh 산출에 추정 폭 사용 가능성)
3. **TAC / 인라인 그림 정합** — 셀 inline shape 좌표 산출에 text_before 폭 사용 → -X px 시프트 가능
4. **paragraph border / 박스 폭** — `wrap=Square` 호스트 박스의 우측 경계 산출

## 2. 정정 옵션 (Stage 1 진단 후 결정)

| 옵션 | 본질 | 영향 범위 | 인덱싱 영향 |
|------|------|----------|-----------|
| **A** | `display_text` 우선 사용 (`run.display_text.as_ref().unwrap_or(&run.text)`) | 좁음 — 호출처 전체 (~10건) 패치 | 영향 없음 (인덱싱은 PUA 1-char 유지) |
| **B** | IR 단계에서 자모 시퀀스로 변환 + `char_offsets` 재계산 | 광범위 — Composer + 다수 모듈 | **인덱싱 변경 필수** (text/index 동조) |
| **C** | Option A + display_text 캐시 + width 캐시 (성능) | A 와 동일 | 영향 없음 |

### 2.1 사전 권장: 옵션 A

이유:
- **인덱싱 불변성 유지** — Task #528 의 trade-off 가 정합 (composer / line_seg / char_offsets 재작업 회피)
- **변경 범위 최소** — 호출처 ~10건만 패치
- **회귀 위험 낮음** — 비-PUA 텍스트는 동일 동작 (`unwrap_or(&run.text)` 로 fallback)
- **HWP 표준 정합** — 한컴이 본질적으로 자모 시퀀스 폭 기준으로 line_seg 산출 (IR 의 cs/lh 가 자모 기준)

옵션 B 는 광범위 영향 + Composer/index 재작업 필요로 회귀 위험 큼. 옵션 C 는 성능 보강이지만 Stage 1 측정 후 필요 시 결정.
