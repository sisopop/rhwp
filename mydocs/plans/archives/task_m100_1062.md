# 수행계획서 — Task #1062: 다단 빈 문단 연속 trailing-ls drift (시험지 본문 하단 overflow)

- 이슈: edwardkim/rhwp#1062 (#1049 후속)
- 브랜치: `local/task1062` (devel 기반, #1049 수정 `09014c63` 포함)
- 마일스톤: M100 (v1.0.0)

## 1. 배경 / 문제

다단(2단) 시험지 문서에서 빈/짧은 문단이 연속될 때, 렌더러가 본문 하단을 넘어 문단을
아래로 적층(`LAYOUT_OVERFLOW`)한다. 한컴 2022 PDF 대비 페이지 수도 부족하다.

| 파일 (HWP·HWPX 동일) | LAYOUT_OVERFLOW item | 최대 초과 | 우리/PDF 쪽수 |
|------|------|------|------|
| 3-09월_교육_통합_2022 | 151 / 150 | ~832px | 21 / 23 |
| 3-09월_교육_통합_2023 | 116 / 116 | ~897px | 19 / 20 |
| 3-10월_교육_통합_2022 | 110 / 110 | ~332px | 16 / 18 |
| 3-11월_실전_통합_2022 |  92 /  92 | ~556px | 19 / 21 |

전체 LAYOUT_OVERFLOW(item) 1081건 중 **937건(87%)** 이 이 4종(×2포맷)에 집중. 전부 `FullParagraph`.

## 2. 근본 원인 (조사 완료)

빈/짧은 단일 줄 문단에서 (`RHWP_TYPESET_DRIFT` 진단):
`vpos_h=12.0` (LINE_SEG 실측), `fmt_total=18.0` (= lh 12 + ls 6), `diff=+6.0` = trailing line_spacing.

| 엔진 | 누적 | 비고 |
|------|------|------|
| 페이지네이터 `typeset.rs:1834-1841` (다단) | `+= height_for_fit` (total_height − trailing_ls) | Task #391 의도적 제외 |
| 렌더러 `height_cursor.rs:120-159` | vpos→y + lazy_base + 조건부 trailing_ls bridge | 문단당 ~18px → +6px 초과 |

문단당 +6px 차이가 빈 문단 다수 페이지(예: page 18, 88문단)에서 ~530px 누적
→ 페이지네이터 "fit", 렌더러 본문 밖 적층. 평균 drift +6.05px/문단.

HWP=HWPX 동일 → 파서 무관, 렌더러↔페이지네이터 측정 계약 문제.

## 3. #1049 와의 경계

- #1049(PR #1054, devel `09014c63` 반영) = **인라인 TAC 표 직후** trailing-ls 이중 차감 해결.
  가드 `vpos_continuous && prev_has_text` 둘 다일 때만 bridge off.
- 본 건 = **빈 문단 연속**(`prev_has_text == false`) → 가드 미발동 → bridge 유지 → drift 잔존.
- 즉 #1049 가 손대지 않은 별개 manifestation.

## 4. 목표

- 대상 4종(×2포맷)의 본문 하단 LAYOUT_OVERFLOW 를 한컴 2022 PDF 정합 수준으로 해소
  (페이지 수 정합 + 빈 문단 연속 구간 +6px drift 제거).
- 정량 지표: 대상 파일 LAYOUT_OVERFLOW item 대폭 감소, 페이지 수 PDF 일치.

## 5. 제약 — 비회귀 (필수 보존)

trailing-ls 처리를 건드리므로 아래를 깨면 안 된다. 특히 **빈 문단 뒤 처리에서 시험지와 정면 충돌**한다.

| 케이스 | 요구 | 출처 |
|------|------|------|
| 복학원서 page1 (빈 문단 뒤 폼 표) | `prev_has_text==false` 시 bridge **유지** | #1049 명시 보존 |
| footnote-01 (vpos gap) | bridge 유지 | #1049 |
| exam_eng 8p 단 채움 | 다단 height_for_fit 누적 | #391 |
| k-water-rfp p3 (311px) | 단단 total_height 누적 | #359 |
| exam_kor p5 표 overflow | lazy_base 보정 | #1022 |

→ "빈 문단 뒤 trailing-ls 제거(시험지)" vs "빈 문단 뒤 bridge 유지(복학원서)" 를 가르는
**더 정밀한 구분 조건**이 본 과제의 핵심 난점이다.
