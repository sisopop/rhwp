## 2. 적용 항목 (작업지시자 결정: A, C, B, F)

### A. 테스트 중복 제거 (ci.yml)
- `Canvas layer parity tests` step 제거.
- **근거 (검증 완료)**: `canvas_layer_tree_matches_legacy*` 는 `src/renderer/canvas.rs` 의
  feature gate 없는 lib 테스트 → `cargo test`(Run tests)에 이미 포함됨. 별도 실행은 순수 중복.
- **주의 (검증으로 확정)**: `native-skia` 는 `default = ["console_error_panic_hook"]` 에 없음 →
  `cargo test` 가 native-skia 테스트/예제(32개 중 required-features 항목)를 포함하지 않음.
  **Native Skia tests step 은 유일 실행 경로이므로 유지.**
- 절감: ~36s.

### B. CodeQL rust cargo 캐시 추가 (codeql.yml)
- rust matrix step 에 `actions/cache` 추가 (ci.yml 과 동일 path, 별도 key namespace
  `${{ runner.os }}-codeql-rust-${{ hashFiles('**/Cargo.lock') }}`).
- cold build → warm build. 절감: CodeQL rust ~4~5분 (러너 분 절약; CodeQL 단독 지연 대비).
- matrix 의 js-ts/python step 은 rust 전용 cache step 에 `if: matrix.language == 'rust'` 로 한정.

### C. Free disk space 축소 (ci.yml: build-and-test + wasm-build job)
- 캐시 정상 작동 확인 → Task #1109 의 전체 정리(76s)를 "빠르고 큰" 항목만 남겨 축소.
- 유지: `/usr/local/lib/android`(대용량), `/usr/share/dotnet`(대용량).
- 제거 검토(느리거나 효과 작음): `sudo docker image prune --all --force`(느림),
  `apt-get clean`, `/opt/ghc`, `/opt/hostedtoolcache/CodeQL`.
- **안전장치**: 보수적으로 축소, 변경 후 첫 CI 로그 `df -h` 로 여유 확인. 부족 시 즉시 복원.
- 절감: ~40~60s.

### F. concurrency 취소 그룹 (ci.yml, codeql.yml, render-diff.yml)
- 각 워크플로우 최상위에 추가:
  ```yaml
  concurrency:
    group: <workflow-name>-${{ github.workflow }}-${{ github.head_ref || github.ref }}
    cancel-in-progress: ${{ github.event_name == 'pull_request' }}
  ```
- **핵심 안전장치**: `cancel-in-progress` 를 **PR 이벤트에서만 true**. devel/main push run 은
  취소하지 않음(머지 검증 run 보존) → F 의 머지 검증 누락 위험 제거.
- `head_ref || ref` 로 PR 단위 그룹핑.
- 효과: 컨트리뷰터 연속 push 시 진행 중 PR run 자동 취소 → 중복 실행 제거.
