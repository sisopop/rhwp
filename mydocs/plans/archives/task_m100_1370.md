## 3. 목표

`RHWP_EN_SSOT=A3` 위에서 break 게이트를 정확 sim 단일 신뢰원으로 재보정해 **회귀 13건을 green** 으로
복원한다. 동시에:
- `issue_1082` 5 exam overflow A3 **5/5 유지**
- 기본(B) 전체 cargo test **무회귀**(A3 게이트 내에서만 수정)
- 전 exam(1082/1139/1189/1209/1284) **동시 green** 을 매 단계 게이트로

완료 후속(별 타스크 가능): A3 기본 승격 + 레거시(per-para) 경로 정리, export-svg CLI 158쪽 폭발, 성능(O(n²)).

## 4. 핵심 제약 (메모리 정합)

- **비단조 cascade** (`tech_endnote_overflow_nonmonotonic_gate`): 큰 추정이 overflow 를 늘릴 수 있음.
  단일 문서 최적화 금지 → 전 exam 동시 green 게이트. 정확 측정만으론 fit-parity 불가, break 게이트
  공동 재보정이 정답.
- **검증 경로**: A3 `export-svg` CLI 는 158쪽으로 폭발(별개 아티팩트). 검증은 **반드시**
  `build_page_render_tree`/`dump_page_items`(통합테스트 경로)·`dump-pages` 로만 한다.
- `trailing` 전면 통일 금지 (`tech_trailing_model_no_ssot`) — 게이트 조건부 재보정만.
- PR 전 `--lib` 아닌 **전체 cargo test** (`feedback_full_cargo_test_before_pr`).
