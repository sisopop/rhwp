# Task #304: samples/21_언어 1쪽 바탕쪽 중복 렌더 + 우측단 누락 — 수행계획서

## 목표

`samples/21_언어_기출_편집가능본.hwp` 문서의 SVG 출력이 PDF 원본과 1쪽부터 일치하도록 수정한다. 2쪽 이후는 이미 정상이므로 1쪽 전용 문제에 집중한다.

## 현황 정리

### 증상

1. **바탕쪽 글상자 중복 렌더 (1쪽)**
   - 바탕쪽(Odd)의 "언어이해" 제목 글상자와 "홀수형" 글상자가 body 표(header 4×5 표)와 겹쳐 그려짐
   - SVG에 "언어이해"가 2회(33pt + 53pt), "홀수형"이 2회(좌상단 + 우상단) 출력됨

2. **1쪽 우측 단(column) 본문 누락**
   - `dump-pages -p 0` 결과: 단0 items=12, 단1 items=2 (페이지네이터는 단1까지 배치 완료)
   - 그러나 SVG 렌더 결과 우측 단 텍스트가 보이지 않음

### 파악된 사실 (덤프 기반)

- 섹션 0 용지: A3 (297×420mm)
- 바탕쪽 2개: `[0] Both`, `[1] Odd` (각각 "언어이해", 페이지번호, "홀수형" 글상자 보유)
- 섹션 플래그: `0xC0080004` → `hide_master_page` 비트(0x0400) **꺼짐**
- 문단 0 컨트롤: [구역정의][단정의][표] — PageDef 없음
- 바탕쪽 글상자 attr:
  - Both: `0x146A2000, 0x146A2000, 0x146A2000, 0x146A2300, 0x142A2B00`
  - Odd: `0x046A2000, 0x042A2000, 0x042A2000, 0x042A2300, 0x042A2300`

### PDF 동작 관찰

- **1쪽**: 바탕쪽에서 **페이지번호만** 표시, "언어이해"·"홀수형" 글상자는 숨김
- **2쪽 이후**: 바탕쪽 전체(`#쪽 | 언어이해 | 홀수형`) 정상 표시

## 원인 가설

### 가설 A1 — 바탕쪽 중복: 글상자 개별 "첫쪽 숨김" 속성
바탕쪽 글상자의 `attr` 비트 중 특정 비트가 "첫쪽에 표시/숨김"을 제어. 현재 파서/렌더러가 이 비트를 무시하고 있음.

### 가설 A2 — 바탕쪽 중복: body 표와 위치 충돌 시 자동 숨김
한컴의 자체 규칙상 body 표/도형이 바탕쪽 글상자 위치를 덮을 경우 바탕쪽을 자동 숨김. 우리 렌더러가 이 규칙을 미구현.

### 가설 B — 우측 단 누락: 단1 시작 y-좌표 계산 오류
상단 표(`wrap=TopAndBottom`, 높이 183px)가 단0 상단을 차지한 후 단1 y-기준점이 표 상단이 아니라 body 상단이어야 하는데 잘못 계산되어 단1이 표 뒤에 가려짐. 또는 단1 x-좌표가 본문영역 밖으로 배치됨.

## 조사 단계 (Stage 1: 원인 특정)

Stage 1-1. **바탕쪽 글상자 attr 비트 의미 조사**
- 한/글 문서파일형식 스펙(5.0) 중 도형개체 속성(`shape_object_attr`) 문서 확인
- 비교 샘플: 정상 렌더되는 다른 HWP의 바탕쪽 글상자 attr와 비교하여 "첫쪽 숨김" 비트 특정
- 결과물: `mydocs/tech/masterpage_glassbox_attr.md`

Stage 1-2. **우측 단 누락 원인 추적**
- `rhwp export-svg --debug-overlay` 로 단1 영역 좌표 확인
- `src/renderer/pagination/engine.rs`에서 `wrap=TopAndBottom` 표 이후 단1 x/y 계산 경로 추적
- 단1 PartialParagraph 렌더 시점에 좌표가 잘려 나가는지 혹은 그려지되 다른 요소에 가려지는지 확인
- 결과물: `mydocs/tech/task304_col1_trace.md`

Stage 1-3. **디버깅 도구 보강** (필요 시)
- `dump` 명령 출력에 바탕쪽/도형 글상자의 `x, y, attr` 16진수 추가
- 이 단계는 Stage 1-1, 1-2 진행 중 필요하다고 판단될 때만 수행

Stage 1-4. **원인 정리 → 구현 계획서 작성**
- Stage 1-1~1-3 결과를 종합하여 `task_m100_304_impl.md` 작성 (수정 파일·함수 식별, 3~6단계 구현 계획)
- 승인 요청
