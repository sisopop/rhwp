## 배경

v5 (`task_m100_321_v5_report.md`) 적용 후 21_언어 page 1 col 0 의 pi=7 본문 테두리
영역에서 두 가지 시각 회귀 보고:

1. pi=7 위에 "두 가로 띠"처럼 보이는 작은 별도 테두리 사각형 (pi=6 빈 문단 bf=7)
2. pi=7 본문이 border 와 너무 가까워 시각적으로 경계가 모호 (border_spacing=0)

devel branch 에서도 동일 현상 확인 → v5 회귀 아닌 **devel 기반 누적 이슈**.
근본 분석 결과 두 가지 원인:

- HWP/PDF 는 인접 문단의 **시각적 stroke 가 동일** 하면 (다른 bf_id 라도) 하나의

  인 샘플에서 default minimum 도 없음)

3 단계.

1. **Stage 7a — 가시성 + 병합 규칙 정밀화**: `any_real_width` (= 보이는 stroke 가 있는

3. **Stage 7c — 회귀 검증 + 보고서**: 992 lib + 6 golden + clippy + 페이지 수.

## 위험

- visual merge 가 너무 공격적이면 의도된 분리 사각형이 합쳐질 수 있음
