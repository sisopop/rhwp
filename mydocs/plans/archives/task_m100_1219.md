## 1. 배경 / 증상

`samples/3-09월_교육_통합_2023.hwp` 6페이지 문26 첫 줄
「공차가 양수인 등차수열 {aₙ}과 등비수열 {bₙ}에 대하여」에서 본문 한글이 겹쳐 렌더링됨.

| | 한글 글자 advance | 비고 |
|--|--|--|
| PDF (한글 2022, 정답지) | ≈ 12px (1.0em, 전각) | 정상 |
| rhwp SVG | **8.96px (0.746em)** | 약 3px씩 겹침 |

페이지 전체 분석 결과 **인라인 수식(treat-as-char)이 포함된 줄만** 압축됨
(문24 0.731 / 문26 0.747 / 문27 0.856 / 문30 0.762). 순수 텍스트 줄(문28 등)은 정상 1.0em.

작업지시자 최초 표현은 "수식 폰트 크기"였으나, 조사 결과 **수식 글리프 자체 문제가 아니라
수식이 든 줄의 본문 한글이 압축**되는 문제임.

## 2. 근본 원인 (계측으로 확정)

`src/renderer/layout/paragraph_layout.rs`

1. 정렬별 간격 분배 로직에 **비정렬(Left/Right/Center) 오버플로우 압축 분기**가 있음
   (`total_text_width > available_width` 이면 음수 자간으로 전 글자 압축).
   - 문26 문단 IR: **align=Left, 자간 0%, 장평 100%** → 압축 의도 없음.

2. 라인0 측정값(계측):
   - `available_width = 313.2px` (= 컬럼 357.2 − 번호 마커 inline_offset 44.0)
   - `total_text_width = 392.2px` → 79px 초과 → −3.04px/char 압축 → advance 8.96px

3. **392.2px가 과대계상된 원인** — `est_x` 측정 루프(라인 1777, 1784-1788):
   ```rust
   let is_last_run_est_tac = run_char_end_est >= (줄 내 run char 합) + char_start;
   ... filter: *pos < run_char_end_est
              || (is_last_run_est_tac && *pos == run_char_end_est)
   ```
   - 라인0의 끝 char 위치(26) == 라인1 첫 수식 `a₁=b₁=1`(폭 55.24px)의 위치.
   - `is_last_run_est_tac`가 참이라 **줄 경계 위치의 수식까지 라인0 폭에 합산**.
   - 세그먼트 합: SEG「공차가 양수인 등차수열 」138 + {aₙ}23 + SEG「과 등비수열 」72
     + {bₙ}22 + SEG「에 대하여 」60 = **실제 라인0 ≈315px**, 여기에 **다음 줄 수식 55px가
     덧붙어 392px**가 됨. (렌더 경로는 해당 수식을 라인1에 정상 배치 → 측정/렌더 불일치)

**결론**: 측정 루프의 줄-경계 TAC 포함 규칙이 **비-마지막 줄**에서 다음 줄의 선두 수식을
잘못 포함시켜 거짓 오버플로우 → 한글 압축.

## 3. 수정 방향

`est_x` 측정 루프의 `is_last_run_est_tac` 경계 포함 규칙을 **렌더 경로의 줄-경계 의미와
일치**시킨다. 즉 줄 끝 위치(`pos == run_char_end_est`)의 TAC는 **그 줄이 문단의 마지막 줄일
때만** 포함하고, 비-마지막 줄에서는 제외한다 (다음 줄로 넘어가는 수식이므로).

- 1차 후보: `is_last_run_est_tac` 조건에 "문단 마지막 줄" 게이트 추가.
- 렌더 경로(그리기 시 TAC→줄 배정)가 이미 올바르므로, 측정만 그 규칙에 맞춘다.
- 부수 확인: 보정 후 라인0 자연폭(≈315px)이 avail(313.2px)을 1.8px 미세 초과하는데,
  이는 −0.07px/char(시각상 0)로 무시 가능. 추가 보정 불요 예상(구현 단계에서 확인).

## 4. 범위 / 비범위

**범위**

**비범위**

- 번호 마커(inline_offset) 폭 계산 (정상).
- 양쪽정렬/배분정렬 압축 로직 (해당 문단은 Left, 무관).
- 다른 줄(문24/27/30)도 동일 원인이면 함께 해소되는지 확인하되, 별도 보정은 추가 안 함.

1. `cargo build --release` 후 `export-svg -p 5` → 문26 첫 줄 한글 advance ≈ 12px 확인.
2. PDF(`pdf/3-09월_교육_통합_2023.pdf` 6쪽)와 시각 비교 — 겹침 해소.
3. 동일 페이지 문24/27/30 동반 개선 확인 (압축 → 정상).
4. `cargo test` 전체 통과 (회귀 없음) — 특히 인라인 TAC/auto_tab_right/목차 관련.
5. 신규 회귀 테스트 추가.

## 6. 리스크

- 줄-경계 TAC 포함 규칙은 과거 타스크(문단 마지막 줄 끝 TAC 포함 목적)에서 도입됨 →
  **마지막 줄 끝에 수식이 오는 문서**의 회귀 가능성. 게이트를 "마지막 줄"로 한정하여 차단.
