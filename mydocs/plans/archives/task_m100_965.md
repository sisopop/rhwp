## 2. Root cause (PR #918 분석 기반)

`src/wmf/converter/svg/mod.rs:2191-2197` (현재 upstream/devel):

```rust
let align_vertical = [
    VerticalTextAlignmentMode::VTA_BOTTOM,
    VerticalTextAlignmentMode::VTA_TOP,    // VTA_TOP = 0x0000
]
.into_iter()
.find(|a| record.text_alignment_mode & (*a as u16) == *a as u16)
.unwrap_or(VerticalTextAlignmentMode::VTA_BASELINE);
```

`mode & VTA_TOP(=0x0000) == 0x0000` 가 **항상 true** → BASELINE (0x0018) 인 mode 도 VTA_TOP 으로 잘못 매핑 → baseline 이 cell-top 보정 (+ascent ~0.8em) 만큼 하강.

## 3. 포팅 대상 (단일 파일 ~50 라인)

`src/wmf/converter/svg/mod.rs`:

1. **`set_text_align` (~2191)**: vertical bits (0x0018 mask) 값 기준 BASELINE → BOTTOM → TOP 우선순위 분기
2. **`ext_text_out` (~813-826)**: baseline y shift 정합:
   - VTA_BASELINE → 0
   - VTA_TOP → +ascent (em × 0.8)
   - VTA_BOTTOM → -descent (em × 0.2)
   - 기존 `font.height < 0 => -font.height` 잘못된 보정 제거
3. **두번째 `ext_text_out` 분기 (~1541-1548)**: 동일 보정 (line 1541)
