# Task #672 수행 계획서

## 1. 이슈 요약

**Issue #672**: TAC 표 비례 축소 시 셀 콘텐츠 클립 — `common.height` vs measured `row_heights` 불일치

**증상**: TAC 표 (`treat_as_char=true`) 에서 측정된 row_heights 합산이 `common.height` (HWP 인코딩 권위값) 을 초과하면 모든 row_heights 가 비례 축소되어, 셀 콘텐츠 일부가 클립됨 (`cell.clip=true` 영향).

| [52] r=13,c=3 "특허 취득" | 3 paragraph 측정 후 비례 축소 → 3번째 paragraph 클립 | 3 paragraph 모두 정상 표시 |

## 4. 본질 위치

`src/renderer/height_measurer.rs:822-830`:

```rust
let table_height = if table.common.treat_as_char && common_h > 0.0 && raw_table_height > common_h + 1.0 {
    let scale = common_h / raw_table_height;
    for h in &mut row_heights {
        *h *= scale;
    }
    common_h
} else {
    raw_table_height
};
```

**메커니즘**: TAC 표에서 측정 row_heights 합 (raw) > common.height (HWP 인코딩) 시 모든 row_heights 비례 축소.

## 5. 한컴 권위 분석

한컴 뷰어가 셀 콘텐츠 3줄 / 3 paragraph 모두 정상 표시 → **한컴은 비례 축소 안 함**. 가능한 한컴 동작:

- (a) common.height 를 동적으로 확장
- (b) 측정값 우선, common.height 무시
- (c) 셀 단위 클립을 적용하지 않음 (셀 밖으로 콘텐츠 표시 허용)

## 6. 정정 방향 옵션

### 옵션 A: TAC 표 비례 축소 제거 + measured 우선

- height_measurer.rs:822-830 의 `if` 분기 제거
- 항상 `raw_table_height` 사용
- 위험: TAC 표가 본문 layout 에 영향 (다른 영역 회귀 가능)

### 옵션 B: 비례 축소 가드 강화 — content_height 보장

- `common.height` 우선이지만, `raw_table_height` 가 약간 클 때 (1~2% 등) 비례 축소 안 함
- 큰 차이 (예: > 5%) 일 때만 비례 축소 (TAC 표 사용자 의도 보존)
- 회귀 위험 좁힘 + 본 case 정정

### 옵션 C: row_heights 비례 축소 시 최소값 보장

- 비례 축소 후 각 row_heights 가 셀 콘텐츠 측정값보다 작아지지 않도록 가드
- raw_table_height 합산은 변할 수 있지만 셀 콘텐츠는 보존

### 옵션 D: cell.clip 동작 완화

- 측정값보다 작은 cell_h 라도 콘텐츠 클립 안 함 (한컴 (c) 동작 모방)
- 위험: 의도적 클립 (overflow 표시) 케이스 영향

### 옵션 우선순위 제안

**옵션 B (가드 강화)** — 회귀 위험 좁힘 + 본 case 정정 양립.

또는 **옵션 C (row_heights 최소값 보장)** — TAC 의 common.height 권위는 보존하면서 셀 콘텐츠도 보장.
