# Task #321 v4 수행 계획서 — col 0 drift 정밀 추적

## 배경

Task #321 v1~v3-정밀화 까지 진행 후, 21_언어_기출_편집가능본.hwp page 1 이 PDF 원본과
다음 차이를 보임이 확인됨.

| 항목 | PDF (정답) | 현재 (task321 HEAD = devel = v2) |
|------|-----------|-----------------------------------|
| col 0 끝 | pi=9 "프로세스 발견이란..." **전체** | pi=9 lines 0..11 (분할) |
| col 1 시작 | pi=10 "적합성 검증이란..." | pi=9 lines 11..14 "도출될 경우..." |

`rhwp dump-pages samples/21_언어_기출_편집가능본.hwp -p 0` 실측:
- col 0 used = 1233.5 px
- hwp_used (LINE_SEG vpos 기반) ≈ 1147.7 px
- **drift = +85.8 px** (= 우리 typeset 이 col 0 에서 86 px 더 누적)

86 px 누적 → pi=9 마지막 3 줄이 col 0 에 들어가지 못함 → mid-paragraph split → col 1
시작이 PDF 와 어긋남.

이 drift 는 v3-정밀화 회귀가 아닌 **devel branch 부터 존재하던 누적 오차** 이며, Task #321
의 본 주제(드리프트 추적·완화) 의 후속 이슈로 분류된다.

## 목표

col 0 drift 의 누적 원인을 pi 단위까지 분해하여 정량화하고, 어떤 항목(spacing_before/after,
line_height, border padding, table height, shape reserve, etc.) 이 얼마나 기여하는지를
식별한다. **본 stage 는 진단·정량화에 한정** 하며, 수정은 별도 stage 에서 다룬다.
