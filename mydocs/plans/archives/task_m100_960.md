X 위치 정확, Y 만 잘못 — off-by-one line 매핑 결함.

## 3. Root cause 후보

### A. composer 의 paragraph 분할 + control 매핑 영역
`src/renderer/composer.rs` — text 를 lines 로 분할 + controls (FFFC) 를 lines 에 매핑.

### B. paragraph_layout 의 inline TAC 처리 영역
`src/renderer/layout/paragraph_layout.rs:1969~` — tac_ci 의 y/baseline 계산.

### C. multi-line equation height 가 다음 line 의 vpos 계산을 corrupt
`layout_box.height` (27.6px, 2 rows) vs `hwp_eq_h` (25.8px) 의 1.8px 차이가 누적되어 line shift 가능성.
