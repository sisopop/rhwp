# Task #30: 문단 삽입/삭제 후 페이지 수 과도 증가 수정

## 수행 목표

문단 분할(Enter) 또는 병합(Backspace) 시 `measure_section_selective`의 off-by-one 인덱싱 버그로 인해 페이지 수가 과도하게 증가하는 문제를 수정한다.

## 현상

- `kps-ai.hwp` 에서 `splitParagraph(0, 199, 0)` 실행 시 페이지 수 78 → 86 (8페이지 과도 증가)
- 한 줄 추가에 페이지가 8개 늘어나는 것은 비정상

## 원인 분석

`measure_section_selective`에서 문단 삽입 후 인덱스 불일치 발생:

1. `insert_composed_paragraph`에서 `dirty_paragraphs` 비트맵에 `bits.insert(para_idx, true)` 실행 → 인덱스 1칸 밀림
2. 하지만 `prev_measured.paragraphs`는 이전 페이지네이션 시점의 인덱스를 유지
3. 결과: 삽입 지점 이후 모든 non-dirty 문단이 **한 칸 밀린 캐시 측정값** 사용
   - 현재 para 201 (= 이전 para 200, 표 624px) → `prev_measured[201]` (이전 para 201, 다른 높이)
4. 수백 개 문단의 높이가 잘못 적용되어 페이지네이션 결과 왜곡

`remove_composed_paragraph`에도 동일한 문제 존재 (문단 병합 시).

## 구현 단계

### 1단계: 버그 수정

- `insert_composed_paragraph`: `dirty_paragraphs[section_idx] = None`으로 변경 (전체 재측정 강제)
- `remove_composed_paragraph`: 동일하게 변경
- `measure_section_selective`에서 `dirty_paras = None`이면 `measure_section_incremental`로 폴백하여 인덱스 불일치 없이 정확한 측정

### 2단계: 회귀 테스트

- `test_split_paragraph_page_count_stability` 테스트 추가
- kps-ai.hwp 로드 → splitParagraph(0, 199, 0) → 페이지 수 증가 ≤ 2 검증

### 3단계: 기존 테스트 통과 확인

- 전체 778개 테스트 통과 확인
