# Task #67: 폰트 폴백 전략 수립 및 적용 — 수행계획서

## 목표

저작권 폰트 woff2 번들링을 제거하고, 이중 폰트 전략을 적용한다.
- **SVG export**: 문서에 사용된 글자만 서브셋 추출하여 SVG 내 임베딩
- **웹 Canvas / VSCode 확장**: OS 기본 폰트 우선 + 오픈소스 폰트 폴백

## 배경

- 함초롬바탕/돋움(한컴), MS 폰트 등의 woff2 번들링은 라이선스 위반 소지
- PDF 서브셋 임베딩과 동일하게, SVG에서도 사용 글자만 서브셋 임베딩은 합법 (출력물)
- 웹 Canvas 실시간 렌더링은 폰트 파일이 필요하므로 오픈소스 폰트 사용 필수

## 대체 폰트 매핑

| 카테고리 | HWP 상용 폰트 | 오픈소스 대체 | 라이선스 |
|----------|--------------|-------------|---------|
| Serif | 한컴바탕, 바탕, 함초롬바탕, HY명조 | Noto Serif KR | SIL OFL 1.1 |
| Sans-serif | 맑은 고딕, 돋움, 함초롬돋움, 한컴돋움 | Pretendard | SIL OFL 1.1 |
| Monospace | 굴림체, 바탕체 | D2Coding | SIL OFL 1.1 |

## 구현 단계

### 1단계: 저작권 폰트 제거 + 오픈소스 폰트 도입

- `web/fonts/`에서 저작권 woff2 파일 제거
- Noto Serif KR, D2Coding woff2 추가 (Pretendard는 이미 있음)
- 폰트 매핑 테이블 구현 (상용폰트명 → 대체폰트명)
- CSS font-family 3계층 폴백 체인 구성

### 2단계: Canvas 폰트 감지 + 동적 폴백

- `document.fonts.check()` 기반 OS 폰트 존재 여부 감지
- OS 폰트 있으면 우선 사용, 없으면 오픈소스 웹폰트 로드
- font-loader.ts 수정: 매핑 테이블 기반 동적 로딩

### 3단계: SVG export 서브셋 임베딩

- SVG 내보내기 시 사용된 글자 수집
- fonttools/pyftsubset 또는 Rust 라이브러리로 폰트 서브셋 생성
- SVG `<style>` 내 `@font-face` + base64 woff2 서브셋 임베딩
- 원본 폰트가 시스템에 있을 때만 서브셋 생성 (없으면 오픈소스 폰트 사용)

### 4단계: 메트릭 보정 + 검증

- 오픈소스 폰트와 원본 폰트의 메트릭 차이 측정
- 줄바꿈 위치, 문자 간격 보정 로직 추가
- 기존 샘플 문서로 렌더링 비교 테스트

## 영향 범위

- `web/fonts/` — 폰트 파일 교체
- `rhwp-studio/src/font-loader.ts` — 폰트 로딩 로직
- `src/renderer/svg.rs` — SVG 서브셋 임베딩
- `src/renderer/web_canvas.rs` — Canvas 폰트 폴백
- `src/renderer/style_resolver.rs` — 폰트 매핑 테이블

## 검증 기준

- 저작권 폰트 woff2 파일이 배포 패키지에 포함되지 않음
- 웹 Canvas에서 한글 문서가 정상 렌더링 (OS 폰트 또는 오픈소스 폰트)
- SVG export에서 오프라인 환경에서도 폰트가 올바르게 표시 (서브셋 임베딩)
- `cargo test` 전체 통과
