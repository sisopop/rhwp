# Task #362 수행계획서 — kps-ai p56 외부 표 안 콘텐츠 클립 결함

## 이슈

[#362](https://github.com/edwardkim/rhwp/issues/362) — `samples/kps-ai.hwp` p56 의 외부 표 (pi=535, 1x1 TAC) 안의 콘텐츠 (11 paragraphs + 7x6 내부 표) 가 외부 표 높이 (865.1 px) 를 초과해 셀 내용이 클립됨.

v0.7.3 에서는 정상.

## 배경

본 task 는 v0.7.3 → main 회귀 정정의 일환:
- Task #359: typeset fit drift + 단독 항목 페이지 차단
- Task #361: TypesetEngine page_num + PartialTable fit 안전마진
- **Task #362**: 외부 표 안의 콘텐츠 클립 (본 task)

근본 origin 후보:
1. **Task #35 (e60ab00)** — TAC 표 height 를 common.height 로 클램프 (`raw_table_height > common_h + 1.0` 인 경우 scale-down). 그러나 v0.7.3 도 포함하므로 단독으로는 회귀 origin 아님.
2. **Task #279 (cell padding aim)** — v0.7.3 → main 사이 height_measurer.rs 의 padding 처리 변경. row_heights 계산 영향 가능성.
3. **두 개의 누적 효과** — Task #279 의 padding 변경으로 raw_table_height 증가 + Task #35 의 scale-down 이 충분치 않거나 셀 안 콘텐츠 별도 계산.

## 증상 정량

### 외부 표 (pi=535)
- TAC 표, common.height=64879 HU (≈865.1 px)
- 셀 [0,0] h=62611 HU (≈834.8 px), padding=(510,510,141,141), border_fill_id=76
- 셀 paras=11, 그 중 p[2] = 7x6 내부 표

### 내부 표 (셀 [0,0] 의 p[2])
- 7x6 표, 셀=42
- 첫 행 셀 h=3840 HU, 데이터 행 셀 h=2660 HU
- raw_table_height ≈ 3840 + 2660*6 = 19800 HU + cell_spacing → 약 19880 HU = **265 px** (HU 가 작으므로)
- 그 외 다른 paragraphs (p[0]~p[10]) 의 vpos 정보로 누적 높이 산출

### 셀 안 누적 vs 외부 표 높이
- 외부 표 height 상한 = 865.1 px
- 셀 안 콘텐츠 누적 (텍스트 + 내부 표) > 865.1 px → 클립
