# Task M050 #938 수행 계획서 — 복학원서 워터마크 투명 배경 사각 영역 제거

## 이슈 정합

- **Issue**: [#938](https://github.com/edwardkim/rhwp/issues/938) — 복학원서 워터마크 투명 배경이 사각 영역으로 렌더링됨
- **마일스톤**: M050 — v0.5.0~v0.7.x 뼈대
- **브랜치**: `local/task938` (`upstream/devel` 기준)
- **대상 문서**: `samples/복학원서.hwp`
- **권위 자료**: `pdf/복학원서-2022.pdf`

## 현상

`samples/복학원서.hwp` 1페이지 중앙의 고려대학교 엠블럼 워터마크 주변에 옅은 사각 배경이 보인다. 한컴 2022 PDF 기준에서는 배경이 투명하게 합성되어 본문 위에 자연스럽게 올라가야 한다.

Task #677에서 워터마크의 회색조/밝기/대비/불투명도는 정정되었지만, 현재 렌더러는 워터마크 이미지 전체에 `opacity=0.17`을 적용한다. 중앙 엠블럼이 JPEG처럼 알파 채널이 없는 이미지이면, 이미지의 흰색 또는 옅은 배경까지 함께 반투명하게 칠해져 사각 영역이 남을 가능성이 높다.

선행 조사 문서 `mydocs/tech/multi_layer_rendering_strategy.md`도 같은 잔존 결함을 “엠블럼 JPEG의 흰색 배경 투명 처리 실패”로 기록했다.

## 1차 가설

1. **이미지 배경 색상키/마스크 미적용**
   - 워터마크 JPEG의 흰색 또는 근백색 배경을 한컴이 투명 배경처럼 처리하는데, rhwp는 원본 JPEG 사각형 전체를 그린다.
   - Task #514의 PCX 변환은 흰색을 투명으로 매핑했지만, JPEG 워터마크에는 같은 처리가 없다.

2. **그림 추가 속성 미해석**
   - HWP5 `SHAPE_COMPONENT_PICTURE`에는 그림 효과 정보와 추가 속성이 남아 있다.
   - 현재 parser는 `raw_picture_extra`로 보존하지만, 이미지 투명도/효과 관련 필드를 렌더링 모델로 해석하지 않는다.

3. **합성 방식 한계**
   - 현재 SVG/Canvas는 워터마크를 하나의 이미지 평면으로 처리한다.
   - 한컴은 배경 제거, 워터마크 필터, 본문 합성을 분리해서 처리할 수 있다.

## 작업 목표

- `samples/복학원서.hwp` 1페이지 중앙 워터마크의 옅은 사각 배경을 제거한다.
- 워터마크 엠블럼 자체의 시각 강도는 Task #677 승인 상태를 유지한다.
- SVG 렌더와 rhwp-studio Canvas 렌더가 같은 의미로 동작하게 한다.
- PCX 학교 로고, 일반 JPEG/PNG 그림, 기존 워터마크 메타정보 회귀를 막는다.

## 포함 범위

### Stage 1 — 재현 및 원인 확정

- 현재 `upstream/devel` 기준 SVG/Canvas 또는 PageLayerTree 출력에서 사각 배경을 재현한다.
- `samples/복학원서.hwp`의 중앙 워터마크 BinData를 추출해 이미지 포맷, 배경 픽셀 분포, 알파 채널 여부를 확인한다.
- `SHAPE_COMPONENT_PICTURE`의 `raw_picture_extra`를 분석해 투명도/효과 관련 필드가 있는지 확인한다.
- `pdf/복학원서-2022.pdf`와 비교해 “투명 처리”가 필요한 배경 영역의 조건을 좁힌다.

**산출물**: `mydocs/working/task_m050_938_stage1.md`

### Stage 2 — 구현 계획서 작성

- 배경 제거 후보를 비교한다.
  - 후보 A: 워터마크 전용 근백색 색상키 → 알파 변환
  - 후보 B: `raw_picture_extra` 기반 이미지 투명도/효과 필드 해석
  - 후보 C: SVG/Canvas 합성 방식을 분리하는 레이어 보강
- 적용 게이트를 확정한다. 일반 이미지 전체에 휴리스틱이 퍼지지 않도록 `ImageAttr::is_watermark()`와 대상 포맷/픽셀 조건을 함께 사용한다.
- SVG, Web Canvas, PageLayerTree/JSON, 테스트 범위를 확정한다.

**산출물**: `mydocs/plans/task_m050_938_impl.md`

### Stage 3 — 구현

- 승인된 구현 계획에 따라 소스 변경을 수행한다.
- 이미지 처리 로직이 renderer별로 흩어져 있으므로 `src/renderer/svg.rs`와 `src/renderer/web_canvas.rs`를 함께 점검한다.
- 필요 시 공통 이미지 전처리 헬퍼로 중복을 줄이되, 범위를 워터마크 배경 처리로 제한한다.

**산출물**: `mydocs/working/task_m050_938_stage3.md`

### Stage 4 — 회귀 검증

- `samples/복학원서.hwp` 1페이지 SVG/Canvas 시각 검증 자료를 생성한다.
- 기존 회귀 테스트를 실행한다.
  - `cargo test --release --test issue_514`
  - `cargo test --release --test issue_516`
  - `cargo test --release --test svg_snapshot`
  - 필요 시 `cargo test --release --lib`
- 새 회귀 가드를 추가한다. 최소 목표는 워터마크 이미지가 투명 배경 처리 경로를 통과하는지 검증하는 단위 또는 통합 테스트다.

**산출물**: `mydocs/working/task_m050_938_stage4.md`

### Stage 5 — 시각 판정 및 최종 보고

- 작업지시자 시각 판정으로 `pdf/복학원서-2022.pdf` 대비 사각 배경 제거 여부를 확인한다.
- 최종 결과보고서를 작성하고 오늘 할일 문서를 갱신한다.

**산출물**: `mydocs/report/task_m050_938_report.md`

## 위험 및 대응

| 위험 | 가능성 | 대응 |
|------|--------|------|
| 근백색 색상키가 일반 이미지에 오탐 적용 | 중간 | `ImageAttr::is_watermark()` + 대상 이미지 특성 + 샘플 sweep으로 제한 |
| JPEG 압축 노이즈로 배경 경계가 지저분하게 남음 | 중간 | Stage 1에서 픽셀 분포를 측정하고 임계값을 구현 계획서에서 확정 |
| SVG와 Canvas가 서로 다른 전처리를 적용 | 중간 | 공통 헬퍼 또는 동일 테스트 fixture로 양쪽 경로 검증 |
| 기존 #677 워터마크 강도 회귀 | 중간 | `issue_516`, `svg_snapshot`, 작업지시자 시각 판정으로 확인 |
| `raw_picture_extra` 해석 오류로 라운드트립 손상 | 낮음 | 읽기 모델 추가 시에도 raw 보존을 유지하고 serializer 변경은 최소화 |
