## 2. 배경

#1183 셀 내부 이미지 저장 정합 처리 중, HWP 저장 경로의 공통 속성 및 문단 계층 필드에 대해 다시 감사할 필요가 확인되었다.

이번 이슈는 새 조판 기능을 추가하는 작업이 아니라, 다음 영역의 의미와 저장 계약을 명확히 정리하는 작업이다.

- `PARA_HEADER`
- `Paragraph.raw_header_extra`
- `Paragraph.control_mask`
- `Paragraph.char_count_msb`
- `HWPTAG_PARA_LINE_SEG`
- HWPX `<hp:lineSegArray>` / `<hp:lineseg>`

## 3. 확인된 불일치

### 3.1 HWPX serializer 주석의 LineSeg 필드 개수 오류

`src/serializer/hwpx/section.rs` 주석은 `Paragraph.line_segs`의 "6개 필드"를 출력한다고 설명한다.
하지만 실제 HWP5/HWPX `LineSeg`는 다음 9개 필드를 가진다.

| HWPX 속성 | IR 필드 |
|---|---|
| `textpos` | `LineSeg.text_start` |
| `vertpos` | `LineSeg.vertical_pos` |
| `vertsize` | `LineSeg.line_height` |
| `textheight` | `LineSeg.text_height` |
| `baseline` | `LineSeg.baseline_distance` |
| `spacing` | `LineSeg.line_spacing` |
| `horzpos` | `LineSeg.column_start` |
| `horzsize` | `LineSeg.segment_width` |
| `flags` | `LineSeg.tag` |

코드는 9개를 모두 출력하므로 코드 동작보다 주석이 뒤처진 상태다.

### 3.2 `LineSeg.tag = 0x00060000` 설명 문제

여러 코드 경로에서 `0x00060000`이 "HWP 기본 플래그"처럼 사용된다.
공식 스펙상 이 값은 단순 기본값이 아니라 다음 비트 조합이다.

- bit 17: 줄의 첫 세그먼트
- bit 18: 줄의 마지막 세그먼트

단일 세그먼트 줄에는 자연스러운 값이지만, 빈 세그먼트(bit 16), 들여쓰기(bit 20), 문단 머리(bit 21)가 필요한 줄까지 포괄하는 일반 기본값으로 설명해서는 안 된다.

### 3.3 `raw_header_extra` 보존 범위 설명 문제

`raw_header_extra`는 `numCharShapes`, `numRangeTags`, `numLineSegs`, `instanceId` 등을 보존한다고 되어 있지만, HWP serializer는 count 3개를 실제 배열 길이로 재계산하고 `raw_header_extra[6..]`만 저장한다.

정확한 계약은 다음과 같다.

- count 필드: 저장 시 실제 데이터 기반 재계산
- `instanceId` 및 후속 extra: `raw_header_extra[6..]` 보존

## 4. 작업 범위

1. `LineSeg.tag` 공식 비트 의미를 모델 주석과 상수/헬퍼로 정리한다.
2. `0x00060000` 주입 지점을 전수 조사해 다음으로 분류한다.
   - 원본 보존
   - HWPX lineSegArray 누락 시 합성
   - 새 빈 문단 생성
   - 편집/reflow 후 합성
   - 테스트 전용
3. `src/serializer/hwpx/section.rs` 주석을 실제 9필드 출력 계약으로 정정한다.
4. `mydocs/tech/hwp_save_guide.md`에 문단 헤더와 LINE_SEG 저장 계약을 보강한다.
5. 공식 스펙과 실제 한컴 동작의 불일치가 발견되는 경우에만 `mydocs/tech/hwp_spec_errata.md`에 분리 기록한다.

## 5. 비범위

- HWP3 `LineSeg.vertical_pos` 누적 계산 정정은 별도 이슈로 분리한다.
- 아직 실측하지 않은 `LineSeg.tag` 비트를 추정으로 조작하지 않는다.
- 조판 로직 자체의 대규모 리팩터링은 이번 이슈에서 하지 않는다.
- HWPX lineSegArray 누락 문단의 완전한 Hancom 재조판 모방은 후속 이슈로 분리한다.
