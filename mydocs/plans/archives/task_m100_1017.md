## 배경

#1016은 워터마크성 JPEG를 `PaintOp::Image` 생성 단계의 resolved visual payload로 일반화했다. 그 결과 `samples/복학원서.hwp` 1페이지 중앙 워터마크는 PageLayerTree JSON에서 원본 JPEG가 아니라 `mime=image/png`, `bakedWatermark=true`로 노출된다.

하지만 #976의 사각형 제거 효과는 resolved bytes만으로 완성되지 않는다. #976/#1016의 baked watermark PNG는 투명 PNG가 아니라 opaque PNG이므로, `BehindText` 의미에 맞게 다음 순서로 합성되어야 한다.

```text
page background
-> wrap=behindText images
-> flow text / tables / normal content
-> wrap=inFrontOfText images
```

현재 Studio Canvas2D 기본 경로는 #516의 HTML Hybrid overlay 방식으로 이 순서를 구현한다. 반면 native Skia PNG export와 browser CanvasKit direct renderer는 PageLayerTree leaf 순서를 그대로 replay하므로, 문서/leaf 순서상 `wrap=behindText` image op가 flow text 뒤에 나오면 워터마크가 글 내용 위에 그려질 수 있다.

이번 작업은 #1016의 image payload contract를 되돌리거나 확장하는 작업이 아니라, PageLayerTree replay 단계에서 `BehindText` / `InFrontOfText` z-order 의미를 backend 공통 정책으로 보존하는 작업이다.

## 작업 목표

- PageLayerTree replay에서 `PaintOp::Image.image.text_wrap`의 `BehindText` / `InFrontOfText` 의미를 보존한다.
- native Skia PNG export가 Studio HTML Hybrid와 같은 기본 z-order 의미를 재현하게 한다.
- browser CanvasKit direct renderer가 Canvas2D overlay fallback 없이 같은 z-order contract를 따르게 한다.
- `wrap=behindText` image op가 flow content 위에 그려지는 회귀를 테스트로 막는다.
- `wrap=inFrontOfText` image op는 flow content보다 앞에 유지한다.
- 기존 #976 / #1016 resolved image payload 회귀 테스트와 Studio Canvas2D overlay 경로를 유지한다.

## 포함 범위

### Stage 1 — 현재 replay 경로와 결함 재확인

- native Skia `PageLayerTree` replay가 leaf `ops` 순서대로 실행되는 지점을 확인한다.
- browser CanvasKit direct renderer가 JSON leaf `ops` 순서대로 실행되는 지점을 확인한다.
- Studio Canvas2D HTML Hybrid 경로의 `flow` / `behind` / `front` 분리 정책을 기준 동작으로 정리한다.
- `samples/복학원서.hwp` 1페이지에서 `wrap=behindText` baked watermark op와 flow op의 상대 순서를 진단한다.
- clip / group / transform context를 잃지 않아야 하는 replay 경계 조건을 정리한다.

**산출물**: `mydocs/working/task_m100_1017_stage1.md`

### Stage 2 — 구현 계획서 작성

- replay 단계 partition, PageLayerTree logical paint plane, normalized replay iterator 후보를 비교한다.
- native Skia와 CanvasKit이 공유할 z-order contract 표현 방식을 확정한다.
- JSON schema minor bump 필요 여부와 additive 호환성 방안을 결정한다.
- 최소 회귀 테스트 구조를 확정한다.
- Stage 3 이후의 구현 단계를 3~6단계로 분해한다.

**산출물**: `mydocs/plans/task_m100_1017_impl.md`

### Stage 3 — Rust replay 정책 구현

- 승인된 구현 계획에 따라 native Skia replay가 `background -> behindText images -> flow ops -> inFrontOfText images` 순서를 보존하도록 구현한다.
- 단순 flatten으로 clip/group/transform context를 잃지 않도록 replay item 또는 tree traversal 정책을 제한적으로 설계한다.
- `wrap=behindText` / `wrap=inFrontOfText` image op와 flow paint op의 분리 replay를 단위 테스트로 고정한다.

**산출물**: `mydocs/working/task_m100_1017_stage3.md`

### Stage 4 — CanvasKit direct replay 연결

- browser CanvasKit direct renderer가 Rust/native Skia와 같은 z-order policy를 따르도록 연결한다.
- Canvas2D overlay fallback으로 숨기지 않고 direct renderer 내부 정책으로 처리한다.
- TypeScript 타입 또는 PageLayerTree JSON contract 변경이 필요한 경우 additive 방식으로 반영한다.

**산출물**: `mydocs/working/task_m100_1017_stage4.md`

### Stage 5 — 검증 및 최종 보고

- `samples/복학원서.hwp` 1페이지 native Skia PNG export에서 중앙 `wrap=behindText` baked watermark가 글 내용 위에 그려지지 않는지 확인한다.
- opaque baked PNG의 흰 사각 배경이 본문 위에 노출되지 않는지 확인한다.
- CanvasKit direct replay source regression 또는 smoke test를 수행한다.
- 기존 #976 / #1016 / #516 회귀 테스트가 유지되는지 확인한다.
- 최종 결과보고서와 오늘할일 문서를 갱신한다.

**산출물**: `mydocs/report/task_m100_1017_report.md`

## 제외 범위

- #1016의 `PaintOp::Image.resolved` payload 구조를 되돌리지 않는다.
- baked watermark PNG를 투명 PNG로 바꾸는 방식으로 우회하지 않는다.
- `Document` IR 또는 parser 단계에서 z-order 전용 필드를 추가하지 않는다.
- Studio Canvas2D 기본 HTML Hybrid overlay 경로를 제거하지 않는다.
- CanvasKit text/glyph parity 전체를 해결하지 않는다.
- HWP layout / pagination / object anchor 위치 보정을 이 이슈에서 다루지 않는다.
- HWP3 전용 분기를 추가하지 않는다.

## 위험 및 대응

| 위험 | 가능성 | 대응 |
|------|--------|------|
| op를 단순 bucket으로 flatten하면서 clip/group/transform context 손실 | 높음 | Stage 1에서 context 경계를 먼저 정리하고, 구현 계획에서 replay item/tree traversal 정책 확정 |
| native Skia와 CanvasKit이 서로 다른 partition 정책을 갖게 됨 | 중간 | contract를 문서화하고 양쪽 테스트를 같은 fixture 의미로 구성 |
| `InFrontOfText`까지 뒤로 밀리는 회귀 | 중간 | behind / flow / front 순서를 별도 테스트로 고정 |
| PageLayerTree JSON schema 변경으로 기존 consumer 영향 | 중간 | 가능하면 additive metadata, schema bump 필요 시 명시 |
| Studio Canvas2D overlay 경로 회귀 | 낮음 | #516 테스트 유지, Canvas2D 경로는 제거하지 않음 |
| native Skia feature 빌드 환경 의존성 | 중간 | 가능한 단위 테스트 먼저 작성하고, feature 검증은 승인된 환경에서 수행 |
