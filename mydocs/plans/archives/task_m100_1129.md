## 1. 작업 배경

rhwp-studio의 상단 도구 막대에는 `격자 보기` 버튼이 있지만 실제 명령이 연결되어 있지 않아 동작하지 않는다. 메뉴의 `격자 설정`은 존재하지만 현재 구현은 표/개체 이동 간격(mm)을 조정하는 기능이며, 한컴오피스의 `격자 보기`처럼 페이지 위에 격자점 또는 격자선을 표시하는 보기 기능과는 다르다.

작업지시자 요청은 한컴오피스와 같이 다음 동작을 제공하는 것이다.

1. 편집 화면에서 격자 보기를 켜고 끈다.
2. 격자 모양과 간격을 설정한다.
3. 쪽/종이 기준 위치와 오프셋을 조정한다.
4. 기존 표/개체 이동 간격 설정과 보기 격자 설정이 혼동되지 않게 정리한다.

## 3. 스펙 검토

HWP5 스펙에는 격자와 줄맞춤에 해당하는 필드가 있다.

1. `mydocs/tech/한글문서파일형식_5.0_revision1.3.md` 표 44 `문단 모양 속성1`
   - `bit 8`: 편집 용지의 줄 격자 사용 여부
2. 같은 문서 표 129 `구역 정의`
   - 세로 줄맞춤 간격: `0 = off, 1 - n = 간격을 HWPUNIT 단위로 지정`
   - 가로 줄맞춤 간격: `0 = off, 1 - n = 간격을 HWPUNIT 단위로 지정`
3. OWPML 스키마
   - `paraPr@snapToGrid`: 편집 용지의 줄 격자 사용 여부
   - `grid@lineGrid`, `grid@charGrid`, `grid@wonggojiFormat`: 줄맞춤 정보

HWP3 스펙(`mydocs/tech/한글문서파일구조3.0.md`)에서는 문서 정보와 문단 모양 자료 구조 안에 용지, 여백, 줄 간격, 탭, 단 정의는 있으나 HWP5의 `lineGrid`, `charGrid`, `snapToGrid`에 직접 대응하는 필드는 확인되지 않았다. HWP3 문서는 사용자 보기 설정 또는 기본값 중심으로 처리한다.

## 5. 작업 범위

이번 작업은 rhwp-studio 편집 화면의 보기 기능을 우선 구현한다.

1. `격자 보기` 버튼을 실제 명령에 연결한다.
2. 페이지 좌표에 맞춰 움직이는 격자 오버레이를 구현한다.
3. 격자 설정 대화상자를 한컴오피스식 설정으로 확장한다.
4. 기존 표/개체 이동 간격은 유지하되, 보기 격자 설정과 구분한다.
5. 최소한 점, 가로선, 세로선, 가로/세로선 모드를 지원한다.
6. 격자 간격과 기준 위치, 오프셋을 설정 가능하게 한다.

문서 저장 포맷 반영은 Stage 2로 분리한다. HWP5/HWPX 스펙 필드 보존은 렌더링 보기 기능과 충돌하지 않게 별도 구조 필드로 추가한다.

## 7. 구현 원칙

1. 한컴오피스 화면처럼 페이지와 함께 움직이는 보기 오버레이로 구현한다.
2. 문서 내용 렌더링 결과 자체를 바꾸지 않는다.
3. 격자 오버레이는 저장/인쇄 결과에 섞이지 않게 한다.
4. `VirtualScroll`의 단일 페이지/다중 페이지 배치 모두에서 좌표가 어긋나지 않게 한다.
5. 표/개체 이동 간격 기능은 기존 동작을 보존한다.
6. HWP3 전용 처리는 `src/parser/hwp3/` 규칙을 지킨다. 단, 이번 보기 오버레이는 포맷 독립적인 UI 상태로 둔다.

## 8. 검증 계획

```bash

```

브라우저 검증:

1. rhwp-studio를 열고 HWP/HWPX 샘플 문서를 로드한다.
2. `격자 보기` 버튼 클릭 시 격자가 표시되는지 확인한다.
3. 다시 클릭하면 사라지는지 확인한다.
4. 격자 설정에서 3mm/가로·세로 간격/점·선 모드 변경 후 즉시 반영되는지 확인한다.
5. 확대/축소, 스크롤, 다중 페이지 그리드 모드에서 페이지 좌표와 격자가 함께 움직이는지 확인한다.
