# 수행계획서 — 페이지 영역 밖 콘텐츠 제거 (페이지밖 콘텐츠 0)

- 타스크: 로컬 task992 (GitHub 이슈 미발행 — 비공개 문서로 발견된 사안, 로컬 진행)
- 브랜치: `local/task992` (`local/devel`에서 분기)
- 마일스톤: M100 (v1.0.0)
- 작성일: 2026-05-19
- 선행: task991 (분할 표 렌더링 정합) — 본 사안을 진단·인계

## 1. 배경

task991에서 비공개 샘플 HWPX를 SVG로 내보내 검증하던 중, 일부 페이지에서 콘텐츠가 **페이지 영역(본문 하단)을 넘어서 그려지는** 현상을 확인했다. task991의 stage 2~4 수정으로 페이지밖 콘텐츠가 6개 페이지 → 2개 페이지로 줄었으나, 남은 2건은 별개 결함이라 본 타스크로 분리했다.

작업지시자 지시: **페이지 영역을 넘어 그려지는 경우가 없도록** 한다.

## 2. 문제 정의

페이지네이터가 한 페이지에 들어갈 콘텐츠 양을 과소 측정하여, 페이지에 다 들어가지 않는 콘텐츠를 추가 분할 없이 배치 → 렌더러가 페이지 본문 하단을 넘어서 그린다.

현재 확인된 2건(181쪽 기준):
- 페이지 143: 2행×1열 표의 연속분. 행 1의 셀에 **문단 51개 + 다중 중첩표**(개인정보 동의서 양식). 텍스트가 y≈1166까지(페이지 높이 1122 초과).
- 페이지 171: 14행×7열 표 연속분 + 다수 빈 문단. 텍스트가 y≈1180까지.

## 3. 원인 (task991 진단 인계)

페이지네이터(`typeset.rs`)의 분할 표 연속분 측정이 셀 실제 높이를 과소 평가한다.

- 페이지 143 pi=308: `MeasuredTable`이 행 1(문단 51개 + 다중 중첩표 셀)을 **1791px**로 측정. content_offset 1642.5 이후 잔여를 **137.4px**로 계산 → "현재 페이지에 들어감" 판정 → 추가 분할 안 함. 그러나 렌더러는 같은 연속분을 **~660px**로 그림.
- 즉 `height_measurer`가 *문단 다수 + 다중 중첩표를 가진 셀*의 높이를 **~500px 과소 측정**한다. 중첩표 문단의 높이가 측정에서 누락/축소되는 것으로 보인다.
- 페이지네이터는 이 과소 측정값으로 "더 분할 불필요" 판단 → 오버플로.

## 4. 작업 범위

- 포함: `height_measurer`의 *중첩표 포함 셀* 높이 측정 정정 → 페이지네이터가 분할 표 연속분을 정확히 재어 필요 시 추가 분할하도록.
- 목표: 비공개 샘플 181쪽에서 **페이지밖 콘텐츠 0**. 가능하면 다른 공개 샘플에서도 오버플로 0.
- 방어 차원에서, 측정 오차가 남더라도 페이지 경계를 넘기지 않도록 하는 가드 도입 여부는 구현계획서에서 검토.
- 제외: 페이지 수 ±쪽 누적 드리프트(별도 사안), HWP3, 파서.

## 5. 검증 방법

- 코드 수정 없이 재현·검증 가능: `export-svg` 후 전 페이지에서 `<text>` y좌표가 viewBox 높이를 초과하는지 스캔. 현재 2개 → 목표 0.
- `dump-pages`로 분할 표 연속분 측정값(`effective_row_height`/`remaining_content_for_row`)이 렌더 높이와 일치하는지 확인.
- 골든 SVG 테스트 + `cargo test` 전체 통과, `cargo clippy` 무경고.
- 다른 공개 분할 표 샘플로 교차 회귀 + 페이지 수 회귀 확인.

## 6. 단계 개요 (상세는 구현계획서)

1. 조사 — `height_measurer`의 중첩표 포함 셀 높이 측정에서 과소 계산 지점 확정.
2. 수정 — 측정 정정 + 단위 검증.
3. 전체 검증(오버플로 0·골든·교차 회귀) + 최종 보고서.

## 7. 비공개 문서 처리

재현용 HWPX/PDF는 비공개 문서로 커밋하지 않는다. 문서에 파일명·기관명을 적지 않는다. 회귀 테스트는 공개 샘플 우선, 불가 시 비커밋 처리 후 보고서에 명시한다.
