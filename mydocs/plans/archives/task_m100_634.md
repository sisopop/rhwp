# Task #634 수행 계획서

**제목**: 한컴 호환 — 첫 NewNumber Page 컨트롤 발화 전 페이지의 쪽번호 미표시

## 1. 배경

작업지시자가 `samples/aift.pdf` 페이지 2 (한컴 PDF) 와 rhwp SVG 페이지 2 비교 중
한컴은 쪽번호 미표시, rhwp 는 "- 2 -" 표시 차이를 발견.

분석 결과 한컴은 **첫 `새번호 (NewNumber Page)` 컨트롤이 등장한 페이지부터**
쪽번호를 표시한다. PageNumberPos 가 등록되어 있어도 NewNumber 발화 전
페이지에는 미표시. rhwp 는 PageNumberPos 가 등록된 시점부터 모든 페이지에
표시 → 표지·요약문·목차에서 한컴과 시각 결과가 달라진다.

## 3. 본질 가설

**가설 H1**: 한컴은 PageNumberPos 등록 후라도 **첫 NewNumber Page 발화 전 페이지에는
쪽번호를 렌더링하지 않는다**. NewNumber 가 "표시 시작 신호" 역할을 겸한다.

**근거**:
- 2건 교차 검증 (aift / 국립국어원) 일치
- HWP5 스펙은 "데이터 포맷" 만 정의하며 한컴 조판 알고리즘은 비공개
  ([hwp_spec_errata.md] 30번 참조)
- "PDF 보조 ref" 로 한정하되, 동일 패턴이 2건에서 일치하므로 채택

**채택 근거 강화 (Stage 0 예정)**:
- 추가 1~2건 한컴 PDF 비교 (별도 샘플) 로 가설 확인
- NewNumber 가 없는 단순 문서가 한컴에서 어떻게 처리되는지 별도 확인

## 5. 단계 분할

### Stage 0 — 사전 분석 + 추가 검증

1. 한컴 PDF 가 NewNumber 없는 단순 문서에서 쪽번호를 어떻게 처리하는지 추가 1~2건 확인 (rhwp 가 모든 페이지 표시인 case 검증)
2. PageNumberAssigner 호출 경로 (`engine.rs::finalize_pages`, `typeset.rs::finalize_pages`) 정합성 확인 — 두 경로 모두 정정 필요
3. 구역간 carry 와 NewNumber 상호작용 (rendering.rs:1227~1248) 정합성 확인
4. PageNumberPos position=0 체크 (layout.rs:1100) 와 새 가드의 상호작용 확인
5. Stage 0 보고서 → 작업지시자 승인 대기

### Stage 1 — TDD 통합 테스트

1. `samples/aift.hwp` 페이지 2 SVG 에 "- 2 -" 미렌더링 검증 통합 테스트 추가
2. `samples/2022년 국립국어원 업무계획.hwp` 페이지 1, 2 SVG 에 footer 쪽번호 텍스트 미렌더링 검증 추가
3. 페이지 7 (aift) / 페이지 3 (국립국어원) 부터는 "- 1 -" 표시 검증 (positive case)
4. 모두 RED 확인 (현재 회귀)
5. Stage 1 보고서 + 커밋

### Stage 2 — Fix 적용

1. PageNumberAssigner 에 numbering_started 상태 + 반환 시그니처 확장
2. PageContent.show_page_number 추가 (default true)
3. engine.rs / typeset.rs finalize_pages 두 경로 정합 정정
4. rendering.rs 구역간 carry 시 numbering_started 도 전달 (이전 구역에서 NewNumber 발화 → 후속 구역도 표시)
5. layout.rs::build_page_number 에 가드 추가
6. 단위 테스트 통과 (page_number.rs 의 기존 5건 테스트 보존 + 새 테스트 추가)
7. Stage 1 통합 테스트 GREEN 확인
8. Stage 2 보고서 + 커밋

### Stage 3 — 광범위 회귀 검증 + 최종 보고서

1. `cargo test --release --lib` 전체 통과
2. 광범위 샘플 회귀 (NewNumber 가 처음부터 있는 문서 / 없는 문서 / PageNumberPos 만 있는 문서 — 표지에 number 표시 안 해야 vs 표시 해야)
3. 한컴 PDF 비교 추가 검증 (가능한 모든 PDF/HWP 짝)
4. Stage 3 보고서 + 최종 보고서 + 커밋
