# 수행계획서: text-align.hwp SVG ↔ 한컴 PDF 렌더링 차이 보정

- **타스크**: [#146](https://github.com/edwardkim/rhwp/issues/146)
- **마일스톤**: M100
- **브랜치**: `local/task146`
- **작성일**: 2026-04-23
- **샘플**: `text-align.hwp` (작업지시자 제공), `text-align.pdf` (한컴 오피스 PDF 출력, 비교 기준)

## 1. 배경

작업지시자가 제공한 `text-align.hwp` 1페이지 문서에 대해 `rhwp export-svg`로 출력한 SVG와 한컴이 직접 PDF로 출력한 `text-align.pdf` 를 150dpi로 나란히 렌더링해 비교한 결과, **페이지 레이아웃 단계부터** 시각적으로 차이가 확인된다.

### 문서 요약 (dump 결과)

- 용지 A4 세로, 여백 좌·우 20mm / 상·하 15mm / 머리·꼬리말 10mm
- 문단 5개 + 표(3행×2열, 위아래 감싸기, treat_as_char=true)
- 모든 문단 `align=Justify`, 음수 `indent` (행 글머리표 hanging indent)
- 제목 문단: `ps_id=30`, `[CS] id=33 (□) spacing=-8%`, `[CS] id=36 (공백) spacing=-4%`
- 본문: `휴먼명조` 계열, 제목: `HY헤드라인M` 계열 사용 (HWP 내부 FaceNames)

### 조사로 확인된 차이

정량 비교(150dpi 렌더링 기준, `output/compare/text-align/` 보관):

| 항목 | 한컴 PDF | rhwp SVG | 차이 성격 |
|------|---------|----------|----------|
| 제목 글자 굵기 | 굵게(display heavy) | 보통 굵기 | **폰트 치환 한계** (HY헤드라인M 부재 시 Malgun Gothic fallback) |
| 제목 "□" 뒤 공백 | 뚜렷한 공백(≈반글자) | **공백이 사라짐** (`□국어` 붙어버림) | **실제 버그 후보 ①** |
| 제목 양끝 정렬 | 오른쪽 여백까지 분산 | 왼쪽 정렬 상태로 끝남 | **실제 버그 후보 ②** (Justify 미적용) |
| 본문 문단 두 번째 줄 hanging indent | "ㅇ" 다음 글자("세대별")와 같은 x 정렬 | x 정렬 위치가 어긋남 | **실제 버그 후보 ③** |
| 본문 줄 오른쪽 끝(시범) | 오른쪽 여백까지 양끝 정렬 | 중간에서 끝남 | 버그 후보 ② 와 동일 원인 |
| 표 구조 | 3×2, 헤더·본문 분리 | 동일 | 일치 |
| "* 3세대 / ** 10지점" 크기 | 본문 크기 | 소폭 작게 보임 | 폰트 치환 영향 |

### 기저 구현 현황 (SVG 렌더러 경로)

| 속성 | CharShape → 레이아웃 | 레이아웃 → SVG |
|------|---------------------|----------------|
| font-size | 반영 (HWPUNIT → px) | `font-size` 속성 출력 O |
| bold | 반영 | `font-weight="bold"` 출력 O |
| italic / underline | 반영 | 출력 O (underline은 별도 `<line>`) |
| ratio(장평) | 반영 | `transform="scale(r,1)"` O |
| spacing(자간) | **부분 반영** — x 좌표에는 포함, 공백 글리프와의 상호작용 검증 필요 | CSS `letter-spacing` 미사용 |
| 공백 문자 | 글리프 렌더 생략 O | 생략 O, **단 x-advance 누락 케이스 존재(후보 ①)** |
| Justify 정렬 | `Alignment::Justify` 저장 | **SVG 출력 단계에서 미반영** (후보 ②) |
| 음수 indent(hanging) | 첫줄/후속줄 x 시작 분기 | 후속 줄 x 기준이 어긋나는 것으로 관찰 (후보 ③) |

## 2. 목표

`text-align.hwp` 렌더링 결과가 한컴 PDF 출력과 **육안상 동등**해지는 수준까지 수렴시킨다. 단, 폰트 실체(한컴·한양 폰트) 자체의 글리프 품질 차이(굵기/폭)는 폰트 폴백 전략(`mydocs/tech/font_fallback_strategy.md`) 범위이므로 본 타스크 범위에서 제외한다.

**범위 (IN)**
- **B-1**: 첫 글자 + 자간(-%) 조합에서 이어지는 공백의 x-advance 누락 원인 규명 및 수정 (후보 ①)
- **B-2**: ParaShape `Alignment::Justify` 를 SVG 글자 x 좌표 재계산으로 반영 (후보 ②)
- **B-3**: 음수 indent(hanging indent) 후속 줄 x 기준점 일관성 검증 및 수정 (후보 ③)
- **B-4**: `text-align.hwp` 를 재현 검증 샘플 (`output/re/`)에 편입해 회귀 방지

**범위 (OUT)**
- 한컴·한양 proprietary 폰트 실체 확보/번들 (라이선스 이슈, 별도 전략 문서 범위)
- 표 외곽·셀 내부 rendering 심화 (#251 등 별건에서 기조치)
- 글머리표(□, ㅇ)의 자동 삽입 로직 (본 문서에선 이미 원문에 포함)

## 3. 구현 단계 (초안 — 별도 구현 계획서로 상세화 예정)

1. **단계 1 — 원인 재현 테스트 작성 및 현상 고정**
   - `text-align.hwp`를 샘플에 추가, 현재 SVG 산출물 golden 으로 저장(실패 예상 지점 주석)
   - B-1/②/③ 각 현상을 분리하는 최소 재현 케이스(unit-level) 도출
2. **단계 2 — 공백 x-advance 누락(B-1) 수정**
   - `paragraph_layout.rs` / `style_resolver.rs` 경로에서 자간-공백 상호작용 정정
3. **단계 3 — Justify 렌더링(B-2) 구현**
   - 레이아웃 단계에서 계산된 Alignment 정보를 줄 내부 글자 x 재배치에 반영
   - 마지막 줄 제외 규칙, 강제 줄바꿈 예외 처리 포함
4. **단계 4 — Hanging indent 후속 줄 기준점(B-3) 보정**
   - `line_segs`의 ts/vpos 및 ParaShape indent 부호 처리 재검
5. **단계 5 — 재현검증·회귀 방지**
   - `re_sample_gen.rs`(또는 동등 경로)에 샘플 편입
   - PDF 이미지 대비 육안 점검, `cargo test` / clippy 통과 확인

(세부 진입점·파일·커밋 단위는 구현계획서 `task_m100_146_impl.md` 로 분리 작성)

## 4. 검증 방법

- `rhwp export-svg text-align.hwp` → 150dpi PNG 변환(Chrome headless) → `text-align.pdf` 150dpi PNG 와 좌우 비교
- 기준: 제목 "□ 국어 변화…" 의 공백이 보이고, 두 본문 문단의 오른쪽 끝이 여백선과 거의 일치, 두 번째 줄이 "세대별" x 위치에서 시작
- `cargo test` 전체 통과, `cargo clippy -- -D warnings` 통과
- 기존 SVG 스냅샷 테스트(svg_snapshot) 영향 검토 — 영향 시 golden 업데이트와 사유 기록

## 5. 리스크·가정

- **가정**: 마일스톤은 현행 진행중인 M100 (기존 `task_m100_*` 연장선). 다른 마일스톤이면 파일명 보정 필요.
- **리스크**: Justify 반영 시 `svg_snapshot` 대량 diff 가능 — 예상 범위 확인 후 golden 재생성 계획 별도 승인 요청.
- **리스크**: hanging indent 수정이 다른 샘플 회귀를 유발할 수 있음 — `samples/` 스모크 스위프 필요.
