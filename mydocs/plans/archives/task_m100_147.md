## 3. 원인 분석

### 파싱 경로
`parse_section_def` → 인식 불가 자식 레코드 → `extra_child_records`
→ `parse_master_pages_from_raw(extra_child_records)` → MEMO LIST_HEADER를 바탕쪽으로 분류

### 핵심 원인
HWP에서 MEMO(주석) 컨트롤은 텍스트 박스 형태로 `LIST_HEADER` + 문단 레코드를 가진다.
이 MEMO의 `LIST_HEADER`가 SectionDef의 `extra_child_records`에 포함되어
`parse_master_pages_from_raw`가 바탕쪽 `LIST_HEADER`로 오분류.

### 판별 근거
실제 바탕쪽 `LIST_HEADER`는 표 139 오프셋에 `text_width(4B) + text_height(4B)` 비-제로 값을 가짐.
MEMO LIST_HEADER는 동일 오프셋에 값이 없거나 0 → **파싱 결과 `text_width=0, text_height=0`**.

한컴 동작: 이 구역에 실제 바탕쪽 없음. 0×0 "바탕쪽"은 MEMO 레코드의 오분류.

## 4. 수정 방향

### 방안: 파서 필터 + 렌더러 가드 (이중 방어)

| 레이어 | 위치 | 수정 내용 |
|--------|------|---------|
| 파서 | `body_text.rs` `parse_master_pages_from_raw` | `text_width == 0 && text_height == 0`이면 바탕쪽 skip |
| 렌더러 | `layout.rs` `build_master_page` | `mp.text_width == 0 && mp.text_height == 0`이면 렌더링 skip |
