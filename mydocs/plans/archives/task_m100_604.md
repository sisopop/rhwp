### 1.3 본질 진단

**근본 원인 1 — HWP3 파서 (`src/parser/hwp3/mod.rs:1399-1407`)**:

let line_cs_sw = current_zone.and_then(|(cs, sw, pgy_start, pgy_end)| {

## 2. 정정 방향 (옵션 C → R3 — wrap zone 본질 재정의)

### 2.1 Stage 1+2 시도 결과 — 옵션 C (cs/sw 만으로 wrap zone 판정) 본질 부적합

Stage 1 (LineSeg 표준 + `is_in_wrap_zone(col_w_hu)` helper) 도입 후 Stage 2 적용 시 회귀
발견 (test_547 fail): HWP5 native passage box 본문 LineSeg `cs=852, sw=30184` 가
`is_in_wrap_zone(box_w-200)` = true 로 false-positive 판정 → 11.4px 추가 → PDF 정합 위배.

**본질 진단**:
- `column_start > 0` 은 wrap zone 외 box inset / cell padding / paragraph indent 로도 사용
- `segment_width < col_w_hu` 은 box 안 본문도 동일 패턴 (정상)
- **LineSeg 단독 검사로는 wrap zone 과 box inset 을 구분 불가능**
- wrap zone 판정의 본질은 **anchor 그림/표와의 컨텍스트 연결**

### 2.2 R3 본질 — typeset wrap_around state machine 결과를 layout 까지 전달

`src/renderer/typeset.rs:478-513` 의 wrap_around state machine 이 이미 anchor ↔ wrap text
매칭 본질 메커니즘:

```
1. anchor 파싱 시 wrap_around_cs/sw 등록 (typeset.rs:625, 650)
2. 후속 문단 LineSeg cs/sw 매칭 → wrap text 로 식별 (typeset.rs:489-491)
3. 매칭 실패 → wrap zone 종료 (typeset.rs:510-512)
```

이 매칭 결과가 **layout 시점에 전달 안 됨** → 본 PR #589 의 wrap_precomputed 플래그
(IR 누설) 우회. Stage 2 시도가 본질 부적합으로 판명되었으므로, R3 가 정합:

| 영역 | 정정 내용 | 본질 |
|------|---------|------|
| **IR 표준** (Stage 1 완료) | `mydocs/tech/document_ir_lineseg_standard.md` + `LineSeg::is_in_wrap_zone(col_w_hu)` helper | 보존 (포맷 독립 표준 + helper) |
| **typeset 출력 메타데이터** (R3 신규) | `ComposedParagraph` 또는 `PageItem::FullParagraph` 에 `wrap_anchor: Option<WrapAnchorRef>` 필드 추가 — typeset 매칭 결과 보존 | wrap context 본질 표현 |
| **렌더러 typeset** (R3 신규) | wrap_around 매칭 시 `wrap_anchor` 메타데이터 설정 후 흡수 skip + FullParagraph 로 통과 | wrap context 보존 |
| **렌더러 layout/paragraph_layout** (R3 신규) | `wrap_anchor.is_some()` 일 때 LineSeg cs/sw 로 정합 렌더 (현 보완6 로직) | 메타데이터 소비 |
| **IR 모델** (R3 신규) | `Paragraph.wrap_precomputed: bool` 필드 **제거** | HWP3 휴리스틱 누설 청산 |

| **렌더러 layout** (Stage 4 그대로) | layout.rs:2957,3345 주석 갱신 (Task #604 인용) | 추적성 |

### 2.3 R3 본질 정정의 위험

- ComposedParagraph 또는 PageItem 의 model 변경 — 직렬화/round-trip 영향 없음 (typeset 출력은 휘발성)
- typeset 의 흡수 흐름 변경 — wrap_around state machine 의 본질을 보존하면서 출력만 추가
- layout 의 wrap context 소비 — 기존 wrap_precomputed 검사를 wrap_anchor 검사로 일대일 교체
