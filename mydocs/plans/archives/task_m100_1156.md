## 1. 본질

2단(multicolumn) 문서에서 차트 컨트롤(OLE/Chart)의 단 내 배치가 한컴 에디터와 다르다.

### 1.1 한컴 동작 (기대)

- 차트 컨트롤이 현재 단(왼쪽)에 들어가면 현재 단에 배치
- 현재 단 영역 부족 시 **오른쪽 단으로 이동**
- **차트가 오른쪽 단으로 이동하면서 비게 된 왼쪽 단 하단 공간에 후속 텍스트(문단)가 배치됨** (작업지시자 추가 명시 — column back-fill / text flow)
- 다음 단/페이지 이동 시 표·문단·차트의 vpos 흐름이 한컴과 일치

### 1.2 rhwp 현 동작 (결함)

- 차트 컨트롤(Shape/OLE) 레이아웃 처리가 다단 column flow 에서 불완전
- 단 1(오른쪽)을 한컴보다 적게 채움 → 빈 공간 텍스트 back-fill 미작동

## 2.4 선결 조건 — 차트 컴포넌트 크기 확정 (작업지시자 지적)

단 이동 판정과 back-fill 공간 계산은 **차트의 확정된 크기**에 의존한다. 크기가 틀리면 "단 영역 부족" 판정과 빈 공간 크기가 모두 잘못된다.

`samples/143E433F503322BD33.hwp` pi=7 차트(OLE)의 크기 정보가 **상충**:

| 출처 | 크기 |
|------|------|
| common (컨트롤 지정, `resolve_object_size`) | **80.0mm × 80.0mm** (22677×22677 HU, Absolute) |
| OLE 요소 (SHAPE_COMPONENT curr) | **25.4mm × 25.4mm** (7200×7200 HU = 1인치) |
| dump `eff` (실제 렌더) | 25.4mm × 25.4mm |

- 레이아웃 점유(`resolve_object_size`): `common.width/height` = **80mm**
- 실제 차트 그림(shape_layout OLE 분기 `render_w/render_h`): 동일 render_w/h 전달이나 OLE curr 기준 그림이 25.4mm 로 축소되어 보일 수 있음

→ **차트 크기가 80mm(common) 인지 25.4mm(OLE curr) 인지 한컴 기준 확정 필요.** 이 확정 없이 단 이동/back-fill 구현 시 공간 계산이 틀림. **Stage 1 의 최우선 항목**으로 둔다.

### 2.4.1 한컴 확정 + HWPX 샘플 분석 (작업지시자 제공)

**작업지시자 확정: 한컴 편집기에서 차트 = 80.0mm × 80.0mm** (= `hp:sz` / common.width 22677 HU). OLE curr/orig 아님.

작업지시자 제공 HWPX 변환본 `samples/hwpx/143E433F503322BD33.hwpx` 의 OLE 차트:
```
<hp:ole textWrap="TOP_AND_BOTTOM" ...>
  <hp:sz width="22677" height="22677" widthRelTo="ABSOLUTE"/>   ← 80mm (한컴 확정값)
  <hp:orgSz width="7200" height="7200"/>                         ← 25.4mm (OLE 원본)
  <hp:curSz width="0" height="0"/>                               ← 0 (미설정)
```

→ 크기 문제의 **두 층위**:
1. **점유 크기**: common 80mm ✅ 정상 (`resolve_object_size` 가 common 사용)
2. **렌더(그림) 크기**: dump `eff=0.0mm` (orig/curr=0) ❌ — 차트 그림이 0 크기로 렌더되어 안 보임/점

**선결 정정**: 차트 그림 렌더 크기를 common 80mm 로 정합 (현재 curSz/orgSz=0 → eff=0). HWPX `materialize_shape_current_size_from_original` (section.rs:2310) 가 curSz=0 시 orgSz fallback 하지만, orgSz 도 0/7200 이라 common(sz=80mm) 우선 fallback 필요.

HWP5(`143E433F503322BD33.hwp`) 와 HWPX 변환본 dump-pages 가 동일 (단0/단1 동일, 차트 vpos=48603) → 같은 결함.

### 2.4.2 한컴 공식 차트 spec 교차 확인 (`mydocs/tech/hwp_chart_spec.md`)

- spec §1: 차트 = GSO(`b"gso "`) 컨트롤. **CTRL_HEADER → SHAPE_COMPONENT (공통 도형 속성: 위치/크기) → HWPTAG_CHART_DATA**.
- → **차트 크기의 권위는 SHAPE_COMPONENT 공통 크기** (= common = HWPX `hp:sz` = 80mm). OLE orgSz/curSz 아님. **작업지시자 확정(80mm) + spec 일치.**
- spec §1 (line 42): "HWP 차트는 내부적으로 MS Graph OLE 기반".
- **본 fixture(`143E...`) 차트는 OLE 경로** (`hp:ole` / SHAPE_COMPONENT_OLE, `[OLE] bin_data_id`) — spec 의 신형 GSO+CHART_DATA 와 다른 구형/MS Graph OLE 차트. 단 크기 권위는 동일 (SHAPE_COMPONENT common).

→ 결론: **차트 그림 크기는 common 80mm 로 정합** (OLE curSz/orgSz=0 fallback 이 아니라 common 우선). spec + 작업지시자 + HWPX `hp:sz` 3중 일치.
