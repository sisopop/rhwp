# 수행 계획서 — Task #1139: 3-09월 교육 통합 2022 본문 수식 이상 문자 렌더링

작업지시자가 `samples/3-09월_교육_통합_2022.hwp`를 한컴오피스 2022와 rhwp-studio에서 비교했을 때, 본문 수식 주변에 한컴 대비 이상한 문자 또는 기호가 포함되어 표시된다고 보고했다.

현재 열린 #922는 미주 내부 수식 미표시와 페이지 간 vpos 리셋 후속이다. 이번 증상은 5쪽 본문 수식 영역에서 관찰되므로 별도 이슈 #1139로 분리한다.

## 초기 재현과 단서

진단 명령:

```bash
./target/release/rhwp dump-pages samples/3-09월_교육_통합_2022.hwp -p 4
./target/release/rhwp dump samples/3-09월_교육_통합_2022.hwp -s 0 -p 278
./target/release/rhwp dump samples/3-09월_교육_통합_2022.hwp -s 0 -p 288
./target/release/rhwp export-svg samples/3-09월_교육_통합_2022.hwp -p 4 -o output/diag_1139 --debug-overlay
```

확인 내용:

- 5쪽 좌측 문23/문24는 섹션 0의 `pi=277~290` 부근에 배치된다.
- 문23 본문 수식: `pi=278`, `script="lim _{x\` -> \`0} {} \` {4  ^{x} -2  ^{x}} over {x}"`
- 문24 본문 수식: `pi=288`, `script=" int _{0} ^{pi } {} x\`cos LEFT ( {pi } over {2} -x RIGHT ) dx"`
- 5쪽 우측 문27에는 `bar{... it}`·`ANGLE`·TAC 그림 컨트롤이 함께 등장한다. 이상 기호가 실제 수식 명령 누락인지, TAC 그림/개체 마커인지, 문서 원본의 의도된 기호인지 한컴 화면과 분리 확인이 필요하다.

## 관련 기존 지식

- `project_equation_always_tac`: 한컴 수식 컨트롤은 항상 TAC이며, `paragraph_layout.rs` 인라인 경로가 핵심이다.
- Task #1061 EQEDIT errata: 수식 record raw byte 계약은 실제 HWP와 hwplib 교차 검증이 필요하다.
- `feedback_visual_judgment_authority`: 한컴 2022 화면이 최종 시각 기준이며 Codex는 정량 측정과 후보 원인 분리만 수행한다.

## 작업 목표

1. 한컴 대비 불필요하게 표시되는 문자가 어느 계층에서 생기는지 분리한다.
2. 수식 스크립트 파싱·AST·SVG/Canvas 출력 중 해당 문자가 새는 경로를 특정한다.
3. 특정 fixture만 덮는 임시 필터가 아니라, 한컴 수식 문법의 명령 처리로 설명 가능한 최소 정정을 적용한다.
4. 5쪽 문23/문24와 우측 문27의 수식/기호가 한컴 화면과 더 가까워졌는지 자동 산출물과 작업지시자 시각 판정으로 확인한다.

1. Stage 1: 원인 계층 진단
   - 문23/문24/문27의 Equation script를 AST와 SVG 텍스트 출력으로 비교한다.
   - 보이는 이상 문자가 수식 명령 문자열, fallback 텍스트, TAC 그림, 개체 마커 중 어디서 나오는지 분류한다.
2. Stage 2: 최소 정정
   - 진단 결과에 따라 `src/renderer/equation/parser.rs`, `symbols.rs`, 또는 인라인 TAC 배치 경로에 국소 수정한다.
   - 한컴 수식 명령으로 해석되어야 할 토큰이 일반 텍스트로 새지 않도록 한다.
3. Stage 3: 검증과 보고
   - 대상 페이지 SVG 재출력, 관련 단위 테스트, `cargo test` 범위 테스트를 수행한다.
   - WASM 변경이 있으면 `wasm-pack build --target web --out-dir pkg`로 검증한다.
   - 자동 검증 후 작업지시자 시각 판정 대기 상태를 보고한다.
