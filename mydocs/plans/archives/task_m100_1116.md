## 3. 기준 자료와 새 전제

### 3.1 작업지시자 제공 기준

작업지시자가 제공한 한컴오피스 캡처는 3쪽 본문 상단의 `I. 사업개요`, `1. 추진목적`, 목적 박스, `2. 추진방향`, 주요 추진내용 구간을 보여준다.

중요한 새 전제:

```text
캡처의 격자는 3mm 단위다.
```

기존 `mydocs/feedback/hwp3-sample16-hwp5-analysis.md`의 1차 측정은 격자를 1mm로 가정했다. 따라서 해당 문서의 수치 비교 중 "한컴 스크린샷 육안 추정" 부분은 그대로 채택하지 않고, 3mm 격자 기준으로 다시 환산한다.

### 3.2 기존 1차 분석에서 유지할 정보

기존 분석 문서에서 유지할 수 있는 것은 rhwp 내부 좌표와 `LINE_SEG` 분해다.

```text
pi=71 ls[0]: vpos=7464,  lh=9764, ls=780   # TAC 글상자 문단
pi=72 ls[0]: vpos=18576, lh=300,  ls=164, spacing_before=568
pi=73 ls[0]: vpos=19892, lh=1600, ls=960, spacing_before=852
```

`pi=71` 글상자 하단에서 `pi=73` 제목 시작까지의 파일상 간격:

```text
19892 - (7464 + 9764) = 2664 HU
2664 HU = 약 9.40mm
```

3mm 격자로 보면 이는 약 3.13칸이다. 캡처에서 같은 구간이 3mm 격자 몇 칸인지 다시 비교해야 한다.

## 5. 현재 가설

### 5.1 p2 목차

p2 전체 높이 차이는 큰 폭보다 작게 관찰되었으므로, 우선순위는 누적 페이지 높이보다 다음 항목이다.

1. right-tab 위치가 본문 폭 또는 문단 기준 폭에 맞게 정렬되지 않는다.
2. leader 점선의 시작점, 종료점, 점 간격이 한컴오피스와 다르다.
3. 같은 문단의 `TopAndBottom + treat_as_char=true` 사각형 컨트롤이 텍스트 폭 또는 탭 계산에 영향을 준다.
4. HWP3 변환본의 `TabDef`와 HWP5/HWPX 일반 문서의 탭 해석 차이가 공통 렌더러에서 섞여 있다.

### 5.2 p3 본문

기존 분석의 핵심은 `pi=71` TAC 글상자 뒤 진행량이다. 파일에는 TAC 문단 뒤 간격이 존재한다.

```text
2664 HU = 약 9.40mm = 3mm 격자 약 3.13칸
```

따라서 이번 조사에서는 "간격 정보가 누락되었다"가 아니라 다음을 검증한다.

1. rhwp가 `LINE_SEG.vpos/lh/ls`를 근거로 문단 진행량을 잡는 기준점이 한컴과 같은가.
2. TAC 글상자 문단의 shape height, line height, line spacing, 다음 빈 문단의 `spacing_before`가 중복 반영되는가.
3. rhwp의 SVG 1mm 격자와 한컴 캡처 3mm 격자를 혼동해 차이를 과소 또는 과대 평가하지 않았는가.
4. p3 상단만 맞추는 보정이 p21/p23 page break와 전체 64쪽 페이지 수를 깨지 않는가.
