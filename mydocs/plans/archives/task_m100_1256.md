# 수행계획서 — Task #1256: 미주 답안 제목 위 between-notes(7mm) 간격 누락

- 이슈: edwardkim/rhwp#1256
- 마일스톤: v1.0.0 (M100)
- 브랜치: `local/task1256` (from `local/devel`)
- 관련: #1253, #1248, #1246, PR #1232
- 재현 문서: `samples/3-09월_교육_통합_2022.hwpx` / `.hwp`

## 1. 배경 / 근본 원인 (조사 완료)

미주 답안 영역에서 `문N)` 제목 **위 줄 간격이 한컴보다 약 7mm(≈23px) 좁다.**

- 정량(96dpi 동일 좌표): 일반 줄간격은 정합(~37px). 그러나 `문6)→문7)`(내용 동일) 구간이
  한컴 PDF 287px vs 우리 264px = **−23px ≈ 7mm 한 줄**.
- 문서 지정값: `미주 사이`(between-notes) = **7mm = 1984 HU**
  (`hp:noteSpacing betweenNotes="1984"`). 파서는 `shape.raw_unknown=1984`로 정확히 읽음.
- **원인**: `src/renderer/height_cursor.rs:431-440`의 between-notes min-gap 보정이
  `prev_is_multiline` 조건으로 게이트되어, **직전 문단이 단일 줄(빈 separator 문단)** 인
  미주에는 7mm 보정이 적용되지 않음. 작업지시자가 지목한 헤더(문5/6/7/12/16/18/19/20/21/
  23/24/25/26/27/28/29)는 모두 **직전이 단일 줄/빈 문단** 케이스 → 보정 스킵.
- `.hwp`/`.hwpx` 양쪽 동일 → 포맷 무관, 공통 미주 레이아웃 경로 문제.

## 2. 목표

- 직전 문단이 단일 줄/빈 문단인 미주에도 between-notes(7mm) 간격이 `문N)` 제목 위에
  올바르게 적용되어, 한컴 PDF와 시각 정합.
- **회귀 0**: 기존 정합 케이스(문13 의도적 소간격, 문15 충분간격, 문22 다줄 prev)와
  `3-09'22.hwp` 페이지 수(23p)·overflow 가드(#1246/#1082) 유지.

## 3. 범위

- 대상: `src/renderer/height_cursor.rs`의 between-notes min-gap 보정 분기,
  필요 시 `src/renderer/typeset.rs`의 between-notes 주입(line_spacing=1984) 경로.
- 비대상: 미주 모양 UI(#1253), 구분선 렌더, 파서/IR. 미주 2D vpos 전면 재설계(별도 이슈)는
  본 타스크에서 다루지 않음 — **국소·안전 보정**에 한정.

## 4. 접근 방향 (구현계획서에서 확정)

1. **단일 줄 prev 이중가산 위험 규명**: 현재 게이트가 단일 줄을 제외하는 이유(주석:
   "trailing 이 이미 sequential y(y_offset)에 포함")가 between-notes(1984) 케이스에도
   성립하는지 검증. 단일 줄 prev의 trailing 에 들어간 값이 일반 줄간격인지 between-notes인지
   typeset 주입 결과로 확인.
2. **안전 조건 확장**: `stored_gap_px`가 between-notes 미적용 수준(≈일반 줄간격)일 때만
   `prev_is_multiline` 무관하게 7mm까지 보정. 이미 7mm 이상이면 무변경(max 의미).
3. **회귀 가드**: 보정 후 `3-09'22.hwp`/`.hwpx` 23p 유지, `미주사이20` 변형 파일의 페이지
   분기, 문13/15/22 케이스를 SVG 좌표로 before/after 비교.

## 5. 검증 방법
