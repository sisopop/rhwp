## 1. 문제 정의

비공개 185페이지 문서에서 일부 페이지 하단의 표/문단이 본문 영역(body bottom)을
넘어 렌더링됨. 현재 native LAYOUT_OVERFLOW **16건** (task977/parent 동일 — task977
무관, 기존 잔여).

사용자 보고(스튜디오/WASM): 28쪽(SIR-002)·35·49·59·125쪽 하단 영역 밖 렌더링.
- 28쪽 = native page=27 pi=242 PartialTable **19.2px**
- 35쪽 = page=34 pi=256 PartialTable 6.9px
- 49쪽 = page=48 pi=290 PartialTable 8.7px
- 59·125쪽: WASM 폰트 폭 폴백으로 페이지네이션이 native와 달라 페이지 번호 어긋남
  (동일 결함, 다른 페이지 번호).

## 2. 근본 원인

페이지네이터 추정/cut 높이 < 렌더러 실측 높이. 예) 28쪽 페이지네이터 `used=875.7px
< 본문 941.1px`로 "여유" 판정 → pi=242 행 0 배치. 렌더러는 누적 ~85px 더 크게 그려
행 0이 본문 바닥(1046.9)을 19.2px 초과. overflow는 `layout.rs:2529`에서 **사후 진단만**
기록될 뿐 이월 로직 없음.

## 3. 목표 (작업지시자 지시)

overflow 감지 시 해당 항목(pi)을 현재 페이지에 렌더링하지 않고 **다음 페이지로 이월**
한다. native·WASM 공통 동작.

**채택 방향: 사후 reflow** (작업지시자 승인). 레이아웃 후 overflow 감지 → 해당 항목
앞에 강제 페이지나눔 → 재페이지네이션.

## 4. 범위

### 포함
- overflow된 항목을 다음 페이지로 이월하는 reflow 메커니즘.
- native(export-svg)·WASM(스튜디오) 동일 경로 적용.
- 무한루프 가드: 본문보다 큰 단일 항목은 이월해도 넘치므로 제외(아래 범위 외).

### 범위 외
- 본문보다 큰 단일 항목(중첩 표/문단, page-larger: pi=567 856.7px 등)의 **내부 분할**.
  어느 페이지로 가도 넘침 → 별도 이슈. 본 타스크는 가드로 무한루프만 방지.
- 페이지네이터↔렌더러 측정 bit 정합(#1022 잔여 systematic).

## 5. 성공 기준

- 사용자 보고 28·35·49쪽(및 WASM 대응 페이지) 하단 항목이 본문 영역 안으로 이월.
- LAYOUT_OVERFLOW 16건 중 page-larger(이월 불가) 제외분이 0 또는 유의미 감소.
- `cargo test --release` 0 failed, 골든 SVG 회귀 무(또는 한컴 PDF 대조 정당 갱신).
- 페이지 수 증가는 정상(이월로 페이지 늘어남) — 한컴 2022 PDF 대조로 정당성 확인.

## 6. 리스크

- reflow 재페이지네이션이 기존 정상 페이지 배치를 흔들 수 있음 → 골든/테스트로 가드.
- 무한루프(이월→여전히 overflow→재이월) → page-larger 가드 필수.
- 두 페이지네이션 엔진(typeset.rs / engine.rs) 중 기본 typeset.rs 대상. fallback 정합 확인.
