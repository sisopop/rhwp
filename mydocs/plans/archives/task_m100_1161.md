# 수행계획서 — Task M100-1161: rhwp-studio 셀 안 picture 복사(Ctrl+C) 미지원

## 1. 이슈

- #1161 (M100, enhancement). 표 셀 안 inline picture(예: `samples/pic-in-table-01.hwp` page 22)를 객체 선택 후 Ctrl+C 로 복사해도 시스템/내부 클립보드 어디에도 그림이 저장되지 않음.
- 선행 #1151 의 `cell_path` 삽입 패턴은 PR #1160(close) → 재제출 **PR #1177 머지(devel `84a09e8c`)** 로 devel 반영 완료. 본 이슈는 그 **복사 경로 대칭** 작업.

## 2. 근본 원인 (진단 완료)

복사 경로의 WASM 함수들이 **본문(outer paragraph) 컨트롤만 접근**하고 셀 경로를 받지 못한다.

- 선택된 picture ref 는 셀 컨텍스트(`cellIdx`/`cellParaIdx`/`outerTableControlIdx`)를 이미 보유 — `cursor.ts:1228`,`:1243`.
- 복사 시 `copyControl(ref.sec, ref.ppi, ref.ci)` 로 본문 인덱스만 전달 — `input-handler-keyboard.ts:1274`.
- `copy_control_native` 는 `sections[sec].paragraphs[para].controls[ci]` 만 접근 — `clipboard.rs:222`. `ref.ppi`=outer 표 문단, `ref.ci`=셀 내부 컨트롤 인덱스라 outer 문단의 `controls[ci]`는 표이거나 범위 초과 → 잘못된 복사/에러.

### 추가 진단 (이슈가 onCopy·copyControl 1곳만 언급, 실제는 광범위)

복사는 **본문 전용 native 4개**를 사용 — 셀 그림을 시스템 클립보드에까지 정상화하려면 모두 셀 인지 필요. 4개 모두 `clipboard.rs` 에 있고 동일한 `para.controls.get(control_idx)` 패턴:

| native (clipboard.rs) | 역할 | 줄 |
|---|---|---|
| `copy_control_native` | 내부 클립보드 | 222 |
| `export_control_html_native` | 시스템 text/html | 924 |
| `get_control_image_data_native` | 시스템 image/png | 1284 |
| `get_control_image_mime_native` | image MIME | 1332 |

동일 `(ref.sec, ref.ppi, ref.ci)` 묶음이 여러 복사/오려두기 경로에 중복:
- onCopy `input-handler-keyboard.ts:1265`, onCut `:1347`
- onKeyDown 복사/오려두기 `:625 :652 :755 :776`
- 컨텍스트 메뉴 `input-handler.ts:2385 :2407`
- 이미지 기록 `writeImageToClipboard` `input-handler-keyboard.ts:178`

추가 발견: `getSelectedPictureRef()` 타입 재선언이 `input-handler.ts:2139` 에서 `outerTableControlIdx` 누락 → cursor.ts 정의와 불일치(타입 보정 필요).

## 3. 접근 (작업지시자 승인: 단일 함수 cell_path 확장 + 전 범위)

작업지시자 결정 2건:
1. **시그니처** — `insert_picture_native` 패턴 정합. 별도 함수 신설 없이 기존 함수에 `cell_path` 인자 추가(빈 = 본문).
2. **범위** — 시스템 클립보드 전체 + 모든 호출 지점.

### 재사용 자산 (신규 코드 최소화)

- `resolve_paragraph_by_path(sec, parent_para, &[(usize,usize,usize)])` — 셀/글상자 문단 탐색 헬퍼 기존재 `cursor_nav.rs:574`.
- `parse_cell_path(json)` — `[{"controlIndex","cellIndex","cellParaIndex"}]` → `Vec<(usize,usize,usize)>` `cursor_nav.rs:403`.
- 네 native 의 컨트롤 접근부를 **공통 private 헬퍼**로 일원화: `resolve_control_para(sec, para, cell_path) -> &Paragraph`(빈 경로=본문 `sections[sec].paragraphs[para]`, 아니면 `resolve_paragraph_by_path`). 이후 각 native 는 `para.controls.get(control_idx)` 만 동일 유지.
- TS cell_path JSON 조립: picture ref 스칼라 → 단일 레벨 `[{controlIndex: outerTableControlIdx, cellIndex: cellIdx, cellParaIndex: cellParaIdx}]`. (중첩 표/글상자는 picture 선택 ref 가 스칼라만 저장 → 단일 레벨 한계는 기존 제약, 신규 회귀 아님.)

## 6. 리스크

- native 4개 시그니처 변경 → 호출처(wasm_api 래퍼)만 영향. 본문 경로는 빈 cell_path 로 기존 동작 보존(테스트로 가드).
- 차용 검사: `resolve_*_by_path` 는 `&self`, 클립보드 기록은 `&mut self` — 컨트롤·문단을 clone/추출 후 불변 차용 종료 → 클립보드 기록(기존 copy_control_native 와 동일 순서) 패턴 유지.
- 다수 호출 지점 중복 → cell_path 조립 로직을 단일 헬퍼로 추출해 누락·불일치 방지.
- `feedback_image_renderer_paths_separate` 정합: 이미지 경로 native 변경이 렌더 경로에 새 분기 유발 안 하도록(클립보드 전용 함수만 수정) 점검.
