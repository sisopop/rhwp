# Task 248 수행 계획서: 조판부호 도형 마커 커서 네비게이션

## 배경

Task 247에서 도형 인라인 삽입 + 한컴 호환 직렬화는 완료.
남은 문제: 조판부호 보이기 모드에서 도형 마커 배치/커서 이동/F11 선택.

## 한컴 구조 (목표)

조판부호 보이기 모드에서:
- 인라인 컨트롤은 `[컨트롤명]` 형태의 **실제 폭을 가진 TextRun**으로 렌더링
- 이 TextRun이 줄 안에서 공간을 차지 → 뒤의 내용이 오른쪽으로 밀림
- 커서는 이 TextRun들 사이를 일반 텍스트처럼 이동
- F11은 커서 위치 이전의 컨트롤을 역순 선택

## 근본 원인 5가지

1. navigable_text_len 과다 계산 → 유효하지 않은 offset 생성
2. getCursorRect가 인라인 컨트롤 위치의 좌표를 반환 못함
3. 조판부호 마커가 텍스트 흐름과 분리됨 (MarkerInsert 후처리)
4. 인라인 컨트롤의 char_pos와 렌더링 좌표 매핑 부재
5. 순방향/역방향 네비게이션 건너뛰기 로직 불일치

## 해결 방향

**composition 단계에서 도형 마커를 실제 텍스트로 통합**

조판부호 모드: 도형 컨트롤 위치에 "[선]" 등 마커 텍스트를 ComposedLine의 run에 삽입
→ 마커가 일반 TextRun으로 렌더링 → getCursorRect 자연 동작 → 커서 이동 자연 동작
