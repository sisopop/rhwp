## 근본 원인 식별

`src/renderer/layout.rs:1991~2003` — TAC 표 분기:

```rust
} else if is_tac {
    let leading = composed.get(para_index)
        .map(|c| compute_tac_leading_width(c, control_index, styles))
        .unwrap_or(0.0);
    Some(col_area.x + effective_margin + leading)  // ← 무조건 좌측 정렬
}
```

비-TAC + Square wrap 분기 (Task #295) 에는 `t.common.horz_align` 반영이 있으나 **TAC 분기에는 ParaShape `alignment` 가 무시**됨. align=Right/Center 인 TAC 표가 단 좌측에 붙어버리는 회귀.

## 목표

ParaShape `alignment` 를 TAC 표 분기에 반영:
- `align=Right` → 단 우측 끝 - 표 폭 - margin_right
- `align=Center` → 단 중앙
- `align=Justify/Left/기타` → 기존 동작 (`base_x` 유지)

## 리스크

- **R1 — 다른 TAC 표 회귀**: align=Center/Right 인 TAC 표가 다른 샘플에 있을 경우 위치 변경 발생. 의도된 개선일 수도 있고 회귀일 수도 있음 → 회귀 스캔에서 사례별 판단 필요.
- **R2 — leading 손실**: 표 앞에 텍스트가 있는 경우 align 정렬로 leading 무시되면 회귀. `.max(base_x)` 로 방어.
- **R3 — `t.common.horz_align` 과 ParaShape `alignment` 우선순위**: 두 값이 다를 경우 어느 것을 따라야 하는가? 본 이슈에서는 `t.common.horz_align=문단(0)` (즉 horz_align 무지정) 이므로 ParaShape 만 고려해도 충분. 추후 다른 케이스 발견 시 재검토.
