# Task 247 수행 계획서: 도형 인라인 컨트롤 구조 전환

## 배경

한컴은 도형(gso)을 문단 내 인라인 컨트롤로 저장한다 (char code 11, 16바이트).
한 문단에 여러 도형과 텍스트가 혼재 가능하며, 컨트롤 사이에 커서 이동/텍스트 삽입이 가능하다.

현재 우리 구현은 도형을 **전용 문단**(1문단=1도형)에 저장하여:
- 컨트롤 간 커서 이동 불가
- F11로 도형 선택 불가 (도형 문단에 커서 진입 못함)
- 한컴 파일 로드 시 구조 차이 발생

## 분석: shape-01.hwp (한컴 작성)

```
Para 1: secd(ch=2) + cold(ch=2) + gso(ch=11) + gso(ch=11) + gso(ch=11) + PARA_END
nChars=41, ctrlMask=0x00000804
```

- 도형 3개가 같은 문단에 연속 배치
- char code 11 = 그리기 개체 (extended control, 16바이트)
- ctrl_id = `gso ` (0x67736F20)

## 목표

1. 도형 삽입 시 기존 문단에 인라인 컨트롤로 추가 (전용 문단 생성 대신)
2. 커서가 도형 컨트롤을 글자처럼 건너뛸 수 있도록 네비게이션 구현
3. F11로 도형 컨트롤 선택 가능

## 산출물

- `create_shape_control_native()` 구조 변경 (인라인 삽입)
- 커서 이동: FFFC 컨트롤 문자 건너뛰기
- F11 자연 동작
- 기존 HWP 파일과의 호환성 유지
