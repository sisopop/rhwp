# 수행계획서 — 분할 표 페이지네이션 모델 교체 (줄 범위 기반)

- 타스크: 로컬 task993 (GitHub 이슈 미발행 — 비공개 문서 검증에서 파생, 로컬 진행)
- 브랜치: `local/task993` (`local/task992` 위에서 분기)
- 마일스톤: M100 (v1.0.0)
- 작성일: 2026-05-19
- 선행: task992 (페이지밖 콘텐츠 제거) — 본 사안을 진단·인계

## 1. 배경

task992는 "페이지 영역 밖 콘텐츠 0"을 목표로 했다. Stage 3에서 페이지 171(페이지보다 큰 부동 표 데코레이션 오분류)을 해결해 오버플로를 4건 → 3건으로 줄였다. 남은 3건(페이지 123·144·176, 분할 표 연속분)은 task992의 v1~v5 시도(측정 보정·측정 경로 통일)로 모두 해소되지 않았고, 진단 끝에 **페이지네이터와 렌더러의 분할 표 모델이 근본적으로 비양립**함을 실측으로 확정했다(상세: `report/task_m100_992_report.md` §3).

## 2. 문제 정의

분할 표의 분할 상태가 두 코드에서 다른 모델로 표현된다.

- 페이지네이터(`typeset.rs`): 분할 상태를 `content_offset`(연속 px 높이)로 추적. "offset을 키우면 연속분이 줄어든다"는 단조성을 가정.
- 렌더러(`table_partial.rs`): `compute_cell_line_ranges`로 **이산 줄 범위**를 산출. 중첩 표는 atomic, HWP의 **LINE_SEG.vpos 리셋**(셀 내부 페이지 분할)은 hard break.

vpos 리셋이 있는 셀은 리셋 이후 콘텐츠가 `content_offset` 크기와 무관하게 항상 visible이 되어 px↔줄범위 매핑이 **비단조**다. 그 결과:

- 페이지네이터가 연속분을 과소 측정 → 추가 분할 누락 → 렌더러가 본문 밖까지 그림(오버플로).
- 측정을 정확히 고치면(task992 v5 후속) 페이지네이터가 리셋 이후를 무한히 분할하려다 **무한 루프**.

또한 누적 위치 계산이 `compute_cell_line_ranges`(vpos 동기화 有)·`calc_visible_content_height_from_ranges`(無)·`table_partial`의 `content_y_accum`로 3중 분기되어 서로 어긋난다.

## 3. 작업 범위

- 포함: 분할 표 분할 상태의 표현을 **연속 px `content_offset` → 이산 줄 범위 컷(per-cell·per-paragraph 줄 인덱스)** 으로 교체. 페이지네이터와 렌더러가 단일 권위 함수로 컷을 산출·소비하도록 통일. vpos 리셋을 자연 분할점으로 처리.
- 목표: 비공개 샘플에서 **페이지밖 콘텐츠 0**(현재 3건). 분할 표 무한 루프 제거. 누적 위치 계산 단일화.
- 제외: HWP3, 파서, 데코레이션 분류(task992 Stage 3 완료분), 폰트 메트릭 자체의 한컴 불일치, 페이지 수 누적 드리프트.

## 4. 검증 방법

- `export-svg` 후 전 페이지 `<text>` y > viewBox 높이 스캔 → 오버플로 0(현재 3건).
- 분할 표 샘플 export가 무한 루프 없이 정상 종료.
- `dump-pages`로 분할 표 연속분 측정이 렌더와 일치하는지 확인.
- `cargo test` 전체 통과 + `cargo clippy` 무경고.
- 골든 SVG 회귀 — 분할 표 문서의 골든이 이동하면 이동 페이지를 한컴 2022 PDF와 대조해 정정/회귀 판정.
- 공개 분할 표 샘플 교차 회귀 + 페이지 수 회귀 확인.

## 5. 단계 개요 (상세는 구현계획서)

1. 설계·조사 — 컷 디스크립터 타입 확정, `PartialTable` 생산자·소비자와 누적 위치 함수 전 호출부 조사.
2. 단일 컷 함수 구현 — `(start_cut, avail_height) → end_cut` (vpos 리셋 hard break, 중첩표 atomic). 단위 검증.
3. 렌더러 정합 — `layout_partial_table`이 컷 디스크립터로 렌더, `content_y_accum` 재판정 제거, `calc_visible_*` 재배선.
4. 페이지네이터 교체 — `typeset_block_table` 분할 루프를 컷 디스크립터로, `PartialTable` 스키마 변경.
5. 전체 검증(오버플로 0·무한 루프 0·골든 PDF 대조) + 최종 보고서.

## 6. 리스크

- `table_partial.rs`는 렌더러 최복잡 파일이며 `PartialTable` 스키마 변경은 `typeset.rs`·`layout.rs`로 파급된다 → 변경을 분할 표 경로로 한정하고 단계별 검증.
- 분할 표 문서의 골든 SVG가 광범위하게 이동 → 페이지별 PDF 대조 검증이 필수.
- 정확한 분할로 페이지 수가 늘 수 있다(오버플로 해소의 정상 귀결).

## 7. 비공개 문서 처리

재현용 HWPX/PDF는 비공개 문서로 커밋하지 않는다. 문서에 파일명·기관명을 적지 않는다. 회귀 테스트는 공개 골든 SVG·공개 샘플 우선, 불가 시 비커밋 처리 후 보고서에 명시한다.
