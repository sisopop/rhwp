## 본질 결함 (분석 단계 사전 확인)

### 결함 #1 — pi=16 인라인 표 라인 누적 (273px 단 하단 초과, 콘텐츠 손실)

**현상**:
```
LAYOUT_OVERFLOW: page=0, col=0, para=16, type=PartialParagraph, y=1357.8, bottom=1084.7, overflow=273.1px
LAYOUT_OVERFLOW: page=0, col=0, para=16, type=Shape,            y=1357.8, bottom=1084.7, overflow=273.1px
```

- pi=16 구성: 3×3 접수증 표 (642×280px, `tac=true` 인라인) + ㊞ 도장 (`wrap=InFrontOfText`) + 본문 2줄 (※ 군필자…)
- 라인 세그먼트: `ls[0]` vpos=55288 lh=1000 (선두 빈 라인) / `ls[1]` vpos=56888 **lh=21296 (~284px)** (인라인 표 포함)
- PDF는 1페이지에 모두 담는데, rhwp는 표 + 본문을 누적해 1357.8px까지 밀어내며 본체 하단 1084.7px를 273px 초과
- 결과: \"복 학 원 서 접 수 증\", \"대학(Name of College)\", \"학번(Student No.)\", \"위 학생의 복학원서를 접수함\", \"년 월 일 ㊞\", \"※ 군필자…\" + \"고 려 대 학 교 총 장 귀 하\" 큰 제목 모두 잘리거나 워터마크에 가려짐

### 결함 #2 — 워터마크 이미지 효과 미적용 (고려대학교 엠블럼 어두움)

- 페이지 중앙 131×131mm 엠블럼이 PDF에서는 흐릿한 회색 워터마크인데, rhwp에서는 진한 회색 솔리드로 본문을 가림
- HWP IR: `effect=GrayScale, brightness=-50, contrast=70, watermark=custom`
- 한컴 표준 워터마크 프리셋은 `brightness=+70, contrast=-50` (`src/model/image.rs:75`) — 부호 반대
- 추정: 파서 부호 처리 결함 또는 한컴 편집기의 워터마크 모드 강제 적용 가능성 (조사 단계에서 확정)

## 접근 (요약, 상세는 구현계획서)

### 결함 #1 접근 후보

후보 A — **layout 레벨 진단**: `paragraph_layout.rs` 의 `text_y` 계산이 `has_tac_shape`/`has_tac_table` 조건에서 `raw_lh - line_height` 만큼 y를 추가 이동시키는데, 이미 `ls[1].vpos` 가 라인 시작 위치를 정확히 표현하므로 추가 이동이 누적 오차를 유발. 인라인 표 라인의 경우 `text_y = y` (조정 없음) 영역으로 좁힌다.

후보 B — **typeset 레벨 진단**: `tac_offsets_px` 가 표/도형 전체 높이를 라인 높이에 누적하지만, 본문 텍스트는 별개 라인에 분리된 상태. 라인 높이 계산을 분리한다.

후보 C — **HWP IR 진단**: `ls[1].lh=21296` 자체가 표 + 본문 합산이 아니라 표만의 높이여야 한다. 파서 영역에서 LineSeg 별 의미 재확인.

진단 단계에서 후보 결정 (ir-diff / dump / 코드 추적).

### 결함 #2 접근 후보

후보 A — **파서 부호 진단**: `src/parser/control/shape.rs:848` 의 `read_i8` 결과 와 한컴 편집기 저장값 비교. 다른 워터마크 적용 fixture 와 교차 검증.

후보 B — **렌더러 워터마크 모드 진단**: `effect=GrayScale` + `brightness/contrast` 비-zero 시 한컴 편집기는 워터마크 프리셋을 강제 적용한다는 가설 검증. `is_watermark()` (`src/model/image.rs:65`) 가 이미 식별 영역으로 존재.

후보 C — **opacity/alpha 진단**: 한컴 워터마크의 시각적 흐림이 brightness/contrast 가 아닌 별도 alpha 채널 또는 SVG `opacity` 속성으로 적용되어야 할 가능성. 한컴 PDF 의 워터마크 색상 분석.

진단 단계에서 후보 결정.
