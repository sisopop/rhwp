# Task #816 수행 계획서

## 1. 이슈

- Issue: #816
- 제목: HWPX export-svg stops after 4 pages; export-pdf renders blank/missing content
- 상태: OPEN
- 마일스톤: m100

## 2. 현상 요약

이슈 등록 시점의 보고는 다음과 같다.

```text
HWPX 10페이지 이상 문서에서 export-svg가 4페이지만 생성된다.
export-pdf는 전체 페이지를 만들지만 5페이지 이후 표 셀 텍스트가 비어 있다.
동일 문서를 HWP binary로 저장하면 정상 출력된다.
```

보고된 환경은 `rhwp v0.7.11 custom build from main branch`이다.

## 3. 현재 판단

현재 `devel`은 v0.7.13 계열이며, 이후 HWPX 렌더링/페이지네이션/표 셀 텍스트 처리 관련 수정이 다수 반영되었다.
따라서 이번 이슈는 새 구현보다 **현행 devel에서 재현 여부를 먼저 판정**한다.

## 4. 검증 대상

명시 샘플이 이슈 본문에 포함되어 있지 않으므로, 저장소에 있는 10페이지 이상 HWPX 샘플 중
표 셀 텍스트와 다중 페이지를 포함하는 문서를 대표로 사용한다.

| 구분 | 파일 | 목적 |
|---|---|---|
| HWPX guard | `samples/hwpx/exam_kor.hwpx` | 20페이지 HWPX, 시험지/표/다단 렌더링 |
| HWP guard | `samples/exam_kor.hwp` | 동일 계열 HWP binary 비교 |
| HWPX guard | `samples/hwpx/aift.hwpx` | 74페이지 HWPX, 후반 표 셀 텍스트 포함 |
| HWP guard | `samples/aift.hwp` | 동일 계열 HWP binary 비교 |

## 5. 판정 기준

```text
1. export-svg가 HWPX 전체 페이지 수만큼 SVG를 생성한다.
2. 5페이지 이후 SVG에 표 셀 텍스트가 포함된다.
3. export-pdf의 후반 단일 페이지 출력에서 텍스트가 보존된다.
4. 동일 HWP binary와 페이지 수가 같은 계열에서는 HWPX도 같은 페이지 수를 낸다.
```
