## 1. 배경

v3 Stage 4 시각 검증 4 시나리오 통과 후, 사용자가 `samples/tac-img-02.hwp` 에서 별도 결함 발견 (2026-05-29):

> 표 셀 안 paragraph 의 inline picture (`bin_id=2, tac=true, wrap=TopAndBottom, vert=Para(off=0), horz=Para(off=0)`, dump line 222) 가 rhwp-studio dev server 에서 클릭되지 않음. picture select 모드 진입 / cursor 이동 모두 안 됨.

v3 fix 이전부터 존재한 **기존 결함** (v3 회귀 아님). Task #1151 "표 + picture 한컴 정합" 묶음의 자연스러운 확장.

### Root cause (3-layer fault, Explore 분석)

| Layer | 파일 | 결함 |
|---|---|---|
| Rust ImageNode 생성 | [src/renderer/layout/paragraph_layout.rs:2688, 3067, 3215](../../src/renderer/layout/paragraph_layout.rs) + [src/renderer/layout.rs:4640](../../src/renderer/layout.rs) | ImageNode 생성 시 `cell_index` / `cell_para_index` 필드 명시 설정 누락 (`..ImageNode::new(...)` 의 Default → None). `tree.set_inline_shape_position(cell_ctx.as_ref(), ...)` 는 inline_shape_positions HashMap 에 cell_path 등록하지만 RenderNode 자체는 cell context 보존 안 함. |
| Rust JSON 직렬화 | [src/document_core/queries/rendering.rs:1495-1499](../../src/document_core/queries/rendering.rs#L1495-L1499) | Image JSON 출력에 `cellIdx` / `cellParaIdx` 미포함 (Rectangle 은 line 1524-1530 에서 정상 처리). studio 가 picture 클릭 시 cell context 를 받지 못해 본문 picture 로만 처리. |
| Rust hit-test | [src/document_core/queries/cursor_rect.rs:1218-1223](../../src/document_core/queries/cursor_rect.rs#L1218-L1223) | `if !cell_path.is_empty() { continue; }` — 셀 내부 inline shape 모두 스킵. 주석 명시: "셀 내부 inline shape 은 cursor hit-test 에서 별도 처리 — 섹션 단위만 검사". 그러나 picture 의 별도 처리 path 가 누락. |

## 2. 목표

셀 안 inline picture (tac=true) 의 click hit-test 가 정상 작동하여:
- picture 위 클릭 → picture select 모드 진입 (또는 cursor 이동)
- 그림 속성 대화상자에서 picture 의 cell context (cellPath, innerControlIdx) 가 정확히 전달
- 셀 내부 inline picture 의 model 위치 (cell.paragraphs[*].controls[*]) 와 click hit-test 가 일치

- `samples/tac-img-02.hwp` 의 표 안 inline picture 위 클릭 → picture select / cursor 이동 정상.
- v1 의 셀 floating picture 클릭 (v1 PR 정합) 회귀 0.
- v2 / v3 의 시나리오 회귀 0 (model 정합 4 통합 테스트 + 시각 4 시나리오).

- `cargo test --lib` 전수 통과 + `cargo clippy --lib -- -D warnings` + `cargo fmt --all -- --check`.

## 6. 위험 요소

- **Risk (낮음)**: cursor_rect.rs 의 `if !cell_path.is_empty() { continue; }` 제거 시 셀 안 inline Shape 의 hit-test 가 잘못 작동할 가능성. cell_path 가 있는 경우의 별도 분기로 명시 처리.
- **Risk (중)**: nested 표 (표 안 inner 표) 안 inline picture 의 cell_path 가 multi-segment. v4 의 hit-test 가 이 케이스를 정확 처리하는지 검증 필요.
- **Risk (낮음)**: studio 측 click handler 가 신규 cellIdx 를 받아 어떻게 사용하는지 검증. picture 속성 대화상자에서 cellPath 전달 path 가 정상 동작하는지.
- **Risk (낮음)**: rendering.rs JSON 출력 변경이 다른 studio path 에 영향. Image JSON 의 추가 필드는 optional 이므로 기존 path 회귀 위험 0.
