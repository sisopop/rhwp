# Task #295: exam_math.hwp 12쪽 좌단 레이아웃 붕괴 — 수행계획서

## 배경

`samples/exam_math.hwp` 12쪽은 2단(段) 다단 레이아웃으로 좌단=29번 문제(본문+표 5×2+수식), 우단=30번 문제(본문+수식+그림)로 구성된다.
PDF(`samples/exam_math.pdf`) 12쪽 대비 우리 SVG 출력에서:

1. **좌단 콘텐츠가 페이지 하단으로 밀려나 압축·겹침** 발생 (주증상)
2. 머리말 페이지번호 `4`(PDF) → `2`(우리) 불일치 (부증상)

## 증상 데이터

`./target/release/rhwp export-svg samples/exam_math.hwp -p 11`:


`dump-pages -p 11`:
- body_area: w=888.2 h=1213.3
- 단 0(19개): pi=22(표 1×1×2) → pi=23(29번 본문+수식4) → pi=24(빈) → pi=25(본문+수식3) → pi=26(빈) → **pi=27(표 5×2 + 수식5)**
- 단 1(23개): pi=28(30번 본문+수식6) … pi=33(그림)

좌단 마지막 항목 pi=27(표 5×2, 146.9×114.4px, wrap=Square)이 단 높이를 약 150px 초과한다.

## 원인 가설

1. **좌단 컬럼 높이 산정/잔여공간 추적 버그** — pi=27 표(wrap=Square)가 좌단 가용 높이를 초과하는데도 같은 컬럼에 강제 배치되어 페이지 하단으로 흘러내림.
2. **wrap=Square 표의 컬럼 경계 처리 누락** — TopAndBottom 표(pi=22)와 달리 Square 랩 모드에서 컬럼 종료 판정이 안 들어가는 가능성.
3. **다단 페이지의 컬럼 분할/페이지 분할 우선순위** — 다단에서는 컬럼이 모두 차면 다음 페이지로 가야 하는데, 단 1로 흘리지 않고 같은 단에 강제 배치되는 패턴.
4. **머리말 페이지번호** — 섹션 시작번호 또는 시작 페이지 산정 오류 (별건). 본 타스크에서 같이 다룰지는 결정 필요.
