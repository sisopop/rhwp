## 1. 배경 / 증상

`samples/3-09월_교육_통합_2022.hwpx` 9쪽 우측 단 "다른 풀이"가 PDF에는 **태그 박스(`〉` 꺾쇠) 안**에 표시되나, 우리 렌더는 **텍스트만 있고 박스 외곽선이 없음**.

## 2. 근본 원인 (확정)

"다른 풀이"는 `<hp:curve id="0">` 도형의 `drawText` 텍스트이며, 박스는 이 curve 도형의 외곽선이다.

HWPX 이 curve 의 geometry 는 **`<hp:seg type="CURVE|LINE" x1 y1 x2 y2>` 417개**로 인코딩됨 (`<hc:pt>` 0개).
그러나 파서 `parse_shape_object()` (`src/parser/hwpx/section.rs`)는 가변 꼭짓점을 **`<hc:pt>`(`b"pt"` 분기)에서만** 수집한다 → `polygon_points` 가 비어 `CurveShape.points` 가 empty → `curve_to_path_commands_scaled()` 가 빈 path 반환 → **외곽선 미렌더**. (drawText 텍스트는 별도 경로로 렌더되어 텍스트만 보임.)

렌더 인프라(stroke → SVG/Skia, `drawing_to_shape_style`)는 **이미 완비**되어 있어, 점만 채우면 외곽선이 그려진다.

### 좌표/구조 확인
- seg 좌표 범위 x:50..3145, y:50..1040 = `orgSz 3175×1090` 로컬 HWPUNIT (렌더러가 기대하는 공간과 동일).
- 모든 seg 가 chain (seg[i].end == seg[i+1].start), 폐곡선(last.end == first.start).
- seg 는 점-대-점 인코딩(제어점 없음) → 폴리라인으로 재구성 (segment_types 비움 → 렌더러 LineTo).

## 3. 수정 범위 (최소, 파서 단일)

`parse_shape_object()` 에 `b"seg"` 분기 추가:
- 첫 seg 에서 `(x1,y1)` push, 모든 seg 에서 `(x2,y2)` push → `polygon_points` 폴리라인 구성.
- `segment_types` 는 비운 채 유지(폴리라인). 모델/렌더러 무변경.

## 4. 검증

1. `cargo build --release` / `cargo test --release` (회귀 0).
2. `export-svg -p 8` → "다른 풀이" 태그 박스 외곽선이 path 로 렌더되는지 확인 (SVG `<path>` stroke).
3. 한글 2022 PDF 9쪽과 시각 대조 (박스 형태 정합).
4. `rustfmt`(변경 파일 한정).

## 5. 리스크

- 낮음. `<hp:seg>` 는 curve 에만 존재 → 타 도형 영향 없음. `<hc:pt>` 기반 curve/polygon 경로 불변(추가 분기).
- 회귀 점검: 기존 polygon/curve 샘플 렌더 유지.

## 6. 범위 외

- `<hp:seg>` 의 CURVE 타입을 진짜 베지어로 처리하는 정밀화는 본 건 범위 아님(점-대-점 폴리라인으로 시각 정합 달성). 필요 시 후속.
