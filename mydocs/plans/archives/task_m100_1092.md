# Task M100-1092 수행 계획서

## 1. 작업 개요

GitHub Issue #1092:

```text
[hwpx2hwp] aift.hwpx 메모 컨트롤 HWP 저장 직렬화
```

목표는 HWPX 문서의 메모 컨트롤을 HWP로 저장할 때 HWP5 control/record contract에 맞게
직렬화하는 것이다.

작업지시자의 현재 판단:

```text
메모 컨트롤만 정상적으로 저장하면 문제가 해결될 것으로 판단한다.
```

이번 작업은 이 판단을 핵심 가설로 삼고, 정답 HWP와 현재 저장 HWP의 메모 control/record 차이를
핀셋 비교한 뒤 필요한 최소 구현만 반영한다.

## 2. 대상 샘플

```text
samples/hwpx/aift.hwpx
samples/aift.hwp
saved/111aift.hwp
```

각 파일의 의미:

```text
samples/hwpx/aift.hwpx
  - 작업지시자가 2026-05-23에 갱신한 HWPX 원본

samples/aift.hwp
  - 한컴 변환/정답 HWP

saved/111aift.hwp
  - 현재 rhwp-studio에서 저장한 재현 HWP
```

## 3. 사전 관찰

소스 검색 기준으로 현재 코드에는 메모 관련 축이 일부 존재한다.

```text
src/serializer/control.rs
  - Control::HiddenComment
  - serialize_hidden_comment

src/parser/tags.rs
  - HWPTAG_MEMO_SHAPE
  - HWPTAG_MEMO_LIST

src/model/document.rs
  - doc_info.memo_shape_count

src/serializer/doc_info.rs
  - ID_MAPPINGS memo_shape_count 직렬화

src/diagnostics/hwp5_contract_analyze.rs
src/diagnostics/hwp5_contract_probe.rs
  - MEMO_SHAPE / ID_MAPPINGS / CTRL_DATA 축 진단 코드
```

따라서 문제는 완전 미구현이라기보다 HWPX 파싱 결과가 HWP5 저장에 필요한 다음 계약을 충분히
만들지 못하는 형태일 가능성이 높다.

```text
1. 메모 control이 본문 문단에 올바른 위치/레벨로 materialize되지 않음
2. MEMO_SHAPE DocInfo record 또는 ID_MAPPINGS memo_shape_count 불일치
3. MEMO_LIST/HiddenComment 본문 목록 직렬화 불일치
4. 메모 anchor control과 메모 내용 list의 참조 관계 불일치
```

## 4. 핵심 가설

이번 이슈의 우선 가설은 다음이다.

```text
HWPX -> HWP 저장 시 메모 컨트롤의 HWP5 record bundle만 정답지와 맞추면,
aift 저장 문제는 해결된다.
```

따라서 표/도형/일반 조판 보정으로 넓히지 않고, 먼저 메모 control/record 계약만 독립적으로
확정한다.

## 5. Stage 1 계획 — 메모 contract trace

Stage 1에서는 구현하지 않고 정답지와 현재 저장본의 차이를 문서화한다.

출력 위치:

```text
output/poc/hwpx2hwp/task1092/stage1_memo_contract_trace/
```

산출물:

```text
memo_record_inventory.md
memo_shape_id_mapping_diff.md
memo_control_bundle_diff.md
memo_source_trace.md
```

확인 항목:

```text
1. samples/aift.hwp의 MEMO_SHAPE, MEMO_LIST, CTRL_HEADER, LIST_HEADER 분포
2. saved/111aift.hwp의 동일 record 분포
3. HWPX XML에서 메모 control/source 위치
4. 정답 HWP의 메모 anchor 문단/char offset/control index
5. 현재 저장본의 누락/오배치 영역
```

## 6. Stage 2 계획 — 최소 구현 후보

Stage 1 결과에 따라 다음 중 필요한 축만 구현한다.

예상 후보:

```text
1. HWPX 메모 요소를 IR의 Control::HiddenComment 또는 대응 control로 materialize
2. DocInfo MEMO_SHAPE record / memo_shape_count 보존 또는 합성
3. 메모 본문 paragraph list를 HWP5 MEMO_LIST/HiddenComment contract로 직렬화
4. 메모 anchor control의 payload/reference 정합
```

예상 수정 파일:

```text
src/parser/hwpx/section.rs
src/parser/hwpx/header.rs
src/model/control.rs
src/serializer/control.rs
src/serializer/doc_info.rs
src/document_core/converters/hwpx_to_hwp.rs
tests/hwpx_to_hwp_adapter.rs 또는 신규 tests/issue_1092_*.rs
```

정확한 수정 범위는 Stage 1 trace 후 확정한다.
