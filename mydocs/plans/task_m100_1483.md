# Task M100 #1483 수행계획서 — 표 줄/칸 지우기 후 커서 cellIndex 보정

- 이슈: #1483 "[Bug] 표 줄/칸 지우기 후 커서 cellIndex 미보정 — updateRect '셀 인덱스 초과' 실패"
- 마일스톤: M100 (v1.0.0)
- 브랜치: `local/task1483`
- 작성일: 2026-06-23

## 1. 배경

확장(크롬) 테스트 중 발견. 표 줄/칸 지우기 후 삭제로 셀 수가 줄면, 커서가 더 이상
존재하지 않는 cellIndex를 가리켜 `updateRect`가 "경로[0]: 셀 인덱스 N 초과 (총 M개)"로
실패한다. PR #1482(표 줄/칸 입력·지우기) 머지 후 드러난 커서 후처리 누락.

## 2. 원인 (소스 조사 완료)

`rhwp-studio/src/command/commands/table.ts` `applyTableDeleteRowColumn` 의
`operation` 콜백이 삭제 후 **`return pos`** 로 삭제 전 커서 위치(원래 cellIndex)를 그대로 반환.

- `deleteTableRow`/`deleteTableColumn` 은 `{ ok, rowCount, colCount }` 반환 (삭제 후 표 크기).
- 삭제로 셀 수가 줄면 원래 cellIndex 가 새 범위를 초과 → `getCursorRectInCell` →
  `field_query.rs`(경로[0] 셀 인덱스 검증)에서 거부.
- 대조: 줄/칸 추가는 셀 수가 늘어 기존 cellIndex 가 유효 → 문제 없음. 표 만들기는
  `cellIndex: 0` 으로 보정.

## 3. 목표

표 줄/칸 지우기 후, 반환 커서 위치를 삭제 후 표의 유효 범위로 보정한다.

## 4. 수정 방향

- 삭제 전 커서의 (row, col) = `cellInfo.row/col`.
- 삭제 후 표 크기 = `deleteTableRow/Column` 반환의 `{ rowCount, colCount }`.
- 보정: `row = min(원래 row, rowCount-1)`, `col = min(원래 col, colCount-1)` 로 clamp.
- clamp 된 (row, col) → cellIndex 는 `getTableCellBboxes`(cellIdx↔row/col 매핑, 셀 병합도 정확)
  로 역조회하여 확정.
- 반환 pos 의 `cellIndex`/`cellParaIndex`(=0) 를 보정값으로 교체.
- 표의 모든 행/열이 삭제되어 표 자체가 사라지는 경우(rowCount==0 또는 colCount==0)는
  표 밖 본문 위치로 폴백(별도 가드).

## 5. 범위

- **포함**: `applyTableDeleteRowColumn` 의 커서 보정. rhwp-studio 한정(TS).
- **제외**: WASM/Rust 변경 없음(반환값·조회 API 기존 것 재사용). 줄/칸 추가 경로는 무영향.

## 6. 검증 기준

1. 표 생성 → 마지막 칸/줄 근처 커서 → 줄/칸 지우기 → `updateRect` 오류 없음(콘솔 무경고).
2. 보정된 커서가 유효 셀을 가리킴(getCursorRectInCell 성공).
3. E2E 또는 단위 테스트로 삭제 후 cellIndex 범위 내 보장.
4. `rhwp-studio` tsc 통과, 기존 표 편집 E2E 회귀 없음.
