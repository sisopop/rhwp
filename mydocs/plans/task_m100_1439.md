# Task M100 #1439 수행계획서 — 드래그&드롭 로컬 파일 로딩 보안 게이트

- 이슈: #1439 "드래그&드롭 로컬 파일 로딩을 기본 동작에서 제외 + 드롭 시 모달 확인 대화상자 (보안 UX)"
- 마일스톤: M100 (v1.0.0)
- 브랜치: `local/task1439` (`local/devel` 분기)
- 작성일: 2026-06-19

## 1. 현황 (조사 완료)

`rhwp-studio/src/main.ts:364~393` drop 핸들러는 `#scroll-container` 에 HWP/HWPX 를
드롭하면 **확인·안내 없이 즉시 `loadFile(file)`** 한다.

- 드롭 시각 피드백은 `drag-over` CSS 영역 강조뿐.
- 상태표시줄(`#sb-message`) 메시지는 "파일 로딩 중...", "HWPX 변환 저장 모드..." 등
  로딩 진행/저장 모드 안내 — **드롭이 로컬 파일을 읽는다는 보안 고지가 아니다**.
- `loadFile(file, {skipUnsavedGuard})` 는 unsaved 가드(`confirmSaveBeforeReplacingDocument`)
  를 거치나, 이는 "저장 안 된 변경" 경고이지 "이 로컬 파일을 여느냐" 동의가 아니다.

참조 모델: `rhwp-studio/src/ui/unsaved-changes-dialog.ts` — `ModalDialog` 상속 +
`showAsync(): Promise<choice>` + 버튼 커스터마이징. 드롭 확인 대화상자도 동형.

## 2. 목표

드래그&드롭 로컬 파일 로딩을 **기본 동작에서 제외**하고, 드롭 시 **모달 확인 대화상자**로
사용자가 명시적으로 [열기]를 눌러야 로딩한다. 기본값(미동의)은 로딩 안 함 — 보안 안전.

## 3. 설계 방향 (구현계획서에서 확정)

- 신규 `rhwp-studio/src/ui/drop-confirm-dialog.ts`:
  `showDropConfirmDialog(fileName): Promise<boolean>` (`ModalDialog` 상속, unsaved 동형).
  본문: "로컬 파일을 엽니다" 고지 + 파일명 + [열기]/[취소]. 기본 포커스 안전(취소).
- `main.ts` drop 핸들러: HWP/HWPX 분기에서 `loadFile` 호출 **전에** 확인 대화상자.
  - [열기] → `loadFile(file)` (이후 unsaved 가드는 기존대로). [취소]/밖 클릭 → 미로딩.
  - 순서: **드롭 확인(보안) → loadFile 내부 unsaved 가드**. 사용자가 먼저 "이 파일 열기"
    동의 후 저장 경고.
- 이미지 드롭/파일 메뉴 경로 처리 범위는 구현계획서에서 확정(우선 HWP/HWPX 보안 게이트).

## 4. 목표와 완료 조건

1. 드롭 시 즉시 로딩되지 않고 모달 확인 대화상자 표시.
2. [열기]에서만 `loadFile` 호출, [취소]/밖 클릭 시 미로딩.
3. 기존 파일 입력(파일 열기 버튼/메뉴) 경로 영향 0.
4. e2e: 드롭 → 확인 표시 / [열기] 로딩 / [취소] 미로딩.

## 5. 단계 개요 (3단계 예정 — 구현계획서에서 확정)

1. `drop-confirm-dialog.ts` 신규 + drop 핸들러 게이트 연결 (HWP/HWPX).
2. unsaved 가드 순서 정합 + 이미지/메뉴 경로 범위 확정 + 빌드/타입체크.
3. e2e 테스트(드롭 확인/열기/취소) + 보안 문서 반영 + 보고서.

| 위험 | 대응 |
