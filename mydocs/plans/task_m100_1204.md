## 2. 근본 원인 (실제 스크립트로 확정)

### A. `root3` — root/sqrt 가 피연산자에 붙은 경우 √ 미렌더 (21 scripts)
`tokenizer.rs:read_command` 의 glued-keyword 분리 목록이 `bold/it/rm/times/sim` 뿐. `root`/`sqrt` 미포함 → `root3` 한 토큰 → 인식 실패 → "root3" 출력.
(과거 Task #576 주석: "sqrt 등은 항상 공백 구분" 가정 — 본 문서의 `root3`/`root5` 가 반례.)

### B. `rm bar {...}` — overline 미렌더 (28 scripts)
`parser.rs:parse_single_or_group` 의 Command 분기가 symbol/function 외 명령을 `EqNode::Text` 로 처리. `rm` 의 body 로 소비된 `bar` 가 Text("bar") 로 leak. (`parse_command` 는 decoration/structure 를 정상 처리하나 이 경로를 안 탐.)

### C. `primeF` — prime 이 글자에 붙은 경우 leak
`F primeF` → "primeF" 한 토큰 → Text leak. (`f prime(x)` 공백은 정상.)

## 3. 수정 범위 (수식 렌더러 단일 영역)

`src/renderer/equation/`:
- **A**: `tokenizer.rs` — `root`/`sqrt`(+대문자) 가 **숫자**에 붙으면 분리 (over/atop digit-guard 패턴 차용, letter-collision 회귀 위험 회피).
- **B**: `parser.rs:parse_single_or_group` — Command 분기에서 symbol/function 외에는 `parse_command` 재귀 (fall-through 가 unknown→Text 이므로 안전).
- **C**: `tokenizer.rs` — `prime`(+대문자) 가 alnum 에 붙으면 분리 (bold/it/rm 패턴 차용).

모델/레이아웃/타 렌더러 무변경 예상.
