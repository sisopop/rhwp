## 1. 배경

`samples/투명도0-50-2nd그림글차처럼off.hwp` 샘플은 한 문단 안에 두 그림이 있다.

- 첫 번째 그림: `글자처럼 취급` 켜짐 (`treat_as_char=true`)
- 두 번째 그림: `글자처럼 취급` 꺼짐 + `자리 차지` (`treat_as_char=false`, `textWrap=TopAndBottom`)

한컴 기준으로 두 번째 그림은 문단 흐름에 공간을 차지하므로, 두 번째 그림의 크기나 배치 속성을 변경하면
문단 레이아웃 전체가 다시 계산되어야 한다. 따라서 같은 문단의 첫 번째 `글자처럼 취급` 그림 위치도 새 흐름에
맞춰 다시 배치되어야 한다.

현재 rhwp-studio에서는 속성 변경 직후 두 그림의 상대 위치가 한컴과 다르게 남는 증상이 확인되었다.

## 2. 현재 확인 내용

- 개체 속성 다이얼로그는 `setPictureProperties` 호출 후 `document-changed` 이벤트를 emit한다.
- 캔버스는 `document-changed` 이벤트에서 `refreshPages()`를 호출한다.
- 따라서 단순 repaint 이벤트 누락보다는 Rust 레이아웃 또는 flow reservation 재계산 경로 문제일 가능성이 높다.
- 샘플 덤프 기준 `ci=2`는 `treat_as_char=true`, `ci=3`은 `treat_as_char=false + TopAndBottom`이며 같은 빈 문단과
  같은 `LINE_SEG` 흐름에 있다.

## 3. 작업 범위

1. `setPictureProperties`가 그림 공통 속성을 변경한 뒤 레이아웃에 stale 상태가 남는지 확인한다.
2. 비-TAC `TopAndBottom` 그림의 크기/배치 변경이 문단 높이와 inline 그림 위치 계산에 반영되도록 보정한다.
3. 샘플 기반 회귀 테스트를 추가해 같은 문단 안의 TAC 그림과 자리차지 그림 위치가 함께 갱신되는지 확인한다.
4. HWP/HWPX 샘플 파일은 회귀 fixture로 보존한다.

## 4. 제외 범위

- 그림 투명도/opacity 재보정
- 모든 floating wrap 모드의 전면 재설계

- 저장 포맷의 신규 속성 추가
