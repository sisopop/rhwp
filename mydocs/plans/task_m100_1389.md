## 2. 현황 분석 (2026-06-14 — 코드·실물 확인 완료)

### 결손 3축

| 요소 | IR 보존 | serializer 결손 | 손볼 범위 |
|------|---------|----------------|-----------|
| **curSz** | ✓ `shape_attr.current_width/height` (parser :2230) | `write_cur_sz(&pic.common)` = sz 값 방출 | **serializer만** |
| **imgRect** | ✓ `pic.border_x/border_y` (parser :2447) | `write_img_rect(&pic.common)` = common 합성 | **serializer만** |
| **imgDim** | ✗ **IR 미적재** (Picture에 필드 없음, parser arm 없음) | `write_img_dim` 간이 계산 → imgClip 의미 오해로 0×0 | **모델+파서+serializer** |

### 실측 (ta-pic-001-r RT)

| 요소 | 원본 pic0 | RT pic0 | RT pic1 |
|------|----------|---------|---------|
| curSz | 13668×12686 | 18425×18160 (변형) | 22792×11932 (정상) |
| imgDim | 49380×45840 | **0×0** | **0×0** (전 pic 변형) |

curSz가 pic1만 정상인 이유는 1단계 정밀 확인 (current_width 적재 0 시 common 폴백
케이스 추정). imgDim은 전 pic 0×0 (간이 계산 결함).

### 게이트 사각

- `diff_documents`는 컨트롤 내부 속성(pic 크기) 미비교 → baseline 통과 (#1379 한계 축).
- 게이트 동승 검토: curSz/imgRect/imgDim을 Picture 비교에 추가할지 1단계 판정.

### 시각 판정 의존

- 이 결함은 **한컴 편집기 시각 판정으로만 최종 확인** 가능 (#1379 선례 — 자기 정합
  ≠ 한컴 호환). roundtrip IR diff 0이어도 한컴 미반영이 본질이었다.

## 3. 목표와 완료 조건

1. curSz → `shape_attr.current_width/height` 방출 (0 폴백 시 common).
2. imgRect → `pic.border_x/border_y` 보존 값 방출.
3. imgDim → IR 적재(모델 필드 + 파서 arm) + serializer 보존 방출 (간이 계산 제거).
4. ta-pic-001-r RT의 세 요소가 원본과 일치, 전수 배치 신규 실패 0.
5. **한컴 편집기 판정: 셀 안 이미지 크기 반영** (작업지시자).

## 4. 단계 개요 (구현계획서에서 상세화 — 3단계)

1. **측정/IR 확정**: curSz pic별 변형 분포(왜 pic1 정상) + imgDim IR 필드 설계
   (적재 위치·타입) + 게이트 동승 판정 + 전수 pic 보유 샘플 영향
2. **모델+파서+serializer**: imgDim IR 필드 + 파서 arm + 세 함수 보존 방출 정정
   + (판정 시) 게이트 동승 + 단위 테스트
3. **전수 검증 + 문서 + 한컴 판정 요청**: 배치, ta-pic RT 세 요소 일치, 매뉴얼,
   최종 보고서, rt 한컴 판정 (셀 그림 크기)

## 5. 위험과 대응

| 위험 | 대응 |
|------|------|
| curSz current_width=0 폴백이 일부 pic에서 의도된 동작 | 1단계 분포 측정 — 0이면 common 폴백 유지(원본도 그 경우 sz=curSz) |
| imgRect border_x가 일부 pic에서 0 (미적재 케이스) | 1단계 확인 — 0이면 common 합성 폴백 |
| imgDim IR 추가가 다른 경로(HWP5 변환) 회귀 | HWPX 한정 필드로 추가, 전수 배치 회귀 검출 |
| 자기 정합 ≠ 한컴 호환 (#1379 선례) | 3단계 한컴 판정 게이트 필수 |
