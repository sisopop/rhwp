# Task M100-1100 수행 계획서

## 1. 작업 개요

GitHub Issue #1100:

```text
[hwpx] exam_social.hwpx HWP와 다른 렌더링 수정
```

`samples/hwpx/exam_social.hwpx`는 같은 문서의 HWP 정답지와 렌더링 결과가 다르다. 이번 이슈는
HWPX를 HWP로 저장하는 문제가 아니라, HWPX 파서/IR/렌더링 경로가 HWP 정답지와 다르게 해석되는
문제를 분리해서 처리한다.

## 2. 대상 파일

```text
source HWPX:
  samples/hwpx/exam_social.hwpx

oracle HWP:
  samples/exam_social.hwp
```

## 3. 목표

```text
1. HWP 정답지와 HWPX 렌더링 차이가 발생하는 페이지/영역을 특정한다.
2. HWP/HWPX parser가 만든 IR 차이를 control, section, master page, paragraph/table 단위로 비교한다.
3. HWPX 쪽에서 누락되거나 잘못 매핑된 속성을 특정한다.
4. HWP 정답지와 같은 시각 결과가 나오도록 HWPX 렌더링 경로를 수정한다.
5. 수정 후 SVG/웹 렌더링 시각 판정용 산출물을 생성한다.
```

## 4. 범위

이번 이슈의 범위:

```text
HWPX -> Document IR -> renderer
```

이번 이슈의 비범위:

```text
HWPX -> HWP 저장
HWP5 저장 record contract
```

HWP 저장 문제가 별도로 발견되면 후속 이슈로 분리한다.

## 5. Stage 1 계획 — HWP/HWPX 렌더링 차이 trace

출력 위치:

```text
output/poc/hwpx/task1100/stage1_exam_social_render_diff/
```

작업:

```text
1. HWP 정답지와 HWPX 파일의 기본 문서 정보를 비교한다.
   - section count
   - page count
   - master page/header/footer presence
   - table/shape/picture count

2. HWP와 HWPX를 같은 조건으로 SVG export한다.
   - 일반 SVG
   - debug overlay SVG
   - 필요 시 특정 페이지만 반복 생성

3. IR 차이를 좁힌다.
   - `rhwp ir-diff samples/hwpx/exam_social.hwpx samples/exam_social.hwp --summary`
   - 차이가 큰 section/page를 우선한다.
   - table, shape, picture, paragraph border/fill, master page 차이를 분리한다.

4. 작업지시자 시각 판정이 필요한 파일 목록을 제시한다.

5. 원인 후보를 3개 이하로 압축한다.
```

산출물:

```text
hwp_info.md
hwpx_info.md
ir_diff_summary.md
render_diff_candidates.md
generated_files.md
```

## 6. Stage 2 계획 — 최소 수정 후보

Stage 1에서 원인 후보가 특정되면 다음 원칙으로 구현한다.

```text
1. HWP 정답지와 HWPX source를 함께 근거로 삼는다.
2. 기존 HWP 렌더링 경로를 깨지 않는다.
3. HWPX 전용 보정이 필요하면 parser/adapter 경계에서 처리한다.
4. 렌더러의 일반 규칙 변경은 회귀 위험이 있으므로 대상 control/type 단위로 검증한다.
```
