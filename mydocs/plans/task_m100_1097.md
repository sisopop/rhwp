# Task M100-1097 수행 계획서

## 1. 작업 개요

GitHub Issue #1097:

```text
[hwpx2hwp] aift 목차 마커와 페이지 표기 저장 누락
```

`samples/hwpx/aift.hwpx`를 HWP로 저장하면 목차 영역에서 마커와 `(페이지 표기)` 부분이 출력되지 않고
마지막 `)`만 출력되는 버그를 수정한다.

이번 작업은 #1092 메모 컨트롤 저장 성공과 #1094 표/페이지 배치 성공을 유지하면서, 목차 문단의
마커/탭/필드/페이지 표기 contract를 정답 HWP와 맞추는 것이 목표다.

## 2. 대상 파일

```text
source: samples/hwpx/aift.hwpx
oracle: samples/aift.hwp
```

비교 대상 후보:

```text
output/poc/hwpx2hwp/task1094/stage5_page10_table_break_mapping/aift-table-break-mapping.hwp
```

## 3. 관찰 증상

작업지시자 제보:

```text
aift.hwpx의 목차에서 마커와 "(페이지 표기)" 부분이 저장된 HWP에서 미출력되고
마지막 ")"만 출력된다.
```

이 증상은 단순 렌더링 문제가 아니라 HWPX -> HWP 저장 과정에서 목차 문단의 field/control/text marker
조합이 HWP5 contract와 다르게 materialize되는 문제로 가정한다.

## 4. 선행 조건

유지해야 할 기존 성공 조건:

```text
1. #1092 메모 컨트롤 저장 성공
2. #1094 2페이지 셀 안쪽 여백 지정 성공
3. #1094 10페이지 pageBreak="TABLE" 표 배치 성공
4. 페이지번호 감추기(pageHiding)와 새 페이지번호 시작(newNum PAGE) 처리 유지
```

## 5. Stage 1 계획 — 목차 문단 anchor trace

출력 위치:

```text
output/poc/hwpx2hwp/task1097/stage1_toc_marker_trace/
```

작업:

```text
1. 현재 devel 기준으로 aift HWP를 생성한다.
2. 정답 HWP와 생성 HWP에서 목차 텍스트 주변 PARA_TEXT, PARA_CHAR_SHAPE, PARA_LINE_SEG,
   CTRL_HEADER, FIELD_BEGIN/FIELD_END record를 추적한다.
3. HWPX 원본의 해당 목차 문단 XML 구조를 추출한다.
4. 누락되는 부분이 text marker인지, field range인지, tab 확장인지, control placeholder인지 분리한다.
```

산출물:

```text

oracle_toc_anchor_trace.md
generated_toc_anchor_trace.md
hwpx_toc_source_trace.md
toc_contract_findings.md
```
