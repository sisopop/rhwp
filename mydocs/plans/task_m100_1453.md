## 1. 배경

OOXML 차트 27종 중 **13종만 렌더, 14종은 "차트 (미지원)" 빈 박스**다. 그 14종은 #1431에서
세 버킷으로 분해됐다 — ① 데이터 OK·라우팅만(7종) ② stock HLC 미구현(2종) ③ scatter 데이터 미추출(5종).
본 작업(C1a)은 ①번 **7종(3D막대 4 + 3D원형 1 + ofPie 2)** 만 다룬다.

이 7종은 파서가 series·값·cats를 **이미 완전 추출**(스파이크 확정)하는데도, 요소명을 인식하지
못해 `chart_type=Unknown` 이 되고 렌더러가 bail(`src/ooxml_chart/renderer.rs:64` → `render_fallback`)
하여 placeholder 가 나온다.

## 2. 사실 검증 (소스 수정 전 확인 완료)

| 항목 | 확인 결과 |
|---|---|
| 파서 인식 요소명 | `handle_start`(`parser.rs:166`)가 `barChart`/`lineChart`/`pieChart`만 인식 |
| 7종 실제 요소명 (HWPX `Chart/chart1.xml` 추출) | `bar3DChart`(barDir=col/bar) · `pie3DChart` · `ofPieChart`(ofPieType=pie/bar) |
| 미인식 결과 | `chart.chart_type=Unknown` (단 `ser`/`val`/`cat`/`tx` 데이터는 plot 타입과 무관하게 추출됨) |
| 렌더러 bail 지점 | `renderer.rs:64` `if chart.chart_type == Unknown → render_fallback` |
| fallback 라벨 | `render_fallback`(`renderer.rs:147`) → `"차트 (미지원)"`, 클래스 `hwp-ooxml-chart-fallback` |
| barDir 후처리 재사용성 | `parser.rs:87-107` 후처리가 `Column`↔`Bar`를 `bar_dir`로 확정 — `bar3DChart`도 `barDir` 핸들러가 그대로 발화하므로 재사용 가능 |
| 샘플·정답지 존재 | `samples/chart/`(hwp+hwpx 7종) + `pdf/chart/`(한컴2022 정답지) 모두 존재 |

→ 이슈의 현상·원인·제안이 모두 사실로 확인됨. 데이터 추출은 손댈 필요 없고 **라우팅만** 추가하면 된다.

## 3. 목표

파서 `handle_start`/`handle_end` 에 `bar3DChart`·`pie3DChart`·`ofPieChart` 요소명을 추가하여
기존 plot 타입(Column/Bar/Pie)으로 라우팅한다. 기존 막대/원형 렌더러가 그대로 그린다.

## 4. 범위

### Part A — 요소명 라우팅 (당초 범위)

- **포함**: `parser.rs` `handle_start`(요소명 3개 추가) + `handle_end`(plot-종료 arm 확장).
  - `bar3DChart` → `Column` (기존 barDir 후처리가 col→Column, bar→Bar 확정)
  - `pie3DChart` · `ofPieChart` → `Pie`
- **2D 근사로 한정**: 3D 입체감·ofPie 보조플롯(원형대원형/원형대막대의 2차 플롯)은 후속(C2).
  ofPie 는 전 데이터를 **단일 원형**으로 근사. (#1431 Non-goals: 픽셀 parity 1차 목표 아님)

### Part B — 막대 누적(stacked/percentStacked) 보정 (범위 확장, 작업지시자 승인 2026-06-21)

3단계 시각판정에서 발견: 현재 `render_bars`가 `c:grouping`을 무시하고 **항상 grouped로 렌더**한다.
→ C1a의 3D 누적막대 2종(3차원누적세로/가로)뿐 아니라 **기존 2D 누적/백프로 막대 4종도 왜곡**
(특히 백프로기준누적이 100% 막대가 아닌 grouped로 나옴). Track C 분해에 누적 슬롯이 없어 C1a에 포함.

- **포함**: 파서 `c:grouping` 파싱 + 모델 grouping 필드 + `render_bars` 누적/백프로 렌더 + 값축 max 재계산.
  - 대상 6종: 3차원누적세로/가로(C1a) + 누적세로/가로(2D 기존) + 백프로기준누적세로/가로(2D 기존).
- **제외**: 꺾은선 누적(render_line — 누적꺽은선 3종)은 별도 후속. 3D 입체감도 제외(C2).

### 변경 면적

- Part A: 파서만 (렌더러·enum 무변경).
- Part B: 파서(grouping 파싱) + 모델(`OoxmlChart` grouping 필드) + 렌더러(`render_bars`·값축).

## 5. 검증 기준

### Part A (라우팅)
1. 파서 단위 테스트: `bar3DChart`(col→Column, bar→Bar)·`pie3DChart`→Pie·`ofPieChart`→Pie + values 추출 단언.
2. 회귀 통합 테스트(`tests/issue_1453_chart_3d_ofpie_routing.rs`): 7종 hwp+hwpx **14파일**
   각각 `render_page_svg(0)` 결과에 `"차트 (미지원)"` **미포함** + `hwp-ooxml-chart` 클래스 **포함**.

### Part B (막대 누적)
3. 파서 단위 테스트: `c:grouping=stacked/percentStacked/clustered` → grouping 필드 매핑.
4. 누적 기하 단언: stacked → 한 카테고리의 시리즈 막대가 **같은 x 공유**(grouped는 x 분리);
   percentStacked → 각 카테고리 누적이 **플롯 전체 높이**(100%) 도달.
5. 대상 6종(3차원누적세로/가로 + 누적세로/가로 + 백프로기준누적세로/가로) 시각판정.
