## 1. 배경

사용자 피드백으로 rhwp-studio의 그림 삽입 기능에서 다음 문제가 보고되었다.

- PNG 이미지를 넣을 때 투명도 처리가 기대와 다르게 보인다.
- `글자처럼 취급`을 끄고 `글씨 위에 표시`하도록 설정하는 동작이 제대로 적용되지 않는 것처럼 보인다.
- 이미지 자체가 종종 삽입되지 않는다.

추가 피드백으로 여러 줄 문단에서 `Shift+Tab`으로 내어쓰기를 첫 줄의 기준 위치에 맞추는 기능이 제안되었다.

현재 코드 확인 결과, 그림 삽입 기능 전체가 미구현인 것은 아니다. 신규 본문 그림은 기본적으로
`treat_as_char=false` floating 개체로 삽입되고, PNG 원본 바이트도 저장 경로에서 보존된다. 다만 그림
속성 변경 시 HWP 저장용 `CommonObjAttr.attr` 비트가 enum 필드와 동기화되지 않는 경로와, Studio의 삽입
실패 안내가 부족한 경로가 있다.

`Shift+Tab` 내어쓰기 단축키는 이미 키 입력 경로와 적용 함수가 존재한다. 그러나 현재 구현은 현재 줄의 시작
위치 대비 커서 x 좌표로 내어쓰기 폭을 계산하므로, 여러 줄 문단에서 "첫 줄 기준 위치에 이후 줄을 맞추는"
요구와 다르게 동작할 가능성이 있다.

## 2. 현재 구현 판정

| 항목 | 현재 상태 | 판단 |
|---|---|---|
| 신규 본문 그림 `글자처럼 취급` 기본값 | `treat_as_char=false`, `text_wrap=Square` | 구현됨 |
| 그림 속성 UI의 `textWrap` 변경 | UI에서 `textWrap` 값을 WASM으로 전달 | 구현됨 |
| HWPX 그림 `textWrap` 저장 | `pic.common.text_wrap` 기반 출력 | 구현됨 |
| HWP 그림 `textWrap` 저장 | `common.attr != 0`이면 기존 attr 우선 사용 | 동기화 결함 가능 |
| PNG 픽셀 알파 | BinData 원본 바이트 보존 | 구조상 보존 가능 |
| 그림 개체 opacity | 모델 필드 없음, HWPX `hc:img alpha="0"` 고정 | 미구현 |
| 파일 선택 후 삽입 실패 안내 | decode/hit-test 실패가 조용히 취소되거나 console warning | 개선 필요 |
| `Shift+Tab` 내어쓰기 호출 | `Tab` + `shiftKey`에서 `applyHangingIndentAtCursor()` 호출 | 구현됨 |
| 여러 줄 문단 첫 줄 기준 맞춤 | 현재 줄 시작점 기준으로 hanging 폭 계산 | 정합성 개선 필요 |

## 3. 작업 범위

### Stage 1: 그림 위치 속성 attr 동기화

- 그림 속성 변경 시 `CommonObjAttr` enum 필드와 HWP 저장용 `attr` 비트를 함께 갱신한다.
- `textWrap=InFrontOfText` 변경이 HWP 저장 경로에서도 유지되도록 한다.
- `treatAsChar`, 위치 기준, 위치 정렬, 겹침/쪽 영역 제한, 크기 보호 등 기존 UI 변경 경로의 attr 일관성을
  함께 점검한다.

### Stage 2: 삽입 실패 안내 개선

- 파일 선택 단계에서 이미지 decode 실패를 사용자에게 알린다.
- 배치 클릭 위치가 유효하지 않아 hit-test가 실패한 경우 조용히 취소하지 않고 안내한다.
- 지원하지 않는 이미지 형식이나 빈 이미지 데이터 오류가 사용자에게 드러나도록 한다.

### Stage 3: PNG 투명도 검증 경로 정리

- PNG 원본 바이트 보존과 그림 개체 opacity 지원을 분리해 판단한다.
- 이번 작업에서는 PNG 픽셀 알파를 훼손하는 코드 경로가 확인될 경우에만 수정한다.
- 그림 개체 opacity 지원은 별도 설계가 필요하면 후속 이슈로 분리한다.

### Stage 4: Shift+Tab 내어쓰기 정합성 개선

- 현재 `Shift+Tab` 단축키가 여러 줄 문단에서 어떤 기준 좌표로 내어쓰기를 계산하는지 재현 테스트를 만든다.
- 첫 줄의 기준 위치와 둘째 줄 이후의 시작 위치를 맞추는 한컴식 기대 동작을 정의한다.
- 본문 문단과 표 셀 문단에서 가능한 범위를 우선 지원한다.
- 글상자/각주/머리말·꼬리말 등 현재 미지원 컨텍스트는 별도 후속 범위로 분리할지 판단한다.

## 4. 제외 범위

- 한컴의 모든 그림 효과/필터 구현
- 그림 전체 opacity UI와 저장 포맷 전체 매핑
- 원본 파일 자동 수정 또는 기존 문서의 대규모 마이그레이션
- 이미지 편집 기능 추가
- 글상자/각주/머리말·꼬리말 내부 Shift+Tab 정합성 전면 구현

## 5. 회귀 방지 기준

- 신규 그림 삽입 기본값인 `treat_as_char=false`, `text_wrap=Square`를 유지한다.
- `InFrontOfText`/`BehindText`/`TopAndBottom` 변경이 HWP 저장용 attr 비트에 반영되어야 한다.
- HWPX 직렬화의 `textWrap` 출력은 기존 동작을 유지한다.
- 파일 선택 후 정상 이미지 삽입 흐름을 깨지 않는다.
- 삽입 실패 안내는 사용자의 문서 상태를 임의로 변경하지 않는다.
- `Shift+Tab`은 일반 Tab 문자 삽입 동작과 폼 필드 이동 동작을 깨지 않는다.
- 여러 줄 문단에서 내어쓰기 적용 후 둘째 줄 이후 시작 위치가 첫 줄 기준 목표 위치와 일치해야 한다.
