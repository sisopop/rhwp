## 1. 배경

#1392(PR #1405)가 `hp:shapeComment` 직렬화를 picture / equation / container / rectangle
4경로에 추가했으나, `render_common_shape_xml` 을 경유하는 legacy 도형
(ellipse / arc / polygon / curve / chart / ole) 은 shapeComment 가 방출되지 않는다.
#1403 에서 같은 경로에 caption 은 추가됐지만 shapeComment 는 동승하지 않았다.

## 2. 사실 검증 (소스 수정 전 확인 완료)

| 항목 | 확인 결과 |
|---|---|
| 원본 `samples/table-vpos-01.hwpx` shapeComment | section0.xml 5건 |
| 재직렬화(`hwpx-roundtrip`) 출력 shapeComment | **3건 (polygon 2건 소실)** |
| `hwpx-roundtrip` diff | **2건** (polygon 2개의 ObjectComment) |
| 누락 지점 | `section.rs:1180` — caption push 직후 `</hp:{tag}>` 닫기 전 shapeComment 없음 |
| 재사용 후보 | `shape.rs:715 write_shape_comment` (단, `Writer<W>` 기반 → String 빌더와 어댑터 필요) |
| 회귀 게이트 | `task1392_shape_comment_loss_in_gate` 존재 — serializer 수정 시 round-trip 자동 보증 |

→ 이슈의 현상·원인·제안이 모두 사실로 확인됨.

## 3. 목표

`render_common_shape_xml` 경유 도형의 `shapeComment` (`CommonObjAttr.description`) 를
OWPML `AbstractShapeObjectType` 순서(`outMargin → caption → shapeComment`)에 맞춰 방출하여
round-trip 보존한다.

## 4. 범위

- **포함**: `render_common_shape_xml` 에 shapeComment 방출 추가 (누락 요소 1종).
  대상 도형: ellipse / arc / polygon / curve / chart / ole.
- **제외**: 알고리즘/스키마 변경 없음. rectangle/equation/picture/container 별도 라이터 무영향.
- 빈 `description` 미방출 (#1392 의 picture/equation 규칙과 동일).

## 5. 검증 기준

1. `samples/table-vpos-01.hwpx` round-trip diff 0 (현 2건 → 0).
2. 재직렬화 출력 shapeComment 5건 (현 3건 → 5).
3. 보존 fixture 회귀 없음 (`aift.hwpx` / `tac-img-02.hwpx` / `business_overview.hwpx` diff 0 유지).
4. `cargo test --profile release-test --tests` 전체 통과 + `cargo fmt --check` + `cargo clippy`.
5. `hwpx_roundtrip_baseline` 회귀 게이트 통과.
