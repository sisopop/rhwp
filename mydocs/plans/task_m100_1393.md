## 1. 배경 + 현황 재측정 (범위 대폭 축소)

#1380 4단계에서 form-002의 표 pageBreak 일괄 TABLE 방출로 검출. **그러나 PR #1405
(physwkim, 6/13 머지)의 커밋 `ad55059f`가 이미 방출·매핑을 정정**했다.

### spot 재측정 (PR #1405 머지 후)

| 항목 | 이슈 보고 | 현재 |
|------|----------|------|
| form-002 pageBreak | RT 일괄 TABLE | **원본 15건 CELL = RT 15건 CELL (보존)** |
| 전수 pageBreak 멀티셋 | — | **43 RT 가능 파일 불일치 0건** |
| serializer 방출 | 고정값 | `table_page_break_str(table.page_break)` 정확 역매핑 (`ad55059f`) |
| 파서 매핑 | — | `CELL↔RowBreak`, `TABLE↔CellBreak` (정합 확인) |

→ 이슈 제안 1(방출)·2(매핑 정오)는 **PR #1405에서 해소**. 잔존은 **게이트 사각**뿐.

## 2. 결함 본체 (잔존)

- `diff_documents`가 표 `page_break`를 비교하지 않음 (grep 0건) — baseline 사각.
- 방출이 회귀하면(미래 변경) baseline이 못 잡는다. **게이트 동승으로 봉인** 필요.

## 3. 매핑 정합 (확인 완료 — 추정 아님)

| HWPX | 파서 → IR | serializer ← IR |
|------|-----------|-----------------|
| CELL/ROW | RowBreak | RowBreak → "CELL" |
| TABLE | CellBreak | CellBreak → "TABLE" |
| (없음) | None | None → "NONE" |

파서·serializer 주석이 한컴 HWPX "CELL"=HWP5 row-break bit 의미를 명시 — 역관계 정합.

## 4. 목표와 완료 조건

1. `diff_documents`가 표 `page_break` 비교 동승 (셀·중첩 표 재귀 포함 검토).
2. 게이트 동승 후 전수 배치 신규 실패 0 (이미 보존되므로 diff 0 유지).
3. pageBreak 보존 회귀 가드 단위 테스트.

## 5. 단계 개요 (구현계획서에서 상세화 — 2단계)

1. **게이트 동승 + 테스트**: `diff_documents` 표 비교에 page_break 추가
   (`IrDifference::TablePageBreak` 또는 기존 비교 확장) + 검출/실샘플 테스트
2. **전수 검증 + 문서**: 배치, baseline, 매뉴얼, 최종 보고서

(측정은 본 수행계획서 1·3절에서 완료 — 별도 1단계 측정 불요)
