# 수행계획서 - Task M100-1197: HWPX 용지 기준 BehindText 그림/표 z-order 보존

- 이슈: #1197
- 제목: HWPX: 용지 기준 BehindText 그림/표의 z-order가 보존되지 않아 텍스트가 겹침
- 브랜치: `local/task1197`
- 기준 커밋: `upstream/devel` `f6ffe9d6`
- 작성일: 2026-06-02
- 재현 문서: `[2027] 온새미로 1 본교재.hwpx` (현재 저장소에는 미포함)

## 1. 현상

HWPX 문서의 주차 안내 페이지에서 같은 위치에 의도적으로 겹쳐 배치된 용지 기준 객체들이
타입별 렌더 경로 차이 때문에 원본 z-order 순서대로 합성되지 않는다.

이슈 본문 기준 주요 객체는 다음과 같다.

| control | 타입 | textWrap | zOrder | 기대 역할 |
|------|------|------|------|------|
| `ci=0` | 표 | `BEHIND_TEXT` | 1 | 낮은 레이어의 이전/대체 텍스트 |
| `ci=1` | 표 | `BEHIND_TEXT` | 10 | 낮은 레이어의 이전/대체 텍스트 |
| `ci=2` | 그림 | `BEHIND_TEXT` | 11 | 전체 페이지 이미지, 낮은 z-order 표를 덮음 |
| `ci=3` | 표 | `BEHIND_TEXT` | 12 | 최종 표시용 하단 텍스트 |
| `ci=4` | 사각형/글상자 | `IN_FRONT_OF_TEXT` | 69 | 중앙 `01` 장식 |

PDF/한컴 기준으로는 `image3`와 zOrder 12 표, zOrder 69 사각형만 최종적으로 보인다.
현재 rhwp SVG에서는 낮은 z-order 표 텍스트가 이미지 위에 다시 남고, 다른 전체 페이지 이미지가
중앙 장식 이미지를 가리며, 하단 텍스트가 겹쳐 보인다.

## 2. 현재 코드 관찰

코드 수정 없이 확인한 현재 구조는 다음과 같다.

- `src/renderer/layout.rs`
  - `paper_images` 이름의 버킷에 용지/페이지 기준 객체 RenderNode를 별도 수집한 뒤 body 이후 root 자식으로 추가한다.
  - 표/도형은 `layout_shapes_for_column`의 2차 패스에서 `z_order` 정렬 대상으로 들어간다.
  - 그림은 문단 레이아웃 중 생성되어 `paper_images`로 들어가는 경로가 있으며, 표/도형과 같은 정렬 리스트에서 항상 함께 비교된다고 보기 어렵다.
- `src/renderer/render_tree.rs`
  - `ImageNode`에는 `text_wrap: Option<TextWrap>`가 있다.
  - `TableNode`에는 `text_wrap` 또는 `z_order` 정보가 없다.
- `src/renderer/svg.rs`
  - #1167 이후 SVG는 자식 노드를 plane 순서로 재정렬한다.
  - 그러나 `node_z_plane()`은 `ImageNode.text_wrap`만 `BehindText`/`InFrontOfText`로 분류하고, 표는 항상 Flow plane이다.
- `src/paint/replay_order.rs`
  - PaintOp replay plane 판정도 `PaintOp::Image` 중심이다.
  - `BEHIND_TEXT` 표/도형이 그림과 같은 레이어 계약을 공유하는지 별도 확인이 필요하다.

즉 #1167은 "BehindText 이미지가 본문 텍스트 뒤로 가야 한다"는 문제를 해결했지만,
#1197은 "용지 기준 Picture/Table/Shape가 타입과 무관하게 같은 z-order 축에서 섞여야 한다"는
더 넓은 문제로 본다.

## 3. 목표

HWPX의 용지/페이지 기준 객체를 Picture/Table/Shape 타입과 무관하게 원본 zOrder 순서대로 렌더링한다.
특히 #1197 재현 페이지에서는 다음 순서를 보존해야 한다.

1. 낮은 zOrder 표들
2. 전체 페이지 그림 `image3`
3. 최종 표시용 표
4. `01` 사각형/글상자

그 결과 SVG 출력에서 중앙 장식 이미지와 최종 주차 안내 텍스트만 보이고,
`PROLOGUE` 및 출처 목록 같은 낮은 z-order 표 텍스트는 전체 페이지 이미지에 의해 가려져야 한다.

## 4. 처리 범위

- HWPX 용지/페이지 기준 anchored 객체의 렌더 순서 계약 정리.
- Picture/Table/Shape가 같은 zOrder 기준으로 정렬되는지 layout/render_tree/SVG 경로에서 확인 및 보정.
- `BEHIND_TEXT` 표/도형이 이미지와 같은 replay plane 계약을 따르도록 필요한 메타데이터 전달 여부 검토.
- #1167 BehindText 워터마크 회귀 방지.
- 재현 문서가 확보되면 이슈 본문 명령 기준 SVG 산출물 검증.

## 5. 비범위

- 본문 흐름 안의 일반 표 배치 알고리즘 변경.
- 표 내부 셀 레이아웃, 줄간격, 텍스트 측정 변경.
- zOrder 값 자체의 파싱/저장 포맷 변경. 단, 조사 중 파서 결함이 확인되면 별도 승인 대상으로 분리한다.
- 특정 파일명이나 control index에만 의존하는 샘플 전용 분기.

## 6. 위험 평가

- 중간 이상. 용지 기준 객체 순서는 페이지 전체 합성 결과에 직접 영향을 주므로, 워터마크/직인/배경 이미지/떠있는 표 회귀가 가능하다.
- `paper_images` 버킷은 이름과 달리 표/도형도 담을 수 있어, 단순 이름 기준 리팩터링보다 실제 렌더 순서 계약을 먼저 고정해야 한다.
- `TableNode`에 메타데이터를 추가하면 JSON/wasm 소비 경로에 영향이 있을 수 있으므로 additive 변경인지 확인한다.
- SVG 경로만 고치면 CanvasKit/PaintOp 경로와 계약이 갈라질 수 있다. 실제 결함 재현 경로를 확인한 뒤 최소 범위를 정한다.
