# Task M100 #1378 수행계획서 — HWPX serializer 문단 run 분할 보존

- 이슈: #1378 "HWPX serializer: 문단 run 분할 보존 — char_shapes 전체를 다중 hp:run으로 출력"
- 부모 이슈: #1315 (HWPX baseline 구축) — 하위 이슈 4건 중 1번째
- 마일스톤: M100 (v1.0.0)
- 브랜치: `local/task1378` (`local/devel` 기준)

## 1. 배경

#1315 4단계에서 확정: serializer가 문단의 `char_shapes[0]`만 `charPrIDRef`로 출력하여
문단 중간 글자모양 전환(여러 run 분할)이 첫 run 서식으로 통일된다.
작업지시자 판정(2026-06-11): "hwpx로 저장할 때 문단, 스타일 처리 부족".

## 2. 현황 분석

### 파서 (역방향 매핑 기준)

`src/parser/hwpx/section.rs` — 각 `<hp:run charPrIDRef>` 경계에서
`CharShapeRef { start_pos, char_shape_id }` 를 기록한다.

- `start_pos`: UTF-16 코드 유닛 위치 (인라인 컨트롤은 8 유닛 폭)
- 연속 동일 `char_shape_id` run 은 dedup — 첫 entry 1개만 유지 (#1058 후속, HWP5 정합)

### serializer (현재 — 세 경로 모두 첫 run 평탄화)

| 경로 | 위치 | 현재 동작 |
|------|------|----------|
| 본문 | `section.rs` `write_section` + `render_run_content` | 문단 전체를 단일 `<hp:run charPrIDRef={char_shapes[0]}>` 으로 감쌈. 내부는 텍스트 조각 + 컨트롤 슬롯(8 유닛) 교차 배치 |
| 표 셀 | `table.rs` `write_sub_list` | 단일 run + `char_shapes.first()` |
| 글상자 | `shape.rs` `write_draw_text_paragraph` | 단일 run + `char_shapes.first()` |

본문 경로 특이점: `write_section` 이 템플릿(`EMPTY_SECTION_XML`) 기반으로
`<hp:run charPrIDRef="0">` anchor 문자열 치환을 사용 — 다중 run 출력 시 이 치환 구조의 정비가 필요.

### 게이트 공백

`diff_documents()` (baseline 게이트)는 문단별 `char_shapes` 를 비교하지 않아
run 평탄화를 검출하지 못한다. 본 타스크에서 게이트 강화를 함께 검토한다.

## 3. 목표와 완료 조건

1. `paragraph.char_shapes` 경계 기준으로 다중 `<hp:run charPrIDRef>` 출력 (본문·셀·글상자 공통)
2. 컨트롤 슬롯·탭·lineBreak 의 run 경계 교차 처리 정의
3. 연속 동일 id dedup 출력 (parser dedup 과 왕복 일치)
4. roundtrip 후 재파싱 시 `char_shapes` 보존 (start_pos·id 일치) — 게이트 강화로 고정
5. baseline 전수(54건)·기존 테스트 무회귀

## 4. 단계 개요 (구현계획서에서 상세화)

| 단계 | 내용 |
|------|------|
| 1 | 재현·측정 — 다중 run 샘플 인벤토리(전수에서 run 분할 빈도), 단위 테스트 케이스 설계, `diff_documents` char_shapes 비교 추가 (강화된 게이트를 먼저 세워 작업 전 실패를 가시화) |
| 2 | 본문 경로 — `render_run_content` 를 char_shapes 경계 분할 출력으로 전환, `write_section` 템플릿 치환 정비 |
| 3 | 셀·글상자 경로 — `write_sub_list`·`write_draw_text_paragraph` 동일 적용 |
| 4 | 전수 검증 + 시각 자료 — baseline 게이트, `hwpx-roundtrip --batch`, 대표 샘플 SVG 비교, 한컴에디터 판정 요청 |

## 5. 위험과 대응

| 위험 | 대응 |

| PR #1366(idaeho) 이 `section.rs`·`table.rs` 수정 — 재제출 시 충돌 | 변경 범위 최소화. 본 타스크는 run 출력 구조에 한정, #1366 범위(pageBreak/margin/colPr/notePr)와 내용 비겹침. 충돌 발생 시 컨트리뷰터에 재안내 |

| 컨트롤 슬롯(8 유닛)과 run 경계 교차 | start_pos 와 슬롯 위치의 우선순위 규칙을 1단계에서 단위 테스트로 먼저 고정 |
| 한컴 호환 — 다중 run 출력의 한컴 인식 | 대표 샘플 한컴에디터 판정 게이트 (자기 검증 ≠ 한컴 호환) |
