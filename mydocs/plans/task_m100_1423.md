# Task M100 #1423 수행계획서 — Cargo.lock git track

- 이슈: #1423 "Cargo.lock 커밋 필요성"
- 마일스톤: M100 (v1.0.0)
- 브랜치: `local/task1423` (`local/devel` 분기)
- 작성일: 2026-06-20

## 1. 제안 (외부 기여자 → 작업지시자 직접 처리)

`Cargo.lock` 이 initial commit 부터 `.gitignore` 에 있어 미추적. Cargo 공식 가이드는
**실행 파일(bin)을 만드는 패키지는 Cargo.lock 을 커밋**하라고 권장한다(재현 가능 의존성).

- rhwp 는 `rhwp` 바이너리 산출(export-svg/pdf/dump 등) → bin 패키지 → 권장 대상.
- 오프라인 환경, CI 캐시, 결정론적 빌드(Nix vendoring), 하위 프로젝트 SSOT 에 유용.
- 참고: https://doc.rust-lang.org/cargo/guide/cargo-toml-vs-cargo-lock.html

## 2. 현재 상태 (진단 완료)

- `.gitignore:4` (# Rust 섹션, `/target/` 아래)에 `Cargo.lock`.
- `git ls-files --error-unmatch Cargo.lock`: 미추적. `origin/main`·`origin/devel` 둘 다 부재.
- 로컬 `Cargo.lock` 존재, `rhwp 0.7.16` 정합 (0.7.16 릴리즈 `cargo update` 반영분이
  gitignore 라 미커밋 상태였음).

## 3. 수정안 (소형 — 2가지)

1. `.gitignore` 에서 `Cargo.lock` 줄 제거.
2. 현재 `Cargo.lock` 을 git add + 커밋. **`cargo update` 로 갱신하지 않고 현 상태 그대로**
   커밋한다(재현성 목적 — 임의 의존성 변경 금지).

## 4. 목표와 완료 조건

1. `Cargo.lock` 이 git 에 추적되고 devel/main 에 반영.
2. `.gitignore` 에서 `Cargo.lock` 제거.
3. `cargo build` / `cargo build --bin rhwp` 가 `--locked` 로 재현 빌드 가능.
4. 커밋된 lock 이 현재 Cargo.toml 과 정합(`cargo metadata --locked` 무오류).

## 5. 단계 개요 (1단계 — 소형 타스크)

1. **수정 + 검증 + 문서**: .gitignore 제거 + Cargo.lock add/commit + `--locked` 빌드 검증
   + (필요 시) CI workflow 가 lock 을 활용하는지 점검 + 최종 보고서.

## 6. 위험과 대응

| 위험 | 대응 |
|------|------|
| lock 이 outdated 의존성 고정 | 현 상태 커밋(재현성). 후속 의존성 갱신은 별도 PR. |
| CI 가 lock 부재 가정 | CI workflow 의 cargo 호출이 `--locked` 강제하지 않는지 확인(현행 유지) |
| 대형 diff(lock 첫 추적) | lock 1파일 추가일 뿐, 코드 무변경 |

## 7. 검증 방법

- `git ls-files Cargo.lock` 추적 확인.
- `cargo build --locked` / `cargo build --bin rhwp --locked` 성공.
- `cargo metadata --locked --format-version 1 >/dev/null` 무오류.

## 8. 산출물

- 수행계획서(본 문서) / 단계별=최종 보고서
- `.gitignore`, `Cargo.lock`(신규 추적)
