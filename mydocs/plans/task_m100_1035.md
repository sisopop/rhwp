## 1. 문제 정의

`samples/hwp3-sample16.hwp` (HWP3 native) 와 `samples/hwp3-sample16-hwp5.hwp` (HWP5 변환본) 의 **총 페이지 수는 동일 (64)** 하나 페이지별 paragraph alignment 차이.

### 1.1 정량 측정 (현 devel a52859de 기준)

| 항목 | 값 |
|------|------|
| 총 페이지 | 64 / 64 (동일) |
| **alignment 정합** | **24 / 64 (37.5%)** |
| **미정합** | **40 / 64 (62.5%)** |

### 1.2 미정합 패턴

HWP5 변환본이 HWP3 보다 일관되게 더 압축 (paragraph 가 +1~+11 늦게 시작):
- p3~p13: HWP5 가 HWP3 보다 2~11 paragraphs 늦음 (누적 증가, p9~p12 +11 최고)
- p29, p31: HWP5 가 FullParagraph 없음 (다른 paragraph 종류)
- p32: HWP5 가 **-1** 역방향 (유일)
- p35~p49: +1~+2 일관

## 2. PR #1009 의 분석 자산 (재활용 가능)

PR #1009 가 광범위 분석 수행 후 close 됨 (sample16-hwp5 over-split 회귀). **분석 내용은 충실, 본 task 의 starting point 로 활용**.

### 2.1 PR #1009 의 본질 메커니즘

> "변환본 encoder 의 `line_segs.vertical_pos` page-reset 신호를 paginator 가 직접 인식"

한컴 변환기 (HWP3 → HWP5) 가 HWP5 측 paragraphs 의 `LineSeg.vertical_pos` 에 page break 신호 인코딩 (페이지 시작 paragraph 의 vpos = 작은 값으로 reset). rhwp paginator 가 이를 인식하지 못해 페이지 alignment 차이 발생.

### 2.2 PR #1009 의 휴리스틱 (4 조건)

```
변환본 + (
  prev_end_vpos > body_height_hu × 0.85       ← 페이지 거의 끝
  AND curr_first_vpos < 1500 HU               ← page-reset 신호 (작은 값)
  AND ((curr 가 empty line_segs 면 < 4000 HU))
  AND ((empty paragraph walk-back/forward + aux_trigger:
        empty bridge ≥ 2 + prev_end > body/2))
)
```

### 2.3 PR #1009 close 사유 (메인테이너 sweep)

| Sweep 결과 | 영향 |
|-----------|------|
| 변환본 9개 중 8개 | no-effect (휴리스틱 미발동) |
| **sample16-hwp5 (타깃)** | devel 64 → with PR **65 (+1 over-split 회귀)** |

→ 순수 증분 0. 유일 발동 케이스에서 회귀. **휴리스틱이 의도된 정합 paragraph 외 추가 1 paragraph 도 trigger** 함.

### 2.4 PR #1009 의 변경 파일 (재활용 가능)

| 파일 | 변경 라인 | 본 task 활용 |
|------|---------|------------|
| `src/renderer/pagination/engine.rs` | +84 | starting point — 휴리스틱 narrow |
| `src/renderer/typeset.rs` | +136 | 동일 로직 (두 경로 정합) |
| `src/renderer/composer.rs` | CHARS_PER_LINE 45→50 | 평가 필요 (이미 #1005 에서 #999 효과 정합) |
| variant 식별 인프라 | (이미 #1005 머지됨) | 재활용 — 무수정 |

본 task 의 fix 위치 후보: PR #1009 의 변경 영역 정합 (pagination/engine.rs + typeset.rs).

## 3. Root cause 가설

PR #1009 의 분석을 ground truth 로 채택:
- 변환본의 page break 신호 = `LineSeg.vertical_pos` reset
- 현 paginator 는 이 신호 미인식
- 인식 휴리스틱 도입 시 sample16-hwp5 에서 over-split (의도 외 1개 추가 split)

본 task 의 핵심: **PR #1009 의 휴리스틱을 narrow 하여 sample16-hwp5 over-split 회피 + 다른 미정합 페이지 정합**.
