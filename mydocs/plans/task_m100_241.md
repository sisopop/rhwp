## 2. 현재 재현/관찰

작업지시자 확인:

- rhwp-studio에서 `samples/hwpx/issue_241.hwpx`를 열고 도장 이미지 속성 대화창을 확인하면,
  세로 위치의 `문단` 기준이 바인딩되지 않는다.

코드 확인:

- Rust/WASM `getPictureProperties()`는 그림 세로 기준을 `"Para"`로 반환한다.
- `setPictureProperties()`도 `"Para"`만 `VertRelTo::Para`로 수용한다.
- 하지만 rhwp-studio `PicturePropsDialog`의 세로 기준 select option 값은 `"Paragraph"`로 되어 있다.

따라서 대화창에서 `props.vertRelTo === "Para"`가 select option과 매칭되지 않아 `문단`이 선택되지 않는다.
사용자가 다시 설정해도 `"Paragraph"`가 전달되어 Rust setter에서 무시될 수 있다.

## 3. 수정 방침

핀셋 수정:

```text
rhwp-studio/src/ui/picture-props-dialog.ts
  세로 기준 option value:
    Paragraph -> Para
```

동일한 속성 체계를 쓰는 다른 UI(`table-cell-props-dialog.ts`)는 이미 `"Para"`를 사용한다.

## 4. 검증

- TypeScript 빌드 또는 wasm/studio 빌드 경로에서 타입 오류 확인
- `samples/hwpx/issue_241.hwpx` 도장 이미지 속성 대화창에서 세로 기준 `문단` 바인딩 확인
- 필요 시 SVG debug overlay로 현재 렌더링 상태 보조 확인
