## 1. 배경

rhwp-studio 상단 메뉴의 단축키 표시는 현재 `index.html`에 `Ctrl+S`, `Ctrl+Shift+S`, `Alt+N` 같은 문자열로
하드코딩되어 있다. 실제 단축키 입력 경로는 macOS에서 `Meta(Command)`를 처리하고, 다른 UI 표면은
이미 `formatShortcutLabel()`을 통해 macOS 표시를 변환한다.

그 결과 macOS에서 커맨드 팔레트/컨텍스트 메뉴/확장 메뉴는 `⌘S`, `⌥N` 형태로 보일 수 있지만,
상단 메뉴는 `Ctrl+S`, `Alt+N`으로 남아 플랫폼별 표시가 불일치한다.

## 2. 확인한 사실

- `rhwp-studio/src/engine/navigation-keymap.ts`
  - `detectPlatformKind()`가 macOS 계열과 기타 플랫폼을 구분한다.
  - `formatShortcutLabel()`이 macOS에서 `Ctrl`/`Alt`/`Shift`를 Apple 기호로 치환한다.
- `rhwp-studio/src/ui/context-menu.ts`
  - `CommandDef.shortcutLabel`을 `formatShortcutLabel()`로 표시한다.
- `rhwp-studio/src/ui/command-palette.ts`
  - 검색/표시 모두 `formatShortcutLabel()`을 사용한다.
- `rhwp-studio/src/command/extension-api.ts`
  - 확장 메뉴 항목 생성 시 `formatShortcutLabel()`을 사용한다.
- `rhwp-studio/src/ui/menu-bar.ts`
  - 현재는 `.md-shortcut` 텍스트를 직접 갱신하지 않는다.
  - `CommandRegistry`를 주입받지 않기 때문에 `data-cmd`와 `CommandDef.shortcutLabel`을 동기화할 수 없다.
- `rhwp-studio/index.html`
  - 상단 메뉴의 `.md-shortcut` 문자열이 직접 하드코딩되어 있다.

## 3. 작업 범위

### 3.1 Studio 상단 메뉴 단축키 표시

- `MenuBar`가 `CommandRegistry`를 주입받도록 한다.
- 초기화 시 상단 메뉴의 `.md-item[data-cmd]`를 순회한다.
- 각 항목의 `data-cmd`에 대응하는 `CommandDef.shortcutLabel`이 있으면 `.md-shortcut` 텍스트를
  `formatShortcutLabel(def.shortcutLabel)` 결과로 갱신한다.
- `CommandDef.shortcutLabel`이 없으면 기존 `.md-shortcut` 텍스트가 있더라도 제거하거나 비우는 대신,
  기존 하드코딩 보조 표시를 보존하지 않는 방향을 우선 검토한다. 이번 범위에서는 단축키가 정의된
  커맨드의 표시 불일치 해소가 핵심이다.

### 3.2 테스트

- `MenuBar` 단위 테스트를 추가한다.
  - macOS override에서 `file:save` 표시가 `⌘S`로 변환되는지 확인한다.
  - macOS override에서 `file:new-doc` 표시가 `⌥N`으로 변환되는지 확인한다.
  - 기타 플랫폼 override에서 `file:save` 표시가 `Ctrl+S`로 유지되는지 확인한다.
  - `index.html`의 하드코딩 문자열보다 `CommandRegistry`의 `shortcutLabel`이 우선되는지 확인한다.
- 기존 `navigation-keymap.test.ts`의 `formatShortcutLabel()` 테스트는 유지한다.
