# Task M100 #1409 수행계획서 — numbering 등록 축 정정 (#1384 동형)

- 이슈: #1409 "HWPX serializer: numbering 등록 축 0/1-based 불일치 — borderFill(#1384) 동형 잠재 결함"
- 마일스톤: M100 (v1.0.0), #1315 하위
- 브랜치: `local/task1409` (`local/devel` 분기)

## 1. 배경

#1384(borderFill 등록 축 정정) 1단계 조사에서 발견. `write_numbering`이 numbering
id 를 `idx + 1`(1-based) 방출하는데 등록은 `idx`(0-based) — borderFill 동형 off-by-one.

## 2. 결함 본체 + 전수 대조 (spot 확정)

ID 풀별 방출 축 vs 등록 축 전수 대조:

| 풀 | 방출 축 | 등록 축 | reference 검사 | 상태 |
|----|---------|---------|---------------|------|
| border_fill | `id+1` (header.rs:300) | `idx+1` (#1384 정정) | 활성 | 정합 |
| **numbering** | **`id+1`** (header.rs:788) | **`idx`** (context.rs:128) | **없음** | **결함** |
| char_shape/para_shape/tab/style | `id` (0-based) | `idx` (0-based) | (일부 활성) | 정합 |

- `id+1` 방출 풀은 border_fill·numbering 둘뿐. border_fill 은 #1384 정정 완료 →
  **numbering 단 1건만 잔존**.
- 실물 numbering id 는 1-based(온새미로 id 1~) — 방출 1-based 가 정본, 등록만 0-based.
- numbering 은 `numbering_ids.reference()` 호출 0건이라 `unresolved` 항상 빈집합 →
  현재 SERIALIZE_FAIL 미표면화 (잠재).

## 3. 수정안 (1줄 — #1384 선례)

`context.rs:128`: `register(idx as u16)` → `register((idx + 1) as u16)`.
방출(`id+1`)·실물(1-based)과 통일. (numbering reference 검사는 없으나, 미래 활성화·
HWP5 변환 경로 혼선 방지 + 등록 축 일관성.)

## 4. 목표와 완료 조건

1. numbering 등록 축을 방출 축(1-based)과 일치.
2. 회귀 가드 단위 테스트 (numbering N개 적재 → id=N resolved, 0 미등록).
3. 전수 배치 신규 실패 0, B=0 유지.
