## 1. 배경

Chrome 0.2.5 / rhwp core 0.7.16 사용자 피드백에서 표 편집 흐름의 한컴 호환성 부족이 확인되었다.

피드백 항목은 다음 네 가지로 나뉜다.

- 표 셀을 마우스로 드래그해서 셀 블록을 선택
- 선택된 셀에서 Ctrl/Alt+방향키로 셀 크기 조정
- 셀 너비를 같게 / 셀 높이를 같게
- Alt+C 모양복사

현재 rhwp-studio는 F5 기반 셀 선택 모드와 Shift/Ctrl 클릭 선택은 갖고 있지만, 한컴처럼 표 셀 위에서
마우스를 누른 채 이동해 셀 블록을 확장하는 경로가 없다. 보호 셀의 경우 #493/#1442에서 단일 셀 클릭 선택과
입력 차단은 구현되었지만, 보호 셀을 포함한 마우스 드래그 셀 선택은 아직 구현되지 않았다.

## 3. 작업 범위

### Stage 1: 마우스 드래그 셀 블록 선택

- 셀 선택 모드에서 일반 좌클릭 후 드래그하면 anchor 셀에서 focus 셀까지 범위를 선택한다.
- 보호 셀을 클릭한 뒤 마우스를 움직여도 기존 보호 셀 입력 차단을 유지하면서 선택 범위가 확장된다.
- 선택 표시는 기존 `CellSelectionRenderer`를 재사용한다.
- 표 경계선 리사이즈, 표 객체 선택, 셀 내부 텍스트 드래그 선택과 충돌하지 않게 분기한다.

### Stage 2: 선택 범위 기반 표 편집 연결

- 마우스 드래그로 선택한 셀 범위에서 기존 Ctrl/Cmd+방향키 크기 조정이 그대로 동작하는지 검증한다.
- 셀 너비/높이 같게 명령이 선택 범위 기준으로 자연스럽게 동작하도록 활성 조건과 적용 범위를 점검한다.
- Alt+방향키를 한컴 호환 셀 크기 조정으로 둘지, macOS/브라우저 충돌을 고려해 제한할지 결정한다.

### Stage 3: 모양복사

- `edit:format-copy`의 현재 비활성 상태를 해소할지 별도 계획으로 분리한다.
- 글꼴 모양과 문단 모양 중 최소 지원 범위를 먼저 확정한다.

## 4. 회귀 방지 기준

- #669에서 정정한 셀 내부 텍스트 드래그 선택을 깨지 않는다.
- 일반 본문 텍스트 드래그 선택과 자동 스크롤을 깨지 않는다.
- 표 경계선 드래그 리사이즈와 외곽선 클릭 표 객체 선택을 깨지 않는다.
- #493/#1442 보호 셀 hover, 입력 차단, 셀 속성 진입을 유지한다.
- F5 셀 선택, 방향키 이동, Shift+클릭 범위 선택, Ctrl/Cmd+클릭 제외 토글을 유지한다.
