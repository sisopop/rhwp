# Task M100 #1426 수행계획서 — rhwp-studio 쪽 테두리/배경 미리보기 버튼 토글 복구

- 이슈: #1426 "rhwp-studio 쪽 테두리/배경 미리보기 버튼 토글 불가"
- 마일스톤: M100 (v1.0.0)
- 브랜치: `local/task1426`
- 작성일: 2026-06-21

## 1. 배경

`rhwp-studio`의 `쪽 > 쪽 테두리/배경` 대화상자에서 테두리 탭 미리보기 주변의
사방 버튼(위쪽/왼쪽/오른쪽/아래쪽)과 우측 하단 전체 버튼이 토글처럼 동작하지 않는다.
현재는 버튼을 클릭하면 항상 현재 선 속성을 대상 방향에 적용하는 켜기 경로만 실행된다.

그 결과 이미 켜진 방향을 다시 눌러도 해제되지 않으며, `선 모양 바로 적용` 상태에서
선 종류/굵기/색을 변경하면 원래 꺼져 있던 방향까지 사방 전체가 켜질 수 있다.

## 2. 사실 검증 (소스 수정 전 확인 완료)

| 항목 | 확인 결과 |
|------|-----------|
| 대상 파일 | `rhwp-studio/src/ui/page-border-dialog.ts` |
| 개별/전체 버튼 클릭 경로 | `sideButton()`이 항상 `applyToSides()`만 호출 |
| 적용 함수 동작 | `applyToSides()`가 현재 선 속성을 덮어쓰고 `borderNoneCheck.checked = false`만 수행 |
| 해제 경로 | 켜진 방향을 `noneBorder()`로 되돌리는 경로 없음 |
| 즉시 적용 경로 | 선 종류/굵기/색 변경 시 항상 `['Left', 'Right', 'Top', 'Bottom']` 전체에 적용 |
| 참고 구현 | `para-shape-tab-builders.ts`는 켜진 방향에만 즉시 적용하고 토글 해제 시 `type: 0`으로 복구 |
| 기존 E2E | `dialog-theme.test.mjs`에 쪽 테두리 preview 접근/line count 확인 헬퍼 존재 |

→ 이슈의 현상·원인·수정 방향이 최신 `upstream/devel` 기준에서도 유효하다.

## 3. 목표

쪽 테두리/배경 대화상자 미리보기 버튼을 한컴식 테두리 토글 조작에 맞게 복구한다.

1. 꺼진 개별 방향 버튼 클릭 시 현재 선 속성으로 해당 방향을 켠다.
2. 켜진 개별 방향 버튼 재클릭 시 해당 방향을 해제한다.
3. 전체 버튼은 사방이 모두 켜진 상태에서는 전체 해제, 일부라도 꺼진 상태에서는 전체 적용한다.
4. `선 모양 바로 적용`은 현재 켜진 방향에만 선 속성 변경을 반영한다.
5. `테두리 사용 안 함` 체크 시 preview뿐 아니라 내부 방향 상태도 모두 해제한다.

## 5. 수정 방향

`borderEdits[side].type !== 0`을 방향별 켜짐 상태로 본다.

- 개별 버튼:
  - 켜짐이면 `noneBorder()`로 해제
  - 꺼짐이면 `currentBorder()`를 적용
- 전체 버튼:
  - 네 방향 모두 켜짐이면 네 방향을 `noneBorder()`로 해제
  - 그 외에는 네 방향에 `currentBorder()`를 적용
- 즉시 적용:
  - `borderEdits`에서 현재 켜진 방향만 골라 선 속성을 갱신
  - 전체가 꺼진 상태에서는 새 방향을 암묵적으로 켜지 않음
- `테두리 사용 안 함`:
  - 체크하면 네 방향을 모두 `noneBorder()`로 초기화
  - 이 상태에서 개별 방향 버튼을 누르면 해당 방향만 적용
  - 이 상태에서 전체 버튼을 누르면 네 방향이 한 번에 적용

## 6. 검증 기준

1. 위쪽 버튼 2회 클릭 후 preview SVG `line` 개수가 클릭 전 상태로 복귀한다.
2. 전체 버튼 2회 클릭 후 preview SVG `line` 개수가 클릭 전 상태로 복귀한다.
3. 위쪽만 켠 상태에서 선 종류/굵기/색 변경 시 preview SVG `line` 개수가 1개로 유지된다.
4. `테두리 사용 안 함` 체크 후 전체 버튼 1회 클릭 시 사방 선이 적용된다.
5. `테두리 사용 안 함` 체크 후 개별 방향 버튼 클릭 시 해당 방향만 적용된다.
6. 관련 TypeScript/E2E 검증이 통과한다.

## 7. 위험과 대응

| 위험 | 대응 |
|------|------|
