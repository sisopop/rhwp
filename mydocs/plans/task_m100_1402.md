# Task M100 #1402 수행계획서 — 열거 속성 표면 표기 정합 검사

- 이슈: #1402 "HWPX serializer: 열거 속성 표면 표기 정합 검사 — 파서 관용 수용에 가려진 비실재 토큰 검출"
- 마일스톤: M100 (v1.0.0), #1315 하위
- 브랜치: `local/task1402` (`local/devel` 분기)
- 작성일: 2026-06-14

## 1. 배경

#1387 한컴 판정에서 autoNum `numType="FIGURE"` 비실재 토큰이 캡션 번호 미출력을
유발(정정 `f14dd6f8`). 근본 구조: 파서가 관용 수용(FIGURE/PICTURE 둘 다)하면
serializer가 어느 표기를 방출해도 IR 게이트는 차이 0 — **게이트는 표면 표기 미비교**.
이 사각을 방어 검사로 봉인하는 것이 본 이슈.

## 2. 측정 결과 (전수 sweep 완료)

### 2.1 serializer 열거 방출 함수

46개 지점 (section/shape/table/header/picture/field). 각 함수의 방출 토큰 집합
수집 완료 (`output/poc/task1402/` 산출 예정). numType은 #1387에서 PICTURE 정정됨.

### 2.2 실물 관측 토큰 (samples/hwpx 전수)

주요 속성별 관측 집합 추출 (`observed_tokens.txt`):
numType: PAGE/FOOTNOTE/ENDNOTE/PICTURE / numberingType: PICTURE/TABLE/EQUATION/NONE /
textWrap: SQUARE/TOP_AND_BOTTOM/BEHIND_TEXT/IN_FRONT_OF_TEXT / vertAlign: TOP/CENTER/
BOTTOM / circleType: CHAR/SHAPE_REVERSAL_RECTANGLE / 등.

### 2.3 명백한 비실재 토큰 — 신규 0건

- numType FIGURE는 이미 정정(#1387).
- `circleType="SHAPE_REVERSAL_TIRANGLE"` (오타 TIRANGLE) — **파서·serializer 양쪽
  동일 철자**라 roundtrip 정합. 실물 corpus에 triangle 사례 없어 한컴 실제 철자
  미확인 (잠재 — owpml 스펙 확인 항목, 신규 결함 아님).
- 나머지 방출 토큰은 실물 관측 집합 또는 owpml 표준값과 정합 (정량 검증은 1단계).

→ 이슈는 **단일 결함 수정이 아니라 방어 검사 인프라 구축**이 본질. 명백한 비실재
토큰은 #1387에서 이미 닫혔고, 본 타스크는 **재발 방지 회귀 테스트**를 만든다.

## 3. 결함 본체 (잔존)

표면 표기 정합 검사가 없어, 미래에 누가 비실재 토큰을 방출하도록 바꿔도 baseline·
단위 테스트가 못 잡는다. **방출 토큰 ∈ 허용 집합 검사 테스트**로 봉인.

## 4. 검사 설계 (1단계에서 확정)

- **허용 집합**: 속성별로 (실물 관측 토큰) ∪ (owpml/한컴 스펙 표준값 — IR 변형
  케이스라 corpus에 없을 수 있는 토큰을 명시 허용). 실물·스펙 교차로 화이트리스트화.
- **검사 방법**: serializer 방출 함수가 각 enum variant에 대해 내는 토큰을 화이트
  리스트와 대조하는 단위 테스트. 화이트리스트 밖 토큰 방출 시 실패.
- numType 선례를 회귀 가드로 포함 (PICTURE 보장 + FIGURE 방출 금지).

## 5. 목표와 완료 조건

1. 핵심 열거 속성(numType/numberingType/pageBreak/textWrap 등 실물 검증 가능한
   것)에 대해 방출 토큰 화이트리스트 정합 테스트 구축.
2. 검사 발견 비실재 토큰(있으면) 케이스별 정정 — 현재 측정상 신규 0건.
3. circleType TIRANGLE 등 미확인 철자는 owpml 교차 확인 후 허용/정정 판정.

## 6. 단계 개요 (구현계획서에서 상세화 — 2단계)

1. **검사 인프라 + 화이트리스트**: 방출 토큰 대조 테스트 + 속성별 허용 집합
   (실물 ∪ 스펙) + numType 회귀 가드 + TIRANGLE owpml 판정
2. **전수 검증 + 문서**: baseline·CI, 매뉴얼(검사 인프라 문서), 최종 보고서

## 7. 위험과 대응

| 위험 | 대응 |
|------|------|
| 화이트리스트가 실물만 보면 IR 변형 토큰을 오탐 | 실물 ∪ owpml 표준값으로 구성 — 스펙 교차 |
