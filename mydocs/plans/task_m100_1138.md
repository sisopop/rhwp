## 1. 배경

표 셀 내부 그림/도형 우클릭 → "개체 속성(P)..." → 다이얼로그 미표시. 콘솔:
```
[CommandDispatcher] 커맨드 실행 실패: insert:picture-props 렌더링 오류: 지정된 컨트롤이 Shape이 아닙니다
```

## 2. Root cause (5-layer)

| Layer | 위치 | 결함 |
|-------|------|------|
| 0. render_tree | `src/renderer/render_tree.rs` | Image/Rectangle/Line/Ellipse/Path 5 노드에 cell 정보 필드 부재 (EquationNode 만 있음) |
| 0+α. layout | `src/renderer/layout/...` | 셀 안 picture/shape 노드 생성 시 cell 정보 안 채워짐 |
| 1. rendering.rs export | `src/document_core/queries/rendering.rs` | 5 노드 export JSON 에 cellIdx/cellParaIdx 부재 |
| 2. WASM API | `src/wasm_api.rs:2364, 2658` | `getPictureProperties` / `getShapeProperties` — cellIdx 매개변수 없음 |
| 3. Handler / Dialog | `rhwp-studio/src/...` | cellPath 전달 안 함 |

→ 4-layer 만 fix 하면 `ref.cellIdx` 가 항상 undefined → cellPath 항상 undefined → 기존 외부 좌표 호출 → 동일 에러.

## 3. 접근 방향

기존 `_by_path` 패턴 (`get_cell_paragraph_count_by_path` 등, `cursor_nav.rs:552 resolve_cell_by_path`) + PR #832 (`ImageNode.header_footer_ref` 패턴) 조합.

### 신규 WASM API 4개

- `getCellPicturePropertiesByPath(sec, parent_para, cell_path_json, inner_control_idx)` + setter
- `getCellShapePropertiesByPath(...)` + setter

### path JSON 스키마

```json
[{"controlIdx": N, "cellIdx": M, "cellParaIdx": P}, ...]
```

- 마지막 segment 로 셀 도달 (기존 `resolve_cell_by_path` 호환)
- `inner_control_idx` 매개변수가 셀 paragraph 내 picture/shape control idx
- nested table 자동 확장 (path 배열 길이 늘림)
