# 수행 계획서 — Task M100-1417: Typeset pagination cursor와 render placement 높이 모델 정합

- 이슈: https://github.com/edwardkim/rhwp/issues/1417
- 제목: Typeset pagination cursor와 render placement 높이 모델 불일치로 TAC 이미지 묶음이 다음 페이지로 밀림
- 작성일: 2026-06-16
- 브랜치: `local/task_m100_1417`

## 1. 문제 요약

`samples/hwpx/pagenation-001.hwpx`에서 2페이지에 함께 출력되어야 할 TAC 이미지 묶음이
3페이지로 넘어간다.

사전 조사 결과, HWPX `lineSegArray` 파싱/보존 문제는 확인되지 않았다. `pi=16`, `pi=27`,
`pi=27` 내부 글상자 문단의 lineSeg 9개 필드는 원본 XML과 내부 `Paragraph.line_segs`
덤프가 일치했고, `hwpx-roundtrip --lineseg-report`에서도 `round1=0`, `round2=0`이었다.

현재 문제는 Typeset pagination cursor가 판단하는 누적 높이와 render placement가 실제
출력하는 `y`/bbox 높이 모델이 서로 어긋나는 현상으로 본다. 디버깅 SVG/render tree 기준으로
2페이지에는 `pi=27` 이미지 묶음이 들어갈 수 있는 공간이 남아 있으나, pagination fit 판정은
이를 다음 페이지로 넘긴다.

## 2. 목표

1. `pagenation-001.hwpx`의 2페이지/3페이지 pagination cursor와 render bbox 차이를 정량화한다.
2. TAC table/shape host paragraph 처리에서 object height, host line advance, trailing spacing이
   fit 판정과 실제 배치에 어떻게 반영되는지 추적한다.
3. HWPX `line_seg`를 authoritative 기준으로 삼지 않고, render placement와 pagination cursor의
   내부 높이 모델을 일관되게 만드는 수정안을 도출한다.
4. 승인된 수정안에 따라 최소 범위로 구현한다.
5. `pi=27` 이미지 묶음이 2페이지에 배치되는지 검증하고, 기존 TAC table/shape 회귀를 확인한다.

## 3. 1차 조사 범위

주요 대상:

- `src/renderer/typeset.rs`
  - TAC table/shape pagination branch
  - `place_table_with_text`
  - paragraph fit height 계산
  - `HeightCursor`와 `vpos_snap_current_height` 연동부
- `src/renderer/height_cursor.rs`
- `src/renderer/height_measurer.rs`
- render tree/SVG bbox 산출 경로

재현/진단 대상:

- `samples/hwpx/pagenation-001.hwpx`
- `output/poc/pagenation-001-debug/`
- `output/poc/pagenation-001-render-tree/`

## 4. 비목표

- HWPX `line_seg`를 pagination의 authoritative 기준으로 삼지 않는다.
- 원본 `lineSegArray` 값을 임의로 보정하거나 재해석하지 않는다.
- HWPX 파서의 lineSeg 보존 로직을 변경하지 않는다.
- 이번 이슈와 무관한 대규모 pagination 리팩터링은 수행하지 않는다.
- 시각 정합 근거 없이 기존 TAC table/shape 특례를 제거하지 않는다.
