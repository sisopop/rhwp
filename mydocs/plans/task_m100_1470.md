## 1. 배경

사용자 피드백으로 스타일 기능과 표 관련 동작에서 다음 문제가 보고되었다.

- 스타일에서 왼쪽 여백을 15로 지정했는데 자동으로 30으로 바뀜
- 스타일 줄간격이 실제 레이아웃에 반영되지 않음
- 표 캡션 기능이 한컴 동작과 다름
- 표가 현재 위치가 아니라 다음 페이지에 생성됨

검토 기준은 한컴 도움말의 스타일, 스타일 적용, 스타일 편집, 새 스타일, 바탕글, 표 캡션 설명이다.

## 2. 한컴 기준 요약

- 문단 스타일은 글자 모양과 문단 모양을 함께 저장하고 적용한다.
- 스타일 편집 시 해당 스타일이 적용된 글자/문단 모양이 함께 갱신된다.
- 블록 선택 상태에서는 여러 낱말/문단에 같은 스타일을 적용할 수 있다.
- 새 스타일은 현재 문단의 글자/문단 모양을 기반으로 추가한다.
- 표 캡션은 표 번호와 제목/설명을 개체에 붙이고 자동 번호 흐름을 유지한다.

## 3. 현재 확인 내용

- 일반 문단 모양 적용 경로는 줄간격 변경 시 `LineSeg`를 재계산하지만, 스타일 적용/편집 경로는 `para_shape_id`와 `char_shapes`만 바꾸고 재흐름 처리가 부족하다.
- 스타일 적용 시 기존 문단의 실효 `ParaShape`를 참조값으로 우선 재사용하는 흐름이 있어, 직접 서식이 스타일 적용값으로 번질 수 있다.
- 새 스타일 생성은 현재 문단 기반이 아니라 바탕글 기반으로 생성되는 경로가 있다.
- Studio의 스타일 적용은 현재 커서 문단 중심이고, 블록 선택 다중 문단 적용은 한컴 기준과 차이가 있다.
- 표 만들기 상세 대화상자에는 `글자처럼 취급`, 너비, 높이 옵션이 있지만 실제 생성 API에는 일부 옵션이 전달되지 않는다.
- 표 캡션 생성 경로는 자동 번호 컨트롤을 literal 텍스트로 치환하는 흐름이 있어, 한컴식 자동 번호/번호 다시 넣기 모델과 다를 수 있다.
- `hasCaption=false`가 기존 표 캡션 삭제로 이어지는지 확인이 필요하다.

## 4. 작업 범위

1. 스타일 적용/편집 후 문단 LineSeg와 페이지 흐름이 즉시 재계산되도록 보정한다.
2. 스타일 문단 여백 단위가 2x로 중복 변환되지 않도록 스타일 전용 회귀 테스트를 추가한다.
3. 새 스타일 생성 기준을 현재 커서 문단의 CharShape/ParaShape 기반으로 맞춘다.
4. 블록 선택 상태에서 문단 스타일이 선택 범위의 여러 문단에 적용되도록 한다.
5. 표 만들기 상세 옵션을 실제 표 생성 API까지 전달한다.
6. 표 캡션 생성/삭제 흐름을 AutoNumber 유지 모델에 맞게 보정한다.
