# 수행 계획서 — Task M100-1418: 첫 페이지 글상자와 표 상단 중앙 overlap 정합

- 이슈: https://github.com/edwardkim/rhwp/issues/1418
- 제목: 첫 페이지 글상자와 표 상단 중앙 overlap 정합
- 작성일: 2026-06-16
- 브랜치: `local/task_m100_1418`

## 1. 문제 요약

다음 샘플에서 첫 페이지 글상자가 표의 상단 중앙 부분을 오버래핑하는 출력이 한컴에디터와 다르게
처리된다.

- `samples/2026_oss_rst.hwp`
- `samples/hwpx/2026_oss_rst.hwpx`

초기 재현 기준:

- HWP: `samples/2026_oss_rst.hwp`는 6페이지로 로드된다.
  - 1페이지 `dump-pages` 기준 `pi=0 ci=2`가 `wrap=InFrontOfText`, `tac=false` 글상자 후보이다.
  - 같은 페이지의 `pi=1 ci=0`은 1x1 대형 TAC 표이며, 표 높이는 약 `852px`이다.
  - 글상자 위치는 문단 dump 기준 가로 `Paper + Center`, 세로 `Paper + Top + 35.3mm`이다.
- HWPX: `samples/hwpx/2026_oss_rst.hwpx`는 1페이지로 로드된다.
  - 초기 `dump-pages`에서는 동일한 글상자 후보가 바로 보이지 않고, 본문/표 flow 중심으로 구성된다.
  - 따라서 HWP/HWPX가 같은 결함의 서로 다른 입력인지, 또는 별도 fixture인지 먼저 판정해야 한다.

baseline 산출물:

- `output/poc/task1418-baseline-hwp/2026_oss_rst_001.svg`
- `output/poc/task1418-baseline-hwpx/2026_oss_rst.svg`

## 2. 목표

1. HWP 첫 페이지의 글상자와 표가 한컴에디터에서 어떤 overlap 정책으로 배치되는지 확인한다.
2. rhwp의 현재 HWP/HWPX 파싱 결과와 render placement를 대조해 parser 차이인지 renderer/layout 차이인지 분리한다.
3. 글상자 `InFrontOfText` / `allow_overlap` / `flow_with_text` / anchor 기준이 표 flow와 상호작용하는 경로를 추적한다.
4. 한컴 호환 결과를 내도록 최소 범위로 수정한다.
5. HWP/HWPX 샘플과 관련 글상자/표 overlap 회귀 테스트를 추가한다.

## 3. 조사 범위

주요 코드 후보:

- `src/parser/hwp5/`
  - shape/common object attr 파싱
  - 글상자 위치 기준, `allow_overlap`, `flow_with_text`, z-order 파싱
- `src/parser/hwpx/`
  - 동일 샘플의 HWPX 구조와 HWP5 IR 차이 확인
- `src/renderer/typeset.rs`
  - 페이지네이션에서 InFrontOfText/TopAndBottom/TAC object가 높이를 소비하는 방식
- `src/renderer/layout.rs`
  - 본문 object와 표 배치 순서, z-order, object lane/overlap 처리
- `src/renderer/layout/shape_layout.rs`
  - 글상자 bbox 및 내부 TextBox layout
- `src/renderer/render_tree.rs`, `src/paint/`
  - 최종 레이어 순서와 clipping/paint order 확인

진단 명령 후보:

```bash
target/debug/rhwp dump-pages samples/2026_oss_rst.hwp -p 0
target/debug/rhwp dump samples/2026_oss_rst.hwp -s 0 -p 0
target/debug/rhwp dump samples/2026_oss_rst.hwp -s 0 -p 1
target/debug/rhwp export-svg samples/2026_oss_rst.hwp -p 0 -o output/poc/task1418-baseline-hwp --debug-overlay
target/debug/rhwp export-render-tree samples/2026_oss_rst.hwp -p 0 -o output/poc/task1418-render-tree-hwp
target/debug/rhwp ir-diff samples/hwpx/2026_oss_rst.hwpx samples/2026_oss_rst.hwp --summary
```

## 4. 비목표

- 한컴 reference 확인 없이 임의로 z-order나 overlap 정책을 전역 변경하지 않는다.
- HWP/HWPX 샘플이 서로 다른 문서로 판정될 경우, 두 샘플을 억지로 같은 expected로 맞추지 않는다.
- 글상자 전체 렌더링 정책을 대규모 리팩터링하지 않는다.
- 표 pagination이나 #1417에서 수정한 TAC host height 모델을 다시 흔들지 않는다.

## 5. 진행 단계

### Stage 1 — 기준점 확정

- HWP/HWPX 첫 페이지 debug SVG와 render tree를 생성한다.
- `pi=0 ci=2` 글상자와 `pi=1 ci=0` 표의 bbox, anchor, z-order, wrap 정책을 정리한다.
- 작업지시자에게 한컴에디터 기준으로 기대 overlap 형태를 확인받는다.

산출물:

- baseline SVG/render-tree 경로
- `mydocs/working/task_m100_1418_stage1.md`

### Stage 2 — HWP/HWPX IR 차이 분리

- HWP5 dump와 HWPX dump/IR diff를 비교한다.
- HWPX 샘플이 같은 문서를 나타내는지, 아니면 이름만 유사한 별도 입력인지 판정한다.
- 같은 문서라면 parser lowering 차이를 후보에 포함한다.
- 별도 입력이라면 이번 이슈의 primary fixture를 HWP로 고정하고, HWPX는 보조 검증으로 둔다.

### Stage 3 — 구현 계획서 작성

Stage 1~2 결과를 바탕으로 수정 지점을 한정한다.

후보 축:

- InFrontOfText 글상자를 pagination height 소비 대상에서 제외하거나 유지하는 조건
- 글상자의 paper-relative center/top anchor 계산
- 표 TopAndBottom/TAC flow와 InFrontOfText object의 z-order/paint order
- `allow_overlap`/`flow_with_text` flag의 parser 및 renderer 의미 정합

산출물:
