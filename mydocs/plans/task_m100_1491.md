## 목표

`셀 너비를 같게` 실행 후 선택 행 또는 선택 셀 범위의 셀 폭이 한컴처럼 독립적으로 같아지도록 회귀를 보정한다.

## 배경

현재 분석상 명령 라우팅은 `table:cell-width-equal`에서 `resizeTableCells`까지 도달한다. 문제는 선택 행의 폭 delta 합이 평균 반올림 때문에 0이 아닌 경우 `local_resize_rows` 힌트가 남지 않고, 렌더러가 행별 독립 x 좌표를 전역 grid로 되돌리는 데 있다.

관련 회귀 후보:

- `57dcc9c6 task 1443: 표 외곽 구조를 한컴 PDF 기준으로 보정`
- `b7b1dd20 task 1443: 표 셀 로컬 resize 회귀 보정`

## 범위

- `table:cell-width-equal`의 선택 행 독립 렌더링 회귀 보정
- 반올림으로 선택 행 delta 합이 0이 아닌 케이스 회귀 테스트 추가
- 가로 병합 셀이 섞인 행에서도 병합 셀을 균등화 대상에 포함
- 셀 선택 모드에서 Shift+마우스 경계 드래그가 단일 셀 resize로 동작
- 경계선 위 hitTest 공백 때문에 세로 컬럼 hover 표시가 끊기는 현상 완화
- 기존 Shift+셀 단독 resize, 일반 컬럼 resize, 표 외곽 크기 보존 동작 유지
- 추후 검증 자료로 샘플 3개를 브랜치 변경분에 포함
  - `pdf/rowbreak-problem-pages-2024.pdf`
  - `samples/rowbreak-problem-pages.hwp`
  - `samples/rowbreak-problem-pages.hwpx`

## 비범위

- PR #1489의 row-break 본문 로직 변경

## 검증 기준

- 선택 행의 bbox 폭이 균등화 후 모두 같아진다.
- 가로 병합 셀이 섞인 선택 행에서도 표시 bbox 폭이 모두 같아진다.
- 비선택 행의 bbox 폭과 x 위치가 유지된다.
- delta 합이 0이 아닌 반올림 케이스에서도 독립 행 렌더링이 유지된다.
- Shift+경계선 클릭은 셀 선택 확장보다 resize 드래그를 우선한다.
- 기존 `issue_493_cell_attrs`, `svg_snapshot`, Studio 타입/테스트가 깨지지 않는다.
