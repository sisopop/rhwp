## 2. 핵심 전략 — scratch 측정 호출
시뮬이 para 높이를 **추정**(v2 휴리스틱: saved line_segs/total_height/las)하는 대신,
**scratch LayoutEngine + 버리는 tree/col_node** 로 `layout_partial_paragraph` 를 호출해
**반환 y(정확 렌더 advance)** 를 사용. 노드는 scratch 로 버려 실제 렌더 무영향.

```
let scratch = LayoutEngine::new(self.dpi);  // 측정 전용
let mut tree = PageRenderTree::default(); let mut col = RenderNode::default();
let y_after = scratch.layout_partial_paragraph(&mut tree, &mut col, para, Some(&comp),
    styles, &col_area, y_top, 0, end_line, section, pi, mcw, Some(bin_data), None);
// advance = y_after - y_top  (정확)
```

## 3. 단계 (다회차)
1. **Stage 1 — 측정 호출 배선**: 시뮬의 per-para advance 를 scratch
   `layout_partial_paragraph` 반환값으로 대체(A2 경로). 필요 컨텍스트(col_area, multi_col_width,
   bin_data_content, section) 를 typeset 미주 루프에서 확보·전달.
2. **Stage 2 — 부작용 격리**: scratch LayoutEngine 의 상태 변이(last_item_content_bottom,
   layout_overflows, auto_counter/numbering) 가 측정에만 머물도록 검증. 필요 시 per-call 재생성
   또는 측정 전용 플래그.
3. **Stage 3 — split/fit 정합**: 정확 높이 위에서 split_endnote_to_fit·컬럼 break 를 측정
   기반으로. v2 의 fit 게이트(a2_overflow_with_para)를 정확 측정으로 구동.
4. **Stage 4 — 성능**: per-para 전체 레이아웃 = O(n²) 측정 비용. 미주 단위 캐싱/증분 측정 검토.
5. **Stage 5 — 승격·검증**: 두 배치 + 전 exam(7건 포함) 무회귀 → A2 기본 승격, `RHWP_EN_SSOT`
   레거시 경로 정리.

## 4. 위험
- **컨텍스트 정합**: col_area/multi_col_width/bin_data 를 렌더와 동일하게 구성해야 측정이
  렌더와 일치. 불일치 시 또 다른 drift.
- **부작용**: scratch 엔진의 numbering/overflow 상태가 실제 렌더에 새지 않도록 격리 필수.
- **성능**: O(n²) 레이아웃. 대용량 미주 문서에서 페이지네이션 지연 가능 → 캐싱 필요.
- **side-effect free 가정 검증**: layout_partial_paragraph 가 측정만으로 안전한지(전역 상태
  변이 없는지) Stage 2 에서 실증.
