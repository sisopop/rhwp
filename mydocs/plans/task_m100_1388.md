## 1. 배경

#1379 4단계 SVG 비교(ta-pic-001-r)에서 검출된 결함. HWPX 저장 시 원본 페이지 여백이
템플릿 고정값으로 변형된다.

- 증상: `hp:margin` left/right=8504(30mm) 고정 방출 — 원본 4252(15mm)가 30mm로 변형,
  본문 전체 +56.7px 균일 시프트
- #1380 4단계에서 영향 범위가 추가 실증됨: RT 파일의 `<hp:margin>`만 원본 값으로
  패치하면 4개 샘플 페이지 수가 전부 일치(10=10, 9=9) — 페이지 수 차이 전량이 본 결함 기인

## 2. 현황 분석 (2026-06-12)

### 결함 구조 (3축)

| 축 | 위치 | 현황 |
|----|------|------|
| serializer | `src/serializer/hwpx/section.rs:1249` `replace_page_pr` | #1166이 pagePr 여는 태그(landscape/width/height)만 동적화. `<hp:margin .../>`은 템플릿(`templates/empty_section0.xml`) 고정값 그대로 방출 |
| parser | `src/parser/hwpx/section.rs:338` `parse_page_margin` | 7필드(left/right/top/bottom/header/footer/gutter) 전부 `PageDef`로 파싱 — **파서는 정상, serializer만 비대칭** |
| 게이트 | `src/serializer/hwpx/roundtrip.rs` `diff_documents` | `PageDef`(여백 포함) 비교 없음 — baseline 통과 사각 |

IR `PageDef`(src/model/page.rs)는 margin 7필드를 모두 보유하므로 모델 변경 불요.

### 인접 항목 (범위 검토)

- `gutterType="LEFT_ONLY"` 역시 템플릿 고정 — IR `PageDef.binding`(SingleSided/DuplexSided/TopFlip)이
  존재하므로 같은 치환 지점에서 동적화 가능. 본 타스크에 포함한다 (같은 pagePr 여는 태그 1곳).
- pagePr height 84186→84188(+2 유닛, #1380 4단계 비고)는 템플릿 계열 별개 증상 — 본 타스크
  조사 범위에 넣되, 원인이 여백 밖이면 별도 이슈로 분리한다.

### 트러블슈팅 사전 검색 결과

- `page_margin_area_calculation.md` — 여백 7필드의 시각 의미(본문 시작 = margin_header + margin_top)
  정리. 직렬화 값 의미 확인에 참고.
- 직렬화 계열(`hwpx2hwp-rule.md`, `task178_*` 등)에 secPr margin 관련 선행 실패 기록 없음.

### 열린 PR 충돌 확인

- PR #1366(idaeho)이 동일 수정(margin 동적 치환)을 포함하나, 2026-06-12 범위 축소 안내
  코멘트에서 "secPr 용지 여백은 #1388로 메인테이너 측 처리 예정"으로 드랍 권장 완료 —
  본 타스크 진행이 합의된 방향. merge 충돌 시 본 타스크가 선행하므로 PR 측이 rebase.

## 3. 목표와 완료 조건

1. HWPX 저장 시 `hp:margin` 7필드가 원본 IR 값으로 방출된다 (템플릿 고정값 제거).
2. `gutterType`이 IR `binding` 값으로 방출된다.
3. `diff_documents`에 섹션 `PageDef` 비교(여백 7필드 + 용지 크기/방향/제본)가 동승하여
   baseline 게이트 사각이 해소된다.
4. `samples/hwpx` 전수 roundtrip에서 여백 diff 0, 기존 xfail 외 신규 실패 0.
5. ta-pic-001-r(#1379 검출 샘플) + #1380 4샘플의 RT 페이지 수가 원본과 일치한다.

## 4. 단계 개요 (구현계획서에서 상세화)

1. **측정**: 전수 배치로 margin 변형 현황 정량화 (원본≠템플릿 샘플 수, 필드별 분포),
   height +2 유닛 증상 원인 조사
2. **serializer 수정**: `replace_page_pr` 확장 — margin 7필드 + gutterType 동적 치환
3. **게이트 동승**: `diff_documents`에 `PageDef` 비교 추가 (#1380 `ParagraphLinesegs` 패턴 준용)
4. **전수 검증 + 문서**: 배치 재실행, 페이지 수 대조, SVG 비교, 매뉴얼 갱신, 최종 보고서
