## 문제 인식

`samples/[2027] 온새미로 1 본교재.hwp` 35쪽에서 한컴 PDF는 우측 상단 그림 주위로
본문 줄이 감싸져 흐르지만, rhwp 렌더링은 본문 줄이 그림 영역을 침범하거나 과도한 자간을
만들어 그림과 텍스트가 겹친다.

초기 비교 자료:

- 한컴 PDF raster: `output/poc/onsamiro_p35_compare/pdf_png/hancom_p35-35.png`
- rhwp raster: `output/poc/onsamiro_p35_compare/rhwp_p35.png`
- render tree: `output/poc/onsamiro_p35_compare/render_tree/render_tree_035.json`

## 조사 범위

1. HWP5 `PARA_LINE_SEG`의 `column_start`/`segment_width`가 35쪽 문단에 정상 파싱되는지 확인한다.
2. `typeset`의 `wrap_anchors` 등록 조건이 해당 그림/문단 조합을 놓치는지 확인한다.
3. `paragraph_layout`에서 `wrap_anchor` 유무와 관계없이 LineSeg wrap zone이 실제 TextLine bbox에 반영되는지 확인한다.
4. 기존 #489/#568/#604/#901의 어울림 보정과 충돌하지 않는 최소 수정 지점을 찾는다.
5. 한컴 PDF 35쪽과 rhwp 수정 전/후 시각 비교 자료를 남긴다.

## 목표

1. 그림 주위 본문 감싸기 줄의 시작 x와 가용 폭을 한컴 PDF와 같은 방향으로 맞춘다.
2. 그림과 본문 TextLine이 같은 y 대역에서 겹치지 않도록 한다.
3. 기존 표/그림 어울림, TAC 표, 다단 필터링 회귀를 막는 테스트를 추가한다.
4. 필요하면 `LineSeg` 표준 문서나 errata에 이번 케이스에서 확인한 의미를 보강한다.
