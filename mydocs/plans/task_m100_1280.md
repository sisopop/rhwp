# Task #1280 수행 계획서

## 이슈

- GitHub Issue: #1280
- 제목: rhwp-studio: 삽입한 글상자가 text_box 없는 Rectangle로 생성되어 커서 진입·타이핑·붙여넣기 실패 (글상자 없음)
- 처리 주체: 컨트리뷰터 (Fork 기반). 마일스톤·orders·이슈 클로즈는 메인테이너 영역.

## 배경

rhwp-studio에서 **새로 글상자(textbox)를 삽입**한 뒤 그 안을 클릭(커서 진입)하거나 텍스트/이미지를
입력·붙여넣으면 모두 실패한다. HWP/HWPX에서 만들어진 **기존** 글상자는 정상 동작하고,
**rhwp-studio가 직접 생성한** 글상자에서만 발생한다. 글상자 기능 전반(텍스트·붙여넣기·이미지)을
사용 불가로 만드는 결함이다.

콘솔 오류 3종(동일 근본원인):

- `지정된 Shape 컨트롤에 텍스트 박스가 없습니다` — 텍스트 입력 (`text_editing.rs:434/1474`)
- `렌더링 오류: 글상자 없음` — 붙여넣기 (`clipboard.rs:512`)
- `셀 커서 위치를 찾을 수 없습니다` / `셀 문단 참조 실패` — 커서 진입 (`cursor_rect.rs`, `text_editing.rs`)

## 재현 대상

1. rhwp-studio 새 문서 생성
2. 입력 → **글상자 삽입** → 마우스 드래그로 글상자 영역 지정
3. 생성된 글상자 안을 클릭(커서 진입) → 텍스트 타이핑 또는 이미지 붙여넣기
4. → 위 콘솔 오류, 글상자에 내용 입력 불가

## 근본 원인 (코드 검증 완료)

글상자 삽입 경로가 도형 타입을 `'textbox'`가 아니라 `'rectangle'`로 백엔드에 전달한다.

1. `rhwp-studio/src/engine/input-handler.ts:513` `enterTextboxPlacementMode()`가
   `this.shapePlacementType = 'rectangle'` 로 **하드코딩**되어 있다(← `'textbox'`여야 함).
   프런트 전체에서 `shapePlacementType`이 `'textbox'`로 설정되는 곳이 **한 군데도 없다**.
2. `input-handler.ts:874` `if (this.shapePlacementType !== 'textbox')` 분기는 **이미 textbox 케이스를
   가정**해 작성됨(textbox면 종이 기준 offset 계산을 스킵 → offset=0). 현재는 항상 참이라 **데드코드**.
3. `input-handler.ts:915` `createShapeControl({ … shapeType: this.shapePlacementType })` → `'rectangle'` 전달.
4. 백엔드 `src/wasm_api.rs:2865` `default_tac = shape_type == "textbox"` → false,
   `src/document_core/commands/object_ops.rs`의 `has_textbox` 및 margin/vertRel/horzRel/attr 게이트
   (모두 `== "textbox"`)가 false → `ShapeObject::Rectangle { drawing.text_box: None }`으로 생성.
5. 입력/붙여넣기 시 `src/document_core/helpers.rs`의 `get_textbox_from_shape(_mut)`가 `None` 반환 →
   세 경로 모두 실패.

**백엔드는 정상이다.** `shapeType == "textbox"`이면 text_box + 글상자 margin(283) + vertRel=Para +
horzRel=Column + treat_as_char=true를 정상 구성한다. 즉 **프런트가 `'textbox'`를 전달하기만 하면**
전체 경로가 의도대로 동작한다(L874 데드코드도 이때 활성화되어 offset=0으로 커서 위치에 인라인 배치).

## 수정 방향

프런트엔드 only. 백엔드(`wasm_api.rs`/`object_ops.rs`/`helpers.rs`) 로직은 변경하지 않는다.

1. `input-handler.ts:513` `enterTextboxPlacementMode()`의 `shapePlacementType`를 `'rectangle'` → `'textbox'`.
2. `input-handler.ts:150` `shapePlacementType` 필드 주석의 가능한 값 목록에 `'textbox'` 추가(실제 값과 정합).
3. 일반 사각형 삽입은 별도 함수 `enterShapePlacementMode('rectangle')`(L520)을 통하므로 영향 없음 —
   두 경로 분리를 회귀 테스트로 함께 고정한다.

## 진행 계획

1. 구현 계획서(`task_m100_1280_impl.md`)를 3~6단계로 작성한다.
2. 프런트 수정(L513, L150)을 적용한다.
3. e2e 회귀 테스트(`rhwp-studio/e2e/issue-1280-textbox-text-input.test.mjs`)를 추가한다.
   `window.__inputHandler`로 **실제 삽입 경로**를 구동(enterTextboxPlacementMode → 마우스 드래그 →
   finishTextboxPlacement)하여 글상자에 텍스트 입력/붙여넣기 round-trip을 검증한다.
4. Rust 단위 테스트(`object_ops.rs`의 `#[cfg(test)]`에 `issue_1280_*`)로 백엔드 계약을 고정한다
   (textbox → text_box 있음 + 텍스트 보존, rectangle → text_box 없음 대조).
5. 네이티브 테스트/타입체크/WASM 빌드/e2e로 검증한다.

## 완료 기준

1. 삽입한 글상자에 커서 진입·타이핑·붙여넣기·이미지가 정상 동작한다.
2. 글상자 삽입 경로가 백엔드에 `shapeType: 'textbox'`를 전달하여 text_box를 가진 도형이 생성된다.
3. e2e 회귀 테스트(삽입 → 텍스트 입력/붙여넣기 round-trip)와 Rust 단위 테스트가 통과한다.
4. 일반 사각형 도형 삽입(`'rectangle'`)은 종전대로 text_box 없이 생성된다(분리 고정).
5. 기존 도형/글상자/시험지 렌더링 회귀가 발생하지 않는다.
