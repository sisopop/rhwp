# 수행계획서 — Task M100-1167: SVG 렌더러 BehindText 워터마크 z-order

## 1. 이슈

- #1167 (PR #1163 / #1017 후속). SVG 출력에서 `samples/복학원서.hwp` 중앙 baked watermark(`wrap=BehindText`)가 본문 텍스트를 덮음.
- PNG(native Skia) / 웹캔버스(CanvasKit)는 PR #1163 의 PaintOp replay plane 으로 정정됨. SVG 만 누락.

## 2. 근본 원인 (진단 완료)

- `src/renderer/svg.rs` 의 `render_node` 는 RenderNode 트리를 **단순 DFS 순회**로 렌더링한다. PaintOp replay plane 같은 z-order 재정렬이 없다.
- 워터마크 ImageNode 는 문단 순회 순서대로 트리에 push 되어, 본문 TextRun 노드 뒤에 위치 → SVG 후순위(=위)로 합성 → 본문을 덮음.
- 정량 확인: `output/poc/pr1163/after/복학원서.svg` 에서 중앙 워터마크 `<image>` 줄 810 > 본문 `<text>` 47~807.

## 3. 접근 (작업지시자 승인: SVG plane multi-pass)

PaintOp replay plane(`PaintReplayPlane`: Background → BehindText → Flow → InFrontOfText)과 **의미 정합**하도록 SVG 순회를 plane 다중 패스로 변경한다.

- `ImageNode` 는 이미 `text_wrap: Option<TextWrap>` 필드 보유(render_tree.rs:865, Task #516). 단 **layout 이 현재 채우지 않음**(기본 None) — 먼저 채워야 plane 판별 가능.
- `paint_op_replay_plane()` 과 동일 분류 규칙(BehindText/InFrontOfText/그 외=Flow) 적용.

## 4. 작업 범위

| 파일 | 변경 |
|------|------|
| `src/renderer/layout.rs` | ImageNode 생성 시 `text_wrap` 채우기 (BehindText/InFrontOfText 보존) |
| `src/renderer/svg.rs` | `render_node` 를 plane 별 multi-pass 순회로 — Background/BehindText 이미지 → Flow(텍스트·표·어울림) → InFrontOfText 이미지 |
| `tests/issue_1167*.rs` (신규) | 복학원서 SVG 에서 BehindText `<image>` 가 본문 `<text>` 앞(아래)에 위치 회귀 가드 |

## 5. 검증 게이트

- 자동: `cargo test --tests` + 신규 회귀 가드 + `cargo fmt --check` + `cargo clippy --lib`
- 정량: 복학원서 SVG 에서 BehindText `<image>` 출현 줄 < 본문 첫 `<text>` 줄
- **시각: 작업지시자 한컴 정답지 대조** — SVG 에서 워터마크가 본문 뒤로 (PNG 와 동일 시각)
- 회귀: PNG/CanvasKit 무영향(svg.rs + layout text_wrap 채움만), skia/canvaskit 테스트 유지

## 6. 리스크

- layout 에서 `text_wrap` 채움이 다른 렌더러(skia/canvaskit)에 영향? → ImageNode.text_wrap 은 현재 None 이고 PaintOp 경로는 별도 image.text_wrap 을 쓰므로 무영향 확인 필요(구현 단계 점검).
- InFrontOfText(직인) 도 같은 plane 순회로 자연 정합되는지 — 복학원서 (인) 서명 확인.
- `feedback_image_renderer_paths_separate`: 이번엔 SVG 단독 정정이나, layout 공통 변경(text_wrap)이 타 경로에 새 분기 유발 안 하도록 점검.

3~4 단계 예상: (1) layout text_wrap 채움 + RED 테스트, (2) svg.rs plane multi-pass, (3) 시각 검증 + 회귀 가드 + 보고서.
