# task 1293 stage100 - 2023-09 p19 tail 후보 판정

## 목적

stage99 이후 `2023-09`는 `flagged=1/20`만 남았다. 잔여 후보는 p19의
`render_tree_frame_tail_overflow`, `line_band_drift`, `column_line_band_drift`, `large_ink_region_drift`이다.

stage100에서는 p19 후보가 실제 flow mismatch인지, 작은 하단 bleed/visual sweep 과검출인지 판정한다.

## 시작 기준

- 브랜치: `local/task_m100_1293`
- 시작 커밋: `1e40bd80 task 1293: 기본 미주 중간 tail gap 흡수 보정`
- Stage99 targeted sweep:
  - `2023-09`: `flagged=1/20`
  - 남은 페이지: p19
  - 대표 후보:
    - `render_tree_frame_tail_overflow`: `pi=935`, `문28`, overflow `6.4px`
    - `column_line_band_drift`: column 1 drift 후보

## 처리 방향

- p19 render-tree와 PDF marker/line-band를 비교한다.
- `pi=935` overflow가 실제 raster frame 밖 ink인지, Stage93의 small tail bleed와 같은 범주의 과검출인지 확인한다.
- 실제 flow mismatch이면 typeset 보정, 과검출이면 sweep 억제 조건으로 분리한다.

## 분석 결과

`2023-09` p19는 실제 문항 흐름 mismatch가 아니라 visual sweep 과검출로 판정했다.

- 문항 marker:
  - `문28`: rhwp y=`412.1px`, PDF y=`412.3px`
  - `문29`: rhwp y=`701.1px`, PDF y=`701.5px`
  - 문항 단/쪽 흐름은 일치한다.
- tail 후보:
  - `pi=935`, `문28`, text line bbox overflow `6.4px`
  - 실제 RHWP raster frame 밖 bleed extent는 `6px`
  - PDF frame 밖 bleed는 `0px`
  - content bottom delta는 `7px`
- annotated 비교에서도 p19 문항과 큰 그림 배치는 맞고, 하단 한 줄의 logical bbox만 frame을 소폭 넘는다.

## 구현 내용

- `scripts/task1274_visual_sweep.py`의 `suppress_tolerated_frame_tail_candidates`에
  `text_logical_box_bleed` 억제 조건을 추가했다.
- 기존 조건은 실제 bleed extent가 6px여도 `rhwp_out_pixels <= 300`을 요구해 p19의 폭 넓은 한 줄
  텍스트(`346px`)를 억제하지 못했다.
- 새 조건은 다음이 모두 참일 때만 tail 후보를 `small_visual_tail_bleed`로 suppress한다.
  - question marker drift 없음
  - RHWP/PDF frame 밖 bleed extent가 허용 범위 안
  - render-tree line overflow가 한 줄 bbox 높이 이하
  - content bottom delta가 기존 drift 한계 안
