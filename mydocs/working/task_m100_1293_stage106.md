# task 1293 stage106 - 2024-09 below20 p14 문22 drift 분석

## 목적

stage105 이후 `2024-09-below20`은 `flagged=3/23`까지 줄었다.
남은 후보 중 가장 이른 p14는 `문22`가 같은 우단 안에서 PDF보다 아래로 배치되는 question drift와 column line drift가 함께 잡힌다.

stage106에서는 p14 `문22`가 이전 미주 tail/제목 gap 보정 대상인지, 또는 같은 쪽 우단에서 별도 단 시작/누적 높이 정합이 필요한 케이스인지 분리한다.

## 시작 기준

- 브랜치: `local/task_m100_1293`
- 시작 커밋: `be83f173 task 1293: below20 제목 tail 렌더 gap 보정`
- 대상:
  - `2024-09-below20`: p14

## 조사 계획

1. stage105 산출물의 p14 `flagged_pages.json`, question flow, render tree, annotated 이미지를 확인한다.
2. `문22` 주변 미주 번호/본문의 RHWP/PDF 위치 차이를 확인한다.
3. 이전 미주 경계 gap, 현재 미주 head/body split, 또는 우단 시작 누적 높이 문제인지 분리한다.

## 조사 결과

- p14 drift 기준은 `문22`이다.
  - stage105 RHWP: p14 우단 `y=577.0`
  - PDF: p14 우단 `y=515.9`
  - 차이: `+61.1px`
- 같은 페이지의 `문21`은 RHWP가 PDF보다 약 12px 위에 있어 페이지 전체 누적 drift는 아니다.
- debug 기준 note22 경계:
  - note22 boundary: `col=2/2 cur=404.60`
  - renderer `HeightCursor`: `pi=785 y_in_rel=486.3 y_out_rel=504.3`
  - pagination 흐름보다 renderer saved-vpos forward jump가 과하게 적용되어 제목이 내려간다.
- 기존 `compact_endnote_page_title_mid_stale_gap` 계열 보정은 `seg.line_spacing >= betweenNotes`인 경우에만 동작한다.
- 이 케이스는 직전 문단 `line_spacing=452HU`로 기본 줄간격만 들고 있고, 공식 `미주 사이 1984HU`는 별도 gap으로 보존되어야 한다. 따라서 순차 y 뒤에 `(betweenNotes - prevLineSpacing)`만 반영하는 page-path 중단 제목 보정이 필요하다.

## 수정 내용

- `src/renderer/height_cursor.rs`
  - 기본 `미주 사이`를 쓰는 page-path 중단 미주 제목에서 직전 lineSeg가 기본 lineSpacing만 갖고 있고 saved-vpos forward jump가 48~120px 범위로 남는 경우를 추가 감지한다.
  - 제목 위치를 `y_offset + (betweenNotes - prevLineSpacing)`로 접고, 후속 같은 미주 본문이 같은 기준을 따르도록 vpos base도 함께 이동한다.
  - 적용 범위는 기본 7mm 계열(`0 < betweenNotes <= 2500HU`), page-path, 문항 제목, 단 높이 35~55% 중단부로 제한했다.
