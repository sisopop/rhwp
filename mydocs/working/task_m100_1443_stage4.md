## 1. Stage 4 목표

`셀 높이를 같게`, `셀 너비를 같게` 명령이 한컴처럼 선택된 셀 범위 기준으로 동작하도록 보정한다.

Stage 2에서는 컨텍스트 메뉴 노출과 활성화까지 확인했지만, 현재 명령 구현은 선택 범위가 있어도 표 전체 행/열을
대상으로 평균을 계산한다. 이슈 #1443 본문에는 “셀 선택 범위가 있을 때 선택 범위 기준으로 동작하는지 확인”이
명시되어 있으므로 이번 단계에서 실제 적용 범위를 정정한다.

한컴 도움말 기준:

- `셀 높이를 같게`: 셀 높이를 같게 만들 셀들을 셀 블록으로 설정한 뒤 실행하면 셀 블록으로 설정한 셀들의 높이가 같아진다.
- `셀 너비를 같게`: 셀 너비를 같게 만들 셀들을 셀 블록으로 설정한 뒤 실행하면 셀 블록으로 설정한 셀들의 너비가 같아진다.
- 두 기능 모두 선택된 셀 블록이 직사각형이어야 하며, 불연속/비직사각형 셀 블록에서는 실행할 수 없다.
- 참고:
  - https://help.hancom.com/hoffice130_assistant/ko-KR/Hwp/table/table(eqheight).htm
  - https://help.hancom.com/hoffice130_assistant/ko-KR/Hwp/table/table(eqwidth).htm

## 2. 현재 판정

- 마우스 드래그 셀 선택: 구현 완료.
- Ctrl/Cmd/Alt+방향키 셀 크기 조정: 구현 완료.
- 보호 셀 포함 드래그 선택과 보호 셀 입력 차단: 유지 확인.
- `셀 높이를 같게` / `셀 너비를 같게`: 메뉴는 뜨지만 구현은 표 전체 기준.
- Alt+C 모양복사: `edit:format-copy`가 비활성 상태로 남아 있어 별도 범위 판단 필요.

## 3. 구현 방향

- 셀 선택 모드이면 `getSelectedCellRange()`를 읽어 선택된 row/col 범위만 대상으로 균등화한다.
- 셀 선택 모드가 아니면 기존처럼 현재 표 전체 기준 동작을 유지한다.
- 병합 셀은 기존 구현과 동일하게 `rowSpan > 1`, `colSpan > 1`인 셀을 균등화 계산에서 제외한다.
- 선택 범위가 1행 또는 1열뿐인 경우는 변경할 대상 축이 부족하므로 아무 작업도 하지 않는다.

## 4. 회귀 방지 기준

- 셀 선택 모드가 아닌 일반 표 내부 명령 동작은 기존 표 전체 균등화로 유지한다.
- 선택 범위 밖의 행/열 크기는 바뀌지 않아야 한다.
- 제외 셀이 있는 불연속/비직사각형 셀 선택에서는 한컴 제한에 맞춰 실행하지 않는다.
- 마우스 드래그로 만든 셀 선택 범위에서도 동일하게 동작해야 한다.
- Stage 1~3의 셀 드래그 선택, 단축키 크기 조정, 보호 셀 클릭 복귀 동작을 깨지 않는다.

## 5. 검증 계획

- 선택 범위 일부에서 `table:cell-height-equal` 실행 후 선택 행만 같은 높이가 되는지 확인한다.

## 6. 구현 결과

- `table:cell-height-equal`, `table:cell-width-equal`이 셀 선택 모드에서는 `getSelectedCellRange()` 범위 안의 셀만 수집하도록 수정했다.
- 셀 선택 모드가 아니면 기존처럼 현재 표 전체 범위를 대상으로 유지했다.
- Ctrl/Cmd+클릭 제외 셀이 있는 선택은 한컴 도움말의 직사각형 제한에 맞춰 실행하지 않도록 했다.
- `InputHandler.hasExcludedCellSelection()`을 추가해 명령 계층에서 비직사각형 셀 선택 여부를 확인할 수 있게 했다.
