## 1. Stage 9 목표

마지막 행/열의 바깥쪽 셀 선을 드래그해 셀 크기를 조정할 수 있도록 한다.

현재 rhwp-studio는 표의 바깥쪽 경계선을 클릭하면 무조건 표 전체 객체 선택으로 전환한다. 이 때문에 한컴처럼 마지막 행의 아래 선이나 마지막 열의 오른쪽 선을 드래그해 마지막 셀 크기를 조절할 수 없다.

추가로, 한 열만 선택한 상태에서 `셀 높이를 같게`를 실행하면 선택 열의 셀 속성만 균등화되어 렌더러의 행 높이 계산과 맞물려 한컴과 다른 화면이 나온다. 한컴처럼 선택 범위가 걸친 행 전체 높이가 같아지도록 보정한다.

## 2. 회귀 방지 기준

- 표 내부/바깥 경계선 드래그로 셀 너비와 높이를 조정할 수 있어야 한다.
- 마지막 행 아래 선과 마지막 열 오른쪽 선도 셀 크기 조정 경로를 타야 한다.
- 단순 표 외곽 클릭으로 표 객체 선택에 진입하는 기존 경로는 유지한다.
- `셀 높이를 같게`는 선택 범위가 걸친 행 전체를 균등화해야 한다.
- Stage 7의 표 객체 선택 후 셀 편집 진입을 깨지 않는다.
- 셀 드래그 선택, 보호 셀 드래그 선택, 셀 내부 텍스트 드래그 선택을 깨지 않는다.

## 3. 구현 방향

- `tableResizeRenderer.hitTestBorder()`로 잡힌 경계선은 외곽선이어도 리사이즈 드래그를 시작한다.
- 표 객체 선택은 경계선 드래그 후보가 아닌 일반 외곽 클릭 경로에서 처리한다.
- 실제 마우스 드래그 검증으로 마지막 행 높이 조정과 마지막 열 너비 조정을 확인한다.
- `table:cell-height-equal`은 선택된 셀 목록이 아니라 선택 범위가 걸친 행 전체를 대상으로 계산/적용한다.

## 4. 구현 결과

- 셀 선택 모드와 일반 표 편집 상태의 경계선 클릭 처리에서 외곽선 즉시 표 객체 선택 분기를 제거했다.
- 모든 경계선은 먼저 리사이즈 드래그 후보로 시작하도록 했다.
- 드래그 이동량이 1px 미만인 단순 외곽 클릭이면 리사이즈를 취소하고 표 객체 선택으로 전환하도록 `finishResizeDrag`에 처리했다.
- 이로써 마지막 행 아래 선과 마지막 열 오른쪽 선은 드래그 시 셀 크기 조정, 단순 클릭 시 표 객체 선택으로 동작한다.
- `table:cell-height-equal`이 선택 범위가 걸친 행 전체의 현재 행 높이 최댓값을 기준으로 평균 높이를 계산하고, 해당 행의 모든 단일 행 셀에 적용하도록 했다.
