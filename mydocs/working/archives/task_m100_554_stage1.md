## 3. 식별 휴리스틱 검증

### 3.1 HWPX — hwpml 스키마 버전

`Contents/header.xml` 의 `<hh:head version="...">`:

| 분류 | 파일 | hwpml | 페이지 회귀 |
|------|------|-------|-------------|
| **HWP3 변환본** | `hwp3-sample-hwpx.hwpx` | **1.4** | +1 |
| **HWP3 변환본** | `hwp3-sample5-hwpx.hwpx` | **1.4** | +4 |
| 한컴 직접 작성 | `표-텍스트.hwpx` | 1.5 | 정상 |
| 한컴 직접 작성 | `2025년 기부·답례품.hwpx` | 1.5 | 정상 |

→ **`hwpml="1.4"` = HWP3 변환본** (6/6 fixture 100% 정확)

### 3.2 HWP5 — 메타데이터 분석

다음 후보를 검증했다:

| 후보 | 결과 | 채택 |
|------|------|------|
| `FileHeader` version (5.0.x.x ~ 5.1.x.x) | 변환본/native 모두 5.1.0.1 발생 — 식별 불가 | ❌ |
| `HWPTAG_COMPATIBLE_DOCUMENT` (표 54) | 모든 fixture가 0 (한글 문서 현재 버전) — 한컴이 변환 시 호환 모드 미표시 | ❌ |
| `HWPTAG_LAYOUT_COMPATIBILITY` (표 56) | 모든 fixture가 동일 hex (`0000...`) | ❌ |
| `paper_height` (84186 vs 84188) | 변환본 모두 84188이지만 native도 84188 다수 | ❌ (단독 사용 불가) |
| **`ParaShape/Paragraph` 비율** | 변환본 0.007~0.036, native 0.076~21.0 | ✅ |
| **`CharShape/Paragraph` 비율** | 변환본 0.005~0.103, native 0.214~9.500 | ✅ |

**검증 데이터 (27 fixture)**:

| 분류 | 파일 | PS/Para | CS/Para |
|------|------|---------|---------|
| **변환본 (3)** | hwp3-sample-hwp5.hwp | 0.036 | 0.103 |
| | hwp3-sample4-hwp5.hwp | 0.019 | 0.016 |
| | hwp3-sample5-hwp5.hwp | 0.007 | 0.005 |
| **직접 작성 (24)** | (가장 가까운 케이스) exam_kor.hwp | 0.076 | 0.214 |
| | (그 외) | ≥ 0.171 | ≥ 0.310 |

→ **`PS/Para < 0.05 AND CS/Para < 0.15`** 휴리스틱이 27 fixture에서 100% 정합.

### 3.3 휴리스틱 안정성

- 한컴이 HWP3→HWP5 변환 시 ParaShape/CharShape를 거의 재사용하지 않고 매우 적은 수만 생성 (paragraph 1931개 → ParaShape 14개)
- 직접 작성본은 작성자가 다양한 스타일을 사용하므로 PS/CS 수가 paragraph와 비슷하거나 더 많음
- 가장 가까운 직접 작성본 (exam_kor.hwp PS/Para=0.076)도 0.05 임계값에서 안전 분리
- 매우 짧은 문서 (paragraph 1~5개)는 비율이 왜곡될 수 있어 **`paragraph_count > 50` 가드 권장**

## 5. 잔존 사항 (Task #554 범위 밖)

1. **hwp3-sample4.hwp 자체 회귀** — HWP3 39 페이지 vs 정답 36 페이지 (+3). HWP3 파서 또는 페이지네이션 별도 결함. 별도 issue 등록 권고.
2. **hwp3-sample 변환본 -1 over-correct** — 방안 C 적용 시 sample은 16 → 15로 회귀. -1600 단일 값으로는 sample/sample5 모두 정확하지 않음. 추가 정밀화 가능성: -1300 ~ -1500 등 다른 값 또는 두 단계 보정.
