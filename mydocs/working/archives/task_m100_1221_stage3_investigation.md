# Stage 3 심화 계측보고서 — Task M100 #1221 (z-표 한 행 밀림)

**브랜치**: `local/task1221`
**결과**: 원인 대폭 좁힘, 단 정확한 메커니즘 미확정 → 전용 사이클 권장. **소스 무변경**.

## 계측으로 배제/확인한 것

1. **텍스트 줄 흐름은 정상**: 셀 valign 계측 결과 z-열(cell_idx=2) y_start=912.6 (lhs=[825,900,900,900]), P-열(cell_idx=3) y_start=912.1 (lhs=[900×4]). 행 위치가 **거의 정렬**(행별 ≤1px 차). → 한 행 밀림은 **텍스트 줄 배치/centering 문제 아님**.
2. **line-height 클램프 무효**: 수식 common.height=825, font_size=900 ≈ 줄높이. 클램프해도 한 행 밀림 안 사라짐(시도 2종 revert).
3. **수식(tac) 렌더 경로 불일치**: `paragraph_layout.rs:3008` 의 셀 인라인 수식 EquationNode 렌더 경로에 계측 로그를 넣었으나, **z-표 수식(y≈912)이 그 경로에 안 나타남**(로그된 셀 수식 최대 y=553). → z-표(어울림 host 인라인 표) 셀 수식은 **다른 렌더 경로/좌표계**로 그려지거나, recompose 가 첫 수식을 다른 줄에 매핑.

## 좁혀진 가설 (전용 사이클 착수점)

- 결함: **다중-줄 셀 문단 p[0](1.0/1.1)의 첫 수식이 ~한 줄 아래로 렌더** → 행1 비고, 행2 에 1.0+1.1 겹침.
- 의심: (a) 어울림 host 인라인 표 셀의 수식 렌더가 표준 경로(3008)와 다른 경로를 타거나, (b) `recompose_for_cell_width` 후 tac_offset→line 매핑이 p[0] 첫 줄(1.0)을 둘째 줄에 할당.
- 다음 단계: z-표 셀 수식이 **실제로 어느 함수에서 어느 y로** 그려지는지(어울림 host 인라인 표 셀 렌더 경로) 추적 → tac→line 매핑/recompose 정합 수정.

## 결론
