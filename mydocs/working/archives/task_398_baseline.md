## 4. 불일치 패턴 분석

### 패턴 A: text_start 오차 (가장 빈번)

- **방향**: 일관적으로 **음수** (-2~-4 UTF-16 단위)
- **의미**: rhwp가 한컴보다 문자 폭을 약간 **좁게** 측정 → 한 줄에 더 많은 글자를 넣음
- **영향**: 줄바꿈 위치 이동, 누적 시 줄 수 차이로 확대
- **원인 추정**: 내장 폰트 메트릭과 한컴 메트릭 간 미세 차이
- **해당 샘플**: lseg-01, lseg-04, lseg-05 (바탕체 가변폭)
- **Task 400 대상**

### 패턴 B: 줄 수 불일치 — 4줄→5줄 (lseg-03-spacing)

- 4개 문단 **모두** 동일한 패턴 (줄간격 130%/160%/200%/고정 20pt 무관)
- 줄간격 종류와 무관하므로 **줄바꿈 알고리즘 자체의 차이**
- 텍스트 폭 측정이 좁아 마지막 단어가 한 줄에 들어가는 경계 문제
- **Task 399 + 400 대상**

### 패턴 C: 다중 크기 혼합 시 line_height/baseline 불일치 (lseg-06)

- 한 문단 내 10pt + 16pt 혼합 시, 한컴은 **줄별로 다른 line_height** 생성
- 현행 `reflow_line_segs()`는 첫 줄의 line_height를 **모든 줄에 동일 적용**
- pi=0: L1에서 line_height +600, baseline +510 (16pt 글자가 있는 줄)
- pi=2: L1에서 line_height -200 (10pt만 있는 줄에 큰 값 적용)
- **Task 401 핵심 대상**

### 패턴 D: segment_width 미세 차이 (2~3 HWPUNIT)

- 거의 모든 문단에서 발생
- px→HWPUNIT 변환 시 반올림 방식 차이로 추정
- 레이아웃에 실질적 영향 없음 (0.01mm 미만)
- **낮은 우선순위**
