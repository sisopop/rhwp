## 완료 내용

### 1. 현재 레이아웃 상수 전수 조사

`layout.rs`의 명시 상수 8개와 코드 내 하드코딩 비율 13개를 전수 목록화했다.

### 2. TeX 표준 파라미터와 비교

| 상수 | rhwp | TeX 표준 | 판정 |
|------|------|---------|------|
| `SCRIPT_SCALE` | 0.7 | 0.7 | ✓ 일치 |
| `FRAC_LINE_THICK` | 0.04 | 0.04em | ✓ 일치 |
| `BIG_OP_SCALE` | 1.5 | ~1.2 | ⚠ 25% 차이 |
| `MATRIX_COL_GAP` | 0.8 | 0.5~1.0 | 한컴 측정 필요 |
| `MATRIX_ROW_GAP` | 0.3 | 0.2~0.5 | 한컴 측정 필요 |

### 3. 구조별 보정 방향 설계

분수, 행렬, 큰 연산자, 첨자, 괄호, 연산자 여백 6개 구조에 대해 현재 방식과 TeX 방식을 비교하고 보정 방향을 제시했다.

### 4. 문단 배치 분석

- HWP 저장값(`common.width/height`)과 레이아웃 엔진 산출값의 불일치 가능성 식별
- 인라인 수식의 베이스라인 정렬 문제(하단 맞춤 vs 수식 축 기준 맞춤) 분석

### 5. 한컴 기준값 측정 절차 수립

표준 수식 12종을 선정하여 한컴 스크린샷 vs rhwp SVG 비교 측정 절차를 수립했다.
