## 2. 한컴 권위 영역 분석

- 작은 차이 (1~2%) 영역: 한컴은 비례 축소 안 함 추정 (셀 콘텐츠 측정값과 common.height 의 미세한 차이는 측정 오차로 간주)
- 큰 차이 (≥5%) 영역: 한컴도 비례 축소 발동 가능성 (TAC 표 본문 압축이 사용자 의도)

본 case (`samples/계획서.hwp`) 1.32% 차이는 한컴이 비례 축소 안 하므로 본 환경도 면제해야 정합.

## 4. 임계값 결정 제안

### 옵션 1: **2%** (보수적, 권장)

- 본 case + 작은 차이 영역 (총 7 건) 만 면제
- 회귀 위험 가장 좁힘
- 5% 이상 영역 (의도적 큰 압축) 그대로 동작
- 2~5% 영역 (4 건) 은 기존 동작 (비례 축소) 유지

### 옵션 2: **5%** (적극적)

- 본 case + 모든 작은~중간 차이 (총 12 건) 면제
- 한컴 권위 영역과 더 큰 정합 (가능성)
- 회귀 위험 약간 큼 — 2~5% 영역 동작 변화

### 옵션 3: **절대값 (예: 5px) + 비례 (예: 2%)**

- `raw - common > max(5px, common * 2%)` 시에만 비례 축소 발동
- 작은 표 (작은 절대값) 도 적절히 면제
- 큰 표 (큰 절대값) 도 비례 임계값으로 안전하게 발동

권장: **옵션 1 (2% 임계값)** — 회귀 위험 가장 좁힘.

## 5. Stage 2 정정 방안

`src/renderer/height_measurer.rs:805-813` 정정:

```rust
// [Task #672] 작은 차이 (~2% 미만) 는 측정값 우선 — 셀 콘텐츠 측정값과
// common.height 의 미세한 불일치 시 비례 축소가 셀 콘텐츠 클립을 발생.
// 본 case (계획서.hwp 1.32%) 정정 + 다른 영역 (>2%) 그대로 동작.
const TAC_SHRINK_THRESHOLD_RATIO: f64 = 0.02; // 2% 임계값

let shrink_threshold = (common_h * TAC_SHRINK_THRESHOLD_RATIO).max(1.0);
let table_height = if table.common.treat_as_char
    && common_h > 0.0
    && raw_table_height > common_h + shrink_threshold {
    let scale = common_h / raw_table_height;
    for h in &mut row_heights {
        *h *= scale;
    }
    common_h
} else {
    raw_table_height
};
```
