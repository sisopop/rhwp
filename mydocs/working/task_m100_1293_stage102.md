# task 1293 stage102 - p10 question drift 공통 후보 조사

## 목적

stage101 전체 visual sweep에서 `2022-11-practice` p10과 `2024-11-practice-above0-between7-below2` p10이 모두 question/line drift로 잡혔다.
두 후보가 같은 배치 원인인지, 실제 렌더 차이인지, sweep 과검출인지 확인하고 필요한 경우 한 스테이지 범위의 보정을 수행한다.

## 시작 기준

- 브랜치: `local/task_m100_1293`
- 시작 커밋: `34e5d7d7 task 1293: 전체 visual sweep 재집계`
- 대상:
  - `2022-11-practice`: p10
  - `2024-11-practice-above0-between7-below2`: p10

## 조사 계획

1. stage101 산출물의 summary/annotated/render-tree를 확인한다.
2. 두 대상의 p10 question marker, line band, large ink drift가 같은 방향/크기인지 비교한다.
3. 실제 본문/미주 흐름 차이면 typeset 보정을 검토한다.
4. 시각적으로 허용 가능한 안정 오차이면 sweep 억제 조건을 검토한다.

## 분석 결과

두 p10 후보는 같은 원인이었다.

- `2022-11-practice`와 `2024-11-practice-above0-between7-below2`의 p10 RHWP/PDF PNG 해시가 각각 동일했다.
- `문10` question marker는 수정 전 RHWP y=`1034.4px`, PDF y=`1079.8px`로 약 `45.4px` 높았다.
- 실제 차이는 수식 줄 자체가 아니라 미주 경계 gap이었다.
  - `문7` 내부 수식 줄은 PDF와 거의 정합했다.
  - `문7 → 문8` 경계에서 `gap=Some(0.0)`으로 계산되어 `미주 사이 7mm`가 빠졌다.
  - `문9 → 문10`도 같은 누적 계열이었다.
- stage99에서 `2023-09` p13의 `문15 → 문16` 이중 gap을 막기 위해 넣은
  중간 단 default gap 흡수 조건이, vpos rewind가 없는 정상 미주 경계까지 포함했다.

`2023-09` p13의 `문15`는 같은 미주 내부에서 `LINE_SEG.vertical_pos`가 0으로 되감기는 tail을 가진 반면,
`2022-11-practice` p10의 `문7`은 되감김 없는 정상 경계다. 따라서 stage99 보정은 직전 미주에
vpos rewind가 있는 경우로 좁히는 것이 맞다.

## 구현 내용

`src/renderer/typeset.rs`에서 두 조건을 좁혔다.

- `skip_default_mid_column_between_notes_trailing`
- `default_visible_tail_absorbed_gap`

두 조건 모두 `boundary_prev_endnote_had_vpos_rewind`가 참일 때만 적용하게 해,
되감김 tail이 만든 이중 gap은 계속 흡수하고 정상 미주 경계에서는 `미주 사이`를 보존한다.
