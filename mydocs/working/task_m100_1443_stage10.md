## 1. Stage 10 목표

`셀 높이를 같게` / `셀 너비를 같게` 실행 범위와 렌더링 결과를 한컴 동작에 맞게 다시 보정한다.

Stage 9에서는 선택된 열이 걸친 행 전체를 균등화하도록 변경했지만, 실제 한컴 동작은 선택된 셀만 높이 균등화 대상이다. 같은 행에 있더라도 선택되지 않은 셀의 높이는 유지되어야 한다.

추가 확인 결과, 선택 셀의 height 속성만 바꾸면 rhwp 렌더러가 행별 최대 높이(`row_y`)로 모든 열의 셀 bbox를 맞추기 때문에 화면에서는 여전히 전체 행 높이가 바뀐 것처럼 보인다. 한컴처럼 선택 열의 가로선이 다른 열과 독립적으로 보이려면 비-TAC 일반 표에서 열별 누적 y를 사용할 수 있어야 한다.

같은 원리로 `셀 너비를 같게`도 선택 셀의 width 속성만 바꾸면 렌더러가 전체 열 폭 기준으로 모든 행의 x 경계를 맞추어 화면이 한컴과 달라진다. 한컴처럼 선택 행의 세로선이 다른 행과 독립적으로 보이려면 비-TAC 일반 표에서 행별 누적 x를 실제 셀 width 기준으로 계산해야 한다.

## 2. 회귀 방지 기준

- 한 열만 선택한 상태에서 `셀 높이를 같게`를 실행하면 선택된 열의 셀 높이만 같아져야 한다.
- 선택되지 않은 다른 열의 셀 높이는 기존 값을 유지해야 한다.
- 렌더링 bbox와 셀 선택 하이라이트도 선택 열에서만 균등 높이를 보여야 한다.
- 한 행만 선택한 상태에서 `셀 너비를 같게`를 실행하면 선택된 행의 셀 너비만 같아져야 한다.
- 선택되지 않은 다른 행의 셀 너비는 기존 값을 유지해야 한다.
- 렌더링 bbox와 셀 선택 하이라이트도 선택 행에서만 균등 너비를 보여야 한다.
- 전체 표가 선택된 경우에는 전체 셀 높이가 같아지는 기존 기대 동작을 유지한다.
- 비직사각형 셀 선택은 기존처럼 명령을 수행하지 않는다.
- TAC(`treat_as_char=true`) 표의 기존 정렬과 레이아웃 테스트를 깨지 않아야 한다.

## 3. 구현 방향

- `table:cell-height-equal`의 적용 대상을 Stage 9의 “선택 범위가 걸친 행 전체”에서 “선택 범위 안의 단일 행 셀”로 되돌린다.
- 평균 높이 계산도 선택된 셀들의 현재 높이를 기준으로 수행한다.
- 비-TAC 일반 표에서 열별 누적 높이가 전역 행 높이와 달라지는 경우 셀 bbox와 테두리를 셀 자체 높이 기준으로 만든다.
- 비-TAC 일반 표에서 행별 누적 너비가 전역 열 폭과 달라지는 경우 셀 bbox와 테두리를 셀 자체 너비 기준으로 만든다.
- TAC 표와 분할 표는 기존 `row_y` 기반 렌더링을 유지한다.
- 자동 검증은 한 열/행 선택 후 선택 범위 bbox만 균등화되고 나머지 bbox가 유지되는지 확인하는 방식으로 보강한다.

## 4. 구현 결과

- `table:cell-height-equal`에서 `targetRows` 기반 행 전체 확장 로직을 제거했다.
- `isCellInRange()`로 실제 선택 범위 안의 단일 행 셀만 수집하도록 변경했다.
- 평균 높이는 선택된 셀들의 현재 높이 합계를 선택 셀 수로 나누어 계산한다.
- `src/renderer/layout/table_layout.rs`에 열별 누적 y 계산 경로를 추가했다.
- `src/renderer/layout/border_rendering.rs`의 `build_row_col_x()`가 비-TAC 일반 표에서 행별 실제 셀 width를 반영하도록 변경했다.
- 비-TAC 일반 표에서 셀별 height가 행별 최대 높이와 달라지면 해당 열의 누적 y로 셀 bbox를 생성한다.
- 비-TAC 일반 표에서 셀별 width가 열별 최대 폭과 달라지면 해당 행의 누적 x로 셀 bbox를 생성한다.
- 독립 높이 표에서는 기존 행 격자 테두리 대신 셀 bbox 기준 테두리 라인을 렌더링해 한컴처럼 선택 열 내부 가로선을 독립적으로 표시한다.
- 행별 독립 너비는 기존 테두리 렌더러의 `row_col_x` 경로를 사용해 한컴처럼 선택 행 내부 세로선을 독립적으로 표시한다.
- TAC 표는 독립 높이 렌더링에서 제외해 기존 수식/인라인 표 정렬 회귀를 막았다.
