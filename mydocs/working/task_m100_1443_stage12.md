## 1. Stage 12 목표

Shift+셀 외곽 드래그가 한컴처럼 경계선 segment 하나만 움직이도록 보정한다.

Stage 11에서는 대상 셀 하나의 width/height 속성만 변경했다. 이 경우 행별/열별 누적 좌표 계산에서 같은 행 또는 같은 열의 뒤쪽 셀이 함께 밀려 표 전체 폭/높이가 달라지는 문제가 있다.

한컴 동작은 대상 셀과 바로 이웃 셀 사이의 경계선만 이동한다. 따라서 대상 셀이 커지면 이웃 셀은 같은 delta만큼 줄어들어야 하며, 그 밖의 셀과 표 외곽은 유지되어야 한다.

## 2. 회귀 방지 기준

- Shift+오른쪽 경계 드래그는 대상 셀 width를 늘리고 오른쪽 이웃 셀 width를 같은 만큼 줄인다.
- Shift+왼쪽 경계 드래그는 대상 셀 width를 줄이고 왼쪽 이웃 셀 width를 같은 만큼 늘린다.
- Shift+아래쪽 경계 드래그는 대상 셀 height를 늘리고 아래쪽 이웃 셀 height를 같은 만큼 줄인다.
- Shift+위쪽 경계 드래그는 대상 셀 height를 줄이고 위쪽 이웃 셀 height를 같은 만큼 늘린다.
- 이웃 셀이 최소 크기에 도달하면 대상 셀 증가도 같은 위치에서 멈춰 표 전체 외곽 밖으로 셀이 빠져나가지 않는다.
- Shift로 한 셀 경계를 이동한 뒤 Shift 없이 같은 분리 segment를 조절하면, 전체 행/열이 아니라 해당 segment만 조절된다.
- Shift로 한 segment가 어긋난 뒤 같은 논리 경계의 공통 좌표에 남아 있는 다른 segment를 Shift 없이 조절하면 전체 경계 리사이즈로 동작한다.
- Shift로 한 segment가 어긋난 뒤 같은 논리 경계의 정렬된 다른 segment를 Shift 없이 조절하면, 한컴처럼 같은 실제 좌표에 남아 있는 segment 그룹만 같이 움직인다.
- Shift로 어긋난 기존 segment는 같은 논리 경계라도 실제 좌표가 다르면 일반 정렬 segment 조절의 영향을 받지 않는다.
- Shift로 직접 만든 segment는 이후 Shift 없이 다시 조절해도 해당 segment만 보상 리사이즈한다.
- Shift로 일부 경계가 어긋난 뒤 실제 셀 경계가 없는 위치에서는 ghost 경계선을 hit-test하지 않는다.
- Shift 없는 균일 내부 경계 리사이즈도 반대편 이웃 셀을 보상해 표 전체 외곽 크기를 유지한다.
- 이웃 셀이 없는 외곽선에서는 기존처럼 대상 셀 단독 변경을 허용한다.
- Shift 없는 경계선 드래그는 기존처럼 행/열 전체 경계를 움직이되, 표 전체 크기는 유지한다.

## 3. 구현 방향

- Shift 리사이즈 완료 시 대상 셀 bbox를 찾아 드래그한 경계선의 반대편 이웃 셀을 찾는다.
- 이웃 셀이 있으면 대상 셀 delta와 이웃 셀 보상 delta를 같은 `resizeTableCells` 배치에 넣는다.
- 자동 검증은 Shift+세로/가로 경계 드래그 후 대상 셀과 이웃 셀만 변경되고 다른 셀은 유지되는지 확인한다.

## 4. 구현 내용

- `rhwp-studio/src/engine/input-handler-table.ts`
  - Shift 단독 셀 리사이즈 대상의 bbox를 기준으로 같은 행/열의 바로 이웃 셀을 찾는 `findSingleCellResizeNeighbor`를 추가했다.
  - Shift 리사이즈 완료 시 대상 셀에는 드래그 delta를 적용하고, 이웃 셀이 있으면 같은 delta를 반대 방향으로 적용한다.
  - Rust `resizeTableCells`의 최소 셀 크기 하한(`200 HWPUNIT`)과 같은 기준으로 단독 셀 리사이즈 delta를 미리 제한한다. 이웃 셀이 더 줄어들 수 없으면 대상 셀도 더 커지지 않는다.
  - Shift 없는 일반 리사이즈는 먼저 클릭한 segment가 같은 논리 경계의 공통 좌표에 남아 있는지 판정한다. 공통 좌표 segment는 전체 행/열을 보상 리사이즈하고, Shift 리사이즈로 공통 좌표에서 벗어난 segment는 해당 segment만 보상 리사이즈한다.
  - Shift로 직접 만든 로컬 segment를 table/논리경계/행 또는 열 span 기준으로 기록한다. 이후 Shift 없이 같은 segment를 잡으면 로컬 리사이즈를 유지한다.
  - Shift 없는 일반 리사이즈의 대상은 논리 경계 전체가 아니라 클릭한 실제 좌표와 같은 segment 그룹으로 제한한다. 따라서 이미 Shift로 분리된 segment는 같은 논리 경계라도 일반 정렬 경계 조절에서 제외된다.
  - 균일한 내부 경계의 전체 행/열 리사이즈에서도 대상 셀과 반대편 이웃 셀을 `+delta/-delta`로 함께 갱신해 표 외곽을 유지한다.
  - 새 문서 생성/문서 열기 시 로컬 segment 기록을 비워 다른 문서의 표에 기록이 섞이지 않도록 했다.
  - 이웃 셀이 없는 표 외곽선에서는 기존처럼 대상 셀만 변경한다.
- `rhwp-studio/src/engine/table-resize-renderer.ts`
  - 경계선 hit-test를 표 전체 min/max 선이 아니라 실제 셀의 상/하/좌/우 segment 기준으로 변경했다.
  - 일부 행/열에만 존재하는 x/y 좌표를 표 전체 높이/폭의 경계선처럼 잡아 ghost 리사이즈가 시작되는 문제를 막았다.
