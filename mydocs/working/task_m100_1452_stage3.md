## 1. 배경

추가 피드백으로 rhwp-studio의 `개체 속성` 창이 탭별 내용 크기에 따라 창 크기와 내부 영역이 달라 보인다는
문제가 보고되었다. 한컴의 개체 속성 창처럼 탭을 바꿔도 같은 크기의 창과 내용 영역을 유지하도록 개선한다.

## 2. 현재 가설

- `PicturePropsDialog`의 탭 패널이 내용 높이를 기준으로 자연 크기 계산을 하면서 탭별로 대화상자 높이가
  달라진다.
- 표 속성 대화상자에서 사용한 방식처럼 body/content 영역에 고정 또는 최소 높이를 부여하고, 각 탭 패널을
  동일한 가용 영역 안에 배치하면 해결 가능하다.

## 3. 작업 범위

- `rhwp-studio/src/ui/picture-props-dialog.ts`
  - 대화상자 shell/body/content 높이를 탭별로 동일하게 유지한다.
  - 탭 패널은 같은 높이 안에서 표시되도록 하고, 내용이 많은 탭은 내부 스크롤을 사용한다.
  - 설정/취소 버튼 영역은 기존처럼 우측에 고정한다.

## 4. 검증 계획

- `cd rhwp-studio && npx tsc --noEmit`
- 필요 시 `cd rhwp-studio && npm test`
- 수동 확인 후보:
  - `개체 속성` 창에서 `기본`, `여백/캡션`, `그림` 탭 전환 시 창 크기가 변하지 않는지 확인
  - `그림` 탭의 긴 내용이 잘리지 않고 내부 스크롤 또는 동일 영역 안에 표시되는지 확인

## 5. 구현 결과

- `picture-props.css`에서 `pp-dialog`를 표 속성 대화상자처럼 고정 높이 flex 레이아웃으로 변경했다.
- 좌측 탭/본문 영역은 남는 높이를 채우고, 각 탭 패널은 같은 높이를 사용하도록 조정했다.
- 내용이 긴 탭은 창 자체를 늘리지 않고 패널 내부 세로 스크롤을 사용한다.
