## 1. Stage 3 목표

한컴 호환 표 셀 크기 조정 단축키를 점검한다.

사용자 피드백의 `표의 셀이 선택된 상태에서 컨트롤이나 알트+방향키로 셀크기 조정` 중 Stage 1/2에서
Ctrl/Cmd+방향키는 드래그 선택과 연결 검증을 마쳤다. 이번 단계에서는 Alt+방향키의 동작 여부와 플랫폼 충돌을
검토하고, 구현 가능한 범위를 확정한다.

## 2. 현재 판정

- Ctrl/Cmd+방향키: 셀 선택 모드에서 기존 구현 동작 확인 완료.
- 마우스 드래그 선택: F5 없이 셀 블록 선택 가능.
- 셀 높이/너비 같게: 컨텍스트 메뉴 노출 및 활성화 확인 완료.
- Alt+방향키: 현재 구현 없음.

## 3. 구현 후보

- 셀 선택 모드에서 Alt+방향키를 Ctrl/Cmd+방향키와 같은 `resizeCellByKeyboard` 경로로 연결한다.
- macOS Option+방향키는 일반 텍스트 편집에서 단어 이동으로 쓰이므로, 표 셀 선택 모드에 한정해 가로채는지 확인한다.
- 일반 셀 텍스트 편집 상태에서는 기존 macOS Option+방향키 단어 이동을 유지한다.

## 4. 구현 결과

- 셀 선택 모드 키 처리에서 Alt+방향키를 기존 Ctrl/Cmd+방향키 셀 크기 조정 경로에 연결했다.
- 일반 텍스트 편집 상태의 navigation keymap은 수정하지 않아 macOS Option+방향키 단어 이동 경로를 유지했다.
- Stage 2 회귀 보정: 셀 선택 모드에서 다른 셀을 단순 클릭하면 셀 선택을 해제하고 클릭한 셀로 텍스트 커서를 이동하도록 했다.
- 보호 셀 선택 모드에서는 보호 셀을 다시 누를 때만 선택을 유지하고, 일반 셀을 누르면 선택을 해제해 텍스트 편집으로 복귀하도록 했다.

## 5. 회귀 방지 기준

- 일반 본문/셀 내부 텍스트 편집에서 macOS Option+방향키 단어 이동을 깨지 않는다.
- 셀 선택 모드의 일반 방향키 이동, Ctrl/Cmd+방향키 크기 조정 동작을 유지한다.
- F5 없이 마우스로 선택한 셀 범위에서도 동일하게 동작해야 한다.

## 6. 검증 결과

- F5 없이 마우스 드래그로 선택한 셀 범위에서 Alt+ArrowRight 크기 조정 확인
  - 선택 셀 폭 `10488 -> 10788`
  - 인접 셀 폭 `10488 -> 10188`
  - 셀 선택 범위 `{ startRow: 0, startCol: 0, endRow: 2, endCol: 2 }` 유지
- 셀 선택 후 다른 셀 단순 클릭 시 커서 이동 확인
  - 셀 선택 모드 해제
  - `.cell-selection-highlight` 0개
  - 클릭한 셀 `row=3, col=0`으로 커서 이동
- 보호 셀 클릭 후 일반 셀 단순 클릭 시 편집 복귀 확인
  - 보호 셀 클릭 직후 단일 셀 선택 유지
  - 일반 셀 클릭 후 셀 선택 모드 해제
  - `.cell-selection-highlight` 0개
  - 클릭한 일반 셀 `row=2, col=2`로 커서 이동
