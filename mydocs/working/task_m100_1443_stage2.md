## 1. Stage 2 목표

F5로 셀 선택 모드에 진입한 뒤에만 마우스 드래그로 여러 셀을 선택할 수 있는 문제를 해결한다.

한컴처럼 일반 셀 편집 상태에서도 셀에서 다른 셀로 마우스를 드래그하면 셀 블록 선택으로 전환되어야 한다.

## 2. 회귀 방지 기준

- 같은 셀 안에서 텍스트를 드래그하면 기존 텍스트 선택으로 남아야 한다.
- 셀 경계선 리사이즈와 표 객체 선택 동작을 방해하지 않아야 한다.
- 보호 셀 클릭/드래그 선택 경로는 Stage 1 동작을 유지해야 한다.
- F5 진입 후 셀 선택 모드 드래그는 Stage 1 동작을 유지해야 한다.

## 3. 구현 메모

- 일반 셀 편집 mousedown 시 셀 블록 드래그 후보를 기록한다.
- mousemove 중 포인터가 같은 표의 다른 셀로 넘어가면 텍스트 드래그를 중단하고 셀 선택 모드로 전환한다.
- 포인터가 같은 셀 안에 머물면 기존 텍스트 드래그 선택 경로를 그대로 사용한다.

## 4. 구현 결과

- `InputHandler`에 일반 셀 편집 상태의 셀 드래그 후보 상태를 추가했다.
- 일반 셀 mousedown 후 같은 표의 다른 셀로 포인터가 넘어가면 텍스트 선택 드래그를 정리하고 셀 선택 모드로 승격한다.
- `CursorState.getCellTableContext()`가 셀 선택 모드 캐시가 없을 때도 현재 일반 셀 위치의 표 컨텍스트를 반환하도록 보강했다.
- 같은 셀 안의 텍스트 드래그는 후보만 유지하다가 mouseup 시 정리되어 기존 텍스트 선택으로 남는다.
- 한컴 컨텍스트 메뉴와 맞추기 위해 표 셀 메뉴에 `셀 높이를 같게`, `셀 너비를 같게` 항목을 추가했다.
- 기존 균등화 커맨드가 셀 선택 모드에서도 활성화되도록 실행 조건을 명시적으로 보강했다.
