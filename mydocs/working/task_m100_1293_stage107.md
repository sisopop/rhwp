# task 1293 stage107 - 2024-09 below20 p16 단 전환 drift 분석

stage106 이후 `2024-09-below20`은 `flagged=2/23`만 남았다.
남은 후보 중 p16은 `문24`가 PDF에서는 좌단 하단에 남지만 RHWP에서는 우단 상단으로 넘어가고, 뒤따르는 `문25`, `문26`도 우단에서 약 103px 내려가는 cascade가 발생한다.

stage107에서는 p16의 첫 단 전환 차이가 이전 미주 tail을 현재 단에 더 남겨야 하는 문제인지, 또는 새 미주 제목/본문을 너무 이르게 다음 단으로 넘긴 문제인지 확인한다.

## 조사 결과

- stage106 기준 p16은 `문24`가 RHWP 우단 상단, PDF 좌단 하단에 있었다.
- debug 로그에서 `문24 ep0`은 `cur=871.13 / avail=1001.56`으로 제목 한 줄과 공식 `미주 사이` gap은 현재 단에 들어가지만, 저장 vpos rewind와 큰 `구분선 아래` head-group 판정 때문에 `large_head_outside=true`, `advance_new=true`가 됐다.
- `문24 ep0~ep2`를 좌단 하단에 남기면 `문25`, `문26`의 우단 y가 PDF와 거의 일치했다.
- 추가로 `문27`은 마지막 단 `cur=972.32 / avail=1001.56`에서 제목 한 줄만 남고 다음 equation-only head 두 문단이 frame tail을 만들었다. PDF는 `문27`을 다음 쪽 좌단 상단에서 시작하므로, 기본 미주 사이 + 큰 `구분선 아래` 마지막 단 제목 orphan을 방지해야 한다.

- `src/renderer/typeset.rs`
  - 기본 미주 사이 + 보이는 구분선 + 큰 `구분선 아래` + vpos rewind 새 문항에서, 공식 `미주 사이` gap과 제목 한 줄이 현재 단에 들어가는 경우 첫 advance를 막아 `문24` head가 좌단 하단에 남도록 했다.
