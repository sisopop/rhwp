## 조사 결과

- 문16 ep0은 fit 단계에서 `large_between_last_column_new_note_tail=true`가 되어 p13 첫 단으로 넘어갔다.
- 문16은 `cur=930.11`, `en_fit=61.13`, `available=1001.56`으로 flow 기준 제목 head가 frame 안에 들어간다.
- new 단계에서도 `endnote_has_vpos_rewind=true`, `render_y=936.23`, `large_head_outside=true` 때문에 advance될 수 있었다.
- PDF 기준 문16은 p12 마지막 단 하단에 남고 p13은 문17부터 시작한다. 따라서 vpos rewind가 있는 제목 head가 flow와 render 양쪽에서 frame 안에 들어가는 경우는 마지막 단 tail로 허용해야 한다.

## 구현

- `src/renderer/typeset.rs`
  - `large_between_last_column_rewind_title_tail_fits` 판정을 추가했다.
  - 큰 `betweenNotes` + visible separator + 마지막 단 + vpos rewind 문항 제목에서, flow 높이와 예측 렌더 y가 frame 안쪽이면 fit/new advance에서 제외한다.
  - `large_between_last_column_new_note_tail`은 vpos rewind가 있고 flow 높이가 frame 안에 들어가는 제목에는 적용하지 않도록 좁혔다.
  - 디버그 출력에 `allow_large_rewind_title`을 추가했다.
