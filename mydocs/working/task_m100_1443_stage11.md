## 1. Stage 11 목표

한컴처럼 Shift+마우스로 셀 외곽선을 드래그하면 해당 셀만 크기를 변경할 수 있도록 한다.

현재 일반 경계선 드래그는 같은 행/열 전체 셀을 함께 조정한다. 한컴은 Shift 키를 누른 채 셀 외곽선을 드래그하면 경계선에 맞닿은 한 셀만 조정되어, 다른 행/열의 경계선과 독립적인 셀 크기 변경이 가능하다.

## 2. 회귀 방지 기준

- Shift 없이 셀 경계선을 드래그하면 기존처럼 행/열 전체 크기를 조정한다.
- Shift+세로 경계선 드래그는 해당 경계선 왼쪽 또는 오른쪽의 한 셀 너비만 조정한다.
- Shift+가로 경계선 드래그는 해당 경계선 위쪽 또는 아래쪽의 한 셀 높이만 조정한다.
- 셀 선택 모드의 기존 선택 셀 리사이즈 동작을 깨지 않는다.
- 외곽선 드래그로 마지막 행/열 크기를 조정하는 Stage 9 동작을 유지한다.
- 단순 외곽선 클릭으로 표 전체 선택에 진입하는 기존 동작을 유지한다.

## 3. 구현 방향

- 리사이즈 드래그 시작 시 Shift 키 상태를 저장한다.
- Shift 리사이즈에서는 경계선에 맞닿은 전체 셀 목록 대신 마우스 위치가 들어있는 셀 하나를 대상으로 삼는다.
- 선택된 셀의 오른쪽/아래쪽 경계뿐 아니라 왼쪽/위쪽 경계 드래그도 해당 셀 크기 변경으로 해석한다.
- 자동 검증은 실제 마우스 이벤트로 Shift+경계선 드래그 후 대상 셀 bbox만 변하는지 확인한다.

## 4. 구현 결과

- `InputHandler.startResizeDrag()` wrapper가 Shift 리사이즈 인자를 `input-handler-table`로 전달하도록 수정했다.
- `startResizeDrag()`에서 Shift 리사이즈인 경우 드래그 시작 위치와 경계선 방향을 기준으로 단일 대상 셀을 계산해 `resizeDragState.singleCellTarget`에 저장한다.
- `finishResizeDrag()`에서 `singleCellTarget`이 있으면 기존 행/열 전체 업데이트 대신 해당 셀 하나에만 width/height delta를 적용한다.
- 왼쪽/위쪽 경계선을 잡은 경우에는 delta 부호를 반대로 적용해 셀 크기 변경 방향을 맞춘다.
- Shift+외곽선 단순 클릭은 표 객체 선택으로 전환하지 않고 리사이즈 후보만 취소하도록 했다.
