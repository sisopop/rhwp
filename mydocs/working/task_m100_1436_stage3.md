# Task 1436 Stage 3

## 목적

크기 고정 개체가 선택됐을 때 리사이즈/회전 핸들이 일반 편집 가능 핸들처럼 보이는 문제를 수정한다. 한컴오피스처럼 크기 고정 상태에서는 핸들 위치에 잠금/금지 표시를 보여 사용자가 크기와 회전 조작이 막힌 상태임을 바로 알 수 있게 한다.

## 범위

- 선택 개체의 `sizeProtect` 상태를 선택 렌더러에 전달
- `sizeProtect` on 상태에서 리사이즈/회전 핸들을 잠금 표시로 렌더링
- `sizeProtect` on 상태에서 핸들 hover 커서가 조작 가능 커서로 바뀌지 않게 유지

## 제외

- 모달 외부 클릭 정책 정리
- 크기 고정과 무관한 선택 핸들 디자인 전면 개편

## 검증 계획

- `npm --prefix rhwp-studio run build`
- `git diff --check`
- 사용자 시각 검증

## 수행 결과

- `TableObjectRenderer.render()`와 `renderMultiPage()`에 잠금 상태 인자를 추가했다.
- 선택 개체의 `sizeProtect`가 켜져 있으면 일반 사각 리사이즈 핸들과 초록색 회전 핸들 대신 원형 금지 표시 핸들을 렌더링하도록 수정했다.
- 잠금 상태에서는 회전 핸들 연결선을 표시하지 않아 조작 가능 상태와 시각적으로 구분되게 했다.
- 다중 선택에서도 선택 개체 중 하나라도 `sizeProtect`가 켜져 있으면 잠금 핸들로 표시한다.

## 검증 결과

- `npm --prefix rhwp-studio run build`: 통과
- `git diff --check`: 통과

## 시각 판단 대기

- 사용자가 Studio에서 크기 고정 개체 선택 시 리사이즈/회전 핸들이 한컴오피스처럼 잠금 상태로 보이는지 확인한다.
