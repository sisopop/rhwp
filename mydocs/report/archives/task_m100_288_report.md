## 경과

### 전제 1: HWP text 에 `\t` 삽입 (단계 1 에서 부인)

이슈 원문의 "HWP composed text 에 `\t` 가 삽입되어 있고 그 때문에 탭 처리가 안 됨" 가설을 실측(`examples/task288_probe.rs`)으로 검증:

```
para.text = "(가) 모든 자연수 에 대하여\n\n이다."
```

- `\t` 문자는 존재하지 않음
- line 1 의 `runs=[]` 는 연속된 `\n\n` 사이에 큰 수식 TAC 이 삽입된 자연스러운 결과

→ 이슈 원문의 접근 (A: 휴리스틱, B: compose `\t` 보존) 모두 **전제 무효**.

### 전제 2: PDF x≈162-182 (단계 2 에서 측정 오류 확정)

PDF content stream 의 Tm 을 직접 추출해 수치 비교:

| 요소 | PDF Tm.e | SVG x | 비율 |
|------|----------|-------|------|
| "(" | 481 | 102.07 | 0.2122 |
| 큰 수식 첫 glyph | 627 | 133.27 | **0.2125** |

- PDF 내 수식이 "(" 보다 146 단위 오른쪽, SVG 는 31.2 px 오른쪽
- 환산: `146 × 0.2125 = 31.03` px ≈ SVG 31.2 px (0.17 px 오차)

→ #287 수정 결과는 **PDF 와 사실상 정확히 일치**. 원래 이슈 본문에 적었던 "PDF x≈162-182" 는 PDF 이미지를 눈대중으로 보던 시절의 **추정 오류**.
