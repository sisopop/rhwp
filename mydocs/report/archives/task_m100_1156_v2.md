## 2. 워터마크 판정 기준 확정 (작업지시자 권위)

초기 정정은 워터마크를 `effect != RealPic && (밝기≠0 \|\| 대비≠0)` 또는 "RealPic 배경 = 항상 워터마크"로 **추정**했다. 이는 자의적 해석이었다.

권위 자료 3종 교차 검증:

- **한컴 공식 파일구조 3.0/5.0** — 그림 정보(표 32)는 밝기·대비·그림효과(REAL_PIC/GRAY_SCALE/BLACK_WHITE)·BinItem ID 만 가진다. 채우기 정보(표 28)에도 워터마크 적용 비트가 없다. (워터마크 비트는 그리기 개체 공통속성 기타옵션 bit 20 에만 있으며, 이는 도형 면 채우기용으로 `<hp:pic>` 그림 컨트롤과 무관.)
- **water-mark.hwp / .hwpx** (작업지시자 제공, 이미지 2개 중 2번째가 워터마크) — 두 그림 비교 시 워터마크 적용/미적용의 **유일한 차이는 밝기·대비 값**. 미적용=(0,0), 워터마크=(70,-50) 또는 (-50,70). effect·별도 플래그·XML 속성 차이 없음.
- **한컴 편집기 UI** (쪽 테두리/배경 > 그림) — "워터마크 효과" 체크박스. 해제 시 밝기·대비를 **0/0 으로 되돌리고**, 적용 시 0 이 아닌 값 부여(기본 70/-50, 사용자 변경 가능).

**확정 기준**: HWP/HWPX 에 워터마크 적용 비트는 **존재하지 않는다**. 워터마크 여부는 **밝기·대비가 둘 다 0 이 아닌 경우**로 판정한다 (effect 무관, 한쪽이라도 0 이면 워터마크 아님). 권위 문서 `mydocs/tech/한글문서파일구조3.0.md` 표 45 직후에 작업지시자가 기준을 명문화함.

## 3. 변경 내용

### 3.1 HWPX 파싱 손실 복구 — `src/parser/hwpx/header.rs`

`<hc:imgBrush><hc:img>` 의 `b"img"` 분기가 `binaryItemIDRef` 만 읽고 **bright/contrast/effect 를 버려서** 쪽 배경 채우기 IR 이 밝기·대비 0/0 으로 손실(→ 워터마크 미판정). bright/contrast/effect(REAL_PIC=0/GRAY_SCALE=1/BLACK_WHITE=2) 파싱 추가.

검증: 143E 배경 채우기 IR `brightness=0,contrast=0` → `brightness=70,contrast=-50` 복구.

### 3.2 판정 기준 일원화 — `is_watermark()` = `밝기≠0 && 대비≠0`

- `src/model/image.rs` `ImageAttr::is_watermark` — `&&` (AND). 특정값 매칭 `is_hancom_watermark_preset` **폐기**, `watermark_preset` 분류 단순화(custom).
- `src/renderer/render_tree.rs` — `PageBackgroundImage::is_watermark` / `ImageNode::is_watermark` 신설 (AND). 미사용 `is_watermark_tone_preset` 2곳 **제거**.

### 3.3 렌더러 3종 판정 통일 (effect 무관 AND)

- `src/renderer/svg.rs` (쪽 배경 + 본문 그림 2곳)
- `src/renderer/web_canvas.rs` (2곳)
- `src/renderer/skia/renderer.rs` — 페이지 배경: 워터마크일 때만 `save_layer_alpha` 반투명 합성. 밝기·대비 0/0(워터마크 아님)은 불투명 그대로 → CI characterization 테스트 정합.

### 3.4 테스트 갱신 — `tests/issue_516.rs`

AND 기준으로 재작성. `(30,0)`·`(0,20)` 같은 한쪽 0 케이스는 워터마크 아님으로 단언.

### 3.5 Node 20 deprecation — `actions/cache@v4` → `@v5`

5개 워크플로(ci/deploy-pages/npm-publish/release-binary/render-diff). v5 는 Node 24 런타임(GitHub-hosted runner 자동 충족, breaking change 없음). 로컬 개발 환경도 nvm Node 24.15.0 으로 정합.
