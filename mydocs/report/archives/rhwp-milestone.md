# rhwp 프로젝트

"모두의 한글"

지금까지는 hwp/hwpx 는 특정 회사의 전유물이었습니다.
rhwp 는 공개된 hwp/hwpx 스펙문서를 기반으로 읽고 보고 쓸 수 있는 모두의 한글 편집기를 AI 에 적합한 방식으로 재구성하고 모두가 참여할 수 있는 워크스페이스를 목표로 합니다.

---

## 로드맵

혼자 뼈대를 세우고, 함께 살을 붙이고, 모두의 것으로 완성한다.

```
0.5 ──── 1.0 ──── 2.0 ──── 3.0
뼈대      조판      협업      완성
역공학    AI 파이프  커뮤니티   공공 자산
```

| 단계 | 방향 | 전략 |
|------|------|------|
| **0.5 → 1.0** | 읽기/쓰기 기반 위에 조판 엔진 체계화 | 핵심 아키텍처를 혼자 견고하게 |
| **1.0 → 2.0** | AI 조판 파이프라인 위에 커뮤니티 참여 개방 | 기여 진입 장벽을 낮추는 구조 |
| **2.0 → 3.0** | 커뮤니티가 채운 기능 위에 공공 자산화 | 한컴 대등 수준 달성 |

### 0.5.0까지 혼자 뼈대를 완성하고 공개하는 이유

커뮤니티가 붙었을 때 방향이 흔들리지 않으려면 핵심 아키텍처가 먼저 견고해야 합니다.
미완성 프로젝트에 기여자가 몰리면 오히려 혼란이 생깁니다.

---

## 이정표

각 버전에서 달성하는 구체적 목표.

### v0.5.0 — 뼈대 (2026-03 공개)

> 역공학 완성, 읽기/쓰기 기반 구축

- HWP 5.0 바이너리 / HWPX 파서 완성
- 문단 레이아웃 (줄간격, 들여쓰기, 정렬, 탭)
- 표 렌더링 (셀 병합, 테두리, 셀 수식)
- 다단 레이아웃, 문단 번호/글머리표
- 수식 파서/레이아웃/렌더러
- 머리말/꼬리말, 바탕쪽, 각주/미주
- 개체 배치 (TopAndBottom, TAC, 글 앞/뒤)
- 페이지네이션 (다단 분할, 표 행 분할)
- SVG 내보내기 (CLI) + Canvas 렌더링 (WASM/Web)
- 웹 에디터 (텍스트 편집, 서식 대화상자, 표 편집)
- hwpctl 호환 API (30 Actions, Field API)
- HWP 직렬화 (저장)
- 755+ 테스트

### v1.0.0 — 조판 엔진

> AI 조판 파이프라인, 뼈대 완성

- LINE_SEG 재계산 + 페이지네이션 재조판 체계화
- 편집 시 동적 재조판 안정화
- AI 기반 문서 생성/편집 파이프라인
- 문서 조판 품질 한컴 뷰어 수준 도달

### v2.0.0 — 협업

> 커뮤니티가 기능을 채워가는 단계, 살 붙이기

- 기여자 친화적 모듈 구조
- 플러그인/확장 아키텍처
- 실시간 협업 편집
- 다양한 출력 포맷 (PDF, DOCX 등)

### v3.0.0 — 완성

> 한컴과 대등한 수준, 완전한 공공 자산

- 전체 HWP 기능 커버리지
- 접근성 (a11y) 완전 지원
- 모바일/태블릿 대응
- 공공기관 실무 투입 가능 수준
