**작성일**: 2026-05-07
**작성자**: 클로드 (페어 프로그래밍 파트너)
**관련 영역**: Issue #652 + #653, 외부 사용자 트윗 (Scope.H @scope_the_hyena, 4월 28일), `samples/hwpx/aift.hwpx`

## 1. 결함 영역 본질

### 1.1 시나리오 영역

**메인테이너 직접 발견 시나리오**:
1. 메인테이너가 본 환경 rhwp-studio 에서 **빈 문서 → 표 생성 → HWPX 저장**
2. 한컴 2020 으로 본 HWPX 영역 열기 → **표 위치 비정상** (표가 문서 구석탱이에 처박힘)
3. 한컴 2020 종료 후 다시 열기 → **HWP 영역과 동일 정상 렌더링 회복**

**외부 사용자 X (Scope.H) 영역 정합**:
> 일례로 폰트 크기부터 제대로 조정하기 힘들고 마우스로 셀 선택이 안되는 건 기본에
> **저장하고 다시 열어봤더니 표가 문서 구석탱이에 처박혔음.**
> **표준이라던 hwpx 저장은 아직 지원조차 안함.**
> 그래서 한컴이 웃을 수 있는 거임.

→ 본 외부 사용자 평가가 본 결함 영역의 권위 본질 영역.

### 1.2 결정적 증거 영역

| 영역 | 결과 |
|---|---|
| **본 환경 → HWPX 저장 → 한컴 2020 열기** | 표 위치 비정상 ⚠️ |
| **한컴 2020 자체 → HWPX 저장 → 한컴 2020 열기** | 정상 ✅ |
| **한컴 자체 (HWP/HWPX 모두)** | 한컴 2010/2020 양쪽 동일 조판 ✅ |
| **본 환경의 HWPX 시리얼라이제이션 영역 정답지** | 한컴 2010 (현행) |

→ **본 환경의 HWPX 시리얼라이제이션 영역이 한컴 2010 정답지로 개발됨**, **한컴 2020 영역에서 결함 발현**.

### 1.3 한컴 영역의 본질 가정 (메인테이너 권위 영역)

> 한컴은 hwpx 를 기존 hwp 의 디시리얼라이즈 파이프라인에 매핑하는 구조로 개발했을 것. 왜? 한컴의 개발자들은 생각보다 부지런하지 않음. 엔진도 2026년인데 여전히 32비트.

→ **한컴 자체에서**:
- HWP → HWP binary deserialize → 내부 IR → renderer
- HWPX → 한컴 자체에서 HWP binary 영역의 동일 IR 으로 변환 → 동일 renderer
- → 한컴 영역에서는 HWP / HWPX 영역의 시각 결과 동일

## 2. 본 결함 영역의 3가지 유력 추론 영역

### 2.1 추론 #1 — HWPX 시리얼라이제이션 영역의 attribute 누락 / 기본값 영역 (가장 유력)

**가설**: 본 환경이 만든 HWPX 영역에 **한컴 2020 이 필수로 요구하는 일부 attribute 영역이 누락**, 한컴 2010 은 누락 영역에 대해 관용적 영역으로 기본값 처리하지만 한컴 2020 은 엄격 영역으로 처리 → 표 위치 영역 부풉.

**근거**:
- 본 환경이 만든 HWPX 영역의 시리얼라이제이션 영역은 한컴 2010 영역으로 검증됨 (영역 정합)
- 한컴 2010 은 하위 호환 영역으로 누락 attribute 영역에 대해 default 영역 적용
- 한컴 2020 영역은 신규 attribute (영역 schema 강화 / pageBreak 영역 / repeatHeader 영역 / cellSpacing 영역 등) 영역에서 default 영역 다르게 처리
- 외부 사용자 평가 영역의 "**표가 구석탱이에 처박힘**" → 표의 위치 영역의 default 값이 한컴 2020 영역에서 0 처리 → 페이지 좌상단 또는 우하단으로 처박힘

**검증 영역**:
- 본 환경 출력 HWPX 의 `<hp:tbl>` element 의 attribute 영역 sweep
- 한컴 2020 자체 출력 HWPX 의 동일 element 영역 sweep
- attribute 차이 영역 정량 측정 (예: `cellSpacing`, `noAdjust`, `lock`, `dropcapstyle` 등)

**예상 누락 영역 후보**:
- `noAdjust` (셀 자동 조정 영역)
- `dropcapstyle` (드롭캡 스타일 영역)
- `lock` (잠금 영역)
- `numberingType` (번호 매기기 영역)
- `textWrap` (감싸기 영역)
- `textFlow` (텍스트 흐름 영역)

### 2.2 추론 #2 — HWPX 시리얼라이제이션 영역의 좌표 / 단위 영역 정합 결함 (메인테이너 가정 정합)

**가설**: 본 환경의 HWPX 시리얼라이제이션 영역이 **vert / horz 영역의 좌표 영역 단위 변환 영역에서 한컴 2010 영역과 한컴 2020 영역 사이 정합 결함** → 표 위치 영역 부풉.

**근거**:
- 본 환경의 HWPX 시리얼라이제이션 영역은 IR 의 좌표 영역을 HWPX XML attribute (vertOffset / horzOffset 등) 영역으로 변환
- 한컴 2010 영역은 본 영역의 좌표 영역을 관용적 영역으로 해석 → 시각 영역 정상
- 한컴 2020 영역은 본 영역의 좌표 영역을 엄격 영역으로 해석 → 좌표 영역의 미세 단위 차이 (HU 영역의 정수 영역 / mm 영역) 가 다른 위치 영역으로 영역 변환

**좌표 영역 변환 영역의 결함 가능성**:
- `vert="PARA"` / `vertRelTo="PARA"` 영역의 default 값 (한컴 2010 정합) 이 한컴 2020 영역에서 다른 영역으로 해석
- `horz="COLUMN"` / `horzRelTo="COLUMN"` 영역의 default 값 영역
- `vertOffset` / `horzOffset` 영역의 단위 영역 (HU vs 1/100 mm 등)
- `treat_as_char=true` 영역과 `wrap=TopAndBottom` 영역의 동시 영역 영역 (Issue #652 의 본질 영역과 정합)

**검증 영역**:
- 본 환경 출력 HWPX 의 `<hp:tbl>` 의 `vert*` / `horz*` attribute 영역 측정
- 한컴 2020 자체 출력 HWPX 의 동일 attribute 영역 비교
- vertOffset / horzOffset 영역의 단위 영역 영역 정합

**메인테이너 가정 영역과의 정합**:
- 한컴이 HWPX 영역을 HWP binary 영역의 동일 IR 영역으로 매핑
- 한컴 2010 영역의 매핑 영역과 한컴 2020 영역의 매핑 영역이 다름 (좌표 영역의 default 영역 차이)
- 본 환경의 HWPX 시리얼라이제이션 영역이 한컴 2010 영역의 매핑 영역과 정합 → 한컴 2020 영역에서 결함

### 2.3 추론 #3 — HWPX 시리얼라이제이션 영역의 elements 순서 / 구조 영역 결함 (XML 영역 schema 강화)

**가설**: 본 환경이 만든 HWPX 영역의 **XML element 영역의 순서 / 구조 영역**이 한컴 2010 영역의 schema 영역과 정합하지만 **한컴 2020 영역의 강화된 schema 영역과 정합 안 됨** → 한컴 2020 영역이 **element 영역을 순서 영역 정합으로 해석하지 못함** → 표 영역의 위치 영역 부풉.

**근거**:
- HWPX 영역은 표준 영역의 OOXML 영역과 유사한 schema 영역 영역
- 한컴 2010 영역은 schema 영역의 element 순서 영역에 대해 관용적 영역
- 한컴 2020 영역은 schema 영역의 element 순서 영역에 대해 엄격 영역 (예: `<hp:tbl>` 영역 내부의 `<hp:sz>` 영역이 `<hp:pos>` 영역 이전에 와야 함 등)
- 본 환경의 출력 영역 element 순서 영역이 한컴 2010 영역 정합 / 한컴 2020 영역 비정합

**구조 영역 결함 가능성**:
- `<hp:tbl>` 영역 내부의 `<hp:sz>` / `<hp:pos>` / `<hp:outMargin>` / `<hp:inMargin>` 영역의 순서
- `<hp:p>` 영역 내부의 `<hp:run>` / `<hp:linesegarray>` 영역의 순서
- `<hp:tc>` (cell) 영역 내부의 `<hp:cellAddr>` / `<hp:cellSpan>` / `<hp:cellSz>` / `<hp:cellMargin>` 영역의 순서

**검증 영역**:
- 본 환경 출력 HWPX 와 한컴 2020 자체 출력 HWPX 의 element 순서 영역 정밀 비교
- diff 영역 자동 점검 (XML canonicalization 후 비교)
- 한컴 2020 schema 영역의 공식 영역 (NDA 영역) 또는 한컴 2020 정합 fixture 영역 sweep

## 6. 결론

본 결함 영역의 가장 유력한 본질 영역은 **추론 #1 (attribute 누락 / 기본값 영역 차이) + 추론 #2 (좌표 / 단위 영역 정합 결함) 결합 영역**.

### 6.1 본질 영역 정의

본 환경의 HWPX 시리얼라이제이션 영역이 **한컴 2010 정답지 영역으로 정합 영역** + **한컴 2020 영역과 비정합 영역**:
- 한컴 2010 영역은 누락 attribute 영역 / 좌표 영역의 미세 영역에 대해 관용적 영역으로 default 처리
- 한컴 2020 영역은 엄격 영역으로 처리 → 표 영역의 위치 영역이 default 영역으로 fallback → "구석탱이에 처박힘"
