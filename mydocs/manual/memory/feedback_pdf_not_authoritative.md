한컴 PDF 출력은 **어느 한컴인지** 가 정의되지 않으면 정답지로 수용할 수 없다. 한컴 한글 2020/2022 **편집기** 의 SaveAs PDF는 정답지로 사용 가능하지만, 한컴 뷰어 / macOS 인쇄 / 외부 변환 / 한글 2010 PDF는 등급 미달이다.

## 정답지 등급 (2026-05-07 PR #670 갱신)

**정답지 가능**
- 한컴 한글 2022 편집기 → PDF 변환 (FileSaveAs_S / SaveAs)
- 한컴 한글 2020 편집기 → PDF 변환 (동일)
- `pdf/` (2022) / `pdf-2020/` 폴더에 영구 보존

**정답지 등급 미달**
- 한컴 한글 2010 PDF: 보조 자료 (일부 케이스 부정확, 이슈 #345 사례)
- 한컴 뷰어 출력: 편집기 아님
- macOS 인쇄 다이얼로그: 한컴 외부 경로
- 외부 변환 도구 (hwp2pdf 등): 환경 의존성 미검증
- 한컴 2018 등 미확인 버전

## Why

### PR #360 정황 (정답지 미달 사례)
동일 HWP (`samples/2022년 국립국어원 업무계획.hwp`) 를 두 환경의 한컴이 출력한 PDF 페이지 수가 다름:
- 컨트리뷰터 (planet6897) 환경: 37 페이지 (한컴 뷰어 + macOS 인쇄)
- 작업지시자 환경: 35 페이지 (한컴 편집기)

→ 한컴 뷰어 + macOS 인쇄는 정답지 미달. 페이지 분할 자체가 환경 의존적이므로 "PDF 와 일치" 검증은 출처가 미정의면 무의미하다.

### PR #670 통찰 (정답지 가능 구간의 근거)
macOS / Linux 컨트리뷰터는 한컴 편집기(Windows 전용)에 접근할 수 없으므로, 한글 2020/2022 PDF가 시각 판정의 권위 자료가 된다. 이는 한컴 미보유 컨트리뷰터의 기여를 받는 다양성 인프라이며 `project_dtp_identity` 의 정체성을 뒷받침한다.

**잠재 원인 (환경 의존성):**
- 한컴 버전 (한글 2010 부정확 / 2018 미확인 / 뷰어)
- 폰트 설치 상태 (함초롬바탕 등 한컴 번들 폰트)
- OS / 프린터 드라이버 / PDF 출력 경로
- 한컴 설정 (페이지 분할 옵션, 빈 줄 감추기 등)
- 출력 방법 (편집기 직접 SaveAs / 인쇄 → PDF 프린터 / 외부 변환)

## How to apply

1. **PR 검토 시**:
   - 컨트리뷰터의 "PDF 와 일치" 검증 결과는 PDF 출처를 먼저 확인
     - 한컴 한글 2020/2022 편집기 SaveAs → 정답지 가능 (특히 맥/리눅스 컨트리뷰터)
     - 한컴 뷰어 / macOS 인쇄 / 한글 2010 PDF → 보조 자료, 재검증 필수
   - 머지 결정은 시각 결과 + cargo test 회귀 0 + svg_snapshot 등 결정적 검증 + 작업지시자 시각 판정

2. **task 검증 시**:
   - "PDF 와 일치" 사용 시 **어느 환경의 한컴인지 명시** (commit / 보고서 / CHANGELOG) — 버전 + 출력 방법 + OS
   - 작업지시자 / 컨트리뷰터 두 환경 모두 시각 점검 권장
   - 가능하면 결정적 기준 (cargo test 회귀, svg_snapshot) 우선

3. **외부 컨트리뷰터 안내**:
   - 맥/리눅스: `pdf/` (한글 2022) 또는 `pdf-2020/` PDF를 정답지로 사용 가능
   - Windows + 한컴: 편집기 직접 시각 판정 1차 + PDF 보조
   - `pdf/README.md` 의 권위 등급 확인

4. **이슈 #253** (한컴 PDF 기준 Visual Diff 파이프라인):
   - 한글 2020/2022 PDF 한정으로 활성화 가능. 그 외 출처는 보류

5. **이미 머지된 task 재점검**:
   - v0.7.x 페이지 분할 정정 task (#347, #356, #359, #361, #362) 가 어느 환경 PDF 기준이었는지 점검
