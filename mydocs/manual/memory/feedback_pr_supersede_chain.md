동일 컨트리뷰터 영역의 두 PR 영역의 supersede 영역의 처리 영역은 두 가지 패턴 영역:

## 패턴 (a): close + 통합 머지

**사례**: PR #649 (Task #517) → PR #650 (Task #517+#518)

PR A 영역이 PR B 영역의 첫 commit 으로 포함된 경우 — PR A close 영역 + PR B 영역에서 두 commit 통합 영역 머지. 단순 cherry-pick 영역의 후속 처리 영역. 시각 판정 게이트 임의 부과 영역 부재 영역의 정합 영역.

## 패턴 (b): 머지 + supersede 머지

**사례**: PR #657 (Task #485) → PR #662 (Task #656)

PR A 영역의 휴리스틱 영역의 정정 영역이 머지된 영역 후 영역, PR B 영역이 PR A 영역의 휴리스틱 영역을 본질 영역으로 대체 영역. 두 PR 영역의 base 영역의 분기 영역으로 동일 함수 동일 hunk 영역의 충돌 영역 발생 영역. **메인테이너 영역의 충돌 해결 영역의 통합 정정 영역** (PR A 의 일부 영역 보존 + PR B 의 본질 정정 영역의 통합) 영역의 정합 영역.

## 패턴 (c): 머지 + 회귀 정정 후속 PR 통합

**사례**: PR #723 (Task #722) → PR #732 (Task #724)

PR A 영역의 본질 정정 (페이지 8/27/48) 영역 머지 후 영역 시각 판정 게이트 영역에서 **별건 영역의 회귀 발견** (exam_science p1/p2/p8 5번/8번/12번 문항 영역 의 지문 왼쪽 경계 클립핑). 회귀 영역 영역 PR A 영역 의 의도 외 영향 영역 영역 동일 컨트리뷰터 영역 의 등록된 후속 PR B (관련 영역 영역 의 추가 정정) 영역 영역 영역 영역 통합 정정 영역 으로 묶음. PR A 영역 rollback 미진행 영역 (본질 정정 보존) + PR B 처리 시 회귀 정정 영역 함께 영역.

**Why:**
- 패턴 (a) 사례 영역에서 "WASM 빌드 + 시각 판정" 잠정 권장 영역을 자동 추가 → 작업지시자 정정 ("이 PR 은 시각 판정없이 앞선 PR #649 에 연결된 처리입니다").
- 패턴 (b) 사례 영역에서 동일 컨트리뷰터 영역의 supersede 의도 영역 인식 부재 → 충돌 영역만 보고 "rebase 요청" 영역의 보류 영역 권장 → 작업지시자 정정 ("이거 같은 컨트리뷰터가 PR 날린겁니다. #657 과 #662. 당신의 리뷰가 처음부터 잘못된거였어요").
- 패턴 (c) 사례 영역에서 회귀 발견 영역 영역 자동 영역 rollback 권장 → 작업지시자 정정 ("동일 컨트리뷰터가 #732 PR 로 관련된 작업을 한 것으로 판단합니다. 이번 PR 은 체리픽해서 처리하겠습니다"). 동일 컨트리뷰터 영역 의 후속 PR 등록 영역 영역 영역 영역 회귀 정정 영역 통합 영역 의 합리 영역 — rollback 보다 본질 보존 + 후속 PR 영역 영역 통합 정정 영역 영역 의 효율 영역.

**How to apply:**

### review 영역 시작 시
1. **컨트리뷰터 일치 영역 점검**: 직전 PR 영역의 컨트리뷰터 영역 + 본 PR 컨트리뷰터 영역 — 동일 컨트리뷰터 영역인지 영역 점검
2. **직전 PR 본문 영역 점검**: 후속 PR 영역의 의도 영역이 명시 (`mydocs/pr/archives/pr_{N-1}_*.md` + PR 본문 + 한국어 댓글) 영역인지 영역 점검
3. **본 PR 본문 영역 점검**: 직전 PR 영역의 supersede / 대체 / 본질 정정 영역의 의도 영역 명시 영역인지 영역 점검

### 패턴 (a) close+통합 머지 영역의 처리
1. close 된 PR 영역의 commit 영역이 본 PR 영역에 포함 영역인지 점검
2. 옵션 권장 영역에 "WASM 빌드 + 시각 판정" 영역 자동 추가 부재 — 직전 PR close 영역의 통합 후속 처리 영역의 단순 cherry-pick 영역의 정합 영역

### 패턴 (b) 머지+supersede 머지 영역의 처리
1. 충돌 영역에서 자동 영역 "rebase 요청" 영역 부재 — 메인테이너 충돌 해결 영역의 통합 정정 영역의 가능 영역 점검
2. 충돌 본질 영역 분석: 직전 PR 영역의 어떤 영역 (휴리스틱) 영역이 본 PR 영역에서 어떤 영역 (본질) 영역으로 대체 영역인지
3. 통합 정정 영역의 합리화: 직전 PR 영역의 보존 영역 (Bug-1 등) + 본 PR 영역의 본질 정정 영역의 통합 영역의 합리
4. 작업지시자 결정 영역 — rebase 요청 vs 메인테이너 통합 영역의 옵션 영역 제시

### 패턴 (c) 머지+회귀 정정 후속 PR 통합 영역의 처리
1. **시각 판정 게이트 영역 의 회귀 발견 영역 영역 자동 rollback 권장 부재** — 동일 컨트리뷰터 영역 의 후속 PR (gh pr list --author) 점검 필수
2. **후속 PR 영역 의 본질 영역 점검**: 동일 영역 (wrap zone / layout 등) 영역 영역 회귀 정정 가능 영역인지 영역
3. 작업지시자 결정 영역 — 옵션 (1) rollback / (2) 추가 정정 시도 / (3) 후속 PR 통합 영역 영역 옵션 제시
4. 후속 PR 통합 영역 결정 영역 영역 — 본 PR 머지 유지 + 후속 PR 처리 시 회귀 정정 통합 영역 영역
