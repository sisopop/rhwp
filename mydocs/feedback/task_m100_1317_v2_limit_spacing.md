# Task M100 #1317 v2 피드백 — 적분 상·하한 간격

## 피드백 (작업지시자)

> SVG에서 적분의 상한·하한 기호가 적분기호에 **너무 붙어 있음**.

## 분석

Stage 1의 적분 attach point는 상·하한을 글리프 갈고리에 바짝(`gap_x=fs*0.02`) 붙였다.
PDF 정답과는 부합했으나, 시각 판정상 상·하한이 적분 줄기에 붙어 보였다.

## 조치 (`src/renderer/equation/layout.rs` 적분 subsup 분기)

반복 피드백을 거쳐 **가로 간격만** 키우는 방식으로 수렴(최종 v4):

| 항목 | 초기 | v2 | v3 | **v4(최종)** |
|------|------|-----|-----|------|
| 가로 여백 `gap_x` | `fs*0.02` | `fs*0.12` | `fs*0.22` | **`fs*0.22`** |
| 상한 세로 | `top_y - sp.h*0.30` | `*0.45` | `*0.85` | **`*0.30`** |
| 하한 세로 | `bottom_y - sb.h*0.72` | `*0.58` | `*0.20` | **`*0.72`** |

- v2(가로 0.12): 여전히 붙어 보임 → 부족.
- v3(세로로도 벌림): 분리됐으나 **적분 박스 높이가 2.7fs→3.45fs로 커져 줄 간격이 위/아래로 넓어짐**(작업지시자 지적).
- **v4: 세로는 컴팩트(초기값) 복귀 + 가로 간격만 `fs*0.22` 유지.** 적분기호는 위 갈고리=우상향, 아래 갈고리=좌하향이라 가로로만 띄워도 첨자가 줄기에서 충분히 분리되며 박스 높이는 커지지 않는다.

→ 상·하한이 적분기호 줄기에서 분리되고 줄 간격도 유지. geom/layout 공유로 SVG/Canvas/Skia 3경로 공통 적용.
